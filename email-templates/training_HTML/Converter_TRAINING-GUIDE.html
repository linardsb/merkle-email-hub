Train from HTML + Screenshot + Figma URL

Upload all three together: the reference HTML, a screenshot of the original design, and the Figma URL. This creates a complete training case.

1

Provide the training materials

All three inputs are shown together. Fill in whichever you have — at minimum the HTML file.

HTML Email File
Drop an HTML email file here or click to browse
Design Screenshot (optional)
Drop a screenshot of the original design (PNG/JPG)

Figma Design URL (optional)

Linking the Figma source lets the system extract design tokens and compare converter output against the original.

2

Name the training case

Give this case an ID and description. Used for regression testing and memory storage.

3

Analyze & register

The system parses the HTML, classifies sections, extracts tokens, and shows quality signals.

"Analyze" parses HTML structure. "Save" creates the case on disk via API. "Backfill" feeds the learning loop via API. "Regression" commands are copied to clipboard.

4

Complete setup (terminal)

After saving, run these commands to wire the case into the learning loop.

# 1. If you have the Figma URL, extract design data python -m app.design_sync.diagnose.extract \ --connection-id CONNECTION_ID \ --node-id NODE_ID \ --output-dir data/debug/CASE_ID # 2. Add the case to the manifest vim data/debug/manifest.yaml # Add: # - id: "CASE_ID" # name: "Case Name" # source: "Source" # figma_node: "NODE_ID" # status: active # 3. Backfill learning data python scripts/backfill-conversion-memories.py --traces-only # 4. Run regression tests make snapshot-test # 5. Check quality baseline make converter-regression

Learning Loop Architecture

Every conversion makes the next one smarter.

Input Converter Learning Figma URL ----> Layout Analysis ----> Memory (PostgreSQL) or Component Match Scaffolder reads this on every HTML File HTML Generation future conversion to avoid | repeating bad template choices. | +----------> Insight Bus | Low-confidence sections (<0.6) | become insights targeting the | Scaffolder agent directly. | +----------> JSONL Traces Append-only audit trail in traces/converter_traces.jsonl Used for regression detection.

What gets stored per conversion

StorageWhatUsed By
Memory Quality warnings, match confidences, design tokens Scaffolder (template selection), Knowledge agent (Q&A)
Insights Low-confidence section matches grouped by type Scaffolder (avoids bad template picks)
Traces Full quality snapshot: score, confidences, warnings Regression detection (make converter-regression)

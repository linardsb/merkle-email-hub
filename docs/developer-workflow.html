Developer Workflow

From campaign brief to ESP delivery — 10 steps with AI-powered quality gates, multi-agent refinement, and cross-client validation.

01 Initiation

Campaign Brief & Project Setup

Define the campaign scope, select target clients, and configure brand constraints within a project workspace.

  • a Create project workspace — set client org, brand guidelines (colours, typography, required elements), and ESP target (Braze/SFMC/Adobe/Taxi)
  • b Write campaign brief — describe layout intent, content sections, personalisation needs, and target audience in natural language
  • c Select target email clients — choose from 25 client profiles (Gmail, Outlook, Apple Mail, Samsung, etc.) to set CSS constraint boundaries
  • d Fetch competitive intelligence — get audience-scoped feasibility report showing which CSS features are safe vs. risky for selected clients

02 Design Import

Design-to-Code Conversion

Import Figma designs or start from scratch — the AI layout analyser converts visual designs into structured email briefs automatically.

  • a Connect Figma file — browse design file tree, select frames to import, auto-extract layout structure and design tokens
  • b AI layout analysis — Figma structure parsed into email sections (hero, content blocks, CTA, footer) with a generated markdown brief
  • c Asset pipeline — images downloaded, resized to email-safe dimensions, served with access control, referenced in the generated template
  • d Component extraction — reusable Figma components detected and saved to the Hub Component Library with versioning

03 Generation

AI Scaffolding & Template Build

The Scaffolder agent generates MSO-first Maizzle HTML from the brief, using the component library and knowledge base for best practices.

  • a Scaffolder agent generates HTML — MSO-first approach using always-loaded VML/conditional skills, Maizzle syntax, and email CSS reference
  • b Component composition — reusable components (header, CTA, hero) injected from the versioned library with slot-based content fills
  • c Maizzle build pipeline — Node.js sidecar compiles Tailwind CSS to inline styles, processes layouts, generates production-ready HTML
  • d Post-generation MSO validation — automatic conditional comment checking, namespace verification, and structured warnings propagated to handoff

04 Refinement

AI Multi-Agent Enhancement

Specialised AI agents refine the email — dark mode support, content polish, personalisation tags, accessibility compliance, and code review.

  • a Dark Mode agent — injects color-scheme meta tags, prefers-color-scheme media queries, Outlook overrides, and colour remapping with deterministic safety net
  • b Content agent — generates subject lines, preheaders, CTA variations, applies brand tone, validates length guardrails per content type
  • c Personalisation agent — injects ESP-specific dynamic content (Liquid for Braze, AMPscript for SFMC, JSSP for Adobe) with syntax balance checking
  • d Code Reviewer + Accessibility agents — flag redundant code, CSS support issues, WCAG AA violations, generate alt text, check contrast ratios
Dark Mode Content Personalisation Accessibility Code Reviewer
05 Quality Gate

11-Point QA Validation

Every email passes through the DOM-parsed QA engine — 11 automated checks covering structure, rendering, accessibility, compliance, and deliverability.

  • a Structural & rendering checks — HTML validation (20 DOM rules), MSO conditional balance, VML namespace integrity, CSS support scoring against 25 clients
  • b Deliverability checks — spam score (59 weighted triggers), file size per client (Yahoo 75KB, Outlook 100KB, Gmail 102KB), link validation, image optimisation
  • c Compliance checks — WCAG AA accessibility (24 rules), dark mode correctness, brand compliance (colours/typography/required elements), personalisation syntax
  • d Auto-repair pipeline — failures route to the Recovery Router for bounded self-correction (max 2 retries) with enriched failure context per agent

06 Manual Editing

Code Editor & Live Preview

Fine-tune the email in the Monaco code editor with syntax highlighting, live multi-viewport preview, and real-time Maizzle compilation.

  • a Monaco editor — full HTML/CSS/Liquid syntax highlighting, intelligent autocomplete, inline error markers from QA checks
  • b Live preview panel — desktop, tablet, and mobile viewports with dark mode toggle, real-time recompilation via Maizzle sidecar
  • c Visual Liquid builder — drag-and-drop dynamic content blocks for non-technical users, generates Liquid/AMPscript behind the scenes
  • d Collaborative editing — Yjs CRDT-based real-time co-editing with live cursors, version history, and side-by-side diff view

07 Testing

Cross-Client Rendering Tests

Validate how the email renders across 90+ email client/device combinations with visual regression detection and persona-driven test profiles.

  • a Rendering test execution — submit to Litmus or Email on Acid, receive screenshots from 90+ client/device combos (Gmail, Outlook, Apple Mail, Samsung, etc.)
  • b Visual regression detection — compare against baseline screenshots, flag layout shifts, colour mismatches, and broken elements per client
  • c Persona-driven testing — 8 subscriber presets (device, email client, dark mode preference, screen reader usage) simulate real-world reading conditions
  • d Compatibility matrix — per-client pass/fail dashboard with deduction breakdown, links to specific rendering issues and suggested fixes

08 Versioning

Template Versioning & Component Save

Save the finalised email as a versioned template, extract reusable components, and persist agent learnings for future campaigns.

  • a Template versioning — save as a versioned email template with metadata, diff against previous versions, soft delete with restore capability
  • b Component extraction — isolate reusable sections (headers, footers, CTAs, hero blocks) into the versioned component library for future campaigns
  • c Agent memory persistence — handoff decisions, QA patterns, and successful fix strategies stored as episodic memory for cross-session recall
  • d Knowledge graph update — new patterns (CSS workarounds, client-specific fixes) promoted to durable semantic knowledge via DCG bridge
Templates Components Agent Memory Knowledge Graph
09 Approval

Client Review & Approval Portal

Share the email with stakeholders through the approval portal — collect structured feedback, track revision rounds, and maintain a full audit trail.

  • a Approval request — generate a shareable review link with multi-viewport preview (desktop/mobile/dark mode) for client stakeholders
  • b Structured feedback — inline annotations on specific sections, categorised feedback (copy, design, compliance), priority tagging
  • c Revision workflow — state machine (draft → review → revision → approved) with automatic status transitions and notification triggers
  • d Audit trail — every approval action logged with timestamp, user, and state change — immutable history for compliance and accountability

10 Delivery

ESP Export & Bidirectional Sync

Push the approved email to your ESP with personalisation tags intact — or sync templates bidirectionally between the Hub and your production platform.

  • a ESP export — one-click push to Braze (Content Blocks with Liquid), SFMC (Content Builder), Adobe Campaign, or Taxi for Email with tag mapping
  • b Bidirectional sync — pull existing ESP templates into the Hub for QA/enhancement, push updated versions back with conflict resolution
  • c Raw HTML download — export compiled production HTML with all styles inlined, images referenced, and personalisation tags preserved
  • d Production trace sampling — async quality monitoring on deployed emails, judge verdicts feed back into the agent improvement loop

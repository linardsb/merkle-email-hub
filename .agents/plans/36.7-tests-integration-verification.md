## Implementation Steps

### Step 1: Create `test_e2e_document_pipeline.py` (~25–30 tests)

**Location**: `app/design_sync/tests/test_e2e_document_pipeline.py`

Factories to define:
- `_make_figma_api_response()` → dict mimicking Figma GET /v1/files response (nodes with FRAME/TEXT/IMAGE types, fills, typography)
- `_make_mjml_email()` → str with `<mjml><mj-head>` + `<mj-body>` containing hero, 2-col content, CTA button, footer
- `_load_golden_html(name)` → load from `app/ai/templates/library/{name}.html`
- `_make_document_from_sections(sections)` → `EmailDesignDocument` with given sections + default tokens/layout

**Test classes:**

```
class TestFigmaE2E:
    # Mock Figma HTTP → build_document() → convert_document() → HTML
    test_figma_build_and_convert_produces_valid_html
    test_figma_document_has_classified_sections
    test_figma_document_tokens_round_trip_through_bridge
    test_figma_convert_with_target_clients
    test_figma_build_document_error_raises_figma_api_error

class TestMjmlImportE2E:
    # MJML string → MjmlImportAdapter.parse() → EmailDesignDocument → convert_document_mjml() → HTML
    test_mjml_import_to_document_to_html
    test_mjml_import_preserves_dark_mode_tokens
    test_mjml_two_column_section_detected
    test_mjml_button_with_href_preserved
    test_mjml_social_section_detected
    test_mjml_hero_section_detected
    test_mjml_compile_error_fallback

class TestHtmlImportE2E:
    # Golden HTML → HtmlImportAdapter.parse() → EmailDesignDocument → convert_document() → HTML
    test_golden_template_roundtrip_section_count
    test_bulletproof_button_detected_as_button
    test_inline_styles_extracted_as_tokens
    test_dark_mode_css_extracted
    test_empty_html_produces_empty_document
    test_ai_disabled_preserves_unknown_sections

class TestCrossFormatEquivalence:
    # Same email as MJML and HTML → both produce EmailDesignDocument with comparable sections
    test_same_email_mjml_vs_html_section_count_matches
    test_same_email_mjml_vs_html_section_types_match

class TestPenpotE2E:
    # Mirror of Figma E2E with Penpot adapter
    test_penpot_build_and_convert_produces_valid_html
    test_penpot_document_has_sections
```

**Key assertions per test:**
- `EmailDesignDocument.validate(doc.to_json()) == []` — schema valid
- `"<!DOCTYPE html" in result.html` — valid email HTML
- `"<table" in result.html` — table-based layout (not div)
- `len(doc.sections) > 0` — sections extracted
- Bridge roundtrip: `doc.to_extracted_tokens()` returns `ExtractedTokens` instance
- Bridge roundtrip: `doc.to_email_sections()` returns list of `EmailSection`

**Mocking strategy:**
- Figma: `patch("httpx.AsyncClient.get")` or `patch.object(FigmaDesignSyncService, "sync_tokens_and_structure")` with `AsyncMock`
- Penpot: same pattern with `PenpotDesignSyncService`
- MJML compile: `patch.object(DesignConverterService, "compile_mjml")` returning `MjmlCompileResult(html="...", errors=[])`
- HtmlImportAdapter: `patch.object(adapter, "_resolve_ai_enabled", return_value=False)` for deterministic tests

## Preflight Warnings

- `test_e2e_pipeline.py` and `test_e2e_mjml_pipeline.py` use the OLD `DesignFileStructure` API — do NOT modify these (backward compat via shim). New E2E tests must use `EmailDesignDocument` API exclusively.
- `HtmlImportAdapter.parse()` is async — all HTML import tests need `@pytest.mark.asyncio`.
- `MjmlImportAdapter.parse()` — check if sync or async before writing tests (36.4 tests suggest sync).
- Golden template paths: use `Path(__file__).resolve().parents[N]` pattern from existing `test_adapter.py`.
- `compile_mjml()` calls the Maizzle sidecar — MUST be mocked in unit tests.
- Rate limiter: disable via `limiter.enabled = False` fixture in route-touching tests.

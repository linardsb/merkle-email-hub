# Plan: 35.3 MJML Generation Backend in Converter

## Context

Add a third conversion path `_convert_mjml()` in `converter_service.py` that generates MJML markup from layout analysis, then compiles via the sidecar's `/compile-mjml` endpoint. Replaces hand-rolled table generation (~800 lines in `converter.py`) for the common case. Benefits: responsive stacking, Outlook ghost tables, CSS inlining — all handled by MJML's battle-tested compiler. The recursive converter remains as fallback for advanced features MJML can't express.

## Research Summary

### Conversion Pipeline Flow

```
Figma/Penpot Import → DesignFileStructure + ExtractedTokens
  → analyze_layout(structure) → DesignLayoutDescription
  → DesignConverterService.convert() dispatches to:
      a) _convert_with_components()  [table-on-table]
      b) _convert_recursive()        [tr-stacking]
      c) _convert_mjml()             [NEW — Phase 35.3]
```

**Section-to-MJML mapping:**

| EmailSectionType | MJML Pattern |
|-----------------|--------------|
| `PREHEADER` | `<mj-preview>{text}</mj-preview>` (in `<mj-head>`, not `<mj-body>`) |
| `HEADER` | `<mj-section>` + `<mj-column>` + logo `<mj-image>` + nav `<mj-text>` |
| `HERO` | `<mj-section background-url=...>` + `<mj-text>` (heading) + `<mj-button>` |
| `CONTENT` | `<mj-section>` + columns + `<mj-text>` body |
| `CTA` | `<mj-section>` + `<mj-button>` with brand primary color |
| `FOOTER` | `<mj-section>` + `<mj-text>` (small, muted color) |
| `SOCIAL` | `<mj-section>` + `<mj-social>` with `<mj-social-element>` items |
| `NAV` | `<mj-section>` + `<mj-navbar>` + `<mj-navbar-link>` items |
| `DIVIDER` | `<mj-section>` + `<mj-divider>` |
| `SPACER` | `<mj-section>` + `<mj-spacer height="Npx">` |
| `UNKNOWN` | `<mj-section>` + columns + content elements |
| Multi-column | `<mj-section>` + N × `<mj-column width="X%">` via `_calculate_column_widths()` |

**Token injection:**
- Colors from `convert_colors_to_palette(tokens)` → `background-color`, `color` on MJML elements
- Typography from `convert_typography(tokens)` → `font-family` (via `_font_stack()`), `font-size`, `font-weight`, `line-height`, `letter-spacing`
- Spacing from `section.padding_*` fields → `padding` attribute on `<mj-section>`/`<mj-column>`
- Dark mode from `tokens.dark_colors` → `<mj-style>` CSS block with `@media (prefers-color-scheme: dark)`

**MJML wrapper template:**
```xml
<mjml>
  <mj-head>
    <mj-attributes>
      <mj-all font-family="{body_font_stack}" />
      <mj-text font-size="{body_size}px" color="{text_color}" line-height="{line_height}" />
      <mj-button background-color="{primary}" color="{btn_text}" font-size="16px"
                  inner-padding="12px 24px" border-radius="4px" />
    </mj-attributes>
    <mj-preview>{preheader_text}</mj-preview>
    <mj-style>{dark_mode_css}</mj-style>
  </mj-head>
  <mj-body width="{container_width}px" background-color="{bg_color}">
    {sections_mjml}
  </mj-body>
</mjml>
```

**Text escaping:** All text content passed through `html.escape()` before injection into MJML. CSS values passed through `_sanitize_css_value()`.

**Data attributes:** Post-process compiled HTML to inject `data-section-type`, `data-node-id` on outermost `<table>` per section using simple string replacement keyed on `<!-- section:{node_id} -->` comment markers in MJML source.

### Step 2: Add `convert_mjml()` as a **separate async method** (DO NOT make `convert()` async)

> **PREFLIGHT CRITICAL:** `convert()` is called synchronously from **34 test callsites** across 12 test files + `import_service.py:191` + 12 penpot integration tests. Making it async would require rewriting all 46+ callers. Instead, add a **separate** `convert_mjml()` public async method.

**2a.** Add new public async method to `DesignConverterService`:
```python
async def convert_mjml(
    self,
    structure: DesignFileStructure,
    tokens: ExtractedTokens,
    *,
    raw_file_data: dict[str, Any] | None = None,
    selected_nodes: list[str] | None = None,
    target_clients: list[str] | None = None,
    connection_config: dict[str, Any] | None = None,
    image_urls: dict[str, str] | None = None,
) -> ConversionResult:
```

This method duplicates the preamble from `convert()` (normalize tree, collect frames, analyze layout, derive container width) then calls the MJML-specific path. Extract the shared preamble into a private `_prepare_conversion()` helper to avoid duplication:

```python
def _prepare_conversion(self, structure, tokens, *, raw_file_data, selected_nodes,
                        connection_config) -> tuple[list[DesignNode], DesignLayoutDescription,
                                                     list[str], ConverterCompatibility, int] | ConversionResult:
    """Shared preamble: normalize, collect frames, analyze layout, derive width."""
```

If it returns a `ConversionResult`, that's the early-exit (no frames). Otherwise returns the tuple of prepared data.

**2b.** Add private `_convert_mjml_from_layout()` async method:
```python
async def _convert_mjml_from_layout(
    self,
    *,
    layout: DesignLayoutDescription,
    tokens: ExtractedTokens,
    warnings: list[str],
    container_width: int,
    target_clients: list[str] | None = None,
) -> ConversionResult:
```

Implementation:
1. Call `generate_mjml(layout, tokens, container_width=container_width)` from `mjml_generator.py`
2. Call `await self.compile_mjml(mjml_str, target_clients=target_clients)`
3. Check `result.errors` — if any, log and add to warnings (non-fatal)
4. Post-process HTML: inject `data-section-type`/`data-node-id` attributes from comment markers
5. Return `ConversionResult(html=result.html, sections_count=len(layout.sections), ...)`

**Fallback logic (in `convert_mjml()`):**
```python
try:
    return await self._convert_mjml_from_layout(...)
except MjmlCompileError:
    logger.warning("design_sync.mjml_fallback", reason="compilation_failed")
    warnings.append("MJML compilation failed, falling back to recursive converter")
    return self._convert_recursive(...)
```

**2c.** Existing `convert()` method is **unchanged** — no signature or behavior changes. Zero test breakage.

## Preflight Warnings

1. **`convert()` MUST stay sync** — 46+ sync callsites (34 test + 12 penpot integration). Plan updated: add separate `convert_mjml()` async method instead of converting `convert()` to async.
2. **`converter_service.py` has 41 pyright errors** — use typed models for all new code, avoid raw dict access.
3. **MJML `<mj-hero>` is deprecated** in MJML 4.x — use `<mj-section>` with `background-url` instead for HERO sections.
4. **`data-*` attributes** are stripped by MJML compiler — must post-process compiled HTML to re-inject section markers. Use comment markers (`<!-- section:node_id:type -->`) in MJML source as anchors.
5. **MJML `<mj-social>` requires network-accessible icon URLs** — for SOCIAL sections, fall back to `<mj-text>` with inline links if no icon URLs available.
6. **`test_e2e_pipeline.py:370`** asserts `sections_count == 4` — safe, MJML path doesn't change existing convert() behavior.
7. **Extract shared preamble** from `convert()` into `_prepare_conversion()` to avoid code duplication between sync and async paths.

# Plan: Binary Judge Prompts (Task 5.2)

## Context

Phase 5 requires binary pass/fail LLM judges for subjective quality criteria that the 10-point QA gate cannot catch. Each of the 3 implemented agents (scaffolder, dark_mode, content) needs one judge module evaluating 3-5 subjective dimensions. The judges consume JSONL traces from `runner.py` and produce structured JSON verdicts.

"""Schemas for LLM judge inputs and outputs."""

from pydantic import BaseModel


class JudgeCriteria(BaseModel):
    """A single evaluation criterion."""

    name: str
    description: str


class JudgeInput(BaseModel):
    """Input to an LLM judge — one trace to evaluate."""

    trace_id: str
    agent: str
    input_data: dict[str, object]
    output_data: dict[str, object] | None
    expected_challenges: list[str]


class CriterionResult(BaseModel):
    """Result for a single criterion."""

    criterion: str
    passed: bool
    reasoning: str


class JudgeVerdict(BaseModel):
    """Structured output from an LLM judge."""

    trace_id: str
    agent: str
    overall_pass: bool
    criteria_results: list[CriterionResult]
    error: str | None = None
```

class Judge(Protocol):
    """Protocol for all agent judges."""

    agent_name: str
    criteria: list[JudgeCriteria]

    def build_prompt(self, judge_input: JudgeInput) -> str:
        """Build the full prompt for the LLM judge call."""
        ...

    def parse_response(self, raw: str, judge_input: JudgeInput) -> JudgeVerdict:
        """Parse LLM response into structured verdict."""
        ...


SYSTEM_PROMPT_TEMPLATE = """\
You are a strict quality evaluator for email development AI agents.

You will receive an agent's input and output. Evaluate the output against each criterion below.

For EACH criterion, respond with:
- "passed": true or false
- "reasoning": 1-2 sentences explaining your judgment

Then give an overall "overall_pass" which is true ONLY if ALL criteria pass.

Respond with ONLY valid JSON in this exact format:
{{
  "overall_pass": true/false,
  "criteria_results": [
    {{
      "criterion": "<name>",
      "passed": true/false,
      "reasoning": "<explanation>"
    }}
  ]
}}

CRITERIA:
{criteria_block}

IMPORTANT:
- Be strict. If in doubt, fail.
- Judge the OUTPUT only. Do not judge the input quality.
- Output ONLY the JSON object. No markdown, no explanation outside JSON.
"""


def build_criteria_block(criteria: list[JudgeCriteria]) -> str:
    """Format criteria list into numbered prompt block."""
    lines: list[str] = []
    for i, c in enumerate(criteria, 1):
        lines.append(f"{i}. **{c.name}**: {c.description}")
    return "\n".join(lines)
```

from app.ai.agents.evals.judges.schemas import (

    JudgeVerdict,

SCAFFOLDER_CRITERIA: list[JudgeCriteria] = [
    JudgeCriteria(
        name="brief_fidelity",
        description=(
            "Does the generated HTML faithfully implement all key elements from the brief? "
            "Check that layout type, sections, content areas, and specific requirements "
            "mentioned in the brief are present in the output. Minor creative additions "
            "are acceptable; missing requested sections are not."
        ),
    ),
    JudgeCriteria(
        name="email_layout_patterns",
        description=(
            "Does the HTML use email-appropriate layout patterns exclusively? "
            "Must use nested <table> with role='presentation' for layout — no flexbox, "
            "no CSS grid, no float-based layouts. Tables must have cellpadding='0' "
            "and cellspacing='0'. Content width must not exceed 600px."
        ),
    ),
    JudgeCriteria(
        name="mso_conditional_correctness",
        description=(
            "Are MSO conditional comments (<!--[if mso]>, <!--[if !mso]>) present "
            "and correctly structured? Must include xmlns:v and xmlns:o namespace "
            "declarations. VML elements (if used for buttons/backgrounds) must be "
            "inside MSO conditionals. Conditional blocks must be properly closed."
        ),
    ),
    JudgeCriteria(
        name="dark_mode_readiness",
        description=(
            "Does the output include dark mode meta tags (color-scheme, "
            "supported-color-schemes), @media (prefers-color-scheme: dark) rules, "
            "and [data-ogsc]/[data-ogsb] Outlook dark mode selectors? "
            "Colors must maintain 4.5:1 contrast ratio in both modes."
        ),
    ),
    JudgeCriteria(
        name="accessibility_baseline",
        description=(
            "Does the HTML include lang attribute on <html>, role='article' on "
            "wrapper, role='presentation' on layout tables, semantic heading "
            "hierarchy, and meaningful alt text on all <img> tags? "
            "Images must have explicit width/height and style='display:block;border:0'."
        ),
    ),
]


class ScaffolderJudge:
    """Binary judge for Scaffolder agent outputs."""

    agent_name: str = "scaffolder"
    criteria: list[JudgeCriteria] = SCAFFOLDER_CRITERIA

    def build_prompt(self, judge_input: JudgeInput) -> str:
        """Build evaluation prompt with brief and generated HTML."""
        criteria_block = build_criteria_block(self.criteria)
        system = SYSTEM_PROMPT_TEMPLATE.format(criteria_block=criteria_block)

        brief = judge_input.input_data.get("brief", "")
        html_output = ""
        if judge_input.output_data:
            html_output = judge_input.output_data.get("html", "")

        user_content = (
            f"## AGENT INPUT (Brief)\n{brief}\n\n"
            f"## AGENT OUTPUT (HTML)\n```html\n{html_output}\n```"
        )
        return f"{system}\n\n---\n\n{user_content}"

    def parse_response(self, raw: str, judge_input: JudgeInput) -> JudgeVerdict:
        """Parse LLM JSON response into JudgeVerdict."""
        try:
            # Strip markdown code fences if present
            cleaned = raw.strip()
            if cleaned.startswith("```"):
                cleaned = cleaned.split("\n", 1)[1]
                cleaned = cleaned.rsplit("```", 1)[0]

            data = json.loads(cleaned)

            criteria_results = [
                CriterionResult(
                    criterion=cr["criterion"],
                    passed=cr["passed"],
                    reasoning=cr["reasoning"],
                )
                for cr in data["criteria_results"]
            ]

            return JudgeVerdict(
                trace_id=judge_input.trace_id,
                agent=self.agent_name,
                overall_pass=data["overall_pass"],
                criteria_results=criteria_results,
            )
        except (json.JSONDecodeError, KeyError, TypeError) as e:
            return JudgeVerdict(
                trace_id=judge_input.trace_id,
                agent=self.agent_name,
                overall_pass=False,
                criteria_results=[],
                error=f"Failed to parse judge response: {e}",
            )
```

def build_prompt(self, judge_input: JudgeInput) -> str:
    criteria_block = build_criteria_block(self.criteria)
    system = SYSTEM_PROMPT_TEMPLATE.format(criteria_block=criteria_block)

    input_data = judge_input.input_data
    operation = input_data.get("operation", "")
    text = input_data.get("text", "")
    tone = input_data.get("tone", "not specified")
    brand_voice = input_data.get("brand_voice")

    output_data = judge_input.output_data or {}
    content_output = output_data.get("content", [])
    # content is a list of alternatives
    formatted_output = "\n---\n".join(
        str(c) for c in content_output
    ) if isinstance(content_output, list) else str(content_output)

    constraints = f"Operation: {operation}\nTone: {tone}"
    if brand_voice:
        constraints += f"\nBrand voice: {brand_voice}"

    user_content = (
        f"## AGENT INPUT\n{constraints}\n\nOriginal text:\n{text}\n\n"
        f"## AGENT OUTPUT\n```text\n{formatted_output}\n```"
    )
    return f"{system}\n\n---\n\n{user_content}"

### Step 7: Create judge runner CLI (`judge_runner.py`)

```python
"""CLI for running LLM judges on collected traces.

Usage:
    python -m app.ai.agents.evals.judge_runner \
        --agent scaffolder \
        --traces traces/scaffolder_traces.jsonl \
        --output traces/scaffolder_verdicts.jsonl
"""
```

**Key implementation details:**

- Reads JSONL traces from `--traces` path
- Skips traces where `output` is null (agent errored)
- Converts each trace to `JudgeInput`
- Calls the agent's LLM provider (via existing `app.ai` infrastructure) with the judge prompt
- Parses response via `judge.parse_response()`
- Writes `JudgeVerdict.model_dump()` as JSONL to `--output`
- Rate limiting: process 5 traces at a time with 2s delay between batches
- Prints summary: total traces, passed, failed, errors, pass rate per criterion

**CLI arguments:**
- `--agent` (required): scaffolder | dark_mode | content | all
- `--traces` (required): path to input JSONL traces file
- `--output` (required): path to output JSONL verdicts file
- `--provider` (optional): override AI provider (default: use settings)
- `--model` (optional): override model (default: use settings)
- `--batch-size` (optional): concurrent traces per batch (default: 5)
- `--delay` (optional): seconds between batches (default: 2.0)

**Core function:**

```python
async def judge_trace(
    judge: ScaffolderJudge | DarkModeJudge | ContentJudge,
    trace: dict[str, Any],
    provider: AIProvider,
) -> JudgeVerdict:
    """Run a single judge on a single trace."""
    judge_input = JudgeInput(
        trace_id=trace["id"],
        agent=trace["agent"],
        input_data=trace["input"],
        output_data=trace["output"],
        expected_challenges=trace.get("expected_challenges", []),
    )
    prompt = judge.build_prompt(judge_input)

    response = await provider.complete(
        messages=[{"role": "user", "content": prompt}],
        temperature=0.0,  # Deterministic for consistency
    )

    return judge.parse_response(response.content, judge_input)
```

## Security Constraints

- Judge prompts must NOT include the agent's system prompt (no leakage)
- Judge prompts must NOT make external API calls (LLM call is via existing provider)
- Judge outputs are structured JSON only (JudgeVerdict schema)
- `traces/` directory must remain in `.gitignore`
- No real client data, PII, or API keys in judge prompts or test fixtures

## Design Decisions

1. **Protocol-based judges** — Each judge implements `build_prompt()` + `parse_response()`, matching the codebase's Protocol pattern (ConnectorProvider, RenderingProvider)
2. **Shared system prompt template** — DRY: all judges use the same structured JSON output format, only criteria differ
3. **temperature=0.0** — Deterministic judge responses for inter-run consistency (>90% agreement target)
4. **Batched execution** — 5 traces at a time with 2s delays to respect rate limits (per TODO.md security requirements)
5. **JSON-only output parsing** — Handles markdown code fences gracefully but rejects freeform text
6. **Separate judge runner** — `judge_runner.py` is independent from `runner.py` (trace collection). Clean separation: collect traces first, then judge them.

# Plan: 45.1 Cron Scheduling Engine

## Context

Multiple features need periodic execution (QA sweeps, ontology sync, rendering baselines, eval regression checks) but currently require manual `make` targets. A shared in-process cron scheduler avoids external dependencies (Celery Beat, external cron) while providing run logging and failure alerting. Feature-gated via `SCHEDULING__ENABLED` (default `false`).

## Implementation Steps

### Step 1: Config (`app/core/config.py`)

Add `SchedulingConfig` near other feature configs:

```python
class SchedulingConfig(BaseModel):
    """Cron scheduling engine settings."""
    enabled: bool = False
    check_interval_seconds: int = 60
    job_timeout_seconds: int = 3600
    max_run_history: int = 100
    run_history_ttl_seconds: int = 86400
```

Add to `Settings`: `scheduling: SchedulingConfig = SchedulingConfig()`

### Step 2: Schemas (`app/scheduling/schemas.py`)

```python
class JobStatus(str, Enum):
    idle = "idle"
    running = "running"
    success = "success"
    failed = "failed"

class JobDefinitionResponse(BaseModel):
    name: str
    cron_expr: str
    callable_name: str
    enabled: bool
    last_run: datetime | None
    last_status: JobStatus | None
    next_run: datetime | None
    run_count: int

class JobUpdateRequest(BaseModel):
    enabled: bool | None = None
    cron_expr: str | None = None

class JobRunResponse(BaseModel):
    job_name: str
    started_at: datetime
    ended_at: datetime | None
    status: JobStatus
    error: str | None
    duration_ms: int | None
```

### Step 3: Registry (`app/scheduling/registry.py`)

Global dict `_JOB_REGISTRY: dict[str, tuple[Callable[..., Awaitable[None]], str]]` mapping `name → (callable, default_cron)`.

`scheduled_job(cron: str)` decorator — registers `func.__name__` + default cron expr. Validates cron syntax at registration time using `croniter.is_valid()`.

`get_registry()` returns immutable copy. `get_job_callable(name: str)` returns callable or raises `NotFoundError`.

### Step 4: Engine (`app/scheduling/engine.py`)

`CronScheduler` class:

| Method | Behavior |
|--------|----------|
| `__init__(settings: SchedulingConfig)` | Store config, init `_task: asyncio.Task | None`, `_running = False` |
| `async start()` | Set `_running = True`, `asyncio.create_task(self._loop())` |
| `async stop()` | Set `_running = False`, cancel task, suppress `CancelledError` |
| `async _loop()` | Sleep `check_interval_seconds`, call `_evaluate_jobs()`, repeat |
| `async _evaluate_jobs()` | Load jobs from Redis, check each cron vs `last_run`, fire if due |
| `async _execute_job(name, callable)` | Record start in Redis, `asyncio.wait_for(callable(), timeout)`, record result |

**Leader election:** Use `scheduling:leader` Redis key with `SET NX EX 90` (1.5× check interval). Only leader evaluates jobs. Follows `DataPoller._acquire_leader_lock()` pattern.

**Redis keys:**

| Key | Type | TTL | Content |
|-----|------|-----|---------|
| `scheduling:jobs:{name}` | Hash | none | `cron_expr`, `callable_name`, `enabled`, `last_run`, `last_status`, `run_count` |
| `scheduling:runs:{name}` | List (capped) | `run_history_ttl_seconds` | JSON-encoded `JobRunResponse` entries, LPUSH + LTRIM to `max_run_history` |
| `scheduling:leader` | String | 90s | Hostname/PID for debugging |

**Job registration flow:** On `start()`, sync `_JOB_REGISTRY` entries → Redis. If key exists, preserve `enabled`/`last_run`/`last_status`. If key doesn't exist, create with defaults from decorator.

### Step 5: Service (`app/scheduling/service.py`)

| Function | Behavior |
|----------|----------|
| `async list_jobs() → list[JobDefinitionResponse]` | Load all `scheduling:jobs:*` from Redis, compute `next_run` via croniter |
| `async get_job(name) → JobDefinitionResponse` | Load single job, raise `NotFoundError` if missing |
| `async update_job(name, req: JobUpdateRequest) → JobDefinitionResponse` | Update `enabled`/`cron_expr` in Redis hash, validate cron syntax |
| `async trigger_job(name) → JobRunResponse` | Look up callable from registry, execute immediately, return run result |
| `async get_run_history(name, limit=20) → list[JobRunResponse]` | LRANGE on `scheduling:runs:{name}` |

### Step 7: Lifespan wiring (`app/main.py`)

In startup block (after existing pollers):
```python
scheduler: CronScheduler | None = None
if settings.scheduling.enabled:
    from app.scheduling.engine import CronScheduler
    scheduler = CronScheduler(settings.scheduling)
    await scheduler.start()
    logger.info("scheduling.scheduler_started")
```

In shutdown block:
```python
if scheduler is not None:
    await scheduler.stop()
    logger.info("scheduling.scheduler_stopped")
```

Include router in `app/main.py` router registration (gated by `settings.scheduling.enabled`).

## Preflight Warnings

- `croniter` has no type stubs — add to pyright `extraPaths` or use `# type: ignore[import-untyped]` on import
- Redis `SCAN` for listing `scheduling:jobs:*` keys — use `async for key in redis.scan_iter(match="scheduling:jobs:*")` not `KEYS` (production-safe)
- Leader election TTL (90s) must exceed `check_interval_seconds` (60s) to avoid split-brain

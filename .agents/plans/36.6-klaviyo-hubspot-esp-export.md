# Plan: 36.6 Klaviyo + HubSpot ESP Export

## Context
Add Klaviyo and HubSpot ESP export providers to complete the Big 5 coverage (joining Braze, SFMC, Adobe Campaign). Both follow the existing `ESPSyncProvider` protocol + `PROVIDER_REGISTRY` dispatch pattern.

### ESPSyncProvider Protocol (6 methods)
```
validate_credentials(credentials: dict[str, str]) -> bool
list_templates(credentials: dict[str, str]) -> list[ESPTemplate]
get_template(template_id: str, credentials: dict[str, str]) -> ESPTemplate
create_template(name: str, html: str, credentials: dict[str, str]) -> ESPTemplate
update_template(template_id: str, html: str, credentials: dict[str, str]) -> ESPTemplate
delete_template(template_id: str, credentials: dict[str, str]) -> bool
```

### Provider Pattern (Braze = simplest, matches both Klaviyo & HubSpot)
- `__init__(settings=None)` → reads `_base_url` from `ESPSyncConfig`
- `_headers(credentials)` → auth header dict
- Each method: `async with httpx.AsyncClient(timeout=30) as client:` → `resilient_request()`
- Map ESP JSON response → `ESPTemplate(id, name, html, esp_type, created_at, updated_at)`

### Current PROVIDER_REGISTRY
```python
{"braze": BrazeSyncProvider, "sfmc": SFMCSyncProvider, "adobe_campaign": AdobeSyncProvider, "taxi": TaxiSyncProvider}
```

### ESPConnectionCreate Validation
`esp_type: str = Field(..., pattern=r"^(braze|sfmc|adobe_campaign|taxi)$")` — must add `klaviyo|hubspot`.

### Step 3: Klaviyo Provider — `app/connectors/klaviyo/sync_provider.py`

**Auth:** `Authorization: Klaviyo-API-Key {api_key}` + `revision: 2025-07-15` header.
**Credentials:** `{"api_key": "pk_..."}`.

| Method | Endpoint | Notes |
|--------|----------|-------|
| `validate_credentials` | `GET /api/accounts/` | 200 = valid |
| `list_templates` | `GET /api/templates/` | JSON:API pagination via `page[cursor]`, unwrap `data[].attributes` |
| `get_template` | `GET /api/templates/{id}/` | Unwrap `data.attributes` |
| `create_template` | `POST /api/templates/` | JSON:API body: `{"data": {"type": "template", "attributes": {"name": ..., "html": ...}}}` |
| `update_template` | `PATCH /api/templates/{id}/` | Same JSON:API format |
| `delete_template` | `DELETE /api/templates/{id}/` | 204 = success |

Pagination: follow `links.next` until absent. Map each item:
```python
ESPTemplate(
    id=item["id"],
    name=item["attributes"]["name"],
    html=item["attributes"]["html"],
    esp_type="klaviyo",
    created_at=item["attributes"]["created"],
    updated_at=item["attributes"]["updated"],
)
```

### Step 4: HubSpot Provider — `app/connectors/hubspot/sync_provider.py`

**Auth:** `Authorization: Bearer {access_token}`.
**Credentials:** `{"access_token": "pat-..."}`.

| Method | Endpoint | Notes |
|--------|----------|-------|
| `validate_credentials` | `GET /account-info/v3/details` | 200 = valid |
| `list_templates` | `GET /marketing/v3/emails/` | Cursor pagination via `after` param, results in `results[]` |
| `get_template` | `GET /marketing/v3/emails/{id}` | Direct object response |
| `create_template` | `POST /marketing/v3/emails/` | `{"name": ..., "content": {"html": ...}, "type": "REGULAR"}` |
| `update_template` | `PATCH /marketing/v3/emails/{id}` | `{"content": {"html": ...}}` |
| `delete_template` | `DELETE /marketing/v3/emails/{id}` | 204 = success (soft delete) |

Pagination: check `paging.next.after`, pass as `?after=` param. Map each item:
```python
ESPTemplate(
    id=str(item["id"]),
    name=item["name"],
    html=item.get("content", {}).get("html", ""),
    esp_type="hubspot",
    created_at=item["createdAt"],
    updated_at=item["updatedAt"],
)
```

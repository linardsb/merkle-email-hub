# Plan: 46.1 Credential Pool with Rotation and Cooldowns

## Context

Single-key configurations are fragile. Batch eval runs exhaust Anthropic rate limits, SFMC campaign pushes hit per-key send limits. A `CredentialPool` with multiple keys per service, round-robin rotation, and automatic cooldown enables graceful degradation instead of hard failures.

### Step 2: Add exception to exceptions.py

After `PromptInjectionError` (line ~44):

```python
class NoHealthyCredentialsError(ServiceUnavailableError):
    """All credentials for a service are cooling down or unhealthy (503)."""

    def __init__(self, service: str) -> None:
        self.service = service
        super().__init__(f"No healthy credentials available for service: {service}")
```

### Step 3: Create `app/core/credentials.py`

Core module with three components:

#### `CredentialLease` — context for a borrowed key

```python
@dataclass
class CredentialLease:
    service: str
    key: str
    key_hash: str
    _pool: "CredentialPool"

    async def report_success(self) -> None:
        """Reset failure state for this key."""

    async def report_failure(self, status_code: int) -> None:
        """Record failure. On 429/401/403: enter cooldown. On 3 consecutive: mark unhealthy."""
```

#### `CredentialPool` — manages keys for one service

```python
class CredentialPool:
    def __init__(self, service: str, keys: list[str], config: CredentialsConfig) -> None:
        self._service = service
        self._keys = keys
        self._config = config
        self._index = 0  # round-robin counter
        self._lock = asyncio.Lock()
        # In-memory fallback when Redis unavailable
        self._fallback: dict[str, _KeyState] = {}

    async def get_key(self) -> CredentialLease:
        """Return next healthy key via round-robin. Raises NoHealthyCredentialsError if none available."""

    async def _get_state(self, key_hash: str) -> _KeyState:
        """Load key state from Redis, fallback to in-memory."""

    async def _set_state(self, key_hash: str, state: _KeyState) -> None:
        """Persist key state to Redis with TTL."""

    async def _record_success(self, key_hash: str) -> None:
        """Reset failure count, clear cooldown."""

    async def _record_failure(self, key_hash: str, status_code: int) -> None:
        """Increment failures, enter cooldown on trigger codes, mark unhealthy at threshold."""

    def pool_status(self) -> dict[str, Any]:
        """Sync snapshot for health check API (future 46.4)."""
```

**Key state** stored per key in Redis (`credentials:{service}:{key_hash}`):

```python
@dataclass
class _KeyState:
    healthy: bool = True
    cooldown_until: float = 0.0  # time.monotonic() timestamp
    failure_count: int = 0
    last_failure_code: int = 0
```

**Redis key format:** `credentials:{service}:{key_hash}` with JSON value, TTL = `unhealthy_ttl_seconds`.

**Cooldown trigger codes:** 429 (rate limit), 401 (unauthorized), 403 (forbidden).

**Backoff formula:** `min(initial * 2^(failure_count - 1), max_seconds)` — progression 30→60→120→300.

**In-memory fallback:** Mirror of quota.py pattern — if Redis is unreachable, use `self._fallback` dict. State is per-process but still functional for single-instance deployments.

#### `get_credential_pool()` — singleton accessor

```python
_pools: dict[str, CredentialPool] = {}

def get_credential_pool(service: str) -> CredentialPool:
    if service not in _pools:
        settings = get_settings()
        keys = settings.credentials.pools.get(service, [])
        if not keys:
            raise NoHealthyCredentialsError(service)
        _pools[service] = CredentialPool(service, keys, settings.credentials)
    return _pools[service]

def reset_pools() -> None:
    """Clear all pools. For testing only."""
    _pools.clear()
```

## Preflight Warnings

- `CredentialsConfig.pools` uses `dict[str, list[str]]` — env var loading requires JSON string (`CREDENTIALS__POOLS='{"anthropic": ["k1", "k2"]}'`). Pydantic v2 handles this via `env_nested_delimiter`.
- Round-robin `_index` is per-process. In multi-worker deployments, each worker rotates independently — acceptable for key distribution but not perfectly balanced.
- The `asyncio.Lock` in `get_key()` prevents concurrent rotation from selecting the same key twice in a race, but only within a single process.

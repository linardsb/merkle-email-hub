# Plan: 48.12 Knowledge Graph Proactive QA Pipeline

## Context

QA failures (e.g. "hero-full-width breaks in Outlook 2019 due to missing VML wrapper") stay in QA reports. Next time an agent uses that component, it has no warning. This feature turns past failures into future prevention by extracting failure patterns into the knowledge graph as queryable relationships and injecting them as proactive warnings before pipeline runs.

The "Cognify phase" from the PRD is **already implemented** via this existing export path.

### Step 2: `ProactiveWarning` dataclass + core logic (`app/knowledge/proactive_qa.py`)

```python
@dataclass(frozen=True)
class ProactiveWarning:
    component: str      # component slug
    client: str         # email client ID
    failure: str        # failure_type / qa_check name
    severity: str       # "critical" | "warning" | "info"
    suggestion: str     # 1-line fix suggestion
    occurrence_count: int = 1
    first_seen: str = ""
    last_seen: str = ""
```

**`FailureExtractor`** class:
- `extract_from_qa_result(check_name: str, passed: bool, details: dict, component_slugs: list[str], client_ids: list[str]) -> list[ProactiveWarning]`
- Only produces warnings when `passed=False`
- Deduplication: `blake2b(component + client + check_name)` hash → skip if seen in current batch
- Maps severity from QA check score: score < 0.3 → "critical", < 0.7 → "warning", else "info"

**`ProactiveWarningInjector`** class:
- `query_warnings(component_slugs: list[str], client_ids: list[str], project_id: int | None) -> list[ProactiveWarning]`
  - Reuses `recall_failure_patterns()` pattern from `failure_patterns.py` — queries semantic memory by component+client terms
  - Filters results where `metadata["source"] == "failure_pattern"` and overlapping client_ids
  - Respects `failure_min_occurrences` config
  - Caps at `proactive_max_warnings`
- `format_warnings_for_prompt(warnings: list[ProactiveWarning]) -> str`
  - Returns markdown-formatted block: "## Known Failure Patterns\n- hero-full-width: Missing VML wrapper breaks Outlook 2019 (seen 3x, fix: Add VML wrapper)"

**Integration with existing `recall_failure_patterns()`:**
The existing function queries by agent+clients. We add a parallel function `recall_component_warnings()` that queries by component+clients instead. Same Redis caching pattern from `knowledge_prefetch.py`.

New private method:
```python
async def _inject_proactive_warnings(self, run_id: str) -> None:
    """Query knowledge graph for proactive warnings and inject into artifact store."""
    try:
        from app.core.config import get_settings
        settings = get_settings()
        if not settings.knowledge.proactive_qa_enabled:
            return

        from app.knowledge.proactive_qa import ProactiveWarningInjector
        injector = ProactiveWarningInjector(settings)

        # Extract component slugs from build plan artifact if available
        plan = self._store.get_optional("plan", BuildPlanArtifact)
        component_slugs = _extract_component_slugs(plan) if plan else []

        # Extract client IDs from project config or empty
        client_ids = _extract_client_ids(self._store)

        if not component_slugs and not client_ids:
            return

        warnings = await injector.query_warnings(
            component_slugs=component_slugs,
            client_ids=client_ids,
            project_id=None,  # extracted from store metadata if available
        )

        if warnings:
            from app.knowledge.proactive_qa import ProactiveWarningInjector
            artifact = ProactiveWarningsArtifact(
                name="proactive_warnings",
                produced_by="proactive_qa",
                produced_at=datetime.now(UTC),
                warnings=tuple(warnings),
                formatted_text=injector.format_warnings_for_prompt(warnings),
            )
            self._store.put("proactive_warnings", artifact)
            logger.info(
                "pipeline.proactive_warnings.injected",
                extra={"run_id": run_id, "count": len(warnings)},
            )
    except Exception:
        logger.warning("pipeline.proactive_warnings.failed", exc_info=True)
```

Agent adapters that want proactive warnings read `proactive_warnings` from the store in `adapt_inputs()` and append `formatted_text` to the system prompt. This is an opt-in pattern — adapters that don't read it simply ignore it.

## Preflight Warnings

- `recall_failure_patterns()` returns raw text — the new `recall_component_warnings()` must parse metadata to reconstruct `ProactiveWarning` objects. Test this parsing thoroughly.
- `CogneeConfig.prefetch_enabled` gates **knowledge prefetch** (prior outcomes). `KnowledgeConfig.proactive_qa_enabled` gates **proactive QA warnings**. Keep these independent — don't conflate.
- Semantic memory search is pgvector-based and fuzzy — component slugs with hyphens (e.g. `hero-full-width`) must be tested for accurate retrieval. May need exact metadata match instead of vector similarity.
- `ArtifactStore.put()` overwrites — if multiple pipeline extensions write `proactive_warnings`, last wins. Use a unique artifact name.

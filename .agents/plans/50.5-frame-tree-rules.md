# Plan: 50.5 Pure FRAME-Tree Rules — Rules 7, 8, 10, 11

## Context

> **Concrete diff targets:** see `.agents/plans/50-current-output-reference.md` § Section 1 (Rules 7, 8, 10) and § Section 3 (Rule 11) for current vs expected markup excerpts.

Per `docs/architecture/opus-figma-to-html-process.md` §8.5:
> *"Rules 7–11 in particular are pure FRAME-tree reads (no PNG required) and should ship first because they have the highest hit rate per minute of engineering investment."*

The four rules in this subtask:

| Rule | Signal | Action |
|------|--------|--------|
| **7** Pill alignment from bbox.x | `pill.bbox.x` vs `parent_column.bbox.x` | `align="left|center|right"` per offset |
| **8** Pill cornerRadius is source of truth | `cornerRadius` field (or absent → 0) | `border-radius: Npx` (0 means square — no "looks like pill, round it" heuristic) |
| **10** Image per-corner radii | `rectangleCornerRadii: [TL, TR, BR, BL]` | 4× `border-{top|bottom}-{left|right}-radius` longhand |
| **11** Inner card width matches dominant child image width | All children share same `max-width` | `width="<NATIVE_PX>"` + `align="center"` + `class="wf"` |

Rules 7+8 fire on tag/pill slots (Phase 51.3 will introduce the slot itself; this phase ships the rule predicates so 51.3 can consume them). Rule 10 fires on any image FRAME. Rule 11 fires on inner cards from Phase 50.4.

## Files to Create/Modify

### New: `app/design_sync/frame_rules.py` (~250 lines)

Module hosting the 4 rule predicates as pure functions. Each takes a `DesignNode` (or relevant subset) and returns a structured result.

```python
"""Pure FRAME-tree rules — codified detection rules 7, 8, 10, 11 from
docs/architecture/opus-figma-to-html-process.md §8.3.

Each rule is a pure function: takes Figma node data, returns a structured
result indicating whether the rule fires and what action to take. No I/O,
no PNG sampling — these rules read FRAME-tree fields directly.
"""

from __future__ import annotations
from dataclasses import dataclass
from typing import Literal

from app.design_sync.protocol import DesignNode

Alignment = Literal["left", "center", "right"]


@dataclass(frozen=True)
class PillAlignment:
    """Output of Rule 7."""
    align: Alignment
    pill_x: float
    parent_x: float


@dataclass(frozen=True)
class CornerRadiusSpec:
    """Output of Rule 8 / Rule 10."""
    scalar: float | None  # None means "no radius (square corners)"
    per_corner: tuple[float, float, float, float] | None  # (TL, TR, BR, BL)


@dataclass(frozen=True)
class CardWidthSpec:
    """Output of Rule 11."""
    fixed_width_px: int  # Native px from dominant child image
    use_class: str = "wf"  # mobile-only width:100% override class


def rule_7_pill_alignment(
    pill: DesignNode,
    parent_column: DesignNode,
    *,
    tolerance_px: float = 4.0,
) -> PillAlignment:
    """Pill / tag alignment from child x-coordinate (not heuristic)."""
    pill_x = pill.absolute_bounding_box.x if pill.absolute_bounding_box else 0
    pill_w = pill.absolute_bounding_box.width if pill.absolute_bounding_box else 0
    parent_x = parent_column.absolute_bounding_box.x if parent_column.absolute_bounding_box else 0
    parent_w = parent_column.absolute_bounding_box.width if parent_column.absolute_bounding_box else 0

    if abs(pill_x - parent_x) <= tolerance_px:
        return PillAlignment(align="left", pill_x=pill_x, parent_x=parent_x)
    if abs((pill_x + pill_w) - (parent_x + parent_w)) <= tolerance_px:
        return PillAlignment(align="right", pill_x=pill_x, parent_x=parent_x)
    return PillAlignment(align="center", pill_x=pill_x, parent_x=parent_x)


def rule_8_corner_radius(node: DesignNode) -> CornerRadiusSpec:
    """Border-radius from cornerRadius / rectangleCornerRadii — never heuristic.

    Missing or 0 → square corners. Don't apply 'looks like pill therefore round it'.
    """
    per_corner = node.rectangle_corner_radii  # tuple[float, float, float, float] | None
    if per_corner and any(c > 0 for c in per_corner):
        return CornerRadiusSpec(scalar=None, per_corner=per_corner)

    scalar = node.corner_radius
    if scalar and scalar > 0:
        return CornerRadiusSpec(scalar=scalar, per_corner=None)

    return CornerRadiusSpec(scalar=None, per_corner=None)


def rule_10_image_corner_radii(image_frame: DesignNode) -> CornerRadiusSpec:
    """Image per-corner border-radius from rectangleCornerRadii (alias for Rule 8 specifically on image FRAMES)."""
    # Same logic as rule_8; the only difference is *who calls it* (image vs pill)
    return rule_8_corner_radius(image_frame)


def rule_11_card_width_from_dominant_image(
    card_frame: DesignNode,
) -> CardWidthSpec | None:
    """Inner card width must match its image children's max-width.

    Fires when all direct image children share the same max-width
    (read from dominant `<img>`'s Figma frame).
    """
    image_widths: list[int] = []
    for child in card_frame.children:
        if not _is_image_frame(child):
            continue
        if not child.absolute_bounding_box:
            continue
        image_widths.append(int(child.absolute_bounding_box.width))

    if len(image_widths) < 2:
        return None

    # All images must share the same width (dominant width pattern)
    if len(set(image_widths)) == 1:
        return CardWidthSpec(fixed_width_px=image_widths[0])

    # Fallback: if 2+ images share the max width, treat as dominant
    from collections import Counter
    counts = Counter(image_widths)
    most_common, count = counts.most_common(1)[0]
    if count >= 2:
        return CardWidthSpec(fixed_width_px=int(most_common))

    return None


def _is_image_frame(node: DesignNode) -> bool:
    """A FRAME with image fill or RECTANGLE with image fill."""
    if not node.fills:
        return False
    return any(f.type == "IMAGE" for f in node.fills)
```

### Modify: `app/design_sync/component_matcher.py:1401-1469` — extend `_build_token_overrides`

```python
# Rule 11 — inner card fixed width (50.5)
if section.inner_card_fixed_width:
    overrides.append(
        TokenOverride("width", "_inner", f"{section.inner_card_fixed_width}px")
    )
    overrides.append(
        TokenOverride("__html_attr_align", "_inner", "center")
    )
    overrides.append(
        TokenOverride("__html_attr_class_add", "_inner", "wf")
    )

# Rule 10 — per-corner image radii (50.5)
for img in section.images:
    if img.corner_radius_spec and img.corner_radius_spec.per_corner:
        tl, tr, br, bl = img.corner_radius_spec.per_corner
        # Per-corner longhand survives more legacy renderers (Outlook 2016, AOL)
        overrides.append(TokenOverride("border-top-left-radius", f"_image_{img.node_id}", f"{tl:.0f}px"))
        overrides.append(TokenOverride("border-top-right-radius", f"_image_{img.node_id}", f"{tr:.0f}px"))
        overrides.append(TokenOverride("border-bottom-right-radius", f"_image_{img.node_id}", f"{br:.0f}px"))
        overrides.append(TokenOverride("border-bottom-left-radius", f"_image_{img.node_id}", f"{bl:.0f}px"))
```

## Implementation Steps

### Step 1: Create `frame_rules.py`

Per spec. Pure functions only — no logging, no I/O. Easy to unit-test.

### Step 2: Wire Rules 8 + 10 into layout_analyzer's image/button extraction

`_extract_images` populates `ImagePlaceholder.corner_radius_spec`. `_extract_buttons` populates `ButtonElement.corner_radius_spec`. Add `corner_radius_spec` field to both dataclasses (frozen — careful with field ordering).

### Step 3: Wire Rule 7 into text block extraction

When tag detection (51.3 will refine; for now use existing role_hint="tag" infrastructure if present) fires, run rule_7_pill_alignment with the tag's parent column → set `TextBlock.layout_align`.

### Step 4: Wire Rule 11 into section construction

After 50.4's `_detect_inner_bg`, if `inner_bg` was set, run `rule_11_card_width_from_dominant_image` on the inner FRAME → set `EmailSection.inner_card_fixed_width`.

### Step 5: Extend `_build_token_overrides`

Add Rule 11 emission (width + align + class), Rule 10 per-corner emission (4 longhand). Rule 7 emission lives in tag-slot fill logic in 51.3 — punt for this plan.

### Step 6: Renderer handlers

- `_apply_html_attr_overrides`: lxml mutation of `align`/`class`. Test with synthetic `<table class="_inner">`.
- `_apply_per_corner_radius`: lxml `select_one('img[data-node-id="..."]')`, apply inline 4 longhand props, ensure parent `<td>` has `overflow:hidden` for WebKit-based clients (per Rule 10 doc note).

### Step 7: 6 component HTMLs add `data-node-id` to `<img>`

Use Jinja-style: `data-node-id="{{ image.node_id }}"`. Component matcher fills the value.

### Step 8: Tests

18 tests. Synthetic `DesignNode`/`ImagePlaceholder`/`TextBlock`/`EmailSection` factories.

# Plan: 11.10 CSS Support — Syntax Validation & Vendor Prefixes

## Context

The current `css_support` check (`app/qa_engine/checks/css_support.py`) is a thin 78-line wrapper that delegates entirely to the ontology (`unsupported_css_in_html()`). It only checks if CSS properties have poor client support. It does NOT:

1. **Parse CSS syntax** — Invalid CSS (e.g., `color: ;`, `font-size: 14`, unclosed brackets) passes silently
2. **Detect vendor prefixes** — `-webkit-`, `-moz-`, `-ms-`, `-o-` prefixes used in emails are pointless in most email clients (Outlook uses Word, Gmail strips styles)
3. **Flag external stylesheets** — `<link rel="stylesheet">` is stripped by all major email clients; must be caught
4. **Validate `@import`** — Stripped by Gmail and most clients; should be flagged
5. **Detect `!important` abuse** — Overused `!important` can conflict with email client dark mode injection

Task 11.10 upgrades the check with `cssutils`-based syntax validation, vendor prefix detection, and external stylesheet flagging. It also adds an L4 agent reference file linking the comprehensive CSS DOM reference doc to the Code Reviewer agent's skill system.

## Design Decisions

### Why YAML rules + custom functions (not pure ontology)

The ontology handles **client support** (property X unsupported in client Y). The new checks are about **CSS syntax quality** — invalid values, vendor prefixes, external resources — which are independent of client support. Use the rule engine pattern (like spam_score, brand_compliance) with custom functions for the complex parsing.

### Why `cssutils` (not `tinycss2` or regex)

- `cssutils` is the most mature Python CSS parser with error reporting
- It provides `CSSStyleSheet` parsing with `log` for syntax errors
- It handles vendor prefixes, `@import`, `@media` natively
- No alternative offers the same combination of parsing + validation

### Scoring approach

Keep the existing ontology-based scoring AS-IS, then ADD syntax/prefix/external deductions on top. The check runs two passes:
1. **Pass 1 (existing)**: Ontology-based client support scan
2. **Pass 2 (new)**: Syntax validation, vendor prefixes, external stylesheets

Both contribute to the same score. New issues are appended to the existing issue list.

#### 4a: CSS parsing helper (module-level)

```python
import io
import logging as _stdlib_logging
import cssutils

# Silence cssutils logging (it logs parse errors to stderr by default)
cssutils.log.setLevel(_stdlib_logging.CRITICAL)

_VENDOR_PREFIX_RE = re.compile(r"-(?:webkit|moz|ms|o)-[\w-]+")
_IMPORTANT_RE = re.compile(r"!important", re.IGNORECASE)
_IMPORT_RE = re.compile(r"@import\s", re.IGNORECASE)
_STYLE_BLOCK_RE_CSS = re.compile(r"<style[^>]*>(.*?)</style>", re.DOTALL | re.IGNORECASE)
_INLINE_STYLE_RE_CSS = re.compile(r'style\s*=\s*"([^"]*)"', re.IGNORECASE)
_LINK_STYLESHEET_RE = re.compile(
    r"<link[^>]+rel\s*=\s*[\"']stylesheet[\"'][^>]*>", re.IGNORECASE
)
# Layout-critical properties that MUST be inlined for Gmail
_LAYOUT_CRITICAL_PROPS = frozenset({
    "display", "width", "max-width", "min-width", "height",
    "float", "position", "margin", "padding", "text-align",
    "vertical-align", "background-color", "color", "font-size",
    "font-family", "line-height", "border",
})


def _parse_css_blocks(html: str) -> list[str]:
    """Extract CSS content from <style> blocks."""
    return [m.group(1) for m in _STYLE_BLOCK_RE_CSS.finditer(html)]


def _parse_inline_styles(html: str) -> list[str]:
    """Extract inline style values."""
    return [m.group(1) for m in _INLINE_STYLE_RE_CSS.finditer(html)]


def _get_cssutils_errors(css_text: str) -> list[str]:
    """Parse CSS with cssutils and return syntax error messages."""
    handler = _stdlib_logging.handlers.MemoryHandler(capacity=1000)  # noqa: F841
    log = io.StringIO()
    handler2 = _stdlib_logging.StreamHandler(log)
    handler2.setLevel(_stdlib_logging.WARNING)

    # Use cssutils parser with custom log handler
    parser = cssutils.CSSParser(
        loglevel=_stdlib_logging.WARNING,
        raiseExceptions=False,
    )
    parser.parseString(css_text)

    # Collect errors from cssutils log
    errors: list[str] = []
    log_output = log.getvalue()
    if log_output.strip():
        for line in log_output.strip().splitlines():
            errors.append(line.strip())
    return errors
```

**Note:** The `_get_cssutils_errors` implementation needs careful testing — cssutils logging is tricky. An alternative approach is to use `cssutils.parseString()` and check the returned sheet's `cssRules` for invalid entries. Test during implementation and adjust.

#### 4b: `css_syntax_errors` function

```python
def css_syntax_errors(
    doc: HtmlElement,
    raw_html: str,
    config: QACheckConfig | None,
) -> tuple[list[str], float]:
    """Detect CSS syntax errors using cssutils parser."""
    blocks = _parse_css_blocks(raw_html)
    if not blocks:
        return [], 0.0

    max_reported = _get_param(config, "max_syntax_errors_reported", 5)
    deduction = _get_param(config, "deduction_syntax_error", 0.15)

    all_errors: list[str] = []
    for block in blocks:
        sheet = cssutils.parseString(block, validate=True)
        # cssutils stores parse errors in its internal log
        # Check for rules that failed to parse
        for rule in sheet:
            if rule.type == rule.UNKNOWN_RULE:
                all_errors.append(f"Unknown CSS rule: {rule.cssText[:60]}")

    # Also check for empty value declarations
    # Pattern: property with no meaningful value
    for block in blocks:
        for line in block.splitlines():
            stripped = line.strip().rstrip(";").strip()
            if ":" in stripped:
                prop, _, val = stripped.partition(":")
                if not val.strip() and prop.strip():
                    all_errors.append(f"Empty value for '{prop.strip()}'")

    if not all_errors:
        return [], 0.0

    issues = all_errors[:max_reported]
    if len(all_errors) > max_reported:
        issues.append(f"... and {len(all_errors) - max_reported} more syntax errors")

    return issues, deduction
```

def css_vendor_prefixes(
    doc: HtmlElement,
    raw_html: str,
    config: QACheckConfig | None,

        issues.append(f"... and {len(unique) - max_reported} more vendor prefixes")

#### 4f: `css_import_rule` function

```python
def css_import_rule(
    doc: HtmlElement,
    raw_html: str,
    config: QACheckConfig | None,
) -> tuple[list[str], float]:
    """Detect @import rules (stripped by Gmail and most clients)."""
    deduction = _get_param(config, "deduction_import_rule", 0.20)

    blocks = _parse_css_blocks(raw_html)
    issues: list[str] = []
    for block in blocks:
        for m in _IMPORT_RE.finditer(block):
            # Get the full @import line
            line_start = block.rfind("\n", 0, m.start()) + 1
            line_end = block.find(";", m.start())
            if line_end == -1:
                line_end = block.find("\n", m.start())
            if line_end == -1:
                line_end = len(block)
            line = block[line_start:line_end + 1].strip()
            issues.append(f"@import rule: {line[:80]}")

    if not issues:
        return [], 0.0

    return issues, deduction
```

def css_important_overuse(
    doc: HtmlElement,
    raw_html: str,
    config: QACheckConfig | None,
) -> tuple[list[str], float]:
    """Detect excessive !important usage (may conflict with dark mode)."""
    threshold = _get_param(config, "important_threshold", 10)
    deduction = _get_param(config, "deduction_important_overuse", 0.10)

    all_css = "\n".join(_parse_css_blocks(raw_html) + _parse_inline_styles(raw_html))
    count = len(_IMPORTANT_RE.findall(all_css))

    if count <= threshold:
        return [], 0.0

    # Check if they're in dark mode context (those are expected)
    dark_mode_css = ""
    for block in _parse_css_blocks(raw_html):
        if "prefers-color-scheme" in block.lower():
            dark_mode_css += block
    dark_mode_count = len(_IMPORTANT_RE.findall(dark_mode_css))
    non_dark_count = count - dark_mode_count

    if non_dark_count <= threshold:
        return [], 0.0

    issues = [
        f"{non_dark_count} !important declarations outside dark mode "
        f"(threshold: {threshold}) — may conflict with client dark mode injection"
    ]
    return issues, deduction
```

def css_non_inline_dependency(
    doc: HtmlElement,
    raw_html: str,
    config: QACheckConfig | None,
) -> tuple[list[str], float]:
    """Detect layout-critical CSS in <style> blocks without inline fallbacks."""
    max_reported = _get_param(config, "max_non_inline_reported", 5)
    deduction = _get_param(config, "deduction_non_inline", 0.10)

    style_blocks = _parse_css_blocks(raw_html)
    if not style_blocks:
        return [], 0.0

    # Parse style blocks for class/selector -> property mappings
    issues: list[str] = []
    all_css = "\n".join(style_blocks)

    # Simple heuristic: check if layout-critical properties appear ONLY in
    # <style> blocks and not also as inline styles
    for prop in _LAYOUT_CRITICAL_PROPS:
        # Check if property is in style blocks (excluding @media and dark mode)
        # This is a simplified check — the full implementation should parse selectors
        prop_in_block = re.search(
            rf"(?<!@media[^{{]*){re.escape(prop)}\s*:", all_css, re.IGNORECASE
        )
        if prop_in_block:
            # Check if same property appears inline
            inline_styles = _parse_inline_styles(raw_html)
            prop_inline = any(
                re.search(rf"{re.escape(prop)}\s*:", style, re.IGNORECASE)
                for style in inline_styles
            )
            if not prop_inline:
                issues.append(
                    f"'{prop}' in <style> block only — no inline fallback for Gmail"
                )

    if not issues:
        return [], 0.0

    reported = issues[:max_reported]
    if len(issues) > max_reported:
        reported.append(f"... and {len(issues) - max_reported} more")

    return reported, deduction
```

def css_syntax_summary(

    vendor_count = len(set(_VENDOR_PREFIX_RE.findall(all_css)))
    important_count = len(_IMPORTANT_RE.findall(all_css))
    external_count = len(_LINK_STYLESHEET_RE.findall(raw_html))
    import_count = len(_IMPORT_RE.findall(all_css))

    parts = []
    if vendor_count:
        parts.append(f"{vendor_count} vendor prefixes")
    if important_count:
        parts.append(f"{important_count} !important")
    if external_count:
        parts.append(f"{external_count} external stylesheets")
    if import_count:
        parts.append(f"{import_count} @import rules")

    summary = f"CSS audit: {len(blocks)} style blocks, {len(inline)} inline styles"
    if parts:
        summary += f" — {', '.join(parts)}"

    return [summary], 0.0

```python
"""CSS email client support check — ontology + syntax validation."""

from __future__ import annotations

import pathlib

from lxml import html as lxml_html
from lxml.html import HtmlElement

from app.knowledge.ontology.query import unsupported_css_in_html
from app.qa_engine.check_config import QACheckConfig
from app.qa_engine.rule_engine import RuleEngine, load_rules
from app.qa_engine.schemas import QACheckResult

_DEFAULT_ERROR_DEDUCTION = 0.2
_DEFAULT_WARNING_DEDUCTION = 0.1
_DEFAULT_MAX_DETAILS = 10

_RULES_PATH = pathlib.Path(__file__).resolve().parent.parent / "rules" / "css_support.yaml"


class CssSupportCheck:
    """Checks for CSS properties with poor email client support + syntax validation.

    Two-pass check:
    1. Ontology scan — flags CSS properties unsupported by major email clients
    2. Syntax validation — cssutils parsing, vendor prefixes, external stylesheets
    """

    name = "css_support"

    def __init__(self) -> None:
        self._rules = load_rules(_RULES_PATH)
        self._engine = RuleEngine(self._rules)

    async def run(self, html: str, config: QACheckConfig | None = None) -> QACheckResult:
        error_deduction: float = (
            config.params.get("error_deduction", _DEFAULT_ERROR_DEDUCTION)
            if config
            else _DEFAULT_ERROR_DEDUCTION
        )
        warning_deduction: float = (
            config.params.get("warning_deduction", _DEFAULT_WARNING_DEDUCTION)
            if config
            else _DEFAULT_WARNING_DEDUCTION
        )
        max_details: int = (
            config.params.get("max_issues_in_details", _DEFAULT_MAX_DETAILS)
            if config
            else _DEFAULT_MAX_DETAILS
        )

        # ── Pass 1: Ontology-based client support scan (existing) ──
        ontology_issues = unsupported_css_in_html(html)

        for issue in ontology_issues:
            if issue["severity"] == "error" and issue["fallback_available"]:
                issue["severity"] = "warning"

        actionable = [i for i in ontology_issues if i["severity"] in ("error", "warning")]
        errors = [i for i in actionable if i["severity"] == "error"]
        warnings = [i for i in actionable if i["severity"] == "warning"]

        detail_parts: list[str] = []
        for issue in actionable[:max_details]:
            prop_str = f"{issue['property_name']}"
            if issue["value"]:
                prop_str += f": {issue['value']}"
            client_count = issue["unsupported_count"]
            fallback = " (fallback available)" if issue["fallback_available"] else ""
            detail_parts.append(f"Unsupported: {prop_str} ({client_count} clients){fallback}")

        if len(actionable) > max_details:
            detail_parts.append(f"... and {len(actionable) - max_details} more")

        ontology_score_loss = len(errors) * error_deduction + len(warnings) * warning_deduction

        # ── Pass 2: Syntax validation via rule engine (new) ──
        doc: HtmlElement = lxml_html.document_fromstring(html)
        syntax_issues, syntax_deduction = self._engine.evaluate(doc, html, config)

        # Merge results
        detail_parts.extend(syntax_issues[:max_details])

        total_deduction = ontology_score_loss + syntax_deduction
        score = max(0.0, round(1.0 - total_deduction, 2))
        has_errors = len(errors) > 0 or syntax_deduction >= 0.25
        severity = "error" if has_errors else ("warning" if detail_parts else "info")

        return QACheckResult(
            check_name=self.name,
            passed=not has_errors,
            score=score,
            details="; ".join(detail_parts) if detail_parts else None,
            severity=severity,
        )
```

# CSS Syntax Validation — L3 Reference

## Syntax Errors to Flag

| Pattern | Severity | Example |
|---------|----------|---------|
| Empty value | warning | `color: ;` |
| Missing semicolon in multi-line | info | `color: red\nfont-size: 14px` |
| Unclosed braces | error | `.class { color: red` |
| Invalid property name | warning | `colr: blue;` |
| Unitless numeric (non-zero) | warning | `font-size: 14` (needs `px`) |

## Vendor Prefixes (Flag All in Email)

| Prefix | Engine | Email Relevance |
|--------|--------|-----------------|
| `-webkit-` | Chrome/Safari | Only Apple Mail uses WebKit |
| `-moz-` | Firefox | No email client uses Gecko |
| `-ms-` | IE/Edge Legacy | Outlook uses Word, not IE |
| `-o-` | Opera Legacy | Dead engine |

**Rule:** Any vendor-prefixed property is wasted bytes in email. Exception: `-webkit-text-size-adjust` in Apple Mail.

## External Resources (Always Error)

| Pattern | Why |
|---------|-----|
| `<link rel="stylesheet">` | Stripped by ALL email clients |
| `@import url(...)` | Stripped by Gmail, most webmail |
| `@font-face` with external URL | Only Apple Mail loads web fonts |

## !important Usage Guidelines

- **Expected in dark mode**: `@media (prefers-color-scheme: dark)` requires `!important`
- **Expected for resets**: Body margin/padding reset
- **Flag if**: >10 non-dark-mode `!important` declarations
- **Why problematic**: Email clients inject their own `!important` styles for dark mode; conflicts cause unpredictable colors

## Non-Inlined CSS Risk

Properties that MUST have inline fallbacks for Gmail:

# CSS Email Reference — L3 Quick Lookup

## Safe Properties (Use Freely)

## Implementation Notes

1. **cssutils logging**: cssutils has a notoriously chatty logging system. Silence it at module level with `cssutils.log.setLevel(logging.CRITICAL)`. Use `raiseExceptions=False` in parser to collect errors without raising.

2. **mso- prefix exclusion**: The vendor prefix regex `_VENDOR_PREFIX_RE` matches `-webkit-`, `-moz-`, `-ms-`, `-o-` but NOT `mso-`. The `mso-` prefix is Outlook-specific and intentional — never flag it.

3. **Progressive enhancement pattern**: The existing fallback downgrade logic (ontology errors → warnings when fallback exists) is preserved. New syntax issues are additive and don't interfere with existing scoring.

4. **`css_non_inline_dependency` is heuristic**: This check is intentionally simplified. A perfect implementation would parse CSS selectors and match them to DOM elements, then check if those elements have inline equivalents. The heuristic approach (check if layout-critical properties appear in `<style>` without also appearing inline) catches the most common case (Gmail style stripping) without over-engineering.

5. **Type stubs**: `cssutils` may not have complete type stubs. If pyright/mypy complains, add `# type: ignore[import-untyped]` on the import and create a minimal stub file if needed.

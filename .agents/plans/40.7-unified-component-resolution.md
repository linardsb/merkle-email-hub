### Merge Precedence (seeds.py:1581â€“1597)

```python
def _build_all_seeds() -> list[dict[str, Any]]:
    inline_slugs = {s["slug"] for s in _INLINE_SEEDS}
    file_seeds = load_file_components()
    merged = list(_INLINE_SEEDS)
    for fs in file_seeds:
        if fs["slug"] not in inline_slugs:
            merged.append(fs)
    return merged
```

## Implementation Steps

### 40.7.1 â€” Map converter matcher to file-based components

1. Audit all 24 inline slugs against existing files in `email-templates/components/`:
   - For each inline slug, check if an HTML file with matching `data-slot` attributes exists
   - Categorize: (A) exact file match, (B) file exists but different slug, (C) no file exists
2. For category (C) â€” extract inline HTML to new file:
   - Copy `html_source` from `_INLINE_SEEDS` dict to `email-templates/components/{slug}.html`
   - Ensure `data-slot` attributes are present
   - Add `<!-- Component: {slug} -->` comment tag
3. For category (B) â€” decide: rename file or add manifest alias
   - Prefer adding manifest entry with matcher's slug + `file:` pointing to existing file
   - Example: `slug: hero-block, file: hero-text.html`

### 40.7.2 â€” Add manifest entries with slot definitions

1. For every converter-emitted slug not yet in manifest, add entry to `component_manifest.yaml`
2. Each entry needs: `slug`, `name`, `description`, `category`, `compatibility`, `slot_definitions`
3. Copy `slot_definitions` from corresponding `_INLINE_SEEDS` entry
4. Verify `slot_definitions` match `data-slot` attributes in the HTML file

### 40.7.3 â€” Auto-detect slots from HTML in file_loader

1. Add `_extract_slots_from_html(html_source: str) -> list[dict[str, Any]]` to `file_loader.py`
2. Regex: `r'data-slot="([^"]+)"'` on trusted HTML
3. In `_load_manifest()`, if manifest entry has no `slot_definitions`, call `_extract_slots_from_html()` as fallback
4. Return format: `[{"id": slot_name, "type": "content", "required": False}]`
5. Add tests in `test_file_loader.py`

### 40.7.4 â€” Delete inline seeds, file-based only

1. Remove all 23 non-shell entries from `_INLINE_SEEDS` list in `seeds.py`
2. Keep only `email-shell` (document wrapper)
3. Simplify `_build_all_seeds()`:
   ```python
   def _build_all_seeds() -> list[dict[str, Any]]:
       from app.components.data.file_loader import load_file_components
       shell = [s for s in _INLINE_SEEDS if s["slug"] == "email-shell"]
   ```

4. Update `test_file_loader.py`: change count assertion from 90 to new total
5. Verify `test_component_renderer.py` still finds all 8 expected slugs

## Interactive CSS Architecture (40.7.6 â€” Deferred)

Interactive components (`accordion`, `carousel`, `anim-*`) embed `<style>` blocks in the HTML body. Email clients require all CSS in `<head>`. Current architecture has no mechanism to hoist body CSS to head.

**Proposed approach (future phase):**
1. During `ComponentRenderer.render_section()`, parse `<style>` blocks from component HTML
2. Collect all extracted CSS into a list on the renderer instance
3. After all sections rendered, inject collected CSS into `email-shell`'s `<slot data-slot="head_styles">`
4. Strip `<style>` blocks from component HTML body after extraction

**Dependency:** Requires `email-shell` to have a `head_styles` slot (currently hardcoded). Separate migration phase needed.

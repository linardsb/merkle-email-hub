Plus per-provider tests verifying:
- Each provider has all 6 protocol methods (`validate_credentials`, `list_templates`, `get_template`, `create_template`, `update_template`, `delete_template`)

- `create_template` sends correct payload, returns `ESPTemplate`
- `update_template` sends correct payload

**Route tests (8 endpoints × 3 scenarios each = 24 tests):**

`BASE = "/api/v1/connectors/sync"`

For each endpoint, patch `ConnectorSyncService.<method>` as AsyncMock:

1. **POST /connections → 201** (developer auth)
   - Success: service returns connection response
   - Auth denied: no auth fixture → 401/403
   - Viewer denied: viewer auth → 403 (require_role("developer"))

2. **GET /connections → 200** (viewer auth)
   - Success: returns list
   - No auth → 401

3. **GET /connections/{id} → 200** (viewer auth)
   - Success: returns connection
   - Not found: service raises `ESPConnectionNotFoundError` → 404

4. **DELETE /connections/{id} → 204** (developer auth)
   - Success: returns 204
   - Viewer denied → 403

5. **GET /connections/{id}/templates → 200** (developer auth)
   - Success: returns template list
   - Provider error: service raises `ESPSyncFailedError` → 500

6. **GET /connections/{id}/templates/{tid} → 200** (developer auth)
   - Success: returns single template

7. **POST /connections/{id}/import → 201** (developer auth)
   - Success: returns `{"template_id": N}`
   - Invalid request body → 422

8. **POST /connections/{id}/push → 201** (developer auth)
   - Success: returns ESPTemplate
   - Provider error → 500

# Plan: 11.25.4 — Agent Pipeline Constraint Injection

## Context

The hub serves many clients. Each has their own design system — completely different palettes, fonts, logos, footers. Client A (Nike) has a palette of 6 colors; Client B (Sephora) has 12. The hub must apply ANY client's design system to ANY template with pixel-perfect fidelity. The hub does not design — it receives a design system and faithfully executes it.

Currently the design system (11.25.1) is validation-only — brand compliance catches violations AFTER generation, causing retry loops. This task makes it generative: every email is built within the client's brand identity from the start.

**Critical finding:** Golden templates use hardcoded inline hex colors (`#e84e0f`, `#1a1a1a`, `#444444`), NOT CSS custom properties. The assembler's `_apply_design_tokens()` replaces `var()` patterns that don't exist. Design token application is a no-op. This plan fixes that.

**Depends on:** 11.25.1 (done), 11.25.2 (done), 11.25.3 (plan exists, assumed complete).

## Architecture: Role-Based Token Mapping

```
Client Design System          Template/Component Metadata     Assembler
┌──────────────────┐     ┌──────────────────────────┐     ┌──────────────────┐
│ colors:           │     │ default_tokens:           │     │                  │
│   cta: "#ff0000"  │────▶│   cta: "#e84e0f"          │────▶│ find "#e84e0f"   │
│   heading: "#002" │     │   heading: "#1a1a1a"      │     │ replace "#ff0000"│
│   body: "#333"    │     │   body: "#444444"         │     │                  │
│   ...any roles... │     │   ...template or component│     │ + brand sweep    │
└──────────────────┘     └──────────────────────────┘     └──────────────────┘
```

1. **Templates AND components** declare their built-in defaults keyed by semantic role name
2. **Clients** provide their colors keyed by the same role names
3. **Assembler** collects defaults from both the template AND any composed component sections, then replaces all
4. **Brand sweep** catches any remaining off-palette colors as safety net

This is fully dynamic — any number of roles, any colors, any client. Any component can be used on any client — the assembler applies that client's design system to the component's defaults.

**Component agnosticism:** A shared "hero section" component declares `default_tokens: {cta: "#e84e0f", heading: "#1a1a1a"}`. Client A (Nike) gets Nike colors applied. Client B (Sephora) gets Sephora colors. Same component, different brand. The component doesn't know or care which client it's used for.

## Well-Known Roles (documented convention, not enforced)

Templates and clients use these role names. Not all are required — use what applies.

**Colors:** `primary`, `secondary`, `accent`, `cta`, `cta_text`, `heading`, `body`, `subheadline`, `muted`, `background`, `surface`, `link`, `border`, `dark_background`, `dark_text`

**Fonts:** `heading`, `body`

**Font sizes:** `base`, `heading`, `small`

**Spacing:** `border_radius`, `button_padding`

Clients can add any additional roles — the system ignores roles it doesn't recognise.

## Existing Model Gap

`BrandPalette` has 8 fixed fields. Templates use ~7 distinct color roles. The mapping doesn't align:

| Template Role | BrandPalette Field | Problem |
|---|---|---|
| `cta` (CTA bg) | `primary`? `accent`? | Ambiguous |
| `heading` (#1a1a1a) | `text` | OK but shares with body |
| `body` (#444444) | `text` | Same field, different color |
| `subheadline` (#666666) | — | No field |
| `muted` (#999999) | — | No field |
| `dark_background` | `dark_background` | OK |
| `dark_text` | `dark_text` | OK |

## Model Change: Add Dynamic Token Map

Add a `colors` dict to `DesignSystem` that holds the complete role→hex map. This is the **single source of truth** the pipeline reads. The existing `BrandPalette` stays for backward compat (existing API, frontend editor, brand compliance bridge). The `colors` dict is populated either directly (new clients/imports) or derived from `BrandPalette` fields (existing clients).

```python
    @field_validator("colors", mode="before")
    @classmethod
    def validate_color_values(cls, v: dict[str, str]) -> dict[str, str]:
```

Add conversion function that merges structured fields + dynamic overrides into a flat color map:

```python
def resolve_color_map(ds: DesignSystem) -> dict[str, str]:
    """Build the complete role→hex color map for pipeline use.

    Merges BrandPalette fields (as base roles) with explicit colors dict
    (which can override or add roles). Works for any client:
    - Simple client: only BrandPalette filled → auto-derived roles
    - Advanced client: colors dict with custom roles
    - Figma import: colors dict populated directly from design tool tokens
    """
    # Start with BrandPalette fields mapped to standard roles
    base: dict[str, str] = {}
    for field_name in BrandPalette.model_fields:
        val = getattr(ds.palette, field_name)
        if val is not None:
            base[field_name] = val

    # Auto-derive common aliases from BrandPalette
    # (clients can override any of these via the colors dict)
    if "primary" in base:
        base.setdefault("cta", base["primary"])       # CTA defaults to primary
        base.setdefault("link", base.get("link", base["primary"]))
    if "text" in base:
        base.setdefault("heading", base["text"])       # heading defaults to text
    if "accent" in base:
        base.setdefault("cta", base["accent"])         # accent as CTA fallback

    # Merge explicit colors dict (overrides base)
    base.update(ds.colors)

    return base


def resolve_font_map(ds: DesignSystem) -> dict[str, str]:
    """Build role→font-stack map."""
    base: dict[str, str] = {
        "heading": ds.typography.heading_font,
        "body": ds.typography.body_font,
    }
    base.update(ds.fonts)
    return base


def resolve_font_size_map(ds: DesignSystem) -> dict[str, str]:
    """Build role→font-size map."""
    base: dict[str, str] = {"base": ds.typography.base_size}
    base.update(ds.font_sizes)
    return base


def resolve_spacing_map(ds: DesignSystem) -> dict[str, str]:
    """Build role→spacing map."""
    base: dict[str, str] = {"border_radius": ds.button_border_radius}
    base.update(ds.spacing)
    return base
```

```python
def design_system_to_brand_rules(ds: DesignSystem) -> dict[str, Any]:
    all_colors = resolve_color_map(ds)
    unique_colors = list(dict.fromkeys(all_colors.values()))  # dedupe, preserve order

    font_map = resolve_font_map(ds)
    fonts = []
    for stack in font_map.values():
        primary = stack.split(",")[0].strip().strip("'\"")
        if primary and primary not in fonts:
            fonts.append(primary)

    required_elements: list[str] = []
    if ds.footer:
        required_elements.append("footer")
    if ds.logo:
        required_elements.append("logo")

    return {
        "allowed_colors": unique_colors,
        "required_fonts": fonts,
        "required_elements": required_elements,
    }
```

```python
@staticmethod
def _build_locked_fills(
    ds: DesignSystem, available_slots: set[str]
) -> dict[str, SlotFill]:
    """Build locked fills from whatever the design system provides.

    Client-agnostic: only locks slots that (a) exist in the template
    and (b) the design system has a value for. Skips everything else.
    """
    locked: dict[str, SlotFill] = {}

    def _lock(slot_id: str, content: str) -> None:
        if slot_id in available_slots and content:
            locked[slot_id] = SlotFill(slot_id=slot_id, content=content, is_personalisable=False)

    if ds.footer:
        _lock("footer_company", ds.footer.company_name)
        _lock("footer_legal", ds.footer.legal_text)
        _lock("footer_address", ds.footer.address)
        _lock("footer_unsubscribe", ds.footer.unsubscribe_text)

    if ds.logo:
        _lock("logo_url", ds.logo.url)
        _lock("logo_alt", ds.logo.alt_text)

    return locked
```

The content pass filters locked slots from LLM's task and informs LLM about them for contextual coherence (same approach as previous plan version).

**7d. Pass 3 (Design) — deterministic from design system, no LLM:**

When a design system exists, the pipeline does NOT call the LLM for design tokens. It builds them directly from the client's resolved maps. For clients without a design system, the existing LLM-based pass runs (returning a dynamic dict now).

```python
async def _design_pass(
    self, brief: str, brand_config: dict[str, object] | None = None,
) -> DesignTokens:
    if self._design_system is not None:
        return self._design_pass_from_system(self._design_system)
    # No design system → LLM generates tokens (existing behavior, adapted for dynamic format)
    return await self._design_pass_llm(brief, brand_config)

@staticmethod
def _design_pass_from_system(ds: DesignSystem) -> DesignTokens:
    """Build DesignTokens deterministically from any client's design system.

    Zero LLM calls. All resolved roles are locked — assembly enforces them.
    Works for any design system regardless of size or tool of origin.
    """
    from app.projects.design_system import (
        resolve_color_map,
        resolve_font_map,
        resolve_font_size_map,
        resolve_spacing_map,
    )

    colors = resolve_color_map(ds)
    fonts = resolve_font_map(ds)
    font_sizes = resolve_font_size_map(ds)
    spacing = resolve_spacing_map(ds)

    # All color roles from the design system are locked
    locked_roles = tuple(colors.keys())

    return DesignTokens(
        colors=colors,
        fonts=fonts,
        font_sizes=font_sizes,
        spacing=spacing,
        button_style=ds.button_style,
        source="design_system",
        locked_roles=locked_roles,
    )

async def _design_pass_llm(
    self, brief: str, brand_config: dict[str, object] | None = None,
) -> DesignTokens:
    """LLM-generated design tokens (no design system configured)."""
    brand_context = ""
    if brand_config:
        brand_context = f"\nBrand guidelines:\n{json.dumps(brand_config, default=str)}"

    system = (
        "You are an email design architect. Choose design tokens for the email.\n\n"
        "Return a JSON object with:\n"
        '- colors: object with keys like "primary", "secondary", "background", "text", '
        '"heading", "body", "muted", "cta", "link" — all hex #RRGGBB values\n'
        '- fonts: object with "heading" and "body" as web-safe font stacks\n'
        '- button_style: "filled" | "outlined" | "text"\n\n'
        "Use web-safe hex colors. Font stacks must include system fallbacks."
        f"{brand_context}\n\n"
        "Respond ONLY with valid JSON, no markdown fences."
    )
    user = f"Campaign brief:\n{brief}"

    parsed = await self._call_json(system, user, max_tokens=1024)

    return DesignTokens(
        colors=_extract_str_dict(parsed.get("colors", {})),
        fonts=_extract_str_dict(parsed.get("fonts", {})),
        font_sizes=_extract_str_dict(parsed.get("font_sizes", {})),
        spacing=_extract_str_dict(parsed.get("spacing", {})),
        button_style=parsed.get("button_style", "filled"),
        source="llm_generated",
    )


def _extract_str_dict(raw: Any) -> dict[str, str]:
    """Safely extract dict[str, str] from LLM JSON output."""
    if not isinstance(raw, dict):
        return {}
    return {str(k): str(v) for k, v in raw.items() if isinstance(v, str)}
```

**8b. New `assemble()` flow:**

```python
def assemble(self, plan: EmailBuildPlan) -> str:
    """Assemble final HTML. 100% deterministic. Design-system-agnostic."""
    template = self._resolve_template(plan)

    # Step 1: Fill slots
    fills = {sf.slot_id: sf.content for sf in plan.slot_fills}
    html = self._registry.fill_slots(template, fills)

    # Step 2: Role-based palette replacement (template defaults)
    if template.default_tokens is not None and plan.design_tokens.colors:
        html = self._apply_palette_replacement(
            html, template.default_tokens, plan.design_tokens,
        )

    # Step 2b: Role-based palette replacement (composed component defaults)
    # When the template is composed from SectionBlocks, each component
    # may have its own default_tokens. Apply replacement for each.
    if plan.design_tokens.colors:
        html = self._apply_component_palette_replacement(
            html, template, plan.design_tokens,
        )

    # Step 3: Font replacement
    if template.default_tokens is not None and plan.design_tokens.fonts:
        html = self._apply_font_replacement(
            html, template.default_tokens, plan.design_tokens,
        )

    # Step 4: Logo dimension enforcement
    if self._design_system is not None and self._design_system.logo:
        html = self._enforce_logo_dimensions(html, self._design_system.logo)

    # Step 5: Social link injection
    if self._design_system is not None and self._design_system.social_links:
        html = self._inject_social_links(html, self._design_system.social_links)

    # Step 6: Dark mode replacement
    if template.default_tokens is not None and plan.design_tokens.colors:
        html = self._apply_dark_mode_replacement(
            html, template.default_tokens, plan.design_tokens,
        )

    # Step 7: Brand color sweep (safety net — catches off-palette colors in content slots)
    if plan.design_tokens.source == "design_system" and plan.design_tokens.colors:
        html = self._brand_color_sweep(html, plan.design_tokens.colors)

    # Step 8: Apply section decisions
    for section in plan.sections:
        html = self._apply_section(html, section)

    # Step 9: Set preheader
    if plan.preheader_text:
        html = self._set_preheader(html, plan.preheader_text)

    logger.info(
        "scaffolder.assembly_completed",
        template=plan.template.template_name,
        slots_filled=len(fills),
        html_length=len(html),
        design_source=plan.design_tokens.source,
        color_roles=len(plan.design_tokens.colors),
    )
    return html
```

**8c. `_apply_palette_replacement()` — the core dynamic replacement:**

```python
def _apply_palette_replacement(
    self,
    html: str,
    defaults: DefaultTokens,
    tokens: DesignTokens,
) -> str:
    """Replace template default colors with client colors, matched by role.

    For each role in the template's default_tokens.colors:
    1. Look up the default hex (what's in the template HTML)
    2. Look up the client's hex for the same role
    3. Global find-replace (case-insensitive)

    Works for any template × any design system combination.
    """
    for role, default_hex in defaults.colors.items():
        # Skip dark mode roles (handled separately)
        if role.startswith("dark_"):
            continue
        client_hex = tokens.colors.get(role)
        if client_hex is None or client_hex.lower() == default_hex.lower():
            continue
        html = re.sub(re.escape(default_hex), client_hex, html, flags=re.IGNORECASE)
    return html
```

**8d. `_apply_component_palette_replacement()` — handle composed component sections:**

```python
def _apply_component_palette_replacement(
    self,
    html: str,
    template: GoldenTemplate,
    tokens: DesignTokens,
) -> str:
    """Apply palette replacement for each composed SectionBlock's default_tokens.

    When a template is composed from SectionBlocks (via TemplateComposer),
    each section may have its own default_tokens (from the component library).
    A shared component uses its own default colors — not the golden template's.
    This method applies the client's colors to each component's defaults.

    For non-composed templates (regular golden templates), this is a no-op.
    """
    if not hasattr(template, "composed_sections") or not template.composed_sections:
        return html

    for section in template.composed_sections:
        if section.default_tokens is not None:
            html = self._apply_palette_replacement(
                html, section.default_tokens, tokens,
            )
    return html
```

```python
def _apply_font_replacement(
    self,
    html: str,
    defaults: DefaultTokens,
    tokens: DesignTokens,
) -> str:
    """Replace template default font stacks with client fonts, matched by role."""
    for role, default_stack in defaults.fonts.items():
        client_stack = tokens.fonts.get(role)
        if client_stack is None or client_stack == default_stack:
            continue
        html = html.replace(default_stack, client_stack)

    # Replace base font size if different
    default_base = defaults.font_sizes.get("base", "16px")
    client_base = tokens.font_sizes.get("base")
    if client_base and client_base != default_base:
        # Only replace in the root element's font-size declaration
        html = html.replace(
            f"font-size: {default_base}",
            f"font-size: {client_base}",
            1,  # Only first occurrence (root element)
        )

    # Replace border-radius
    default_radius = defaults.spacing.get("border_radius", "4px")
    client_radius = tokens.spacing.get("border_radius")
    if client_radius and client_radius != default_radius:
        html = html.replace(
            f"border-radius: {default_radius}",
            f"border-radius: {client_radius}",
        )

    return html
```

```python
def _enforce_logo_dimensions(self, html: str, logo: "LogoConfig") -> str:
    """Set width/height attributes on logo img tags."""
    logo_pattern = re.compile(
        r'(<img\b[^>]*data-slot=["\'](?:logo_url|hero_logo|logo)["\'][^>]*?)(/?>)',
        re.IGNORECASE,
    )
    def _enforce(m: re.Match[str]) -> str:
        tag = m.group(1)
        close = m.group(2)
        tag = re.sub(r'\bwidth=["\']\d+["\']', f'width="{logo.width}"', tag)
        if "width=" not in tag:
            tag += f' width="{logo.width}"'
        tag = re.sub(r'\bheight=["\']\d+["\']', f'height="{logo.height}"', tag)
        if "height=" not in tag:
            tag += f' height="{logo.height}"'
        return tag + close

    return logo_pattern.sub(_enforce, html)
```

```python
def _inject_social_links(self, html: str, social_links: "tuple[SocialLink, ...]") -> str:
    """Inject social links into social_links slot if present."""
    if not social_links:
        return html

    parts: list[str] = []
    for link in social_links:
        if link.icon_url:
            parts.append(
                f'<a href="{link.url}" style="display:inline-block;margin:0 8px;">'
                f'<img src="{link.icon_url}" alt="{link.platform}" width="24" height="24" '
                f'style="display:block;border:0;"></a>'
            )
        else:
            parts.append(
                f'<a href="{link.url}" style="display:inline-block;margin:0 8px;'
                f'font-size:12px;text-decoration:none;">{link.platform.title()}</a>'
            )

    social_html = '<div style="text-align:center;padding:16px 0;">' + "".join(parts) + "</div>"

    # Inject at social_links slot if it exists
    if 'data-slot="social_links"' in html or "data-slot='social_links'" in html:
        pattern = re.compile(
            r"(<(\w+)\b[^>]*\bdata-slot=[\"']social_links[\"'][^>]*>)(.*?)(</\2>)",
            re.DOTALL,
        )
        return pattern.sub(lambda m: m.group(1) + social_html + m.group(4), html, count=1)

    return html  # No slot → no injection (template doesn't support social links)
```

**8h. `_brand_color_sweep()` — safety net:**

```python
_HEX_IN_STYLE_RE = re.compile(
    r"((?:color|background-color|background|border-color)\s*:\s*)(#[0-9a-fA-F]{6})",
)

def _brand_color_sweep(self, html: str, client_colors: dict[str, str]) -> str:
    """Final safety net: replace off-palette colors with nearest palette match.

    Scans inline CSS color properties. Any hex NOT in the client's palette
    → replaced with nearest match (Euclidean RGB distance). Catches off-brand
    colors from LLM-generated content slots.
    """
    allowlist: set[str] = {c.lower() for c in client_colors.values()}
    allowlist.update({"#ffffff", "#000000"})  # Structural colors always allowed

    palette_rgb = {c: _hex_to_rgb(c) for c in allowlist}

    def _replace(m: re.Match[str]) -> str:
        prefix, color = m.group(1), m.group(2).lower()
        if color in allowlist:
            return prefix + color
        return prefix + _nearest_color(color, palette_rgb)

    return _HEX_IN_STYLE_RE.sub(_replace, html)


def _hex_to_rgb(h: str) -> tuple[int, int, int]:
    h = h.lstrip("#")
    return (int(h[0:2], 16), int(h[2:4], 16), int(h[4:6], 16))


def _nearest_color(target: str, palette: dict[str, tuple[int, int, int]]) -> str:
    t = _hex_to_rgb(target)
    best, best_d = target, float("inf")
    for hex_c, rgb in palette.items():
        d = sum((a - b) ** 2 for a, b in zip(t, rgb))
        if d < best_d:
            best, best_d = hex_c, d
    return best
```

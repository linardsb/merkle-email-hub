# Plan: 33.4 Spacing Token Pipeline & Auto-Layout → Table Mapping

## Context

Figma auto-layout spacing is extracted (`DesignNode.item_spacing`, `counter_axis_spacing`, `padding_*`) and spacing tokens are validated (`token_transforms._validate_spacing()`), but **none of it reaches the HTML output or Scaffolder**:

1. `node_to_email_html()` applies padding from `_NodeProps` on the `<table>` element — but padding on `<table>` is unreliable across email clients. Must move to `<td>` cells.
2. `_group_into_rows()` uses y-position proximity only — ignores `layout_mode` (HORIZONTAL/VERTICAL) entirely.
3. `item_spacing` and `counter_axis_spacing` from `DesignNode` are never consumed — no spacer rows or cell padding generated.
4. `_build_design_context()` in `import_service.py` omits spacing tokens — Scaffolder never sees them.
5. No `convert_spacing()` function exists to normalize spacing tokens into a scale.

**Knowledge rules encoded (not runtime lookups):**
- Padding works ONLY on `<td>` cells (Outlook ignores padding on `<table>`/`<p>`/`<div>`/`<a>`)
- Spacer rows MUST include `font-size:1px;line-height:1px;mso-line-height-rule:exactly` (prevents Outlook expanding to 18px default)
- Layout tables MUST have `cellpadding="0" cellspacing="0" border="0" role="presentation"`
- Centering uses `align="center"` HTML attribute, never `margin:0 auto`
- Max nesting depth 6 (already enforced at `converter.py:394`)

### Step 2: Create `convert_spacing()` (`converter.py`, after `convert_typography` ~line 233)

```python
# Standard spacing scale — values that align to 4px/8px multiples get named
_SPACING_SCALE: dict[int, str] = {
    4: "2xs",
    8: "xs",
    12: "sm",
    16: "md",
    20: "md-lg",
    24: "lg",
    32: "xl",
    40: "xl-2",
    48: "2xl",
    64: "3xl",
}


def convert_spacing(spacing: list[ExtractedSpacing]) -> dict[str, float]:
    """Convert extracted spacing tokens to a named spacing scale.

    Maps numeric spacing values to semantic names. Values that align to
    the standard 4px/8px scale get standard names (xs, sm, md, lg, xl).
    Non-standard values keep their original name.

    Returns:
        Mapping of semantic name → pixel value.
    """
    result: dict[str, float] = {}
    for token in spacing:
        px = token.value
        int_px = int(px)
        # Use standard scale name if value matches
        if int_px in _SPACING_SCALE:
            name = _SPACING_SCALE[int_px]
        else:
            # Normalize name: strip "spacing-" prefix, lowercase
            name = token.name.lower().removeprefix("spacing-").removeprefix("space-")
            if not name:
                name = f"{int_px}px"
        # Avoid overwriting a scale entry with a different value
        if name not in result:
            result[name] = px
    return result
```

Import `ExtractedSpacing` from `app.design_sync.protocol` (already imported for `ExtractedColor`, `ExtractedTypography`).

        layout_dir = props.layout_direction if props else None
        # Also check DesignNode directly (takes priority)

**3c. Wrap children in a padding `<td>` and insert spacer rows.**

Replace the current row-rendering loop (lines 429–460) with:

```python
        if not node.children:
            lines.append(f"{pad}  <tr><td>&nbsp;</td></tr>")
        else:
            # Wrap all rows in a padding <td> if the node has padding
            if has_padding:
                lines.append(
                    f'{pad}  <tr><td style="{padding_css}">'
                )
                lines.append(
                    f'{pad}    <table width="100%" cellpadding="0" cellspacing="0"'
                    f' border="0" role="presentation"'
                    f' style="border-collapse:collapse;'
                    f'mso-table-lspace:0pt;mso-table-rspace:0pt;">'
                )

            for row_idx, row in enumerate(rows):
                # Insert vertical spacer between rows (not before first)
                if row_idx > 0 and gap > 0 and layout_dir in ("column", None):
                    gap_px = int(gap)
                    lines.append(
                        f'{pad}    <tr><td style="height:{gap_px}px;'
                        f"font-size:1px;line-height:1px;"
                        f'mso-line-height-rule:exactly;" '
                        f'aria-hidden="true">&nbsp;</td></tr>'
                    )

                lines.append(f"{pad}    <tr>")
                for cell_idx, child in enumerate(row):
                    child_html = node_to_email_html(
                        child,
                        indent=indent + (4 if has_padding else 2),
                        props_map=props_map,
                        parent_bg=effective_bg,
                        parent_font=effective_font,
                        section_map=section_map,
                        button_ids=button_ids,
                        text_meta=text_meta,
                        current_section=section,
                        body_font_size=body_font_size,
                        compat=compat,
                        _depth=_depth + 1,
                    )

                    if child.type != DesignNodeType.TEXT:
                        # Build <td> style
                        td_styles: list[str] = []
                        if effective_font:
                            td_styles.append(f"font-family:{effective_font}")
                        # Horizontal gap: padding-left on all cells except first
                        if layout_dir == "row" and cell_idx > 0 and gap > 0:
                            td_styles.append(f"padding-left:{int(gap)}px")
                        # Cross-axis padding
                        if cross_gap > 0:
                            if layout_dir == "row":
                                td_styles.append(f"padding-top:{int(cross_gap)}px")
                            elif layout_dir == "column":
                                td_styles.append(f"padding-left:{int(cross_gap)}px")
                        td_style = f' style="{";".join(td_styles)}"' if td_styles else ""
                        lines.append(f"{pad}      <td{td_style}>")
                        lines.append(child_html)
                        lines.append(f"{pad}      </td>")
                    else:
                        lines.append(child_html)
                lines.append(f"{pad}    </tr>")

            if has_padding:
                lines.append(f"{pad}    </table>")
                lines.append(f"{pad}  </td></tr>")
```

**Key design decisions:**
- Padding wraps as `<tr><td style="padding:..."><table>...</table></td></tr>` — the outer `<td>` carries padding (Outlook-safe), inner table carries children.
- Vertical spacers use the established pattern from inter-section spacers (`converter_service.py:160–164`) — `font-size:1px;line-height:1px;mso-line-height-rule:exactly`.
- Horizontal gaps use `padding-left` on `<td>` cells (not first child).
- Cross-axis spacing applied as padding on the perpendicular axis.
- Falls back to `_group_into_rows()` when no `layout_mode` — preserves existing behavior.

class TestConvertSpacing:
    """Tests for convert_spacing() scale mapping."""

    def test_standard_scale_values(self) -> None:
        tokens = [
            ExtractedSpacing(name="spacing-8", value=8),
            ExtractedSpacing(name="spacing-16", value=16),
            ExtractedSpacing(name="spacing-32", value=32),

        result = convert_spacing(tokens)

    def test_non_standard_keeps_original_name(self) -> None:
        tokens = [ExtractedSpacing(name="spacing-10", value=10)]

        tokens = [

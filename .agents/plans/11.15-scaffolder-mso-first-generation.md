# Plan: 11.15 Scaffolder Agent — MSO-First Generation

## Context

The Scaffolder agent currently generates email HTML that often fails MSO/Outlook validation. The Outlook Fixer agent then has to repair it downstream in the blueprint pipeline. Task 11.15 shifts the Scaffolder to an **MSO-first generation** strategy: always load MSO patterns, run post-generation MSO validation, and emit structured warnings — so the Scaffolder produces Outlook-correct HTML from the start, reducing recovery router trips.

### Current gaps:
1. `mso_vml_quick_ref` skill only loads when brief mentions "outlook"/"mso"/"vml"/"button" etc. — most briefs don't mention these, so the LLM generates without MSO guidance
2. `css_email_reference` is listed in SKILL.md `references` but missing from `SKILL_FILES` dict — never loaded
3. No post-generation MSO validation — the agent can't self-correct MSO issues
4. Blueprint node doesn't emit MSO-specific warnings in handoff — recovery router gets no signal until full QA runs
5. SKILL.md has MSO rules but they're brief; more patterns from Outlook Fixer skills would help

## Files to Create/Modify

1. `app/ai/agents/scaffolder/prompt.py` — Always load MSO skill, add `css_email_reference` to SKILL_FILES
2. `app/ai/agents/scaffolder/service.py` — Add post-generation MSO validation, emit `mso_warnings` in response

### Step 3: Add post-generation MSO validation to ScaffolderService

**File:** `app/ai/agents/scaffolder/service.py`

Override `_post_process` to run MSO validation after HTML extraction, storing warnings for the response:

```python
from app.ai.agents.base import BaseAgentService
from app.ai.agents.scaffolder.prompt import (
    build_system_prompt as _build_system_prompt,
    detect_relevant_skills as _detect_relevant_skills,
)
from app.ai.agents.scaffolder.schemas import ScaffolderRequest, ScaffolderResponse
from app.ai.shared import extract_html, sanitize_html_xss
from app.core.logging import get_logger
from app.qa_engine.mso_parser import validate_mso_conditionals
from app.qa_engine.schemas import QACheckResult

logger = get_logger(__name__)


class ScaffolderService(BaseAgentService):
    agent_name = "scaffolder"
    model_tier = "complex"
    stream_prefix = "scaffold"

    def __init__(self) -> None:
        super().__init__()
        self._mso_warnings: list[str] = []

    def build_system_prompt(self, relevant_skills: list[str]) -> str:
        return _build_system_prompt(relevant_skills)

    def detect_relevant_skills(self, request: Any) -> list[str]:
        req: ScaffolderRequest = request
        return _detect_relevant_skills(req.brief)

    def _build_user_message(self, request: Any) -> str:
        req: ScaffolderRequest = request
        return req.brief

    def _post_process(self, raw_content: str) -> str:
        """Extract HTML, run MSO validation, sanitize XSS."""
        html = extract_html(raw_content)
        html = sanitize_html_xss(html)

        # MSO-first: validate generated HTML for Outlook issues
        mso_result = validate_mso_conditionals(html)
        self._mso_warnings = [
            f"[{issue.severity}] {issue.category}: {issue.message}"
            for issue in mso_result.issues
        ]

        if self._mso_warnings:
            logger.warning(
                "agents.scaffolder.mso_validation_issues",
                issue_count=len(self._mso_warnings),
                categories=list({i.category for i in mso_result.issues}),
            )

        return html

    def _build_response(
        self,
        *,
        request: Any,
        html: str,
        qa_results: list[QACheckResult] | None,
        qa_passed: bool | None,
        model_id: str,
        confidence: float | None,
        skills_loaded: list[str],
        raw_content: str,
    ) -> ScaffolderResponse:
        warnings = self._mso_warnings
        self._mso_warnings = []  # Reset for next request
        return ScaffolderResponse(
            html=html,
            qa_results=qa_results,
            qa_passed=qa_passed,
            model=model_id,
            confidence=confidence,
            skills_loaded=skills_loaded,
            mso_warnings=warnings,
        )

    # Backward-compat aliases
    async def generate(self, request: ScaffolderRequest) -> ScaffolderResponse:
        return await self.process(request)

    async def stream_generate(self, request: ScaffolderRequest) -> AsyncIterator[str]:
        async for chunk in self.stream_process(request):
            yield chunk
```

**Important:** The `_mso_warnings` instance variable is set during `_post_process()` and read during `_build_response()`. Both are called sequentially in `base.py:process()`, so this is safe. Reset after read to avoid stale state.

### Step 5: Expand `mso_vml_quick_ref.md` with common bug fixes

**File:** `app/ai/agents/scaffolder/skills/mso_vml_quick_ref.md`

Append the top 5 most common Outlook bugs that the Scaffolder should avoid during generation (extracted from the Outlook Fixer's `mso_bug_fixes.md`). Add this after the existing content:

```markdown
## Common Outlook Bugs to Avoid During Generation

### Bug: Font Fallback (Outlook uses Times New Roman)
Outlook ignores `font-family` on `<div>`, `<p>`, `<span>`. Always set font on `<td>`:
```html
<td style="font-family:Arial,Helvetica,sans-serif; font-size:16px; line-height:24px; color:#333333;">
  <p style="margin:0; font-family:Arial,Helvetica,sans-serif;">Your text here</p>
</td>
```

### Bug: Line-Height Inconsistency
Outlook has its own line-height calculation. Fix with MSO-specific property:
```css
mso-line-height-rule: exactly;
line-height: 24px;
```

### Bug: Table Gaps (1px White Lines)
Outlook inserts gaps between table rows. Prevent with:
```html
<table role="presentation" cellpadding="0" cellspacing="0" border="0"
  style="border-collapse:collapse; mso-table-lspace:0pt; mso-table-rspace:0pt;">
```

### Bug: Image DPI Scaling
Outlook on high-DPI displays scales images. Always set both HTML `width` attribute AND CSS `width`:
```html
<img src="image.jpg" width="600" height="200" alt="Hero"
  style="display:block; width:600px; height:auto; border:0; -ms-interpolation-mode:bicubic;">
```

### Bug: Padding Not Supported on Some Elements
Outlook ignores `padding` on `<p>`, `<div>`, `<a>`. Use `padding` only on `<td>`:
```html
<!-- WRONG: padding on <a> -->
<a style="padding:12px 24px;">CTA</a>

<!-- RIGHT: padding on wrapping <td> -->
<td style="padding:12px 24px;">
  <a style="color:#ffffff; text-decoration:none;">CTA</a>
</td>
```
```

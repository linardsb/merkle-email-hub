## Implementation Steps

### Step 1: Create `app/design_sync/quality_contracts.py`

Define a lightweight warning dataclass and 3 pure check functions:

```python
@dataclass(frozen=True)
class QualityWarning:
    """Single quality contract violation."""
    category: Literal["contrast", "completeness", "placeholder"]
    severity: Literal["error", "warning", "info"]
    message: str
    context: dict[str, str | int | float] = field(default_factory=dict)
```

**1a. `check_contrast(html: str) -> list[QualityWarning]`**
- Parse HTML with `lxml.html.document_fromstring()`
- For each element with `style` containing `color:` — extract text color
- Walk up ancestors for `background-color:` or `background:` on `<td>` cells
- Compute ratio via `_relative_luminance()` + `_contrast_ratio()` (import from `converter.py`)
- Warn if ratio < 4.5:1 (normal text) or < 3.0:1 (large text ≥18px or bold ≥14px)
- `context`: `{"text_color": "#333", "bg_color": "#FE5117", "ratio": 1.8, "element": "td.hero"}`

**1b. `check_completeness(html: str, input_section_count: int, input_button_count: int) -> list[QualityWarning]`**
- Count `<!-- section: ... -->` markers in HTML (from `inject_section_markers()`)
- Compare to `input_section_count` — warn if fewer
- Count `<a` tags with `role="button"` or class containing "button" or `<!--[if mso]>` button patterns
- Compare to `input_button_count` — warn if fewer
- `context`: `{"expected": 12, "found": 11, "type": "section"}`

**1c. `check_placeholders(html: str) -> list[QualityWarning]`**
- Extract visible text from parsed HTML (`.text_content()` per element)
- Run `_PLACEHOLDER_PATTERNS` regex (import from `component_matcher.py`)
- One warning per match with the matched text snippet
- `context`: `{"text": "Image caption — describe...", "pattern": "image caption"}`

**1d. Orchestrator:**
```python
def run_quality_contracts(
    html: str,
    input_section_count: int = 0,
    input_button_count: int = 0,
) -> list[QualityWarning]:
    """Run all post-conversion quality contracts. Pure, sync, no I/O."""
    if not html:
        return []
    warnings: list[QualityWarning] = []
    warnings.extend(check_contrast(html))
    warnings.extend(check_completeness(html, input_section_count, input_button_count))
    warnings.extend(check_placeholders(html))
    return warnings
```

## Design Decisions

1. **Separate module, not qa_engine checks.** Quality contracts are converter-internal (run synchronously inline, no async, no config YAML). QA engine checks are post-build validation triggered by API. Different lifecycle.

2. **`QualityWarning` dataclass, not `QACheckResult`.** QACheckResult is a Pydantic model with `score: float` — quality contracts are binary (pass/warn) with richer context dicts. Lighter weight for inline use.

3. **No config toggle.** Checks are pure, fast, zero I/O. Always run. No reason to disable — they only produce warnings, never block conversion.

4. **Sync, not async.** All 3 checks are CPU-only HTML parsing. No benefit from async. Keeps integration simple (no `await` at return sites).

5. **Import existing utilities.** Reuse `_relative_luminance`, `_contrast_ratio` from `converter.py` and `_PLACEHOLDER_PATTERNS` from `component_matcher.py`. Don't duplicate.

## Preflight Warnings

- `ConversionResult` is frozen — adding fields is safe (default factory), but ensure all existing `replace()` calls (if any) are updated.
- `converter.py` functions `_relative_luminance`, `_contrast_ratio` are module-private (underscore prefix). Need to either make public or re-export.
- Inline `style` parsing in `check_contrast` must handle shorthand `background` (not just `background-color`) and `rgb()`/`rgba()` formats.

## Security Checklist

- No new endpoints — read-only analysis of converter output
- No user input paths — operates on internally generated HTML
- No SQL, no subprocess, no eval
- No secrets or credentials
- lxml parsing: already used throughout codebase, safe for trusted HTML

### Design Token System (`cms/packages/ui/src/tokens.css`)

3-tier token architecture fully implemented:
- **Tier 1 — Primitives:** oklch color palette (neutral, primary, success, warning, danger)
- **Tier 2 — Semantic:** role-based tokens (surface, foreground, border, interactive, status)
- **Tier 3 — Component:** scoped tokens (sidebar, card, input, badge)

shadcn/ui aliases map shadcn-expected variable names to semantic tokens:
- `--color-background` → `var(--color-surface)`
- `--color-primary` → `var(--color-interactive)`
- `--color-destructive` → `var(--color-status-danger)`
- etc.

### Dark Mode

- `.dark` class overrides Tier 2 semantic tokens in `tokens.css`
- Tier 3 + shadcn aliases auto-resolve via `var()` chains (no duplication)
- `ThemeProvider` wraps app (next-themes, attribute="class", enableSystem)
- `ThemeToggle` component in sidebar footer

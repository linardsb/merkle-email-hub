# Plan: Task 0.4 Completion — API Client Error Handling + Domain Hooks

## Context

Task 0.4 is partially done. What exists:
- `authFetch()` — JWT injection for server + client (`src/lib/auth-fetch.ts`)
- `swr-fetcher.ts` — basic SWR fetcher, throws generic `Error` on non-ok
- `api-error.ts` — `ApiError` class with status/code/helpers (unused by fetcher)
- `sdk.ts` — SDK client with auth interceptor (server-side only)
- `use-projects.ts`, `use-orgs.ts` — SWR hooks for projects/orgs

What's missing (from TODO 0.4 requirements):
1. **SWR fetcher doesn't parse backend error shape** — throws generic `Error`, not `ApiError` with backend's `{error, type, detail}` body
2. **No 401 handling** — expired tokens don't redirect to login
3. **No 429 rate limit retry** — rate-limited requests just fail
4. **No request timeouts** — no AbortController timeout (spec: 30s default, 120s for AI/build)
5. **Missing domain hooks** — only projects/orgs exist, need components, email engine, QA, personas, approvals for Sprint 1+

**Backend error response shape** (from `app/core/exceptions.py` + `app/shared/schemas.py`):
```json
{ "error": "Human-readable message", "type": "NotFoundError", "detail": "optional" }
```

**No backend changes needed.** This plan is 100% frontend.

## Design Decisions

1. **Enhance `swr-fetcher.ts`** to parse backend error JSON → `ApiError`. This gives all existing hooks structured errors automatically.
2. **Add SDK response interceptor in `sdk.ts`** for 401 → redirect to login (client-side only). The SDK interceptor handles API calls made outside SWR too.
3. **Add `authFetch` timeout support** via `AbortController`. Default 30s. Callers can override.
4. **429 retry** in SDK response interceptor — respect `Retry-After` header, max 2 retries.
5. **New hooks use same pattern** as existing `use-projects.ts` — `useSWR` + `fetcher` + URL key + SDK types.
6. **Mutation hooks** (build, preview, QA run, export) use `useSWRMutation` from `swr/mutation` for POST operations.

## Implementation Steps

### Step 1: Enhance `auth-fetch.ts` with timeout support

Add timeout parameter to `authFetch`. Default 30s. Uses `AbortController`.

```typescript
// cms/apps/web/src/lib/auth-fetch.ts

const DEFAULT_TIMEOUT_MS = 30_000;
const LONG_TIMEOUT_MS = 120_000;

export async function authFetch(
  input: RequestInfo | URL,
  init?: RequestInit & { timeoutMs?: number }
): Promise<Response> {
  const token = await getAccessToken();
  const headers = new Headers(init?.headers);
  if (token) {
    headers.set("Authorization", `Bearer ${token}`);
  }

  const timeoutMs = init?.timeoutMs ?? DEFAULT_TIMEOUT_MS;
  const controller = new AbortController();
  const timeoutId = setTimeout(() => controller.abort(), timeoutMs);

  try {
    return await fetch(input, {
      ...init,
      headers,
      signal: init?.signal ?? controller.signal,
    });
  } finally {
    clearTimeout(timeoutId);
  }
}

export { LONG_TIMEOUT_MS };
```

Keep the rest of `auth-fetch.ts` unchanged (getAccessToken, clearTokenCache).

### Step 2: Enhance `swr-fetcher.ts` with structured error parsing

Parse backend JSON error body into `ApiError`. Handle non-JSON errors gracefully.

```typescript
// cms/apps/web/src/lib/swr-fetcher.ts

import { authFetch } from "./auth-fetch";
import { ApiError } from "./api-error";

export async function fetcher<T>(url: string): Promise<T> {
  const res = await authFetch(url);

  if (!res.ok) {
    let message = "Request failed";
    let code: string | undefined;

    try {
      const body = await res.json();
      if (body.error) message = body.error;
      if (body.type) code = body.type;
    } catch {
      // Non-JSON error response — use status text
      message = res.statusText || message;
    }

    throw new ApiError(res.status, message, code);
  }

  return res.json();
}
```

### Step 3: Add SDK interceptors for 401 redirect + 429 retry

Add response interceptors to the SDK client in `sdk.ts`.

```typescript
// cms/apps/web/src/lib/sdk.ts

import { client } from "@email-hub/sdk";
import { authFetch } from "./auth-fetch";

// Configure the SDK client
client.setConfig({
  baseUrl: process.env.NEXT_PUBLIC_API_URL || "/api/proxy",
});

// Auth interceptor (existing — server-side token injection)
client.interceptors.request.use(async (request) => {
  if (typeof window === "undefined") {
    try {
      const { auth } = await import("@/auth");
      const session = await auth();
      if ((session as any)?.accessToken) {
        request.headers.set(
          "Authorization",
          `Bearer ${(session as any).accessToken}`
        );
      }
    } catch {
      // Auth not available
    }
  }
  return request;
});

// 401 response interceptor — redirect to login on expired session (client-side only)
client.interceptors.response.use(async (response) => {
  if (response.status === 401 && typeof window !== "undefined") {
    const { signOut } = await import("next-auth/react");
    await signOut({ callbackUrl: "/login" });
  }
  return response;
});

// 429 rate limit retry interceptor
client.interceptors.response.use(async (response, request) => {
  if (response.status === 429) {
    const retryAfter = response.headers.get("Retry-After");
    const delayMs = retryAfter ? parseInt(retryAfter, 10) * 1000 : 2000;

    // Only retry once to avoid infinite loops
    const retryCount = (request as any).__retryCount ?? 0;
    if (retryCount < 1) {
      await new Promise((resolve) => setTimeout(resolve, Math.min(delayMs, 10_000)));
      (request as any).__retryCount = retryCount + 1;
      // Re-fetch with the original request
      return fetch(request);
    }
  }
  return response;
});

export { client };
export { authFetch };
```

**Note:** Need to verify `@hey-api/client-fetch` interceptor signature. The response interceptor receives `(response, request)`. If the API differs, adjust accordingly. Check the package docs before implementing.

"use client";

import useSWR from "swr";
import { fetcher } from "@/lib/swr-fetcher";
import type {
  PaginatedResponseComponentResponse,
  ComponentResponse,
  ComponentVersionResponse,
} from "@email-hub/sdk";

interface UseComponentsOptions {
  page?: number;
  pageSize?: number;
  category?: string;
  search?: string;
}

export function useComponents(options: UseComponentsOptions = {}) {
  const { page = 1, pageSize = 20, category, search } = options;

  const params = new URLSearchParams({
    page: String(page),
    page_size: String(pageSize),
  });
  if (category) params.set("category", category);
  if (search) params.set("search", search);

  return useSWR<PaginatedResponseComponentResponse>(
    `/api/v1/components/?${params.toString()}`,
    fetcher
  );
}

export function useComponent(componentId: number | null) {
  return useSWR<ComponentResponse>(
    componentId ? `/api/v1/components/${componentId}` : null,
    fetcher
  );
}

export function useComponentVersions(componentId: number | null) {
  return useSWR<ComponentVersionResponse[]>(
    componentId ? `/api/v1/components/${componentId}/versions` : null,
    fetcher
  );
}

// cms/apps/web/src/hooks/use-email.ts
"use client";

import useSWRMutation from "swr/mutation";
import { authFetch, LONG_TIMEOUT_MS } from "@/lib/auth-fetch";
import { ApiError } from "@/lib/api-error";
import type { BuildRequest, BuildResponse, PreviewResponse } from "@email-hub/sdk";

type PreviewRequest = Pick<BuildRequest, "source_html" | "project_id" | "template_name">;

async function postJson<T>(url: string, { arg }: { arg: unknown }): Promise<T> {
  const res = await authFetch(url, {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify(arg),
    timeoutMs: LONG_TIMEOUT_MS,
  });

  if (!res.ok) {
    let message = "Request failed";
    let code: string | undefined;
    try {
      const body = await res.json();
      if (body.error) message = body.error;
      if (body.type) code = body.type;
    } catch {}
    throw new ApiError(res.status, message, code);
  }

  return res.json();
}

export function useEmailBuild() {
  return useSWRMutation<BuildResponse, ApiError, string, BuildRequest>(
    "/api/v1/email/build",
    postJson
  );
}

export function useEmailPreview() {
  return useSWRMutation<PreviewResponse, ApiError, string, PreviewRequest>(
    "/api/v1/email/preview",
    postJson
  );
}

### Step 11: Extract shared `postJson` helper

The `postJson` mutation fetcher is duplicated across hooks. Extract to shared util:

```typescript
// cms/apps/web/src/lib/mutation-fetcher.ts
import { authFetch } from "./auth-fetch";
import { ApiError } from "./api-error";

/**
 * Generic POST mutation fetcher for useSWRMutation.
 * Usage: useSWRMutation("/api/v1/foo", mutationFetcher)
 */
export async function mutationFetcher<T>(
  url: string,
  { arg }: { arg: unknown }
): Promise<T> {
  const res = await authFetch(url, {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify(arg),
  });

  if (!res.ok) {
    let message = "Request failed";
    let code: string | undefined;
    try {
      const body = await res.json();
      if (body.error) message = body.error;
      if (body.type) code = body.type;
    } catch {}
    throw new ApiError(res.status, message, code);
  }

  return res.json();
}

/**
 * Same as mutationFetcher but with 120s timeout for long-running operations.
 */
export async function longMutationFetcher<T>(
  url: string,
  { arg }: { arg: unknown }
): Promise<T> {
  const res = await authFetch(url, {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify(arg),
    timeoutMs: 120_000,
  });

  if (!res.ok) {
    let message = "Request failed";
    let code: string | undefined;
    try {
      const body = await res.json();
      if (body.error) message = body.error;
      if (body.type) code = body.type;
    } catch {}
    throw new ApiError(res.status, message, code);
  }

  return res.json();
}
```

# Plan: 16.4 Template/Component Retrieval

## Context
When the query router classifies a query as `template` intent (16.1), the current `_search_components` stub just redirects to the knowledge base with `domain="best_practices"`. This means agents asking "Show me a CTA button component" search text chunks instead of the actual `Component` table which has tested, QA'd HTML with per-client compatibility data. This feature connects the retrieval pipeline to the existing component library.

## Decisions
- **SearchResult compatibility**: Use synthetic values (`document_id=0`, `document_filename="component:{slug}"`) like ontology results do — no schema changes needed.
- **Embedding column**: Add `search_embedding` Vector(1024) column + migration + `search_by_embedding` repo method now. Column is nullable, unused until a population path is added, but infrastructure is ready.
- **Compatibility filter**: Use `ComponentVersion.compatibility` JSON column (simpler, no extra join to `ComponentQAResult`, same data shape).

### Step 3: Add repository methods to `ComponentRepository`
**File:** `app/components/repository.py`

Add two methods:

**3a. `search_with_compatibility`** — extends existing `list()` pattern:
```python
async def search_with_compatibility(
    self,
    *,
    search: str | None = None,
    category: str | None = None,
    compatible_with: list[str] | None = None,
    limit: int = 5,
) -> list[tuple[Component, ComponentVersion]]:
```
- Start with `select(Component, ComponentVersion)` joining `ComponentVersion` on `component_id` with `version_number` subquery (latest version only)
- Filter `Component.deleted_at.is_(None)`
- If `search`: ILIKE on `Component.name` and `Component.description` using `escape_like()`
- If `category`: exact match on `Component.category`
- If `compatible_with`: for each client in the list, add `.where(ComponentVersion.compatibility[client].as_string() != "none")` — only include components where the latest version doesn't have `"none"` for those clients. Components with `NULL` compatibility pass (no QA data = don't exclude).
- Order by `Component.name`, limit
- Return list of `(Component, ComponentVersion)` tuples

Latest version subquery pattern:
```python
latest_version_sq = (
    select(
        ComponentVersion.component_id,
        func.max(ComponentVersion.version_number).label("max_version"),
    )
    .group_by(ComponentVersion.component_id)
    .subquery()
)
query = (
    select(Component, ComponentVersion)
    .join(latest_version_sq, Component.id == latest_version_sq.c.component_id)
    .join(
        ComponentVersion,
        sa.and_(
            ComponentVersion.component_id == Component.id,
            ComponentVersion.version_number == latest_version_sq.c.max_version,
        ),
    )
    .where(Component.deleted_at.is_(None))
)
```

### Step 4: Create `ComponentSearchService`
**File:** `app/knowledge/component_search.py` — **NEW**

```python
"""Component search service for template-intent queries."""

from __future__ import annotations

from sqlalchemy.ext.asyncio import AsyncSession

from app.components.models import Component, ComponentVersion
from app.components.repository import ComponentRepository
from app.core.logging import get_logger
from app.knowledge.schemas import SearchResult

logger = get_logger(__name__)


class ComponentSearchService:
    """Search reusable email components with optional compatibility filtering."""

    def __init__(self, db: AsyncSession) -> None:
        self.repo = ComponentRepository(db)

    async def search_components(
        self,
        query: str,
        *,
        category: str | None = None,
        compatible_with: list[str] | None = None,
        limit: int = 5,
    ) -> list[SearchResult]:
        """Search components by text + optional compatibility filter.

        Returns backward-compatible SearchResult objects.
        """
        results = await self.repo.search_with_compatibility(
            search=query,
            category=category,
            compatible_with=compatible_with,
            limit=limit,
        )

        logger.info(
            "component_search.completed",
            query=query,
            category=category,
            compatible_with=compatible_with,
            result_count=len(results),
        )

        return self._format_as_search_results(results)

    @staticmethod
    def _format_as_search_results(
        components: list[tuple[Component, ComponentVersion]],
    ) -> list[SearchResult]:
        """Convert component + latest version to SearchResult objects."""
        results: list[SearchResult] = []
        for component, version in components:
            # Build content: component name + description + HTML preview
            content_parts = [f"## Component: {component.name}"]
            if component.description:
                content_parts.append(component.description)
            content_parts.append(f"\n**Category:** {component.category}")
            if version.compatibility:
                compat_items = [
                    f"{client}: {level}"
                    for client, level in version.compatibility.items()
                ]
                content_parts.append(f"**Compatibility:** {', '.join(compat_items)}")
            content_parts.append(f"\n```html\n{version.html_source}\n```")

            results.append(
                SearchResult(
                    chunk_content="\n".join(content_parts),
                    document_id=0,
                    document_filename=f"component:{component.slug}",
                    domain="components",
                    language="en",
                    chunk_index=0,
                    score=1.0,
                    metadata_json=None,
                )
            )
        return results
```

### Step 5: Rewrite `_search_components` in `KnowledgeService`
**File:** `app/knowledge/service.py`

Replace the current stub (lines 565-575):
```python
async def _search_components(
    self, request: SearchRequest, classified: ClassifiedQuery
) -> SearchResponse:
    """Template/component search: search Component table, merge with knowledge results."""
    from app.knowledge.component_search import ComponentSearchService

    component_service = ComponentSearchService(self.db)

    # Extract category hint from entities if available
    category: str | None = None
    for entity in classified.extracted_entities:
        if entity.entity_type == "category":
            category = entity.raw_text
            break

    # Extract client names for compatibility filtering
    compatible_with: list[str] | None = None
    client_entities = [
        e.ontology_id for e in classified.extracted_entities if e.entity_type == "client"
    ]
    if client_entities:
        compatible_with = client_entities

    component_results = await component_service.search_components(
        request.query,
        category=category,
        compatible_with=compatible_with,
        limit=5,
    )

    # Supplement with top-3 knowledge base results
    knowledge_request = SearchRequest(
        query=request.query,
        domain="best_practices",
        language=request.language,
        limit=3,
    )
    knowledge_response = await self.search(knowledge_request)

    # Component results first, then knowledge supplement
    all_results = component_results + knowledge_response.results

    logger.info(
        "knowledge.search_components.completed",
        component_count=len(component_results),
        knowledge_count=len(knowledge_response.results),
    )

    return SearchResponse(
        results=all_results,
        query=request.query,
        total_candidates=len(all_results),
        reranked=False,
    )
```

# Plan: 43.5 Calibration Delta Tracking and Regression Gate

## Context

Judge calibration (37.5) measures TPR/TNR per criterion against human labels. The existing regression gate (`regression.py`) tracks **agent pass rates** but not **judge accuracy**. A prompt tweak that fixes one criterion may silently degrade another's TPR/TNR. This closes the gap: every calibration run is compared to a baseline, and regressions >5pp are caught before propagating.

### Calibration Report Format (`traces/{agent}_calibration.json`)

```json
{
  "all_meet_targets": bool,
  "target_tpr": 0.85, "target_tnr": 0.80,
  "details": [{ "agent", "criterion", "tpr", "tnr", "meets_targets", "confusion": {tp,tn,fp,fn}, "total_labels" }],
  "needs_attention": [{ "agent", "criterion", "tpr", "tnr" }]
}
```

### Pattern to Follow: `regression.py`

The calibration tracker mirrors `regression.py` closely:
- `regression.py` compares `{agent: {criterion: pass_rate}}` dicts, flags drops >tolerance
- New tracker compares `{agent:criterion: {tpr, tnr}}` dicts, flags drops >5pp
- Same CLI pattern: `--current`, `--baseline`, `--update-baseline`
- Same exit(1) on failure, shutil.copy2 for baseline snapshots
- Same `build_*_report()` → JSON-serializable dict pattern

## Implementation Steps

### Step 1: Add `CalibrationDelta` to `schemas.py`

Add after `CalibrationResult` (line ~60):

```python
@dataclass(frozen=True)
class CalibrationDelta:
    """Per-criterion TPR/TNR change between calibration runs."""

    agent: str
    criterion: str
    tpr_before: float
    tpr_after: float
    tpr_delta: float  # after - before
    tnr_before: float
    tnr_after: float
    tnr_delta: float  # after - before
    regressed: bool  # True if tpr_delta < -threshold OR tnr_delta < -threshold
```

### Step 2: Create `calibration_tracker.py`

Mirror `regression.py` structure. Key functions:

```python
"""Calibration delta tracking: compare TPR/TNR against baseline.

Compares per-criterion TPR/TNR from current calibration against stored
baseline. Flags regression if any criterion's TPR or TNR drops >5pp.

CLI: python -m app.ai.agents.evals.calibration_tracker \
       --current traces/ \
       --baseline traces/calibration_baseline.json \
       [--threshold 0.05] \
       [--update-baseline]
"""

CALIBRATION_REGRESSION_THRESHOLD = 0.05  # 5 percentage points

def load_calibration_reports(traces_dir: Path) -> dict[str, dict[str, Any]]:
    """Load all {agent}_calibration.json files from traces directory.
    
    Returns: {agent: calibration_report_dict}
    """

def extract_metrics(reports: dict[str, dict[str, Any]]) -> dict[str, dict[str, float]]:
    """Extract flat {f"{agent}:{criterion}": {"tpr": x, "tnr": y}} from reports."""

def load_baseline(path: Path) -> dict[str, dict[str, float]]:
    """Read calibration baseline JSON. Returns {} if file missing."""

def save_baseline(metrics: dict[str, dict[str, float]], path: Path) -> None:
    """Snapshot current metrics as new baseline."""

def compare_calibration(
    current: dict[str, dict[str, float]],
    baseline: dict[str, dict[str, float]],
    threshold: float = CALIBRATION_REGRESSION_THRESHOLD,
) -> list[CalibrationDelta]:
    """Compute per-criterion TPR/TNR deltas. Flag regressions >threshold."""

def build_delta_report(deltas: list[CalibrationDelta]) -> dict[str, Any]:
    """JSON-serializable delta report with has_regression flag."""

def main() -> None:
    """CLI entry point — argparse, load, compare, report, exit(1) on regression."""
```

**`load_calibration_reports`:** Glob `traces_dir / "*_calibration.json"`, extract agent name from filename prefix.

**`extract_metrics`:** Iterate `report["details"]`, build `{f"{d['agent']}:{d['criterion']}": {"tpr": d["tpr"], "tnr": d["tnr"]}}`.

**`compare_calibration`:** For each key in union of current+baseline keys:
- Compute `tpr_delta = current_tpr - baseline_tpr`, same for TNR
- `regressed = tpr_delta < -threshold or tnr_delta < -threshold`
- New criteria (not in baseline): `regressed=False`, deltas from 0.0

**`build_delta_report`:** Same pattern as `build_regression_report`:
```json
{
  "has_regression": bool,
  "threshold": 0.05,
  "criteria_checked": N,
  "criteria_regressed": N,
  "criteria_improved": N,
  "details": [{ "agent", "criterion", "tpr_before", "tpr_after", "tpr_delta", "tnr_before", "tnr_after", "tnr_delta", "regressed" }]
}
```

**CLI `main()`:**
- `--current` (required): traces directory path
- `--baseline` (required): baseline JSON path
- `--threshold` (default 0.05): max acceptable drop
- `--update-baseline`: save current as baseline if no regressions
- First run (no baseline file): save baseline, exit 0
- Print per-criterion summary like `regression.py:171-178`
- `sys.exit(1)` if `has_regression`

## Preflight Warnings

- `regression.py` uses `analysis.json` (pass rates). Calibration tracker uses `*_calibration.json` (TPR/TNR) — different data sources. Don't confuse them.
- `test_calibration_knowledge.py` has `_make_calibration()` helper but it builds full report dicts. The new tests should build the lighter `{key: {tpr, tnr}}` metric dicts directly.
- Baseline file (`traces/calibration_baseline.json`) is a new artifact. Add to `.gitignore` if traces/ is already ignored; otherwise it should be committed as a CI artifact.

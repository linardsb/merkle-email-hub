# Plan: 4.11 Custom Persona Creation

## Context

Only preset personas (8 hardcoded email client profiles) are available in the workspace preview toolbar dropdown. The PRD requires "Custom persona creation" — users should be able to create their own test personas with custom email client, viewport, dark mode, and OS settings. The backend `POST /api/v1/personas/` endpoint already exists and accepts `PersonaCreate`.

## Files to Create/Modify

| File | Action | Purpose |
|------|--------|---------|
| `cms/apps/web/src/components/workspace/create-persona-dialog.tsx` | **Create** | Dialog with form for creating custom personas |
| `cms/apps/web/src/components/workspace/persona-selector.tsx` | Modify | Add "Create custom..." trigger at bottom of dropdown |
| `cms/apps/web/src/hooks/use-personas.ts` | Modify | Add `useCreatePersona()` SWR mutation hook |
| `cms/apps/web/src/lib/demo/mutation-resolver.ts` | Modify | Handle `POST /api/v1/personas` in demo mode |
| `cms/apps/web/messages/en.json` | Modify | Add ~15 i18n keys for persona creation form |

## Data Shape Reference

```typescript
// SDK types (already exist in @merkle-email-hub/sdk)
type PersonaCreate = {
  name: string;          // required, 1-200 chars
  slug: string;          // required, 1-100 chars (auto-generated from name)
  description?: string | null;
  email_client?: string; // default "gmail"
  device_type?: string;  // "desktop" | "mobile", default "desktop"
  dark_mode?: boolean;   // default false
  viewport_width?: number; // 200-2000, default 600
  os_name?: string;      // default "macOS"
};

type PersonaResponse = PersonaCreate & {
  id: number;
  is_preset: boolean;
  created_at: string;
  updated_at: string;
};
```

### Step 2: Add `useCreatePersona()` mutation hook

In `cms/apps/web/src/hooks/use-personas.ts`, add:

```typescript
import useSWRMutation from "swr/mutation";
import { mutationFetcher } from "@/lib/mutation-fetcher";
import type { PersonaCreate } from "@merkle-email-hub/sdk";

export function useCreatePersona() {
  return useSWRMutation<PersonaResponse, Error, string, PersonaCreate>(
    "/api/v1/personas",
    mutationFetcher
  );
}
```

Keep existing `usePersonas()` and `usePersona()` hooks unchanged.

### Step 4: Create `create-persona-dialog.tsx`

Create `cms/apps/web/src/components/workspace/create-persona-dialog.tsx`:

**Structure:**
- Controlled `Dialog` (open/onOpenChange props)
- Form with fields:
  - **Name** — `<Input>` (required, max 200 chars)
  - **Description** — `<Input>` (optional)
  - **Email client** — `<Select>` with options: `gmail`, `outlook_365`, `outlook_2019`, `apple_mail`, `ios_mail`, `samsung_mail`, `yahoo`, `thunderbird`
  - **Device type** — `<Select>` with two options: `desktop`, `mobile`
  - **Viewport width** — `<Input type="number">` (min 200, max 2000, default 600)
  - **Dark mode** — Checkbox (`<Checkbox>` from shadcn)
  - **OS** — `<Select>` with options: `macOS`, `Windows`, `iOS`, `Android`, `Linux`
- Auto-generate `slug` from name: `name.toLowerCase().replace(/[^a-z0-9]+/g, "-").replace(/^-|-$/g, "")`
- Submit calls `useCreatePersona().trigger(data)`, then mutate the `"/api/v1/personas"` SWR cache, show success toast, close dialog
- Error handling: show error toast on failure

**Component interface:**
```typescript
interface CreatePersonaDialogProps {
  open: boolean;
  onOpenChange: (open: boolean) => void;
  onCreated?: (persona: PersonaResponse) => void;
}
```

**Form state:** Use `useState` for each field (simpler than a form library for 7 fields). Reset on dialog close.

**Imports needed:**
- `Dialog, DialogContent, DialogHeader, DialogTitle, DialogDescription, DialogFooter` from ui
- `Input` from ui
- `Select, SelectContent, SelectItem, SelectTrigger, SelectValue` from ui
- `Checkbox` from ui
- `Button` from ui
- `Label` from ui
- `useCreatePersona` from hooks
- `useTranslations` from next-intl
- `toast` from sonner
- `useSWRConfig` from swr
- `PersonaCreate, PersonaResponse` from SDK

**Layout:** Two-column grid for the 6 main fields (email client + device type on one row, viewport + OS on one row, name and description full-width, dark mode as a checkbox row). Dialog width: `max-w-[28rem]`.

### Step 2: SWR Hooks

**`cms/apps/web/src/hooks/use-outlook-analysis.ts`**
```typescript
"use client";

import useSWRMutation from "swr/mutation";
import { longMutationFetcher } from "@/lib/mutation-fetcher";
import type {
  OutlookAnalysisRequest,
  OutlookAnalysisResponse,
  OutlookModernizeRequest,
  OutlookModernizeResponse,
} from "@/types/outlook";

export function useOutlookAnalysis() {
  return useSWRMutation<OutlookAnalysisResponse, Error, string, OutlookAnalysisRequest>(
    "/api/v1/qa/outlook-analysis",
    longMutationFetcher,
  );
}

export function useOutlookModernize() {
  return useSWRMutation<OutlookModernizeResponse, Error, string, OutlookModernizeRequest>(
    "/api/v1/qa/outlook-modernize",
    longMutationFetcher,
  );
}
```

### Step 3: OutlookAdvisorPanel Component

**`cms/apps/web/src/components/outlook/OutlookAdvisorPanel.tsx`**

Structure:
- Header: Search icon + "Outlook Advisor" title + "Analyze" button
- Empty state when no data
- When data exists:
  - **Summary bar**: total dependencies count, removable count, byte savings estimate
  - **Severity breakdown**: colored badges grouping deps by severity (high=red, medium=yellow, low=green)
  - **Type counts**: horizontal stat row showing counts per type (vml_count, ghost_table_count, etc.) with i18n labels
  - **Dependency list**: collapsible section showing each dependency with:
    - Type badge, severity badge, line number
    - Code snippet in monospace `<code>` block
    - Modern replacement suggestion if available
  - **Modernize section**: target selector dropdown (new_outlook / dual_support / audit_only) + "Modernize" button
  - **Post-modernize results** (when modernize data exists):
    - Before/after byte comparison bar
    - Changes applied count
    - Re-analysis summary

Props:
```typescript
interface OutlookAdvisorPanelProps {
  html: string;
  onHtmlUpdate?: (html: string) => void; // callback to update editor with modernized HTML
}
```

Key implementation details:
- Use `useOutlookAnalysis()` and `useOutlookModernize()` hooks
- "Analyze" button triggers analysis: `trigger({ html })`
- "Modernize" button triggers modernize: `modernizeTrigger({ html, target: selectedTarget })`
- On successful modernize, call `onHtmlUpdate?.(modernizeData.html)` to push new HTML to editor
- Severity color helper: `high` → `text-badge-danger-text bg-badge-danger-bg`, `medium` → `text-badge-warning-text bg-badge-warning-bg`, `low` → `text-badge-success-text bg-badge-success-bg`
- Dep type icons from lucide-react: `Code2` for VML, `Table` for ghost tables, `MessageSquare` for MSO conditionals, `Paintbrush` for MSO CSS, `Image` for DPI images, `ExternalLink` for external class, `WrapText` for word wrap
- Default target: `"dual_support"` (safest)
- Byte savings formatted as KB: `(bytes / 1024).toFixed(1)`

### Step 5: CSSCompilerPanel Component

**`cms/apps/web/src/components/email-engine/CSSCompilerPanel.tsx`**

Structure:
- Header: Paintbrush icon + "CSS Compiler" title + "Compile" button
- Empty state when no data
- When data exists:
  - **Size comparison bar**: original_size vs compiled_size with reduction_pct badge
    - Two horizontal bars (before/after) with size labels in KB
    - Reduction percentage in green badge: `"-{reduction_pct}%"`
  - **Compile time**: `{compile_time_ms}ms` in muted text
  - **Warnings section** (if warnings.length > 0): amber alert box listing warnings
  - **Removed properties section**: horizontal chip/tag list of removed CSS properties
  - **Conversions section**: collapsible list of conversions, each showing:
    - `{original_property}: {original_value}` → `{replacement_property}: {replacement_value}`
    - Reason text in muted
    - Affected clients as small badges

Props:
```typescript
interface CSSCompilerPanelProps {
  html: string;
  onHtmlUpdate?: (html: string) => void; // callback to update editor with compiled HTML
}
```

Key implementation details:
- Use `useCSSCompile()` hook
- "Compile" button: `trigger({ html })`
- On success, offer "Apply" button that calls `onHtmlUpdate?.(data.html)`
- Size display: format bytes as KB with 1 decimal: `(bytes / 1024).toFixed(1) + " KB"`
- Progress bar width for size comparison: `width: ${(compiled_size / original_size) * 100}%`
- Property chips: `rounded-full bg-surface-muted px-2 py-0.5 text-xs text-foreground-muted`
- Conversion arrows: use `→` character between original and replacement

# Plan: 43.4 Knowledge Agent Integration for Cross-Judge Learnings

## Context

After each calibration cycle, auto-generate a knowledge base document from calibration results and disagreement patterns. Store in Knowledge agent's RAG corpus so developers can query "what do our judges struggle with?" and get grounded answers. Bridges calibration feedback loop (43.1 corrections, 43.3 skills) with human-facing Knowledge agent.

## Research Summary

### Calibration Data Schema (`traces/{agent}_calibration.json`)

```json
{
  "all_meet_targets": bool,
  "target_tpr": 0.85, "target_tnr": 0.80,
  "details": [{ "agent", "criterion", "tpr", "tnr", "meets_targets", "confusion": {tp,tn,fp,fn}, "total_labels" }],
  "needs_attention": [{ "agent", "criterion", "tpr", "tnr" }]
}
```

Generated by `calibration.py:build_calibration_report()` (lines 106-151). Inputs: `traces/{agent}_verdicts.jsonl` + `traces/{agent}_human_labels.jsonl`.

### Knowledge Agent Indexing Pipeline

- `app/knowledge/service.py:ingest_document()` — creates `Document` record, chunks text (512-char default), generates embeddings, stores in `document_chunks` table with pgvector
- `app/knowledge/models.py:Document` — `domain` field (indexed, 50 chars) filters searches; use `"calibration"` domain
- `app/knowledge/schemas.py:SearchRequest` — supports `domain` filter
- Physical files stored at `data/documents/{document_id}/{filename}` (config: `settings.knowledge.document_storage_path`)
- **No `data/knowledge/` directory exists** — the RAG corpus lives in the database. The script generates a markdown file, then ingests it via `KnowledgeService`

## Implementation Steps

### Step 1: Create `scripts/generate-calibration-knowledge.py`

Entry point script following `scripts/eval-skill-update.py` pattern.

**CLI interface:**
```
uv run python scripts/generate-calibration-knowledge.py \
  --traces-dir traces/ \
  --output data/knowledge/judge_calibration_insights.md \
  [--ingest]  # optional: auto-ingest into KnowledgeService
```

**Core logic:**

```python
"""Generate knowledge base document from judge calibration data."""
from __future__ import annotations

import argparse
import json
import sys
from datetime import datetime, timezone
from pathlib import Path

from app.core.logging import get_logger

logger = get_logger(__name__)
```

**Functions to implement:**

1. `load_all_calibrations(traces_dir: Path) -> list[dict]`
   - Glob `traces/*_calibration.json`, load each with `json.loads()`
   - Skip files that fail to parse (log warning)
   - Return list of calibration report dicts

2. `load_all_verdicts_with_reasoning(traces_dir: Path) -> dict[tuple[str, str], list[dict]]`
   - Glob `traces/*_verdicts.jsonl`, load verdict lines
   - Key by `(agent, criterion)` → list of verdict dicts with reasoning
   - Used for disagreement example extraction

3. `load_all_human_labels(traces_dir: Path) -> dict[tuple[str, str], list[dict]]`
   - Glob `traces/*_human_labels.jsonl`
   - Key by `(agent, criterion)` → list of label dicts

4. `extract_cross_agent_patterns(details: list[dict]) -> list[dict]`
   - Group calibration details by `criterion`
   - Find criteria that fail (`meets_targets=False`) across ≥2 agents
   - Return `[{"criterion": str, "agents": list[str], "avg_tpr": float, "avg_tnr": float}]`

5. `extract_early_warnings(details: list[dict]) -> list[dict]`
   - Find criteria where `tpr < 0.90` or `tnr < 0.85` but still `meets_targets=True`
   - These are approaching failure threshold — early warning

6. `extract_disagreement_examples(verdicts: dict, labels: dict) -> dict[tuple[str, str], list[str]]`
   - For each `(agent, criterion)`, find trace_ids where judge and human disagree
   - Cap at 3 examples per key, return trace IDs only (no HTML)

7. `generate_markdown(calibrations: list[dict], cross_patterns: list[dict], warnings: list[dict], examples: dict) -> str`
   - Produce structured markdown with sections:

   ```markdown
   # Judge Calibration Insights
   
   > Auto-generated on {date}. Query via Knowledge agent.
   
   ## Per-Agent Calibration Summary
   
   ### {Agent Name}
   
   | Criterion | TPR | TNR | Status | Labels |
   |-----------|-----|-----|--------|--------|
   | {name}    | {v} | {v} | {pass/warn/fail} | {n} |
   
   ## Criteria Approaching Failure Threshold
   
   | Agent | Criterion | TPR | TNR | Risk |
   |-------|-----------|-----|-----|------|
   
   ## Cross-Agent Patterns
   
   ### {criterion} — fails in {N} agents
   Agents: {list}. Average TPR: {v}, TNR: {v}.
   {Brief explanation of what this criterion tests}
   
   ## Common Disagreement Patterns
   
   ### {Agent}: {Criterion}
   - Type: {FP-heavy / FN-heavy / mixed}
   - Example traces: {trace_id_1}, {trace_id_2}
   ```

8. `async def ingest_to_knowledge(output_path: Path) -> None`
   - Import `KnowledgeService`, `get_db_context`
   - Create/update document with `domain="calibration"`, `source_type="markdown"`
   - Uses existing `ingest_document()` pipeline
   - Only runs when `--ingest` flag is passed

9. `async def main() -> None`
   - Parse args, load data, generate markdown, write file
   - Optionally ingest if `--ingest`
   - Exit 0 on success, 1 on no calibration files found

**Key constraints:**
- No PII — trace IDs and criterion names only, never full HTML
- Idempotent — regenerating overwrites previous output cleanly
- Cap total document at ~2000 lines to keep chunk count reasonable
- Use `from __future__ import annotations` for forward refs

def _make_calibration(agent: str, details: list[dict], needs_attention: list[dict] | None = None) -> dict:
    return {
        "all_meet_targets": len(needs_attention or []) == 0,
        "total_criteria": len(details),
        "passing_criteria": sum(1 for d in details if d["meets_targets"]),
        "failing_criteria": sum(1 for d in details if not d["meets_targets"]),
        "target_tpr": 0.85,
        "target_tnr": 0.80,
        "details": details,
        "needs_attention": needs_attention or [],
    }

def _make_detail(agent: str, criterion: str, *, tpr: float = 0.95, tnr: float = 0.90,
                 meets: bool = True, tp: int = 19, tn: int = 18, fp: int = 2, fn: int = 1) -> dict:
    return {
        "agent": agent, "criterion": criterion,
        "tpr": tpr, "tnr": tnr, "meets_targets": meets,
        "confusion": {"tp": tp, "tn": tn, "fp": fp, "fn": fn},
        "total_labels": tp + tn + fp + fn,

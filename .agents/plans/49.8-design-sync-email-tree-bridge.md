# Plan: 49.8 Design-Sync → EmailTree Bridge

## Context

The design-sync pipeline currently renders HTML via `ComponentRenderer` (regex-based slot filling + token overrides — fragile). Phase 48.6 introduced `EmailTree` (typed tree schema) and Phase 48.8 introduced `TreeCompiler` (deterministic lxml-based compilation). This task bridges design-sync output to `EmailTree`, eliminating regex rendering for the design-sync path.

### Data Flow: Current vs. Tree Bridge

```
Current:  layout → match_all() → ComponentMatch[] → ComponentRenderer (regex) → HTML fragments → COMPONENT_SHELL
Tree:     layout → match_all() → ComponentMatch[] → tree_bridge → EmailTree → TreeCompiler (lxml) → CompiledEmail
```

### Key Type Mappings

| SlotFill.slot_type | → SlotValue type | Extraction logic |
|--------------------|------------------|------------------|
| `"text"` | `TextSlot(text=value)` | Direct value passthrough |
| `"image"` | `ImageSlot(src=value, alt=..., width=..., height=...)` | Dimensions from `attr_overrides` or matched `ImagePlaceholder` |
| `"cta"` | `ButtonSlot(text=..., href=value, bg_color=..., text_color=...)` | Colors from matched `ButtonElement`, text from section buttons |
| `"attr"` | `HtmlSlot(html=value)` | Raw HTML slot (rare, used for complex content) |

### TokenOverride → style_overrides

`list[TokenOverride]` → `dict[str, str]`: each `TokenOverride.css_property: TokenOverride.value`. The `target_class` is not needed in tree mode (TreeCompiler applies overrides to section root element).

#### `build_email_tree()`

**Signature:**
```python
def build_email_tree(
    layout: DesignLayoutDescription,
    matches: list[ComponentMatch],
    tokens: ExtractedTokens,
    *,
    groups: list[RepeatingGroup] | None = None,
    group_map: dict[int, RepeatingGroup] | None = None,
    subject: str = "",
) -> EmailTree:
```

**Logic:**
1. Build `TreeMetadata`:
   - `subject` from parameter (default empty string — caller provides)
   - `preheader` from first PREHEADER section's text content (if any)
   - `design_tokens` from `_build_design_tokens(tokens)`
2. Build `TreeSection` list:
   - Track `rendered_group_ids: set[int]` to deduplicate repeating groups
   - For each match at index `i`:
     - If `group_map` has entry for `i` and group not yet rendered:
       - Collect all matches belonging to this group
       - Create wrapper `TreeSection` with `children` list
       - `style_overrides` includes `background-color` from `group.container_bgcolor`
       - Mark group as rendered
     - Else: create single `TreeSection` via `_match_to_tree_section(match)`
3. Return `EmailTree(metadata=metadata, sections=sections)`

#### `_match_to_tree_section(match: ComponentMatch) -> TreeSection`

```python
TreeSection(
    component_slug=match.component_slug,
    slot_fills=_convert_slot_fills(match.slot_fills, match.section),
    style_overrides=_convert_token_overrides(match.token_overrides),
)
```

#### `_convert_slot_fills(fills: list[SlotFill], section: EmailSection) -> dict[str, SlotValue]`

Map each `SlotFill` by `slot_type`:
- `"text"` → `TextSlot(text=fill.value)` — strip HTML tags if present
- `"image"` → `ImageSlot(src=fill.value, alt=fill.attr_overrides.get("alt", ""), width=int(fill.attr_overrides.get("width", 600)), height=int(fill.attr_overrides.get("height", 400)))` — clamp to 1-2000
- `"cta"` → `ButtonSlot(text=_extract_cta_text(fill, section), href=fill.value, bg_color=_extract_button_color(fill, section), text_color=_extract_button_text_color(fill, section))` — default colors `#000000`/`#FFFFFF`
- `"attr"` → `HtmlSlot(html=fill.value)` — for complex/raw content slots

**Helper: `_extract_cta_text(fill, section)`** — check `fill.attr_overrides.get("text")`, then first `section.buttons[].text`, fallback `"Click here"`.

**Helper: `_extract_button_color(fill, section)`** — check `fill.attr_overrides.get("bg_color")`, then first `section.buttons[].fill_color`, fallback `"#000000"`.

**Helper: `_extract_button_text_color(fill, section)`** — check `fill.attr_overrides.get("text_color")`, fallback `"#FFFFFF"`.

#### `_convert_token_overrides(overrides: list[TokenOverride]) -> dict[str, str]`

```python
return {o.css_property: o.value for o in overrides}
```

Deduplicate: last wins (dict insertion order). Validate CSS property names against `TreeSection` validator pattern.

#### `_build_design_tokens(tokens: ExtractedTokens) -> dict[str, dict[str, str]]`

Build `design_tokens` dict for `TreeMetadata`:
- `"colors"`: `{color.name: color.hex for color in tokens.colors}`
- `"typography"`: `{t.name: t.family for t in tokens.typography}` (primary font info)
- `"dark_palette"`: `{c.name: c.hex for c in tokens.dark_colors}`

Only include non-empty sub-dicts.

```python
if output_format == "tree":
    from app.design_sync.tree_bridge import build_email_tree
    email_tree = build_email_tree(
        layout=layout,
        matches=matches,
        tokens=tokens,
        groups=groups if sibling_detection_enabled else None,
        group_map=group_map if sibling_detection_enabled else None,
    )
    # Compile to HTML via TreeCompiler for validation/output
    tree_html: str = ""
    try:
        from app.components.tree_compiler import TreeCompiler
        compiler = TreeCompiler()
        compiled = compiler.compile(email_tree)
        tree_html = compiled.html

        log.warning("tree_bridge.compile_fallback", msg="TreeCompiler unavailable, using legacy renderer")
        # Fall through to legacy rendering for HTML

    if tree_html:
        return ConversionResult(
            html=tree_html,
            sections_count=len(email_tree.sections),
            warnings=warnings,
            layout=layout,
            tree=email_tree.model_dump(mode="json"),
            # ... propagate other fields
        )
    # If TreeCompiler failed, fall through to legacy rendering
```

## Preflight Warnings

- `TreeSection.component_slug` pattern is `^([a-z][a-z0-9-]*|__custom__)$` — ensure all `ComponentMatch.component_slug` values from `COMPONENT_SEEDS` conform (they should, but verify)
- `ImageSlot.width`/`height` are `int` with `gt=0, le=2000` — `attr_overrides` values are strings, need `int()` conversion with clamping
- `ButtonSlot.bg_color`/`text_color` require `^#[0-9a-fA-F]{6}$` — fallback colors must be valid hex
- `SlotFill.slot_type` values: `"text"`, `"image"`, `"cta"`, `"attr"` — handle unknown types gracefully (log warning, skip or use HtmlSlot)
- `EmailTree.sections` requires `min_length=1` — handle empty match lists with early return or error

## Security Checklist

No new endpoints. No user input paths. All data flows from internal Figma pipeline → tree bridge → compiler. No injection vectors.

- `HtmlSlot` values come from `SlotFill(slot_type="attr")` — these are already sanitized by `ComponentRenderer` pipeline. TreeCompiler's `_fill_html()` uses `sanitize_component_html()` as additional defense.
- Button `href` values validated by TreeCompiler (`http`/`https`/`mailto`/`tel` only).
- No `eval()`, `subprocess`, or external API calls.

# Plan: Fix HTML Editor — Content Not Fully Visible and Not Editable

## Context
The Monaco HTML editor in the workspace shows only partial template content. The full HTML template is not visible, not scrollable, and interaction (editing) is broken. This is a CSS layout issue — the editor container doesn't get a properly constrained height, so Monaco can't render its viewport correctly.

## Root Cause
Classic flexbox height collapse. In `monaco-editor.tsx`:

```tsx
<div className="flex h-full flex-col overflow-hidden">
  <EditorToolbar ... />          {/* fixed h-8 */}
  <div className="flex-1">      {/* ← PROBLEM: missing min-h-0 */}
    <Editor ... />
  </div>
</div>
```

In a `flex-col` layout, `flex-1` sets `flex: 1 1 0%` but the child's `min-height` defaults to `auto` (content-based sizing). Monaco renders a large internal content area, so the div's min-height becomes unconstrained, breaking the scrollable viewport. The editor can't determine its available height, so it renders incorrectly — content gets clipped and mouse events may not reach the correct editor coordinates.

## Files to Modify
- `cms/apps/web/src/components/workspace/editor/monaco-editor.tsx` — Fix editor container sizing

## Implementation Steps

### Step 1: Fix the editor container in `monaco-editor.tsx`

Change line 118 from:
```tsx
<div className="flex-1">
```
to:
```tsx
<div className="relative min-h-0 flex-1">
```

- `min-h-0` — overrides the default `min-height: auto` so the flex item can shrink below its content height, allowing Monaco to calculate its viewport correctly
- `relative` — Monaco editor uses absolute positioning internally; having a positioned ancestor ensures it measures dimensions from this container

### Step 2: Add explicit height to the Editor component

On the `<Editor>` component (line 119), add the `height` prop:
```tsx
<Editor
  height="100%"
  ...
```

This ensures Monaco explicitly sizes to its container rather than relying on its default height calculation.

# Plan: 1.1 Project Dashboard Page — Backend

## Context

Task 1.1 requires backend APIs to power the project dashboard at `/[locale]/(dashboard)/page.tsx`. The dashboard needs: project overview grid, stats (total projects, active orgs), and project CRUD.

## Status: Backend Already Complete

The `app/projects/` module fully implements all required endpoints. The frontend hooks (`use-projects.ts`, `use-orgs.ts`) are already wired and consuming these APIs.

### Existing Endpoints

| Method | Endpoint | Auth | Rate Limit | Purpose |
|--------|----------|------|------------|---------|
| `GET` | `/api/v1/projects` | any auth | 30/min | List projects (page, page_size, client_org_id, search) |
| `GET` | `/api/v1/projects/{id}` | any auth | 30/min | Get single project |
| `POST` | `/api/v1/projects` | developer+ | 10/min | Create project (auto-adds creator as admin member) |
| `PATCH` | `/api/v1/projects/{id}` | developer+ | 10/min | Update project |
| `DELETE` | `/api/v1/projects/{id}` | admin | 10/min | Soft delete project |
| `GET` | `/api/v1/orgs` | any auth | 30/min | List client orgs (page, page_size) |
| `POST` | `/api/v1/orgs` | admin | 10/min | Create client org (unique slug check) |

### Existing VSA Structure

```
app/projects/
├── models.py       — ClientOrg, Project, ProjectMember (with SoftDeleteMixin)
├── schemas.py      — ClientOrgCreate/Response, ProjectCreate/Update/Response
├── repository.py   — ClientOrgRepository, ProjectRepository (search with escape_like)
├── service.py      — ProjectService (list_projects, list_orgs, create, update, delete)
├── exceptions.py   — ProjectNotFoundError, ClientOrgNotFoundError, ClientOrgAlreadyExistsError
└── routes.py       — All endpoints with RBAC + rate limiting
```

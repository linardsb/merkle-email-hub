### Key Schemas

**Trace:** `{id, agent, dimensions, input, output: {html, qa_results, qa_passed}, expected_challenges, elapsed_seconds, error, timestamp}`

**Verdict:** `{trace_id, agent, overall_pass, criteria_results: [{criterion, passed, reasoning}], error}`

**Comparison output:** `{generated_at, summary: {total_criteria, high_flip_criteria, threshold}, criteria: {agent:criterion: {total, flips, flip_rate, pass_to_fail, fail_to_pass, pre_pass_rate, post_pass_rate}}, priority_review: [...]}`

### Runner skip_existing Logic

- `--skip-existing` → loads existing trace IDs, appends only new (file mode `a`)
- Without flag → truncates file, regenerates all (file mode `w`)
- Same pattern in `judge_runner.py`

### Regression Tolerance

- Per-criterion: 10% (CLI `--tolerance`)
- Per-agent overall: `AGENT_REGRESSION_TOLERANCE = 0.03` (3pp hard-coded)
- Agent regressed if: any criterion regressed OR overall pass rate dropped >3pp

## Implementation Steps

### 37.4.1 — Back up current verdicts as pre-file-based baseline

```bash
mkdir -p traces/pre_file_based
cp traces/*_verdicts.jsonl traces/pre_file_based/
```

Verify: `ls traces/pre_file_based/` shows 9 verdict files. Keep `traces/pre_golden/` intact.

### 37.4.2 — Re-generate agent traces with file-based components

```bash
uv run python -m app.ai.agents.evals.runner --agent all --output traces/
```

**No `--skip-existing`** → full regeneration (truncates, rewrites all traces).

~112 test cases × 9 agents. Requires LLM API access.

**Spot-check:** After completion, verify `traces/scaffolder_traces.jsonl` contains:
- `data-slot` attributes in input/output HTML
- `<!-- Component: -->` markers
- NOT old inline seed patterns (hardcoded HTML strings without markers)

### 37.4.3 — Re-run judges against new traces

```bash
make eval-rejudge
```

Equivalent to: `judge_runner --agent all --traces traces --output traces --mode hybrid` (no `--skip-existing`).

~9 agents × ~12 traces × 5 criteria = ~540 LLM judge calls (hybrid mode uses QA checks for some criteria, reducing LLM calls).

Golden references from 37.1–37.3 automatically injected via `format_golden_section()` in `judges/base.py`.

### 37.4.4 — Compare verdicts: inline-seed vs file-based

```bash
uv run python scripts/eval-compare-verdicts.py \
  --pre-dir traces/pre_file_based \
  --post-dir traces \
  --output traces/verdict_comparison_file_based.json
```

**Expected outcomes:**
- Scaffolder: verdicts may improve (better template HTML with `data-slot` markers)
- Code reviewer: verdicts may shift (file-based HTML has different patterns to flag)
- Outlook fixer: verdicts may shift (VML `v:rect`/`v:roundrect` patterns differ from inline)
- Criteria with >20% flip rate flagged in `priority_review` for 37.5 human review

### 37.4.5 — Run analysis and regression check

```bash
make eval-analysis    # Regenerate traces/analysis.json
make eval-regression  # Check pass rates vs AGENT_REGRESSION_TOLERANCE (3pp)
```

If regression detected:
1. Check `traces/verdict_comparison_file_based.json` — is the flip a true quality change or judge calibration artifact?
2. True quality change → agent output genuinely worse on file-based HTML → investigate agent prompt
3. Calibration artifact → judge is more/less discriminating on new HTML structure → adjust golden refs or judge criteria

Final gate: `make eval-check` must pass.

## Preflight Warnings

1. **LLM required:** Steps 37.4.2 and 37.4.3 make real LLM API calls. `--dry-run` mode produces mock verdicts with ~20% failure rate — useful for testing pipeline but produces 0% flip rate (deterministic).
2. **Knowledge + Innovation verdicts:** May have been empty/mock from previous run. Comparison script handles missing pre-files gracefully (returns empty results for those agents).
3. **Token budget:** Golden reference section capped at 2000 tokens. Large agent outputs may approach context limit — judge may fail on oversized outputs. Monitor for errors in verdict files.
4. **Timing:** Full trace regeneration (~112 cases) + judging (~540 hybrid calls) takes significant time. Run sequentially: traces must complete before judging starts.

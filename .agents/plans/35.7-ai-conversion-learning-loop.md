# Plan: 35.7 AI Conversion Learning Loop

## Context

When AI agents (Outlook Fixer, Dark Mode, Code Reviewer) repeatedly fix the same converter output patterns, those corrections should be tracked, aggregated, and surfaced as converter rule suggestions. This creates a self-improving pipeline — validated agent corrections feed back upstream, reducing future agent work and improving first-pass quality.

## Implementation Steps

### Step 1: Data Models & Core Tracker (`app/design_sync/correction_tracker.py`)

Define frozen dataclasses and core logic:

```python
@dataclass(frozen=True)
class CorrectionPattern:
    agent: str                          # "outlook_fixer", "dark_mode", etc.
    pattern_hash: str                   # SHA-256 of (agent, element_tag, attr_name, change_type)
    element_tag: str                    # "td", "table", "style", etc.
    attribute: str                      # "style", "class", "width", etc.
    change_type: str                    # "property_added", "property_changed", "element_wrapped", etc.
    input_snippet: str                  # Representative input (truncated)
    output_snippet: str                 # Representative output (truncated)
    occurrences: int
    first_seen: datetime
    last_seen: datetime
    confidence: float                   # Consistency: same change_type ratio across occurrences

@dataclass(frozen=True)
class ConverterRuleSuggestion:
    id: str                             # UUID
    description: str                    # "Expand shorthand padding to longhand on <td>"
    agent_source: str
    pattern: CorrectionPattern
    suggested_code: str                 # Python snippet for converter.py
    status: Literal["suggested", "approved", "rejected", "applied"]
```

`CorrectionTracker` class:
- `__init__(self, data_dir: Path)` — accepts data directory, creates JSONL path
- `async record_correction(self, agent: str, original_html: str, corrected_html: str) -> int` — returns number of diffs recorded
  - Use `app/rendering/sandbox/dom_diff.py:compute_dom_diff()` to get structural diff
  - For each changed element/attribute, compute `pattern_hash`
  - Append to JSONL log as one line per atomic change
  - Return count of changes recorded
- `get_frequent_patterns(self, *, min_occurrences: int = 5, min_confidence: float = 0.9) -> list[CorrectionPattern]` — sync, reads JSONL, aggregates by `pattern_hash`, filters by thresholds
- `suggest_converter_rules(self) -> list[ConverterRuleSuggestion]` — generates human-readable suggestions with Python code snippets from frequent patterns
- `approve_rule(self, pattern_hash: str) -> None` — updates status in `correction_rules.json`
- `_compute_pattern_hash(self, agent: str, element_tag: str, attribute: str, change_type: str) -> str` — SHA-256 hash
- `_append_to_log(self, entry: dict[str, object]) -> None` — append one JSONL line
- `_load_log(self) -> list[dict[str, object]]` — read all JSONL entries
- `_aggregate_patterns(self, entries: list[dict[str, object]]) -> list[CorrectionPattern]` — group by hash, compute occurrences + confidence

### Step 2: Structural Diff Adapter

Adapt `dom_diff.py` pattern for correction tracking. The existing `DOMDiff` tracks `removed_elements`, `modified_styles`, `removed_css_properties`, `added_elements`, `removed_attributes`.

For correction tracking, we need a lighter extraction that identifies:
1. **Style property changes** on elements (e.g., shorthand→longhand padding)
2. **Attribute additions/removals** (e.g., `align` added, `class` removed)
3. **Element wrapping** (e.g., MSO conditional `<!--[if mso]>` ghost tables added)
4. **CSS injection** (e.g., dark mode `<style>` block added/modified)

Implement `extract_correction_diffs(original: str, corrected: str) -> list[CorrectionDiff]` in `correction_tracker.py`:
- Parse both with `html.parser.HTMLParser` (stdlib, no new dependency)
- Walk both DOMs in parallel, record divergences
- Each `CorrectionDiff` is a frozen dataclass: `element_tag`, `attribute`, `change_type`, `old_value`, `new_value`

### Step 4: Blueprint Engine Hook (`app/ai/blueprints/engine.py`)

Add correction recording after agent nodes return modified HTML. Hook location: in `_run_node()` or equivalent, after `result = await node.execute(context)`:

```python
# After node execution, record correction if HTML changed
if (
    settings.correction_tracker.enabled
    and context.html
    and result.html
    and context.html != result.html
    and result.handoff  # Only record successful agent corrections
):
    try:
        tracker = CorrectionTracker(data_dir=Path("data"))
        await tracker.record_correction(
            agent=result.handoff.agent_name,
            original_html=context.html,
            corrected_html=result.html,
        )
    except Exception:
        logger.warning("correction_tracker.record_failed", exc_info=True)
```

This is fire-and-forget — never blocks or crashes the pipeline (same pattern as insight_bus LAYER 17).

### Step 5: API Endpoints (`app/design_sync/routes.py`)

Add 3 admin-only endpoints:

```python
@router.get("/correction-patterns", response_model=list[CorrectionPatternResponse])
async def list_correction_patterns(
    min_occurrences: int = Query(default=5, ge=1),
    min_confidence: float = Query(default=0.9, ge=0.0, le=1.0),
    agent: str | None = Query(default=None),
    user: User = Depends(require_role("admin")),
) -> list[CorrectionPatternResponse]:
    tracker = CorrectionTracker(data_dir=Path("data"))
    patterns = tracker.get_frequent_patterns(
        min_occurrences=min_occurrences, min_confidence=min_confidence
    )
    if agent:
        patterns = [p for p in patterns if p.agent == agent]
    return [CorrectionPatternResponse.from_pattern(p) for p in patterns]

@router.get("/correction-patterns/suggestions", response_model=list[ConverterRuleSuggestionResponse])
async def get_rule_suggestions(
    user: User = Depends(require_role("admin")),
) -> list[ConverterRuleSuggestionResponse]:
    tracker = CorrectionTracker(data_dir=Path("data"))
    suggestions = tracker.suggest_converter_rules()
    return [ConverterRuleSuggestionResponse.from_suggestion(s) for s in suggestions]

@router.post("/correction-patterns/{pattern_hash}/approve")
async def approve_correction_rule(
    pattern_hash: str = Path(...),
    user: User = Depends(require_role("admin")),
) -> dict[str, str]:
    tracker = CorrectionTracker(data_dir=Path("data"))
    tracker.approve_rule(pattern_hash)
    return {"status": "approved", "pattern_hash": pattern_hash}
```

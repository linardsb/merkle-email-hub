# Plan: 52.7 Rule 9 — Dark-Mode Contrast on Nested Cards + Identity Exception — STUB

**Status:** Deferred. Detailed plan promotes when 52.7 starts.

## Summary

Every nested coloured surface gets its own `class="card-bg-N"` + matching `@media (prefers-color-scheme: dark) { .card-bg-N { background-color: <darkColor> !important } }`. Light→dark mapping from `design_system.py:dark_palette` (fallback: shift L by –40, hue preserved).

**Identity exception**: physical-card surfaces (50.7's `is_physical_card_surface=True`) opt OUT — stay white in dark mode (membership cards with barcode + logo, credit cards, boarding passes).

**User-info row** text on dark-flipped bg gets `class="footer-strong"` + `!important` color overrides (name/number stays readable; divider pipe `|` stays a neutral mid-grey).

## Approximate Files Affected

- `app/design_sync/component_renderer.py` — emit `card-bg-N` classes on each nested coloured surface; skip when `is_physical_card_surface=True`
- `app/design_sync/component_renderer.py` — emit matching dark-mode CSS rules in `<style>` block
- `app/projects/design_system.py` — already has `dark_palette`; extend with deterministic L-40 fallback if no mapping
- `email-templates/shells/*.html` — `.footer-strong` CSS rule for user-info row
- `app/design_sync/component_matcher.py` — emit dark-mode metadata to renderer (which sections need flip)
- New: `app/design_sync/tests/test_rule_9.py`

## Dependencies

- **Depends on:** 50.4 (`inner_bg`/`container_bg` distinction), 50.7 (`is_physical_card_surface` flag), 52.3 (`add_class` mutator)
- **Blocks:** Nothing downstream

## Open Questions

- `card-bg-N` class naming: numbered per section, or hashed from bg color for de-duplication?
- Anchor tags inside class-controlled parents: needs `.parent-class a { color: ... !important }` override (doc note). Sweep for all `<a>` inside themed sections.
- Stat / metric numbers with inline `color: #000000` need their own dark classes — extend to all `<td>` with inline color.

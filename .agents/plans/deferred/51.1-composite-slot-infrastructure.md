# Plan: 51.1 Composite Slot Type Infrastructure (Part 4 §3) — STUB

**Status:** Deferred. Detailed plan promotes when 51.1 starts. Phase 50 surfaces shape decisions.

## Summary

`SlotFill` extended with a discriminated union: `string | composite[ComponentMatch]`. Adds a sub-renderer that recurses into composite slot values. Component HTMLs gain `data-slot-composite="<slot_name>"` markup. Builder/renderer consume the recursive form. This is the foundational infrastructure that lets cards contain tag-pills + CTAs, zigzag rows contain spec-lists, footers contain logo + barcode + social-row.

Per `docs/architecture/opus-figma-to-html-process.md` Part 4 §3: *"This is the single biggest leverage point — Gaps 2, 3, 4 collapse to 'add a composite slot'."*

- `app/design_sync/component_matcher.py` — `SlotFill` type extension, recursive fill builders
- `app/design_sync/component_renderer.py` — sub-renderer for composite values

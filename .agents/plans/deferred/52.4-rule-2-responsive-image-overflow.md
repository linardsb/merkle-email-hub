# Plan: 52.4 Rule 2 — Responsive Image Overflow on Stacked Columns — STUB

**Status:** Deferred. Detailed plan promotes when 52.4 starts.

## Summary

When `class="column"` triggers `display:block;width:100%`, emit `class="bannerimg"` on every child image. Class sets `width:100% !important; max-width:100% !important; height:auto !important` at `max-width: 599px`.

Hub has `.bannerimg` in every shell template but historically without `max-width:100% !important` — the same bug we just fixed for LEGO.

- `app/design_sync/component_renderer.py` — emit `bannerimg` on every image inside `.column`-stacked parent
- `app/design_sync/component_matcher.py` — emit `__html_attr_class_add` override (50.5 convention) on column-layout sections
- `email-templates/shells/*.html` — verify `.bannerimg` CSS includes `max-width:100% !important`
- New: `app/design_sync/tests/test_rule_2.py`

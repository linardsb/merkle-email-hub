## Summary

`CorrectionResult` extended with `wrap_in: dict | None` (parent tag + style) and `add_class: list[str]` per target element. Applicator can wrap existing elements and add classes, not just mutate styles/content/images on existing elements.

Today's `correction_applicator.py:apply_corrections()` only mutates styles/content/images on existing elements. Categories like *card-with-N-children collapsing* (Rule 1) and *responsive-image-overflow* (Rule 2) require **structural mutations**.

# Plan: 53.4 Transition Band Detection (Gap 8) — STUB

**Status:** Deferred. Detailed plan promotes when 53.4 starts.

## Summary

Single image with `width ≥ 95% of container_width` + bottom edge color matches next section's top edge color (50.2 supplies via boundary classifier) + dimensions < 70px tall → `transition_band` type, zero padding on outer cell.

Today: torn-paper/wave decorative bands are classified as `image-block` or `full-width-image` (correct), but the renderer surrounds them with default 32px padding which breaks the seamless wrapper-to-wrapper transition.

## Approximate Files Affected

- `app/design_sync/figma/layout_analyzer.py:EmailSectionType` — add `TRANSITION_BAND` value
- `app/design_sync/component_matcher.py:_score_candidates` — heuristic for transition band classification
- `app/design_sync/component_matcher.py:_build_token_overrides` — emit zero-padding overrides on transitions
- New: `app/design_sync/tests/test_transition_band.py`

## Dependencies

- **Depends on:** 50.2 (boundary classifier — bottom/top color match)
- **Blocks:** Nothing downstream

## Open Questions

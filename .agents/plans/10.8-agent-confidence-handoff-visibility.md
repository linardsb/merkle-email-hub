# Plan: 10.8 — Agent Confidence & Handoff Visibility

## Context

Agent responses (both chat mode and blueprint runs) include confidence scores (0.0–1.0) and structured handoff metadata (decisions, warnings, component refs). Currently this data is only visible in the blueprint run detail dialog. This task surfaces confidence scores and handoff decisions directly on chat message bubbles and blueprint pipeline views, so users can quickly assess agent quality without digging into detail dialogs.

**Two surfaces:**
1. **Chat message bubbles** — Show confidence badge + "needs review" indicator on agent (non-chat) responses
2. **Blueprint pipeline view** — Expandable per-node handoff decision panel inline in the pipeline timeline

## Backend Data Available (no backend changes needed)

- All 9 agent response schemas include `confidence: float | None`
- Chat SSE streaming sends `[DONE]` signal — confidence can be sent as a final metadata chunk
- Blueprint `HandoffSummary` has: `agent_name`, `decisions[]`, `warnings[]`, `component_refs[]`, `confidence`
- Blueprint `BlueprintProgress` has: `node_name`, `node_type`, `status`, `iteration`, `summary`, `duration_ms`
- `HandoffSummary` is already imported from `@email-hub/sdk` in blueprint components

**Key insight:** In chat mode, the streaming API uses OpenAI-compatible SSE format. Confidence is embedded as `<!-- CONFIDENCE: X.XX -->` in the HTML output. The frontend can parse this from the final message content after streaming completes, rather than requiring a separate API field. For non-HTML agents (content, knowledge), confidence is in the JSON response but not in the SSE stream — so chat-mode confidence display is limited to agents that return via the non-streaming (scaffolder generate) or JSON paths. For the MVP, we extract confidence from completed message content using the same regex the backend uses.

### Step 2: Extract confidence from completed messages in useChat

**File:** `cms/apps/web/src/hooks/use-chat.ts`

Add a helper function to extract confidence from HTML content (mirrors backend's `extract_confidence`):

```typescript
const CONFIDENCE_RE = /<!--\s*CONFIDENCE:\s*([\d.]+)\s*-->/;

function extractConfidence(content: string): number | null {
  const match = content.match(CONFIDENCE_RE);
  if (!match?.[1]) return null;
  const val = parseFloat(match[1]);
  return val >= 0 && val <= 1 ? val : null;
}

function stripConfidenceComment(content: string): string {
  return content.replace(CONFIDENCE_RE, "").trim();
}
```

When streaming completes (the `setMessages` call that sets `isStreaming: false`), also extract and set confidence + strip the comment from content:


### Step 3: Create ConfidenceIndicator component for chat

**File:** `cms/apps/web/src/components/workspace/chat/confidence-indicator.tsx`

A compact inline badge that shows confidence score color-coded. Reuses the same threshold logic as `ConfidenceBar` in `blueprint/shared.tsx` but in a compact chat-friendly format:

```typescript
"use client";

import { useTranslations } from "next-intl";
import { AlertTriangle } from "lucide-react";
import { Badge } from "@email-hub/ui/components/ui/badge";

const REVIEW_THRESHOLD = 0.5;

interface ConfidenceIndicatorProps {
  confidence: number;
  showReviewBadge?: boolean;
}

export function ConfidenceIndicator({ confidence, showReviewBadge = true }: ConfidenceIndicatorProps) {
  const t = useTranslations("workspace");
  const pct = Math.round(confidence * 100);

  const colorClass =
    confidence > 0.8
      ? "text-chart-2"          // green — high confidence
      : confidence >= REVIEW_THRESHOLD
        ? "text-muted-foreground"  // gray — medium
        : "text-destructive";      // red — low / needs review

  const needsReview = confidence < REVIEW_THRESHOLD;

  return (
    <div className="mt-1.5 flex items-center gap-1.5">
      <span className={`text-[10px] font-medium ${colorClass}`}>
        {t("chatConfidence", { pct })}
      </span>
      {needsReview && showReviewBadge && (
        <Badge variant="destructive" className="gap-1 px-1.5 py-0 text-[10px]">
          <AlertTriangle className="h-2.5 w-2.5" />
          {t("chatNeedsReview")}
        </Badge>
      )}
    </div>
  );
}
```

### Step 5: Create inline NodeHandoffPanel for pipeline timeline

**File:** `cms/apps/web/src/components/workspace/blueprint/node-handoff-panel.tsx`

An expandable panel that can be toggled per-node in the pipeline timeline. Shows handoff decisions/warnings/components for that specific node:

```typescript
"use client";

import { useState } from "react";
import { useTranslations } from "next-intl";
import { ChevronDown, ChevronRight, AlertTriangle } from "lucide-react";
import { Badge } from "@email-hub/ui/components/ui/badge";
import type { HandoffSummary } from "@email-hub/sdk";
import { ConfidenceBar } from "./shared";

interface NodeHandoffPanelProps {
  handoff: HandoffSummary;
}

export function NodeHandoffPanel({ handoff }: NodeHandoffPanelProps) {
  const t = useTranslations("blueprintRun");
  const [expanded, setExpanded] = useState(false);

  const hasContent =
    handoff.decisions.length > 0 ||
    handoff.warnings.length > 0 ||
    handoff.component_refs.length > 0;

  if (!hasContent && handoff.confidence === null) return null;

  return (
    <div className="mt-1">
      {/* Confidence inline (always visible) */}
      {handoff.confidence !== null && (
        <div className="flex items-center gap-2">
          <span className="text-[10px] text-muted-foreground">{t("confidence")}</span>
          <ConfidenceBar value={handoff.confidence} />
          {handoff.confidence < 0.5 && (
            <Badge variant="destructive" className="gap-1 px-1.5 py-0 text-[10px]">
              <AlertTriangle className="h-2.5 w-2.5" />
              {t("status.needs_review")}
            </Badge>
          )}
        </div>
      )}

      {/* Expandable decisions/warnings */}
      {hasContent && (
        <>
          <button
            type="button"
            onClick={() => setExpanded((v) => !v)}
            className="mt-0.5 flex items-center gap-1 text-[10px] text-muted-foreground transition-colors hover:text-foreground"
          >
            {expanded ? (
              <ChevronDown className="h-3 w-3" />
            ) : (
              <ChevronRight className="h-3 w-3" />
            )}
            {t("nodeHandoffToggle")}
          </button>

          {expanded && (
            <div className="mt-1 space-y-1.5 rounded border border-border bg-muted/30 p-2">
              {handoff.decisions.length > 0 && (
                <div>
                  <p className="text-[10px] font-medium text-muted-foreground uppercase tracking-wide">
                    {t("handoffDecisions")}
                  </p>
                  <ul className="mt-0.5 space-y-0.5">
                    {handoff.decisions.map((d, i) => (
                      <li key={i} className="flex items-start gap-1 text-[10px] text-foreground">
                        <span className="mt-1 h-1 w-1 shrink-0 rounded-full bg-muted-foreground" />
                        {d}
                      </li>
                    ))}
                  </ul>
                </div>
              )}

              {handoff.warnings.length > 0 && (
                <div>
                  <p className="text-[10px] font-medium text-destructive uppercase tracking-wide">
                    {t("handoffWarnings")}
                  </p>
                  <ul className="mt-0.5 space-y-0.5">
                    {handoff.warnings.map((w, i) => (
                      <li key={i} className="flex items-start gap-1 text-[10px] text-destructive">
                        <AlertTriangle className="mt-0.5 h-2.5 w-2.5 shrink-0" />
                        {w}
                      </li>
                    ))}
                  </ul>
                </div>
              )}

              {handoff.component_refs.length > 0 && (
                <div>
                  <p className="text-[10px] font-medium text-muted-foreground uppercase tracking-wide">
                    {t("handoffComponents")}
                  </p>
                  <div className="mt-0.5 flex flex-wrap gap-1">
                    {handoff.component_refs.map((ref) => (
                      <Badge key={ref} variant="secondary" className="text-[9px] px-1 py-0">
                        {ref}
                      </Badge>
                    ))}
                  </div>
                </div>
              )}
            </div>
          )}
        </>
      )}
    </div>
  );
}
```

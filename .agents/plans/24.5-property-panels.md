# Plan: Phase 24.5 — Visual Builder Property Panels & Section Configuration

## Context

The visual builder (Phase 24.4) has drag-drop canvas, component palette, preview, and undo/redo — but no way to edit section content or styling. `BuilderSection.slotFills` and `tokenOverrides` are populated as empty objects on drop. Property panels make sections useful by exposing slot content editing, design system–driven style controls, responsive toggles, and advanced options.

**Key constraint:** Panels are driven by `slot_definitions` from `ComponentVersion` (backend), so they dynamically reflect each component's configuration surface. The design system palette restricts color choices to on-brand values.

**i18n note:** i18n infrastructure was removed from the project. Use hardcoded English strings (no `useTranslations()`).

## Architecture Decisions

1. **Property panel as right sidebar** — follows existing `w-80` (320px) sidebar pattern (ToolSidebar, PresencePanel, DesignReferencePanel). Appears inside `VisualBuilderPanel` when `selectedSectionId` is set.
2. **Radix Tabs** for Content | Style | Responsive | Advanced — matches shadcn/ui pattern.
3. **No rich text editor dependency** — use `contentEditable` div with toolbar buttons (bold/italic/link) for rich text slots. Avoids adding Tiptap/Plate.js (~150KB). Output sanitized via DOMPurify. If richer editing is needed later, can upgrade to Tiptap.
4. **Slot definitions fetched alongside component HTML** — the `/api/v1/components/{id}/versions` endpoint already returns `slot_definitions` and `default_tokens`. Store on `BuilderSection` at drop time.
5. **Live preview** — `updateSection()` triggers re-render via `useBuilderPreview` → assembled HTML updates → preview iframe refreshes. No separate "Apply" button needed; changes are immediate.
6. **Token overrides** scoped to design system — `tokenOverrides` keys are semantic roles (`primary`, `heading_font`, etc.) not raw CSS properties.

### 2. `cms/apps/web/src/components/builder/panels/content-tab.tsx`
Dynamic slot editing based on `section.slotDefinitions`.

- Iterates `slotDefinitions` array → renders `SlotEditor` for each
- Current values from `section.slotFills[slot.slot_id]`
- On change: calls `onUpdate({ slotFills: { ...section.slotFills, [slotId]: value } })`
- Empty state: "No editable content slots" message

### 3. `cms/apps/web/src/components/builder/panels/slot-editor.tsx`
Generic input renderer based on `SlotDefinition.slot_type`.

Slot type → control mapping:
| slot_type | Control |
|-----------|---------|
| `headline`, `subheadline` | `Input` (single-line text) |
| `body` | `Textarea` with optional bold/italic/link toolbar |
| `cta` | Two inputs: button text + URL, plus style selector (filled/outlined/text) |
| `image` | URL input + "Upload" button placeholder + alt text input |
| `preheader`, `footer` | `Textarea` |
| `nav` | `Textarea` (comma-separated links) |
| `social` | Read-only note ("Configured in project design system") |
| `divider` | Height input (number, px) + color picker (from palette) |

- Shows `Label` with `slot.label || slot.slot_id`
- Shows required indicator (`*`) if `slot.required`
- Character count if `slot.max_chars` is set
- Placeholder from `slot.placeholder`

### 4. `cms/apps/web/src/components/builder/panels/style-tab.tsx`
Design token override controls.

- **Color overrides**: For each color role in `defaultTokens.colors`, show a swatch button that opens a popover with the design system palette colors as clickable swatches. NO freeform color input — palette only.
- **Font override**: `Select` dropdown with design system fonts (heading_font, body_font + any from `fonts` map).
- **Font size override**: `Select` with design system font sizes or custom number input.
- **Spacing**: Margin/padding inputs (top/right/bottom/left) with link toggle for uniform values. Values map to `tokenOverrides.spacing.*`.
- **Alignment**: Button group (AlignLeft/AlignCenter/AlignRight icons) for text alignment.
- **Background**: Palette color picker (same as above).

All changes write to `section.tokenOverrides`.

### 5. `cms/apps/web/src/components/builder/panels/responsive-tab.tsx`
Responsive behavior toggles.

- **Stack on mobile**: `Switch` toggle → `responsive.stackOnMobile`
- **Full width images on mobile**: `Switch` toggle → `responsive.fullWidthImageOnMobile`
- **Mobile font size**: `Input` (number + "px" suffix) → `responsive.mobileFontSize`
- **Preview toggle**: Two buttons (Monitor/Smartphone icons) that set a local `previewMode` state, passed up to parent for preview iframe width change

All changes write to `section.responsive`.

### 6. `cms/apps/web/src/components/builder/panels/advanced-tab.tsx`
Power user controls.

- **Custom CSS class**: `Input` with validation — only allow classes matching `/^[a-zA-Z_-][\w-]*$/`. Show warning if class not in design system.
- **MSO conditional**: `Checkbox` → `advanced.msoConditional`. Description: "Wrap in `<!--[if mso]>` for Outlook"
- **Dark mode overrides**: List of key-value pairs. Key = element description (text input), value = palette color picker. Add/remove rows.
- **HTML attributes**: Key-value list. Key validated: block `on*` attributes, block `javascript:` in values, block `style` attribute. Add/remove rows.
- **View Source**: `Button` that opens a `Dialog` with read-only CodeMirror showing the section's HTML.

Security validations on HTML attributes:
```typescript
const BLOCKED_ATTR_PREFIXES = ["on"];
const BLOCKED_ATTR_NAMES = ["style"];
const BLOCKED_VALUE_PATTERNS = [/^javascript:/i, /^data:/i, /^vbscript:/i];
```

# Plan: 25.10 Template Learning Pipeline — Self-Serve HTML Upload

## Context
Currently adding a new golden template requires hand-coding slot definitions, registering in `TemplateRegistry`, and creating eval cases. This bottleneck means only engineer-curated templates exist. Email teams have hundreds of battle-tested production templates that encode hard-won knowledge about client quirks, layout patterns, and ESP-specific tricks. Self-serve upload turns every production template into agent training data — the system gets smarter with use.

**Dependencies:** Phase 24.9 import annotator (`app/ai/agents/import_annotator/service.py`), Phase 11.25 design system (`app/ai/templates/models.py` — `DefaultTokens`), Knowledge base (`app/knowledge/`).

### 6. `app/templates/upload/analyzer.py` — Static HTML analysis (core)
```python
"""Static HTML analysis — detect sections, slots, tokens, ESP platform, complexity."""

from app.core.logging import get_logger
from lxml import html as lxml_html
import re

logger = get_logger(__name__)

class TemplateAnalyzer:
    """Analyzes raw HTML to extract template structure without LLM calls."""

    def analyze(self, sanitized_html: str) -> AnalysisResult:
        """Run full analysis pipeline. Returns structured result."""
        tree = lxml_html.fromstring(sanitized_html)
        sections = self._detect_sections(tree)
        slots = self._extract_slots(tree, sections)
        tokens = self._extract_tokens(tree)
        esp = self._detect_esp_platform(sanitized_html)
        complexity = self._measure_complexity(tree)
        layout = self._infer_layout_type(tree, sections, complexity)
        return AnalysisResult(sections=sections, slots=slots, tokens=tokens,
                              esp_platform=esp, complexity=complexity, layout=layout)

    def _detect_sections(self, tree) -> list[SectionInfo]:
        """Reuse import annotator's section detection heuristics.
        Look for: top-level <table> blocks, <!--[if mso]> wrappers,
        semantic containers, role='presentation' tables.
        Each distinct top-level structure = one section.
        Detect component type from content: hero (large image + CTA),
        header (logo/nav), footer (small text, unsubscribe), content (text blocks),
        columns (multi-column tables), CTA (button-heavy), divider (hr/spacer)."""
        ...

    def _extract_slots(self, tree, sections: list[SectionInfo]) -> list[SlotInfo]:
        """Identify fillable content regions within sections.
        Rules:
        - Text blocks (p, h1-h6, td with >20 chars text): text slot
        - Images (img with src, not tracking pixel <2x2): image slot
        - Links with button styling (padding, bg-color, border-radius): cta slot
        - Links in body text: url slot
        - Repeating structures (3+ siblings with same tag pattern): slot with max_items
        - Required: hero image, primary CTA. Optional: social links, preheader.
        - max_chars: estimate from container width (td width / ~8px per char)."""
        ...

    def _extract_tokens(self, tree) -> TokenInfo:
        """Extract DefaultTokens from inline styles.
        Colors: collect all hex values (#RGB, #RRGGBB), count frequency.
          - Most-used background-color → 'background'
          - Most-used color on text → 'text'
          - Button/CTA background-color → 'cta'
          - Link color → 'link'
          - Secondary text color → 'secondary'
          - Border color → 'border'
        Fonts: extract font-family stacks, assign 'heading' vs 'body' by context.
        Font sizes: map heading/body/small sizes.
        Spacing: extract common padding/margin values."""
        ...

    def _detect_esp_platform(self, html: str) -> str | None:
        """Detect ESP from personalisation syntax.
        - Liquid: {{ var }}, {% if %} → 'braze' or 'shopify'
        - Handlebars: {{#if}}, {{var}} (no spaces) → 'mailchimp' or 'sendgrid'
        - AMPscript: %%var%%, %%=func()=%% → 'salesforce'
        - ERB: <%= var %> → 'erb'
        - Braze-specific: {{${first_name}}}, connected_content → 'braze'
        - Mailchimp merge tags: *|FNAME|* → 'mailchimp'"""
        ...

    def _measure_complexity(self, tree) -> ComplexityInfo:
        """column_count, nesting_depth, mso_conditional_count, total_elements,
        table_nesting_depth, has_vml, has_amp."""
        ...

    def _infer_layout_type(self, tree, sections, complexity) -> str:
        """Return: newsletter, promotional, transactional, retention.
        Newsletter: 4+ sections, repeating content patterns, table of contents.
        Promotional: hero + CTA dominant, fewer sections, sale/discount language.
        Transactional: order/receipt/confirmation patterns, data tables.
        Retention: personalization-heavy, fewer images, re-engagement language."""
        ...
```

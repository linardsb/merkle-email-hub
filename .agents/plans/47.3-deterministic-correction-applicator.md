# Plan: 47.3 Deterministic Correction Applicator

## Context

Phase 47.2 (`visual_verify.py`) produces `list[SectionCorrection]` — VLM-identified CSS/content mismatches between Figma design screenshots and rendered HTML sections. Most corrections are simple inline style value changes (wrong color, padding, font-size). This module applies them deterministically without LLM calls. Only complex layout restructuring falls back to the existing `correct_visual_defects()` LLM path.

### Step 2: Section extraction helper

```python
_SECTION_RE = re.compile(r"<!--\s*section:(\S+)\s*-->")

def _extract_section_html(html: str, node_id: str) -> tuple[int, int] | None:
    """Return (start, end) char offsets for the section matching node_id.

    Sections are delimited by consecutive `<!-- section:ID -->` markers
    or by the marker and end-of-body.
    """
```

Logic:
1. Find all marker positions via `_SECTION_RE.finditer(html)`
2. Locate the marker whose group(1) starts with `node_id` (prefix match — handles `hero_1` matching `<!-- section:hero_1:content -->`)
3. Section spans from that marker's end to the next marker's start (or `</body>` / EOF)
4. Return `(start, end)` char offsets into the original HTML string

_DETERMINISTIC_TYPES: frozenset[str] = frozenset({"color", "font", "spacing", "image"})

Per-correction dispatch:

| `correction_type` | Handler |
|---|---|
| `color`, `font`, `spacing` | `_apply_style_correction()` — inline style property replacement |

| `image` | `_apply_image_correction()` — replace `width`/`height` attrs on `<img>` |

### Step 4: `_apply_style_correction()`

```python
_LAYOUT_SIMPLE_PROPS: frozenset[str] = frozenset({
    "width", "max-width", "min-width", "text-align", "vertical-align",
})

def _apply_style_correction(
    section_html: str,
    correction: SectionCorrection,
) -> str | None:
    """Replace a CSS property value in the matched element's inline style.

    Returns modified section HTML, or None if element/property not found.
    """
```

Algorithm:
1. Parse `section_html` as lxml fragment
2. Use `CSSSelector(correction.css_selector)` to find elements
3. For each matched element, read `style` attribute
4. Parse style string into `dict[str, str]` (split on `;`, then on `:`)
5. If `correction.css_property` exists and value matches `correction.current_value` (normalized), replace with `correction.correct_value`
6. If property exists but value differs from `current_value`, still replace (VLM may have approximated the current value)
7. Serialize style dict back to string, update element's `style` attribute
8. Serialize fragment back to string via `lxml.html.tostring()`

CSS value sanitization: reuse pattern from `converter.py:_sanitize_css_value()` — strip `; { } ' " \` and dangerous functions. Inline a simplified version to avoid import coupling:
```python
_UNSAFE_CSS_RE = re.compile(r"(?:expression|javascript|url\s*\(|@import)", re.I)

def _sanitize_css_value(value: str) -> str:
    if _UNSAFE_CSS_RE.search(value):
        return ""
    return re.sub(r'[;<>{}\'"\\]+', "", value).strip()
```

### Step 7: Section-scoped application glue

In `apply_corrections()`, for each correction:
1. Extract section HTML via `_extract_section_html(html, correction.node_id)`
2. If section not found → log warning, add to `skipped`, continue
3. Call appropriate handler on the section substring
4. If handler returns `None` → add to `skipped`, continue
5. Replace section substring in full HTML with modified version
6. Update section offsets for subsequent corrections (since HTML length may change)

Key: work on the full HTML string, splicing section substrings. This preserves everything outside sections exactly.

## Preflight Warnings

- `lxml.html.tostring()` may add `<html><body>` wrappers when serializing fragments — use `fragment_fromstring()` + `tostring()` with care, or work with string slicing for section replacement
- `CSSSelector` from `lxml.cssselect` supports a subset of CSS3 selectors — complex selectors (`:nth-child`, attribute selectors) may fail. VLM corrections use simple selectors (`h1`, `td`, `img`, `.class`)
- Section markers may contain colons in node IDs (e.g., `<!-- section:header:header -->`). The regex `(\S+)` captures the full ID including colons
- `lxml` normalizes HTML entities and attribute quoting — section replacement via string slicing (not full-document serialization) avoids unwanted normalization outside the target section

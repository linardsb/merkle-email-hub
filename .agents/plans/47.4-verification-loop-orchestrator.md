# Plan: 47.4 Verification Loop Orchestrator

## Context
Single-pass VLM comparison (47.2) catches obvious errors but may introduce new ones via corrections. A 2–3 iteration render→compare→fix cycle allows cascading corrections to converge toward design fidelity. The loop also detects regressions and stops before making things worse.

### Step 3: Add `run_verification_loop()` to `visual_verify.py`

After `compare_sections()` (line 164), add the loop function. Logic:

```python
async def run_verification_loop(
    html: str,
    design_screenshots: dict[str, bytes],
    sections: list[EmailSection],
    *,
    max_iterations: int | None = None,
    render_client: str = "gmail_web",
    viewport_width: int = 680,
) -> VerificationLoopResult:
```

**Per-iteration flow:**
1. Late-import `LocalRenderingProvider` and `crop_section` (avoid circular)
2. `await provider.render_screenshots(html, clients=[render_client])` → get `image_bytes`
3. Build `rendered_screenshots: dict[str, bytes]` by cropping full screenshot per section using `crop_section(image_bytes, int(s.y_position or 0), int(s.height or 200), viewport_width)`
4. `await compare_sections(design_screenshots, rendered_screenshots, html, sections, iteration=i)`
5. Record `VerificationResult` in `iterations` list
6. **Convergence check:** if `result.converged` or `result.fidelity_score >= target_fidelity` → break
7. **Apply corrections:** `apply_corrections(html, result.corrections, confidence_threshold=threshold)` → `CorrectionResult`
8. **Regression check:** if `result.fidelity_score < prev_fidelity` → revert to `prev_html`, set `reverted=True`, break
9. Update `html = correction_result.html`, accumulate `total_corrections_applied`

**Early returns:**
- If `vlm_verify_enabled` is False → return single-iteration empty result
- If `max_iterations` is None → read from config `vlm_verify_max_iterations`

**Error handling:**
- Rendering failure → log warning, break loop, return best result so far
- `compare_sections` or `apply_corrections` exceptions → log, break, return best

**Logging events:**
- `design_sync.verify_loop.starting` (max_iterations, section_count)
- `design_sync.verify_loop.iteration_complete` (iteration, fidelity, corrections_count)
- `design_sync.verify_loop.converged` (iteration, fidelity)
- `design_sync.verify_loop.regression_detected` (iteration, prev_fidelity, curr_fidelity)
- `design_sync.verify_loop.completed` (iterations, initial_fidelity, final_fidelity, total_corrections)

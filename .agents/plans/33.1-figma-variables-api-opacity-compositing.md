# Plan: 33.1 Figma Variables API + Opacity Compositing

## Context

`FigmaDesignSyncService` (`app/design_sync/figma/service.py`) uses only the legacy Styles endpoint. Figma's Variables API (GA since late 2023) is the default token source — files using it extract zero tokens today. Also: `_rgba_to_hex()` discards alpha, node `opacity` is ignored, gradients vanish, and strokes pollute the fill palette.

### Step 3: Add `_rgba_to_hex_with_opacity()` helper (after line 70)

Signature: `def _rgba_to_hex_with_opacity(r, g, b, fill_alpha=1.0, node_opacity=1.0, bg_hex="#FFFFFF") -> str`

Logic: `eff_alpha = fill_alpha * node_opacity`. If `>= 0.999`, delegate to `_rgba_to_hex`. Otherwise alpha-composite each channel: `final = src * eff_alpha + bg * (1 - eff_alpha)`.

### Step 4: Add `_gradient_midpoint_hex()` helper

Signature: `def _gradient_midpoint_hex(gradient_stops: list[dict]) -> str | None`

Average RGB of first and last stop. Return `None` if `< 2` stops or malformed.

### Step 5: Add `_fetch_variables()` method on `FigmaDesignSyncService`

Signature: `async def _fetch_variables(self, file_ref, access_token) -> dict | None`

- Call `GET /v1/files/{file_ref}/variables/local` + `/variables/published`
- 403 → log + return `None` (free plan, graceful fallback)
- 429 → raise `SyncFailedError` (consistent with existing pattern, see line 174)
- Other non-200 → log warning + return `None`
- Return `{"local": local_json, "published": pub_json}`

### Step 6: Add `_resolve_variable_alias()` method

Signature: `def _resolve_variable_alias(self, value, variables_by_id, mode_id, depth=0) -> Any`

Walk alias chain: if `value` is `{"type": "VARIABLE_ALIAS", "id": "..."}`, look up target var's `valuesByMode[mode_id]` and recurse. Max depth `_MAX_ALIAS_DEPTH = 10` — return `None` + log warning on exceed. Fall back to first available mode if `mode_id` not present.

### Step 7: Add `_parse_variables()` method

Signature: `def _parse_variables(self, raw, bg_hex="#FFFFFF") -> tuple[list[ExtractedColor], list[ExtractedTypography], list[ExtractedVariable], dict[str, str]]`

Parse `raw["local"]["meta"]["variableCollections"]` and `["variables"]`. Build mode name lookup from collections. For each variable:
1. Resolve values per mode via `_resolve_variable_alias`
2. Record as `ExtractedVariable`
3. If `resolvedType == "COLOR"`: resolve default mode value → `_rgba_to_hex_with_opacity` → add to colors (dedup by hex)
4. Variable name display: strip leading `"color/"` prefix if present

Typography from Variables is future work (33.3) — return empty list. Return `global_modes` dict for the first collection.

### Step 8: Update `sync_tokens_and_structure()` (line 161)

After fetching `file_data` + `styles_data` (line 187), add Variables-first logic:

1. Read `settings.design_sync.figma_variables_enabled` and `opacity_composite_bg`
2. If enabled: `await self._fetch_variables(file_ref, access_token)` in try/except
3. If variables returned and `_parse_variables` yields colors → use them, set `variables_source=True`
4. Else fallback to `self._parse_colors(file_data, styles_data, bg_hex=bg_hex)`
5. Build `ExtractedTokens` with new fields (`variables_source`, `modes`, `stroke_colors`, `variables`)

Add `ExtractedVariable` to the protocol import block at top of file.

### Step 9: Refactor `_walk_for_colors()` (line 329)

Three changes:
1. **Separate strokes** — add `stroke_colors` + `seen_stroke_hex` keyword args; strokes go to separate list, not `colors`
2. **Opacity compositing** — read `node.get("opacity", 1.0)`, pass to `_rgba_to_hex_with_opacity`; add `bg_hex` kwarg
3. **Gradient midpoints** — for `GRADIENT_LINEAR` fills, call `_gradient_midpoint_hex(gradientStops)`, add with name `"{node_name} (gradient midpoint)"`
4. **Topmost visible fill** — iterate `reversed(fills)` (last = topmost in Figma), skip `visible: False`, take first solid

### Step 10: Update `_parse_colors()` return type (line 217)

Change return to `tuple[list[ExtractedColor], list[ExtractedColor]]` — `(fill_colors, stroke_colors)`.

- Add `bg_hex` keyword param
- Use `_rgba_to_hex_with_opacity` for published style colors too
- Pass `stroke_colors` + `seen_stroke_hex` + `bg_hex` to `_walk_for_colors`
- Return `(colors, stroke_colors)`

### Step 11: Fix `_parse_node()` fill extraction (lines 558–576)

- Read `node_data.get("opacity", 1.0)`
- Iterate `reversed(raw_fills)` instead of forward

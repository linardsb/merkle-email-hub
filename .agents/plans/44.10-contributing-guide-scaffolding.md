# Plan: 44.10 Contributing Guide & New-Feature Scaffolding

## Context

30+ backend modules, 50+ make targets, 9 AI agents, 11 ESP connectors — but no contributor guide. The vertical slice pattern is consistent but implicit. Making it explicit with a guide + scaffolding tool reduces onboarding from days to hours.

## Research Summary

### Vertical Slice Pattern (from 3 slices: projects, components, connectors)

| File | Purpose |
|------|---------|
| `__init__.py` | Package marker (empty) |
| `models.py` | SQLAlchemy ORM models (`Base`, `TimestampMixin`, `SoftDeleteMixin`) |
| `schemas.py` | Pydantic I/O schemas |
| `exceptions.py` | Feature-specific exceptions (inherit `AppError`) |
| `repository.py` | Data access layer (`AsyncSession`-based) |
| `service.py` | Business logic, validation, logging |
| `routes.py` | FastAPI `APIRouter` with DI, rate limiting, auth |
| `tests/conftest.py` | `make_*()` factories, `mock_db`, `sample_*` fixtures |
| `tests/test_service.py` | Unit tests for service layer |
| `tests/test_routes.py` | Route/endpoint tests |

### Route Registration (`app/main.py:334-403`)

```python
# Top-level import
from app.{feature}.routes import router as {feature}_router
# Include
app.include_router({feature}_router)
# Conditional (feature-flagged)
if settings.{feature}.enabled:
    from app.{feature}.routes import router as {feature}_router
    app.include_router({feature}_router)
```

### AI Agent Structure (`app/ai/agents/{name}/`)

| File | Purpose |
|------|---------|
| `service.py` | Extends `BaseAgentService` — `agent_name`, `sanitization_profile`, `model_tier`, `build_system_prompt()`, `_post_process()` |
| `schemas.py` | `{Agent}Request`, `{Agent}Response` |
| `routes.py` | POST `/api/v1/agents/{name}/process` |
| `prompt.py` | `build_system_prompt()`, `detect_relevant_skills()` |
| `SKILL.md` | L1+L2 skill documentation |
| `skills/` | L3 skill files (loaded on-demand) |
| `skill-versions.yaml` | Skill versioning config |

Judge: `app/ai/agents/evals/judges/{name}.py` — 5 `JudgeCriteria`, `{Name}Judge` class with `build_prompt()` + `parse_response()`.

### ESP Connector Structure (`app/connectors/{provider}/`)

Protocol: `ConnectorProvider` in `app/connectors/protocol.py` — `async def export(html, name, credentials) -> str`.
Registration: `SUPPORTED_CONNECTORS` dict in `app/connectors/service.py`.
Mock ESP: `services/mock-esp/{provider}/routes.py`.

## Test Landscape

- **asyncio_mode = "auto"** — no need for explicit `@pytest.mark.asyncio`
- **Markers**: `integration` (real DB), `benchmark` (perf), `visual_regression` (screenshots)
- **Default run**: `make test` excludes integration, benchmark, visual_regression
- **Factory pattern**: `make_{model}(**overrides)` in per-feature `conftest.py`
- **Fixtures**: `mock_db` (AsyncMock), `sample_{model}`, `sample_{models}` (list)
- **Test linting relaxed**: S101, ANN, ARG, D, FBT, SIM, PERF, RET ignored in test files
- **14 conftest.py** files across features, root `conftest.py` provides `client`, `mock_settings`, `_clear_auth_cache`

## Implementation Steps

### Step 1: Create `scripts/scaffold-feature.sh`

Generates 8 files in `app/{name}/`:
- `__init__.py` — empty
- `models.py` — SQLAlchemy model stub with `TimestampMixin`
- `schemas.py` — Pydantic `{Name}Create`, `{Name}Response` stubs
- `exceptions.py` — `{Name}NotFoundError` inheriting `NotFoundError`
- `repository.py` — `{Name}Repository` with `get()`, `list()`, `create()`
- `service.py` — `{Name}Service` with constructor, `get_logger(__name__)`
- `routes.py` — `APIRouter` with `prefix="/api/v1"`, `tags=["{name}"]`, one GET endpoint
- `tests/__init__.py` — empty
- `tests/conftest.py` — `make_{name}()` factory, `mock_db` fixture

Script must:
- Accept feature name as `$1`, validate it (lowercase, no spaces)
- Derive PascalCase name for class names
- Create directory structure with `mkdir -p`
- Use heredocs with proper variable substitution
- Print next-steps instructions (register routes, create migration)
- Be idempotent (refuse to overwrite existing directory)

### Step 2: Add Makefile target

```makefile
scaffold-feature:  ## Scaffold a new vertical slice (name=feature_name)
	@bash scripts/scaffold-feature.sh $(name)
```

### Step 3: Create `CONTRIBUTING.md`

Three guided workflows, each with numbered steps referencing actual file paths and make targets:

**Section 1: Adding a Feature Slice**
1. Run `make scaffold-feature name=billing`
2. Edit generated models/schemas/service/routes
3. Register routes in `app/main.py` (show exact pattern)
4. Create migration: `make db-revision m="add billing tables"`
5. Run migration: `make db-migrate`
6. Write tests using generated `conftest.py` factories
7. Verify: `make check`

**Section 2: Adding an AI Agent**
1. Create `app/ai/agents/{name}/` directory structure
2. Extend `BaseAgentService` in `service.py` (required attrs: `agent_name`, `sanitization_profile`, `model_tier`)
3. Create `SKILL.md` with L1+L2 skill docs
4. Create `schemas.py` with request/response models
5. Create `routes.py` with POST endpoint
6. Register router in `app/main.py`
7. Create judge: `app/ai/agents/evals/judges/{name}.py` with 5 `JudgeCriteria`
8. Register judge in `JUDGE_REGISTRY` (`judges/__init__.py`)
9. Add synthetic test data: `app/ai/agents/evals/synthetic_data_{name}.py`
10. Verify: `make eval-golden`

**Section 3: Adding an ESP Connector**
1. Create `app/connectors/{esp}/` with `service.py` implementing `ConnectorProvider`
2. Add to `SUPPORTED_CONNECTORS` in `app/connectors/service.py`
3. Create mock endpoint in `services/mock-esp/{esp}/routes.py`
4. Register mock router in `services/mock-esp/main.py`
5. Add integration tests with mock ESP
6. Verify: `make check`

**Additional sections:**

- Code style summary (strict mypy+pyright, ruff 26 rule sets, structured logging pattern)

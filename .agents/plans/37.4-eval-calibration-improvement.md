## Implementation Steps

### Step 1: Create `scripts/eval-compare-verdicts.py`

Core comparison script with these functions:

```python
# scripts/eval-compare-verdicts.py
"""Compare pre/post golden-reference verdicts and report flip rates.

Usage:
    python scripts/eval-compare-verdicts.py \
        --pre-dir traces/pre_golden/ \
        --post-dir traces/ \
        --output traces/verdict_comparison.json
"""
```

**Functions:**

1. `load_verdicts(path: Path) -> list[dict]` — Load JSONL, return list of verdict dicts
2. `extract_criterion_verdicts(verdicts: list[dict]) -> dict[tuple[str,str], bool]` — Key: `(trace_id, criterion)`, value: `passed`. Flatten `criteria_results` array.
3. `compare_verdicts(pre_dir: Path, post_dir: Path) -> dict[str, dict]` — For each `*_verdicts.jsonl` in both dirs, compute per-criterion flip stats:
   - `total` — number of matched (trace_id, criterion) pairs
   - `flips` — count where `pre[passed] != post[passed]`
   - `flip_rate` — `flips / total` (0.0 if total == 0)
   - `pass_to_fail` — count of `True → False`
   - `fail_to_pass` — count of `False → True`
   - `pre_pass_rate` — pass rate before golden refs
   - `post_pass_rate` — pass rate after golden refs
4. `print_summary(results: dict) -> None` — Rich table to stdout, flag criteria with flip_rate > 0.20
5. `main()` — argparse CLI, calls compare + print + write JSON

**Output:** `traces/verdict_comparison.json`
```json
{
  "generated_at": "2026-03-29T...",
  "summary": {"total_criteria": 35, "high_flip_criteria": 3, "threshold": 0.20},
  "criteria": {
    "scaffolder:brief_fidelity": {"total": 12, "flips": 1, "flip_rate": 0.083, ...},
    ...
  },
  "priority_review": ["scaffolder:email_layout", "dark_mode:color_coherence", ...]
}
```

### Step 2: Add Makefile targets

```makefile
eval-rejudge: ## Re-run judges WITHOUT skip-existing (overwrites verdicts)
	uv run python -m app.ai.agents.evals.judge_runner --agent all --traces traces --output traces --mode hybrid

eval-compare: ## Compare pre/post golden-reference verdicts
	uv run python scripts/eval-compare-verdicts.py --pre-dir traces/pre_golden --post-dir traces --output traces/verdict_comparison.json
```

### Step 3: Create tests `app/ai/agents/evals/tests/test_verdict_comparison.py`

Test cases (using `tmp_path` for JSONL fixtures):

1. `test_no_flips` — identical pre/post → flip_rate=0.0
2. `test_all_flips` — every verdict flipped → flip_rate=1.0
3. `test_partial_flips` — mix → correct counts for pass_to_fail, fail_to_pass
4. `test_missing_post_trace` — trace in pre but not post → excluded from comparison
5. `test_empty_files` — graceful handling of empty JSONL
6. `test_multiple_agents` — aggregates across scaffolder + dark_mode files
7. `test_priority_review_threshold` — flip_rate > 0.20 → appears in priority_review list
8. `test_pass_rate_calculation` — pre/post pass rates computed correctly

### Step 4: Execution workflow (manual, documented in plan)

```bash

# 2. Re-run judges (golden refs now in prompts)

# 3. Compare

# 4. Record improvement (manual per-agent)

```

## Preflight Warnings

- Current verdicts are mock/dry-run data ("Pass: mock evaluation of ..."). Re-running with `--dry-run` will produce identical mock verdicts → 0% flip rate. Must run in `hybrid` or `llm` mode with real LLM to measure actual golden reference impact.
- `--skip-existing` in Makefile `eval-judge` target would skip everything. The new `eval-rejudge` target intentionally omits this flag.
- knowledge + innovation have no existing verdicts → no pre/post comparison possible for those 2 agents. Script should handle missing pre files gracefully.

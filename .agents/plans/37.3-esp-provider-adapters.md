# 37.3: ESP Provider Adapters (Mailchimp, SendGrid, ActiveCampaign, Iterable, Brevo)

## Goal

Add 5 new ESP provider adapters following the existing `ESPSyncProvider` Protocol pattern. Each adapter: ~150-170 lines, implements validate/list/get/create/update/delete.

## Depends On

- Nothing — can run in parallel with 37.1 and 37.2

- Protocol: `app/connectors/sync_protocol.py` — `ESPSyncProvider` with 6 methods
- Existing providers to mirror: `app/connectors/klaviyo/sync_provider.py` (169 lines), `app/connectors/hubspot/sync_provider.py` (157 lines)
- HTTP resilience: `app/connectors/http_resilience.py` — `resilient_request()` with retry logic
- Registry: `app/connectors/sync_service.py` — `PROVIDER_REGISTRY` dict
- Config: `app/core/config.py` — `ESPSyncConfig`

## Implementation Pattern

Each provider follows this structure (copy from Klaviyo/HubSpot):

```python
class {Esp}SyncProvider:
    """ESPSyncProvider implementation for {ESP}."""

    def __init__(self, base_url: str | None = None) -> None:
        self._base_url = base_url or get_settings().esp_sync.{esp}_base_url

    def _headers(self, credentials: dict[str, str]) -> dict[str, str]:
        ...  # ESP-specific auth header

    async def validate_credentials(self, credentials: dict[str, str]) -> bool:
        ...  # GET a lightweight endpoint, return True if 2xx

    async def list_templates(self, credentials: dict[str, str]) -> list[ESPTemplate]:
        ...  # GET + paginate + map to ESPTemplate

    async def get_template(self, template_id: str, credentials: dict[str, str]) -> ESPTemplate:
        ...  # GET by ID + map

    async def create_template(self, name: str, html: str, credentials: dict[str, str]) -> ESPTemplate:
        ...  # POST + map

    async def update_template(self, template_id: str, html: str, credentials: dict[str, str]) -> ESPTemplate:
        ...  # PATCH/PUT + map

    async def delete_template(self, template_id: str, credentials: dict[str, str]) -> bool:
        ...  # DELETE + return success
```

All HTTP via `resilient_request()` from `http_resilience.py`. All responses mapped to `ESPTemplate` dataclass. Structured logging per operation.

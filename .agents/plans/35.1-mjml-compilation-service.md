# Plan: 35.1 MJML Compilation Service in Maizzle Sidecar

## Context
Add MJML (`mjmlio/mjml`, MIT) as an npm dependency to the Maizzle sidecar and expose `POST /compile-mjml`. Wire a Python `compile_mjml()` method on `DesignConverterService` to call it. MJML handles responsive columns, ghost tables, MSO conditionals, CSS inlining — replacing ~60% of hand-rolled HTML generation in `converter.py`.

### Step 2: Add `/compile-mjml` endpoint to sidecar

**`services/maizzle-builder/index.js`**:

1. Create `services/maizzle-builder/mjml-compile.js` with pure `compileMjml(mjmlStr, options)` function (see Step 3 tests for expected behavior).

2. Import at top of `index.js` (after existing imports, ~L19):

The `mjml-compile.js` module reads MJML package version internally.

3. Update `GET /health` (L125) — add `mjml_version: mjmlVersion` to response.

4. Add endpoint after `/preview` (~L258, before `app.listen`):
```javascript
app.post("/compile-mjml", async (req, res) => {
  const start = Date.now();
  const { mjml, options = {}, target_clients } = req.body;

  if (!mjml || typeof mjml !== "string") {
    return res.status(400).json({ error: "mjml field is required and must be a string" });
  }

  try {
    const result = compileMjml(mjml, options);

    let html = result.html;
    let optimization = null;

    if (ontologyVersion && target_clients?.length) {
      const r = await optimizeCss(html, target_clients);
      html = r.optimizedHtml;
      optimization = r.optimization;
    }

    res.json({
      html,
      errors: result.errors || [],
      build_time_ms: Date.now() - start,
      ...(optimization && { optimization }),
    });
  } catch (err) {
    console.error("MJML compilation failed:", err.message);
    res.status(500).json({
      error: "MJML compilation failed",
      detail: err.message,
      build_time_ms: Date.now() - start,
    });
  }
});
```

```javascript
import mjml2html from 'mjml';
export function compileMjml(mjmlStr, options = {}) {

    keepComments: false,
    beautify: options.beautify || false,

}
```

```python
    """Result of MJML compilation via Maizzle sidecar."""

```

2. Add imports at top of `converter_service.py`:

```python
from app.core.config import get_settings

```

3. Add method to `DesignConverterService` (standalone async — class has no `__init__`):
```python
async def compile_mjml(
    self,
    mjml: str,
    *,
    minify: bool = True,
    target_clients: list[str] | None = None,
) -> MjmlCompileResult:
    """Compile MJML markup to email HTML via the Maizzle sidecar."""
    url = get_settings().maizzle_builder_url
    payload: dict[str, object] = {
        "mjml": mjml,
        "options": {"minify": minify},
    }
    if target_clients:
        payload["target_clients"] = target_clients

    try:
        async with httpx.AsyncClient(timeout=30.0) as client:
            response = await client.post(f"{url}/compile-mjml", json=payload)
            response.raise_for_status()
            data = response.json()

            errors = [
                MjmlError(
                    line=e.get("line", 0),
                    message=e.get("message", ""),
                    tagName=e.get("tagName", ""),
                )
                for e in data.get("errors", [])
            ]
            return MjmlCompileResult(
                html=str(data["html"]),
                errors=errors,
                build_time_ms=float(data.get("build_time_ms", 0)),
                optimization=data.get("optimization"),
            )
    except httpx.ConnectError as exc:
        logger.error("design_sync.mjml_compile_unavailable", error=str(exc))
        raise MjmlCompileError("Cannot connect to maizzle-builder service") from exc
    except httpx.HTTPStatusError as exc:
        logger.error("design_sync.mjml_compile_error", status_code=exc.response.status_code)
        raise MjmlCompileError("MJML compilation failed") from exc
```

## Preflight Warnings

- **`index.js` does NOT export `app`** — L258 calls `app.listen()` directly. Existing tests only test pure functions from separate modules. New MJML logic must be extracted into `mjml-compile.js` for testability.
- **`converter_service.py` has no `httpx` import** — must add `import httpx` and `from app.core.config import get_settings`.
- **`converter_service.py` has no `__init__`** — `DesignConverterService` is a plain class with only methods. Use `get_settings().maizzle_builder_url` inside the method, not `self._builder_url`.
- **No build error types in `design_sync`** — `BuildServiceUnavailableError`/`BuildFailedError` are in `email_engine/exceptions.py`. Add `MjmlCompileError(AppError)` to `design_sync/exceptions.py` instead of cross-feature import.
- MJML v4 vs v5: v4 is stable; v5 is alpha. Use `^4.15.0`.
- `ontologyVersion` is a module-level variable in `index.js` — loaded at startup from `data/ontology.json`. The `/compile-mjml` endpoint can reference it directly (same closure).
- `optimizeCss()` in `index.js` extracts `<style>` blocks from HTML — MJML outputs standard HTML with `<style>`, so this works.

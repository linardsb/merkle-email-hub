## Context

## Implementation Steps

### Step 1: Manifest schema (`app/design_sync/tests/manifest_schema.py`)

```python
from __future__ import annotations
from pydantic import BaseModel

class CTAExpectation(BaseModel):
    text: str
    bg_color: str | None = None
    has_vml: bool = False

class ComponentExpectation(BaseModel):
    index: int | None = None
    match_by: str = "content"          # "content" | "index" | "type"
    content_hint: str = ""
    expected_component: str
    repeat_count: int | None = None
    container_bgcolor: str | None = None

class TokenExpectations(BaseModel):
    primary_font: str | None = None
    banned_fonts: list[str] = []
    text_color: str | None = None
    banned_colors: list[str] = []

class SectionExpectations(BaseModel):
    count: int
    tolerance: int = 1
    components: list[ComponentExpectation] = []

class CaseManifest(BaseModel):
    name: str
    figma_file: str | None = None
    figma_node: str | None = None
    description: str = ""
    reference_only: bool = False       # True = assertions run against expected.html, not converter output
    sections: SectionExpectations
    tokens: TokenExpectations | None = None
    ctas: list[CTAExpectation] = []
    required_content: list[str] = []
    forbidden_content: list[str] = []
    patterns: list[str] = []
```

### Step 3: Data-driven test file (`app/design_sync/tests/test_converter_data_regression.py`)

Structure:
```python
_DEBUG_DIR = Path(__file__).resolve().parents[3] / "data" / "debug"

def _discover() -> list[Path]:
    return sorted(p.parent for p in _DEBUG_DIR.glob("*/manifest.yaml"))

# Parametrize fixture
@pytest.fixture(params=_discover(), ids=lambda p: p.name)
def case(request) -> tuple[Path, CaseManifest, ConversionResult]:
    case_dir = request.param
    manifest = load_case_manifest(case_dir)
    result = run_case_conversion(case_dir)
    return case_dir, manifest, result
```

**Universal assertions** (no manifest needed, run on every case):

| Test | Assertion |
|------|-----------|
| `test_no_nested_p_tags` | No `<p><p>` or `</p></p>` |
| `test_no_empty_sections` | No section markers with zero content between them |
| `test_valid_html_structure` | lxml parses without error, all `<table>` properly closed |
| `test_no_bare_layout_divs` | No `<div>` with `width`/`flex`/`float`/`display:block` in style |
| `test_mso_conditionals_balanced` | Count `<!--[if mso]>` == count `<![endif]-->` |
| `test_images_have_dimensions` | All `<img>` have `width` attr (height optional for responsive) |
| `test_slot_fill_rate` | `data-slot` with non-default content >= 80% (warning, not fail) |

**Manifest-driven assertions** (skip if field not set):

| Test | Reads from manifest |
|------|-------------------|
| `test_section_count` | `sections.count` ± `sections.tolerance` |
| `test_component_selection` | `sections.components[]` — find by `content_hint` or `index`, assert `component_slug` |
| `test_repeating_groups` | `components[].repeat_count` — assert RepeatingGroup with correct count |
| `test_required_content` | `required_content[]` — all strings present in HTML |
| `test_forbidden_content` | `forbidden_content[]` — no strings present in HTML |
| `test_font_family` | `tokens.primary_font` present, `tokens.banned_fonts` absent |
| `test_text_color` | `tokens.text_color` on text `<td>`s, `tokens.banned_colors` absent |
| `test_cta_colors` | `ctas[].text` → find CTA, assert `bg_color` |
| `test_cta_vml` | `ctas[].has_vml` → assert `v:roundrect` with matching `fillcolor` |
| `test_container_bgcolors` | `components[].container_bgcolor` → assert wrapper table bgcolor |

**Structural diff** (soft — metadata only):
- If `expected.html` exists, compute section-by-section diff
- Write `StructuralDiffResult` to `report.json`, not a hard fail

**Metrics** (`report.json` per case):
```python
@dataclass(frozen=True)
class CaseMetrics:
    section_count_accuracy: float   # 1 - abs(actual - expected) / expected
    component_match_accuracy: float # correct / total expectations
```

### Step 8: Handle missing structure.json/tokens.json

**Problem:** `data/debug/reframe/` has `raw_figma.json` + `expected.html` but no `structure.json`/`tokens.json`. The extract pipeline (`extract.py:256-290`) can generate `structure.json` offline from `raw_figma.json` via `_parse_node()`, but `tokens.json` requires a **live Figma API call** (full-file fetch for style/variable extraction). There is no offline path for tokens.

**Solution — two parts:**

**(a) Skip guard in `run_case_conversion()`:**
```python
def run_case_conversion(case_dir: Path) -> ConversionResult | None:
    structure_path = case_dir / "structure.json"
    tokens_path = case_dir / "tokens.json"
    if not structure_path.exists() or not tokens_path.exists():
        return None  # caller does pytest.skip()
    ...
```

In the fixture:
```python
if result is None:
    pytest.skip(f"Missing structure.json or tokens.json in {case_dir.name} — run extract first")
```

**(b) One-time generation step (documented, not automated):**
```bash
# Requires FIGMA_TOKEN env var
python -m app.design_sync.diagnose.extract \
  --figma-url "https://www.figma.com/design/VUlWjZGAEVZr3mK1EawsYR/..." \
  --node-id 2833-1491 --label reframe
```
This generates `structure.json` + `tokens.json` in `data/debug/reframe/`. Commit the generated files so CI never needs a Figma token.

**(c) Reference-only mode for cases with only `expected.html`:**
Cases that have `manifest.yaml` + `expected.html` but no converter inputs can still run content-based assertions (`test_required_content`, `test_forbidden_content`, `test_font_family`, etc.) directly against `expected.html`. Add a `reference_only: bool` field to `CaseManifest` — when `true`, assertions run against `expected.html` instead of converter output. This lets REFRAME be useful immediately even before generating structure/tokens.

## Preflight Warnings

- **Existing `test_snapshot_regression.py`** must NOT be broken — the new data-driven tests are additive. The old file stays as-is for backward compat with the existing 3 cases.
- **`_run_conversion()` is synchronous** — calls `DesignConverterService().convert()` which is sync. No async needed in the new tests.
- **REFRAME case missing structure.json/tokens.json** — the test framework must handle cases that only have `manifest.yaml` + `raw_figma.json` (parse structure from raw Figma JSON, or skip gracefully).
- **Existing `data/debug/manifest.yaml`** is a top-level manifest used by the old snapshot tests. The new per-case `manifest.yaml` files are separate — do NOT break the top-level manifest. Both systems coexist.
- **lxml dependency** — already in the project (used by `correction_applicator.py`). Safe to use for HTML validation assertions.

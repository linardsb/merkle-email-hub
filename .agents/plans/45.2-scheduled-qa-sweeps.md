# Plan: 45.2 Scheduled QA Sweeps Across Active Templates

## Context

Ontology updates (`make sync-ontology`) can silently break CSS compatibility scores. Dependency updates (Renovate) can change Maizzle/PostCSS behavior. A daily QA sweep catches regressions before users encounter them.

### Step 3: Create `app/scheduling/jobs/qa_sweep.py`

Core structure:

```python
@scheduled_job(cron="0 6 * * *")
async def qa_sweep() -> None:
    """Run QA checks on all active project templates, detect regressions."""
    settings = get_settings()
    redis = await get_redis()
    threshold = settings.scheduling.qa_sweep_regression_threshold
    check_names = settings.scheduling.qa_sweep_checks
    started_at = datetime.now(UTC)

    # 1. Load previous sweep baseline from Redis
    prev_raw = await redis.get("scheduling:qa_sweep:latest")
    prev_scores: dict[str, dict[str, float]] = json.loads(prev_raw) if prev_raw else {}

    # 2. Query active templates with latest version
    async with get_db_context() as db:
        rows = await db.execute(
            select(Template, TemplateVersion)
            .join(TemplateVersion, TemplateVersion.template_id == Template.id)
            .where(
                Template.deleted_at.is_(None),
                Template.status != "archived",
            )
            .distinct(Template.id)
            .order_by(Template.id, TemplateVersion.version_number.desc())
        )
        # Use subquery or window function for "latest version per template"
        # See implementation for exact approach

    # 3. Run QA checks (bounded concurrency via semaphore)
    sem = asyncio.Semaphore(5)
    results = []

    async def check_template(template, version):
        async with sem:
            async with get_db_context() as db:
                svc = QAEngineService(db)
                req = QARunRequest(
                    html=version.html_source,
                    template_version_id=version.id,
                    project_id=template.project_id,
                )
                return await svc.run_checks(req)

    # 4. Gather results, build score map, detect regressions
    # 5. Store in Redis: scheduling:qa_sweep:{date} + scheduling:qa_sweep:latest
    # 6. Log regressions, emit event placeholder for 45.5
```

**Redis storage keys:**
- `scheduling:qa_sweep:{YYYY-MM-DD}` — full sweep result JSON (TTL 30 days)
- `scheduling:qa_sweep:latest` — score map for next sweep's baseline comparison

**Score map structure** (stored in `latest`):
```json
{
  "tmpl:{template_id}": {
    "html_validation": 0.95,
    "css_support": 0.88,
    "css_audit": 0.92
  }
}
```

**Sweep result structure** (stored per date):
```json
{
  "sweep_date": "2026-04-02T06:00:00Z",
  "duration_ms": 45000,
  "total_templates": 42,
  "regressions": [
    {
      "template_id": 142,
      "template_name": "Newsletter",
      "project_id": 5,
      "check_name": "css_support",
      "previous_score": 0.95,
      "current_score": 0.87,
      "delta": -0.08
    }
  ],
  "scores": { "tmpl:142": { ... } }
}
```

**Regression detection:**
- For each `(template, check)`: if `prev_score - current_score > threshold` → regression
- Log each regression at `WARNING` level
- Collect into result for notification system (45.5 placeholder)

## Preflight Warnings

- `TemplateVersion` has `html_source` not `compiled_html` — use `html_source` for QA checks
- `Template` uses `SoftDeleteMixin` → must filter `deleted_at IS NULL`
- No `enabled` column on `Template` — filter by `status != "archived"` instead
- `QAEngineService.__init__` takes `db: AsyncSession` — need session per invocation
- The "latest version per template" query needs care with SQLAlchemy async — use `distinct(Template.id)` with `order_by(version_number.desc())` or a subquery approach

## Security Checklist

- No new endpoints (uses existing `/api/v1/scheduling/` endpoints which have auth + rate limiting)
- No user input processed — job queries internal DB data only
- Redis keys use controlled prefixes, no user-controlled segments
- Template HTML from DB is passed to QA checks (read-only analysis, no execution)

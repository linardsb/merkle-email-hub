## Implementation Steps

### Step 1: Create directory
```bash
mkdir -p email-templates/components/golden-references/
```

### Step 2: Create 14 templates (grouped by category)

**Every template MUST include:**
1. Frontmatter: `<!-- golden-ref: criteria=[...], agents=[...], verified=2026-03-29 -->`
2. Full DOCTYPE + `<html lang="en">` with VML namespaces where applicable
3. Inline styles only (no external CSS except `<style>` in `<head>` for responsive/dark mode/interactive)
4. `<table>` layout (no div/p for structure per CLAUDE.md rules)
5. `role="presentation"` on layout tables
6. `cellpadding="0" cellspacing="0"` on all tables
7. `mso-table-lspace: 0pt; mso-table-rspace: 0pt` on tables
8. MSO conditionals where relevant
9. Dark mode meta + CSS where appropriate
10. Accessibility: `lang`, `alt` text, heading hierarchy
11. Static fallback for interactive/innovation templates
12. < 8KB each (composability within 102KB Gmail limit)

**Category A — VML / MSO (templates 1-4):**

**Template 1: `vml-background-image.html`**
- Pattern from `vml-bg.html`: `v:rect` with `v:fill type="frame"` + `v:textbox`
- CSS `background-image` fallback for non-Outlook
- MSO conditional wrapper: `<!--[if gte mso 9]>...<![endif]-->`
- Content overlay with heading + CTA button inside VML textbox
- VML namespaces on `<html>`: `xmlns:v` + `xmlns:o`

**Template 2: `vml-rounded-button-variants.html`**
- 3 button variants: primary (filled), secondary (outline), small
- Each uses `v:roundrect` with `arcsize`, `fillcolor`, `strokecolor`, `href`
- `<center>` alignment inside VML + CSS fallback `<a>` with `border-radius`
- MSO conditional per button: `<!--[if mso]>...<![endif]-->` / `<!--[if !mso]><!--...<!--<![endif]-->`

**Template 3: `nested-mso-conditionals.html`**
- Demonstrates 4 conditional patterns:
  - `<!--[if mso]>` — all Outlook
  - `<!--[if gte mso 9]>` — Outlook 2007+
  - `<!--[if !mso]><!-->` — non-Outlook
  - `<!--[if (gte mso 9)|(IE)]>` — Outlook + IE
- Nested: outer `<!--[if mso]>` with inner version-specific blocks
- Ghost table structure for 2-column Outlook layout
- Each pattern clearly commented

**Template 4: `complex-hybrid-layout.html`**
- 3-column hybrid: MSO ghost table + `display:inline-block` + `font-size:0`
- `dir="rtl"` on parent for desktop column reversal, `dir="ltr"` on each inner column
- Responsive: columns stack on mobile via `max-width` + `@media` width override
- Full ghost table pattern from `column-layout-3.html`

**Category B — Dark Mode / Accessibility (templates 5-6):**

**Template 5: `dark-mode-complete.html`**
- Meta: `<meta name="color-scheme" content="light dark">`, `<meta name="supported-color-schemes" content="light dark">`
- CSS: `:root { color-scheme: light dark; }`, `@media (prefers-color-scheme: dark) { ... }`
- Outlook dark mode: `[data-ogsc]` (text), `[data-ogsb]` (background) selectors
- Utility classes: `.dark-bg`, `.dark-text`, `.dark-img-swap`
- Image swap technique: light/dark logo variants
- Contrast-safe palette: all text/bg combos ≥ 4.5:1 in both modes
- Full email structure: header, hero, 2-column content, CTA, footer

**Template 6: `accessibility-compliant.html`**
- `<html lang="en">`, `role="article"`, `aria-roledescription="email"`, `aria-label="Newsletter"`
- `role="presentation"` on ALL layout tables
- Heading hierarchy: `<h1>` → `<h2>` → `<h3>` (no skips)
- Descriptive alt text on informative images, `alt=""` on decorative spacers/borders
- Link text: descriptive (not "click here")
- `<p style="margin:0 0 10px 0;">` for body text (screen reader paragraph navigation)
- `font-size` ≥ 14px body, ≥ 22px headings
- Color contrast ≥ 4.5:1 for normal text, ≥ 3:1 for large text
- MSO conditional wrapping for Outlook

**Category C — ESP Tokens (templates 7-10):**

Each ESP template uses a common email structure (header + personalized greeting + conditional content block + CTA + footer) with platform-specific syntax.

**Template 7: `esp-braze-liquid.html`**
```
{{ first_name | default: 'Friend' }}
{% if user.custom_attribute.vip_status == true %}...{% endif %}
{% connected_content https://api.example.com/recs :save recs %}
{% for item in recs.products %}...{% endfor %}
{{ content_blocks.${header_block} }}
```

**Template 8: `esp-sfmc-ampscript.html`**
```
%%[SET @fname = AttributeValue("FirstName")]%%
%%=IIF(Empty(@fname), "Friend", @fname)=%%
%%[IF @segment == "vip"]%%...%%[ELSE]%%...%%[ENDIF]%%
%%=Lookup("ProductRecs", "Name", "ContactId", @contactId)=%%
%%=ContentBlockByKey("header_block")=%%
```

**Template 9: `esp-adobe-campaign.html`**
```
<%= recipient.firstName ? recipient.firstName : 'Friend' %>
<% if (recipient.segment == 'vip') { %>...<%  } else { %>...<%  } %>
<% var recs = sqlSelect("SELECT name FROM ProductRecs WHERE id='" + recipient.id + "'"); %>
<%= include('header_block') %>
```

**Template 10: `esp-klaviyo-django.html`**
```
{{ first_name|default:"Friend" }}
{% if person|lookup:"VIP Status" %}...{% endif %}
{% for item in catalog|catalog_items:"products" %}...{% endfor %}
{% block "header_block" %}
```

All ESP templates include:
- Syntactically valid placeholder syntax (no real API keys/endpoints)
- Fallback/default values for every variable
- Conditional content block (if/else)
- Loop/iteration example
- Content block/include reference

**Category D — Innovation (templates 11-14):**

**Template 11: `innovation-css-carousel.html`**
- Pattern from `carousel.html` + `carousel-animated.html`
- Hidden radio inputs for slide state: `<input type="radio" name="carousel" id="slide-1" checked>`
- `<label for="slide-1">` navigation dots
- CSS: `#slide-1:checked ~ .slides .slide-1 { display: block; }`
- `-webkit-` animation for transitions
- Static fallback: first slide visible, all navigation hidden, for non-supporting clients
- Gmail/Outlook: `[data-ogsc]` / `u + .body` selectors for client-specific fallback

**Template 12: `innovation-accordion-dropdown.html`**
- Pattern from `accordion.html`
- Checkbox per section: `<input type="checkbox" id="acc-1">`
- `:checked` toggle: `#acc-1:checked + .acc-header + .acc-content { max-height: 500px; }`
- `overflow: hidden` + `max-height: 0` default → `max-height + transition` on checked
- Yahoo fallback: `@media screen yahoo { .acc-content { max-height: none !important; } }`
- Outlook: static expanded view via MSO conditional
- 3 accordion sections with heading + body content

**Template 13: `innovation-amp-email.html`**
- AMP boilerplate: `<script async src="https://cdn.ampproject.org/v0.js">`, `<style amp4email-boilerplate>`
- `<amp-carousel>` with 3 slides
- `<amp-form>` with submit action
- `<amp-bind>` for dynamic state (`[text]="greeting"`)
- `<amp-list>` for dynamic content
- MIME note comment: `<!-- AMP requires multipart MIME: text/x-amp-html + text/html fallback -->`
- Full HTML fallback section (static version of all AMP content)

**Template 14: `innovation-kinetic-hover.html`**
- `:hover` CTA reveal: secondary CTA appears on hover
- Animated underline: `border-bottom` transition on links
- Color transition: background-color ease on buttons
- `@keyframes` subtle pulse on primary CTA
- Touch fallback: both CTAs always visible at mobile breakpoint
- Outlook: static layout via MSO conditional (no hover support)

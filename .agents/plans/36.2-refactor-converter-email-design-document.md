# Plan: 36.2 ‚Äî Refactor Converter to Consume EmailDesignDocument

## Context

The converter currently accepts `(DesignFileStructure, ExtractedTokens)` + 8 keyword args ‚Äî a Figma-centric signature. Phase 36.1 introduced `EmailDesignDocument` as the canonical contract. This phase adds `convert_document()` as the primary entry point consuming `EmailDesignDocument`, turns existing `convert()`/`convert_mjml()` into thin shims, and wires `import_service.py` to prefer the document path when `document_json` is available.

## Implementation Steps

### Step 1: Add `from_legacy()` classmethod to `EmailDesignDocument`

`email_design_document.py` ‚Äî add classmethod after existing bridge methods:

```python
@classmethod
def from_legacy(
    cls,
    structure: DesignFileStructure,
    tokens: ExtractedTokens,
    *,
    raw_file_data: dict[str, Any] | None = None,
    selected_nodes: list[str] | None = None,
    connection_config: dict[str, Any] | None = None,
    source_provider: str = "figma",
) -> EmailDesignDocument:
```

Logic:
- Import and call `normalize_tree(structure, raw_file_data=raw_file_data)`
- Collect frames (reuse `DesignConverterService._collect_frames` logic ‚Äî extract as module-level `collect_frames()`)
- Extract `layout_kwargs` from `connection_config` (naming_convention, section_name_map, button_name_hints)
- Call `analyze_layout(structure, **layout_kwargs)` ‚Üí `DesignLayoutDescription`
- Derive `container_width` from config override or `layout.overall_width` (clamped 400‚Äì800)
- Build `DocumentTokens` from `ExtractedTokens` (reverse of `to_extracted_tokens()`)
- Build `DocumentSection[]` from `layout.sections` (reverse of `to_email_section()`)
- Build `DocumentLayout(container_width=..., overall_width=layout.overall_width, naming_convention=...)`
- Return `EmailDesignDocument(version="1.0", tokens=..., sections=..., layout=..., source=DocumentSource(provider=source_provider))`

**Note:** Need a `DocumentTokens.from_extracted_tokens(tokens)` classmethod + `DocumentSection.from_email_section(section)` classmethod for the reverse bridge. Add these as private helpers or classmethods.

### Step 4: Add `convert_document()` and `convert_document_mjml()` to `DesignConverterService`

`converter_service.py` ‚Äî new methods:

```python
def convert_document(
    self,
    document: EmailDesignDocument,
    *,
    target_clients: list[str] | None = None,
    use_components: bool = True,
    image_urls: dict[str, str] | None = None,
    connection_id: str | None = None,
) -> ConversionResult:
```

Logic:
- `tokens = document.to_extracted_tokens()`
- `validate_and_transform(tokens, target_clients=target_clients)` ‚Üí validated tokens + warnings
- `layout = document.to_layout_description()`
- `container_width = document.layout.container_width`
- `compat = ConverterCompatibility(target_clients=target_clients)`
- Compute section hashes if caching enabled
- Branch: `_convert_with_components(...)` or `_convert_recursive(...)`
- No `normalize_tree` / `analyze_layout` ‚Äî document already contains pre-analyzed layout

```python
async def convert_document_mjml(
    self,
    document: EmailDesignDocument,
    *,
    target_clients: list[str] | None = None,
    connection_id: str | None = None,
) -> ConversionResult:
```

Logic:
- Same token/layout extraction as above
- Call `_convert_mjml_from_layout(layout=..., tokens=..., ...)`
- Fallback to `_convert_recursive()` on `MjmlCompileError`
- Need `frames` for fallback ‚Äî extract from document (use `collect_frames()` on a reconstructed structure, or pass `frames=[]` and skip recursive fallback, or store frames in the document). **Decision:** For the MJML fallback path, reconstruct minimal frames from `layout.sections` since `_convert_recursive()` only uses frames for `node_to_email_html()` per-frame. Add a `_frames_from_layout()` helper that creates stub `DesignNode` frames from `EmailSection` data.

### Step 5: Refactor `convert()` ‚Üí shim

`converter_service.py` ‚Äî replace `convert()` body:

```python
def convert(self, structure, tokens, *, raw_file_data=None, selected_nodes=None,
            target_clients=None, use_components=True, connection_config=None,
            image_urls=None, connection_id=None) -> ConversionResult:
    document = EmailDesignDocument.from_legacy(
        structure, tokens,

        document,

        use_components=use_components,
        image_urls=image_urls,
        connection_id=connection_id,
    )
```

### Step 7: Wire `import_service.py` to prefer document path

```python

snapshot = await self._repo.get_latest_snapshot(design_import.connection_id)
```

## Preflight Warnings

- `_convert_recursive()` requires `frames: list[DesignNode]` ‚Äî the document path needs a fallback strategy for this (stub frames from layout, or skip recursive fallback for document path).
- `convert()` currently does `normalize_tree()` + `analyze_layout()` inline (not via `_prepare_conversion()`). The shim must ensure `from_legacy()` produces equivalent results.
- `_prepare_conversion()` is used ONLY by `convert_mjml()` ‚Äî `convert()` has its own inline copy of the same logic. After shimming, `_prepare_conversion()` becomes dead code. Remove it.
- `connection_config` carries `naming_convention`, `section_name_map`, `button_name_hints`, `container_width` ‚Äî all must be threaded through `from_legacy()`.
- Existing tests mock at service level (`converter.convert()`) ‚Äî shim preserves this interface, so no test changes needed.

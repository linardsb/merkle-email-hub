# Plan: 47.5 — Verification Loop Pipeline Integration + Configuration

## Context

Wire the VLM verification loop (`run_verification_loop()` from 47.4) into the converter pipeline so converted HTML is automatically compared against Figma design screenshots and iteratively corrected. Must be opt-in (`vlm_verify_enabled`) with graceful skip when screenshots are unavailable.

**Key constraint**: `convert_document()` is sync but `run_verification_loop()` is async. Integration goes into async paths only (`convert_document_mjml`, `convert_mjml`). Sync `convert_document()` accepts screenshots but defers verification — callers can use standalone `verify_conversion()`.

**Design screenshots**: Not captured in converter pipeline today. Caller provides `dict[str, bytes]` (node_id → PNG). Unavailable → graceful skip.

### Step 4: Add `_apply_verification()` async method

Add after `convert_document_mjml()` (around line 343). This is a private method on `DesignConverterService`:

```python
async def _apply_verification(
    self,
    result: ConversionResult,
    design_screenshots: dict[str, bytes],
    sections: list[EmailSection],
    container_width: int,
) -> ConversionResult:
    """Run VLM verification loop on conversion result if enabled."""
    ds = get_settings().design_sync
    if not ds.vlm_verify_enabled or not design_screenshots:
        return result

    from app.design_sync.visual_verify import run_verification_loop

    try:
        vr = await run_verification_loop(
            html=result.html,
            design_screenshots=design_screenshots,
            sections=sections,
            max_iterations=ds.vlm_verify_max_iterations,
            render_client=ds.vlm_verify_client,
            viewport_width=container_width,
        )
    except Exception:
        logger.warning("design_sync.verification_loop_failed", exc_info=True)
        return result

    return ConversionResult(
        html=vr.final_html,
        sections_count=result.sections_count,
        warnings=result.warnings,
        layout=result.layout,
        compatibility_hints=result.compatibility_hints,
        images=result.images,
        cache_hit_rate=result.cache_hit_rate,
        quality_warnings=result.quality_warnings,
        match_confidences=result.match_confidences,
        figma_url=result.figma_url,
        node_id=result.node_id,
        design_tokens_used=result.design_tokens_used,
        verification_iterations=len(vr.iterations),
        verification_initial_fidelity=vr.initial_fidelity,
        verification_final_fidelity=vr.final_fidelity,
    )
```

## Preflight Warnings

- `ConversionResult` is **frozen** — cannot mutate fields. Must construct new instance with all fields (as shown in Step 4).
- `convert_document()` is **sync** — cannot call `_apply_verification()`. Verification only available on async paths.
- `run_verification_loop()` internally uses `LocalRenderingProvider` — tests must mock this or it will try to launch Playwright.
- Import at call site (`from app.design_sync.visual_verify import ...`) to avoid circular imports.

## Security Checklist

No new endpoints. No user input handling changes. No auth changes. Design screenshots are internal (Figma API bytes), not user-uploaded.

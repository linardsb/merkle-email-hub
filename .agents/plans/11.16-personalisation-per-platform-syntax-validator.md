# Plan: Personalisation Agent — Per-Platform Syntax Validator

## Context

The Personalisation agent (task 11.16) generates ESP-specific template syntax (Liquid, AMPscript, JSSP, etc.) but has **no post-generation validation**. The `personalisation_validator.py` module in `qa_engine/` already provides comprehensive syntax analysis (`analyze_personalisation()`) but is only consumed by the QA check — the agent itself is blind to syntax errors it introduces.

**Current eval pass rate: 58% logic match failure.** The most common failure: LLMs generate plausible-looking but syntactically broken template code (e.g., `{% if %}` without `{% endif %}`, `%%[` without `]%%`).

**Current gaps:**
1. `ESPPlatform` in `schemas.py` only covers 3 of 7 supported platforms
2. `SKILL_FILES` in `prompt.py` only registers 4 of 8 L3 skill files
3. No `_post_process()` override — no syntax validation after generation
4. No `contextvars` for thread-safe warning storage
5. No `syntax_warnings` field in `PersonalisationResponse`
6. Blueprint node emits `warnings=()` — no validation warnings for Recovery Router
7. No programmatic repair capability

**Pattern reference:** Follows the established pattern from tasks 11.13 (Outlook Fixer), 11.14 (Dark Mode), and 11.15 (Scaffolder) — `_post_process()` override + contextvars + blueprint node integration.

### Step 3: Add Syntax Validation to Service (`service.py`)

**File:** `app/ai/agents/personalisation/service.py`

1. Add `contextvars` import and module-level ContextVar (follows scaffolder/dark_mode pattern):
```python
import contextvars
from app.qa_engine.personalisation_validator import analyze_personalisation
from app.core.logging import get_logger

logger = get_logger(__name__)

_syntax_warnings_var: contextvars.ContextVar[list[str] | None] = contextvars.ContextVar(
    "personalisation_syntax_warnings", default=None
)
```

2. Override `_post_process()` to run syntax validation after HTML extraction:
```python
def _post_process(self, raw_content: str) -> str:
    """Post-process LLM output: extract HTML, sanitize, then validate syntax."""
    html = super()._post_process(raw_content)

    analysis = analyze_personalisation(html)
    warnings: list[str] = []

    for err in analysis.unbalanced_delimiters:
        warnings.append(f"[error] delimiter_balance: {err}")
    for err in analysis.unbalanced_conditionals:
        warnings.append(f"[error] conditional_balance: {err}")
    for err in analysis.syntax_errors:
        warnings.append(f"[error] syntax: {err}")
    for err in analysis.nested_depth_violations:
        warnings.append(f"[warning] nesting_depth: {err}")
    for err in analysis.empty_fallbacks:
        warnings.append(f"[warning] empty_fallback: {err}")
    if analysis.is_mixed_platform:
        platforms = ", ".join(str(p) for p in analysis.detected_platforms)
        warnings.append(f"[error] mixed_platform: detected {platforms}")

    _syntax_warnings_var.set(warnings)

    if warnings:
        logger.warning(
            "agents.personalisation.syntax_validation_issues",
            issue_count=len(warnings),
            issues=warnings[:10],
        )

    return html
```

3. Update `_build_response()` to read from the ContextVar and populate `syntax_warnings`:
```python
def _build_response(
    self,
    *,
    request: Any,
    html: str,
    qa_results: list[QACheckResult] | None,
    qa_passed: bool | None,
```

```python
def format_syntax_warnings(html: str) -> list[str]:
    """Run personalisation syntax validation and return formatted warnings."""
    analysis = analyze_personalisation(html)

        warnings.append(f"[error] delimiter_balance: {err}")

        warnings.append(f"[warning] nesting_depth: {err}")
    for err in analysis.empty_fallbacks:
        warnings.append(f"[warning] empty_fallback: {err}")
    if analysis.is_mixed_platform:
        platforms = ", ".join(str(p) for p in analysis.detected_platforms)
```

### Step 6: Update SKILL.md

**File:** `app/ai/agents/personalisation/SKILL.md`

Add a section after "Core Rules" emphasising balanced-tag requirements:

```markdown
## Post-Generation Validation (AUTO-APPLIED)

After generation, your output is automatically validated for:
- **Delimiter balance** — Every `{{` has `}}`, every `{%` has `%}`, every `%%[` has `]%%`, etc.
- **Conditional balance** — Every `{% if %}` has `{% endif %}`, every `IF` has `ENDIF`, etc.
- **Syntax correctness** — No empty tags, no dangling filter pipes, no missing @ prefixes
- **Nesting depth** — Max 3 levels of conditional nesting
- **Fallback presence** — Every output variable should have a default value
- **Platform purity** — No mixed ESP syntax in a single template

Violations are surfaced as warnings and may trigger a self-correction retry.
Aim for ZERO syntax warnings on first generation.
```

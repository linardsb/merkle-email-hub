# Plan: 11.23 Inline Eval Judges — Selective LLM Judge on Recovery Retries

## Context

The QA gate catches structural issues (HTML validation, CSS support, file size, etc.) but misses semantic quality (brief fidelity, tone accuracy, colour coherence). Eval judges assess 5 nuanced criteria per agent but cost ~3,200 tokens per call. Running judges on every handoff is cost-prohibitive (+67% per run). Running them **only on recovery retries** (`iteration > 0`) bounds the cost (max 2 retries × 1 judge = ~6,400 extra tokens) and targets the moment where the signal is most valuable — the agent already failed QA once, and a nuanced verdict prevents wasted retry loops.

After 11.22 (template-first), retries are rare (~5% of runs), making the cost negligible.

## Files to Create/Modify

### New Files
1. `app/ai/blueprints/inline_judge.py` — Adapter between `JUDGE_REGISTRY` judges and live `NodeContext`/`BlueprintRun`. Builds `JudgeInput` from live context, calls judge via provider registry with `temperature=0.0` and lightweight model tier.
2. `app/ai/blueprints/tests/test_inline_judge.py` — Unit tests for the inline judge adapter.

### Modified Files
3. `app/ai/blueprints/engine.py` — After agentic node execution when `iteration > 0` and `judge_on_retry` is enabled, call `run_inline_judge()`. If verdict fails → `needs_review` + break. If passes → proceed to QA gate normally.
4. `app/ai/blueprints/protocols.py` — Add `judge_verdict: JudgeVerdict | None` field to engine state (not protocols — the `BlueprintRun` dataclass is in `engine.py`).
5. `app/ai/blueprints/schemas.py` — Add `InlineJudgeVerdictResponse` schema and `judge_verdict` field to `BlueprintRunResponse`.
6. `app/ai/blueprints/service.py` — Pass `judge_on_retry` config to engine construction.
7. `app/core/config.py` — Add `judge_on_retry: bool = False` to `BlueprintConfig`.

### Step 2: Create inline judge adapter — `app/ai/blueprints/inline_judge.py`

This is the core new module. It bridges eval judges (designed for offline traces) into live blueprint context.

```python
"""Inline LLM judge adapter for live blueprint recovery retries.

Bridges the eval JUDGE_REGISTRY into the blueprint engine, converting
live NodeContext + BlueprintRun into a JudgeInput that eval judges accept.
Only invoked on recovery retries (iteration > 0) to bound cost.
"""

from __future__ import annotations

import uuid
from typing import TYPE_CHECKING

from app.ai.agents.evals.judges import JUDGE_REGISTRY
from app.ai.agents.evals.judges.schemas import JudgeInput, JudgeVerdict
from app.ai.protocols import CompletionResponse, Message
from app.ai.registry import get_registry
from app.ai.routing import resolve_model
from app.core.config import get_settings
from app.core.logging import get_logger

if TYPE_CHECKING:
    from app.ai.blueprints.engine import BlueprintRun
    from app.ai.blueprints.protocols import NodeContext

logger = get_logger(__name__)


def _build_judge_input(
    agent_name: str,
    context: NodeContext,
    html_output: str,
    run: BlueprintRun,
) -> JudgeInput:
    """Build a JudgeInput from live blueprint context.

    Maps live context fields to the trace-based JudgeInput schema that
    eval judges expect:
    - input_data: brief + QA failures (what the agent was asked to fix)
    - output_data: the HTML the agent produced
    - expected_challenges: derived from QA failure details
    """
    return JudgeInput(
        trace_id=f"inline-{run.run_id}-{uuid.uuid4().hex[:8]}",
        agent=agent_name,
        input_data={
            "brief": context.brief,
            "qa_failures": context.qa_failures,
            "iteration": context.iteration,
        },
        output_data={
            "html": html_output,
        },
        expected_challenges=[
            f.check_name for f in run.qa_failure_details
        ],
    )


async def run_inline_judge(
    agent_name: str,
    context: NodeContext,
    html_output: str,
    run: BlueprintRun,
) -> JudgeVerdict | None:
    """Run the eval judge for an agent inline during a recovery retry.

    Returns the JudgeVerdict, or None if:
    - No judge is registered for this agent
    - The LLM call fails (failure-safe: log and return None)

    Uses the lightweight model tier and temperature=0.0 for determinism.
    """
    judge_cls = JUDGE_REGISTRY.get(agent_name)
    if judge_cls is None:
        logger.debug(
            "blueprint.inline_judge_skipped",
            agent=agent_name,
            reason="no_judge_registered",
        )
        return None

    judge = judge_cls()
    judge_input = _build_judge_input(agent_name, context, html_output, run)
    prompt = judge.build_prompt(judge_input)

    try:
        settings = get_settings()
        registry = get_registry()
        provider = registry.get_llm(settings.ai.provider)
        model = resolve_model("lightweight")

        response: CompletionResponse = await provider.complete(
            [Message(role="user", content=prompt)],
            temperature=0.0,
            model=model,
        )

        verdict = judge.parse_response(response.content, judge_input)

        logger.info(
            "blueprint.inline_judge_completed",
            agent=agent_name,
            overall_pass=verdict.overall_pass,
            criteria_count=len(verdict.criteria_results),
            run_id=run.run_id,
        )

        return verdict

    except Exception:
        logger.warning(
            "blueprint.inline_judge_failed",
            agent=agent_name,
            run_id=run.run_id,
            exc_info=True,
        )
        return None
```

Key design decisions:
- **Failure-safe**: Returns `None` on any error so the pipeline continues without the judge signal.
- **Lightweight model**: Uses `resolve_model("lightweight")` — Haiku-tier for cost efficiency.
- **temperature=0.0**: Deterministic verdicts for reproducibility.
- **`_build_judge_input`**: Bridges live context → trace schema. The `trace_id` is prefixed with `inline-` to distinguish from offline eval traces.

In `BlueprintEngine.run()`, insert the inline judge check **after** scope validation and **before** recording progress — specifically after the scope validation block (line ~243) and before `run.progress.append(...)` (line ~245). The inline judge runs only when:
1. `self._judge_on_retry` is True
2. Node is agentic
3. `iteration > 0` (this is a recovery retry)
4. Result status is "success" (the fixer produced output; we want to judge quality before sending to QA gate)

```python
            # --- Inline judge on recovery retries ---
            if (
                self._judge_on_retry
                and node.node_type == "agentic"
                and iteration > 0
                and result.status == "success"
                and result.html
            ):
                from app.ai.blueprints.inline_judge import run_inline_judge

                agent_name = node.name.removesuffix("_node")
                verdict = await run_inline_judge(
                    agent_name=agent_name,
                    context=context,
                    html_output=result.html,
                    run=run,
                )
                if verdict is not None:
                    run.judge_verdict = verdict
                    # Track judge token usage
                    # Judge uses lightweight model — estimate ~1,500 tokens
                    # Actual usage not available from judge call; tracked via provider
                    if verdict.overall_pass is False:
                        run.status = "needs_review"
                        logger.warning(
                            "blueprint.inline_judge_rejected",
                            agent=agent_name,
                            run_id=run.run_id,
                            failed_criteria=[
                                cr.criterion
                                for cr in verdict.criteria_results
                                if not cr.passed
                            ],
                        )
                        break
                    else:
                        logger.info(
                            "blueprint.inline_judge_approved",
                            agent=agent_name,
                            run_id=run.run_id,
                        )
```

Insert this block right after the scope validation block and before the `# Record progress` comment.

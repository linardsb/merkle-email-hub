# Plan: 11.4 Fallback Check — MSO Conditional Parser

## Context

The current fallback check (`app/qa_engine/checks/fallback.py`) is a 36-line stub that does two string-in-string checks:
1. `"<!--[if mso" in html.lower()` — presence-only
2. `"xmlns:v=" not in html` — namespace presence-only

Eval data shows **50% MSO conditional correctness failure** — the single worst failure cluster. The check passes HTML that will render incorrectly in Outlook because it never validates _correctness_ of MSO conditionals (balanced pairs, valid syntax, VML placement).

### Decision: QA Check vs Agent Skill File

The user's MSO reference doc (`docs/outlook-mso-fallback-reference.md`, 980 lines, 300+ patterns) serves two purposes:
- **11.4 (this task)**: Informs the _validation rules_ — what patterns to check for balanced pairs, valid VML elements, namespace requirements, valid conditional syntax. The reference is distilled into code/YAML rules.
- **11.13 (Outlook Fixer agent)**: The reference is agent _knowledge_ — loaded as L3 skill files so the LLM knows MSO vocabulary. That task will import the `validate_mso_conditionals()` function exported here.

**Recommendation**: Keep the reference doc as-is for the agent (11.13 will split/distill it into L3 files). For 11.4, extract validation patterns into `rules/mso_fallback.yaml` + custom check functions. Export `validate_mso_conditionals()` as a standalone function for agent reuse.

## Architecture

Follow the established pattern from `html_validation.py` and `accessibility.py`:
- YAML rules for declarative checks (namespace presence, basic VML patterns)
- Custom check functions for complex MSO parsing that can't be expressed declaratively (balanced pair counting on raw HTML, VML orphan detection across comment blocks, ghost table structure validation)
- Rule engine orchestrates both

**Why custom functions dominate here**: MSO conditionals are HTML comments — `lxml` strips them during DOM parsing. Most checks need raw HTML regex parsing, which maps to `custom` check type, not declarative YAML selectors.

## Implementation Steps

### Step 1: Create `app/qa_engine/mso_parser.py` — Standalone MSO Parser

This is the core logic, extracted as a standalone module so the Outlook Fixer agent (11.13) can import it directly.

```python
"""MSO conditional comment parser and validator.

Provides validate_mso_conditionals() for use by:
- QA fallback check (via rule engine custom functions)
- Outlook Fixer agent (direct import for post-generation validation)

All parsing operates on raw HTML strings because lxml strips HTML comments.
"""

from __future__ import annotations

import re
from dataclasses import dataclass, field

from app.core.logging import get_logger

logger = get_logger(__name__)


@dataclass(frozen=True)
class MSOIssue:
    """A single MSO validation issue with location context."""
    category: str          # "balanced_pair" | "vml_orphan" | "namespace" | "ghost_table" | "syntax"
    message: str
    severity: str = "warning"  # "error" | "warning" | "info"
    position: int = -1     # Approximate char offset in raw HTML (-1 = unknown)


@dataclass
class MSOValidationResult:
    """Aggregate result of MSO conditional validation."""
    issues: list[MSOIssue] = field(default_factory=list)
    opener_count: int = 0
    closer_count: int = 0
    vml_element_count: int = 0
    has_vml_namespace: bool = False
    has_office_namespace: bool = False

    @property
    def is_valid(self) -> bool:
        return len(self.issues) == 0
```

**Core parsing functions** (all operate on raw HTML string):

1. **`_find_conditional_blocks(html)`** — Returns list of `(start_pos, end_pos, condition_expr)` for all `<!--[if ...]>` blocks. Uses regex:
   - Opener: `r'<!--\[if\s+([^\]]+)\]>'`
   - Closer: `r'<!\[endif\]-->'`
   - Also handles non-MSO closer: `r'<!--<!\[endif\]-->'`

2. **`_validate_balanced_pairs(html)`** — Count openers vs closers. Two patterns:
   - MSO-only: `<!--[if mso...]>` paired with `<![endif]-->`
   - Non-MSO: `<!--[if !mso]><!-->` paired with `<!--<![endif]-->`
   - Track positions for each opener/closer. Report unmatched with position.

3. **`_validate_conditional_syntax(html)`** — Check each condition expression against valid patterns:
   - Valid: `mso`, `!mso`, `mso XX`, `gte mso XX`, `gt mso XX`, `lte mso XX`, `lt mso XX`, `mso | IE`, `!mso]><!-->`
   - Valid versions: 9, 10, 11, 12, 14, 15, 16
   - Invalid: `mso 13`, `mso 17`, malformed operators

4. **`_validate_vml_nesting(html)`** — Find all VML elements (`<v:rect`, `<v:roundrect`, `<v:oval`, `<v:line`, `<v:shape`, `<v:image`, `<v:group`, `<v:polyline`, `<v:arc`, `<v:curve`, `<v:fill`, `<v:stroke`, `<v:shadow`, `<v:textbox`, `<v:imagedata`, `<v:path`, `<v:formulas>`) and Office elements (`<o:OfficeDocumentSettings`, `<o:AllowPNG`, `<o:PixelsPerInch`, `<w:anchorlock`, `<o:lock>`). Verify each is inside an `<!--[if ... mso ...]>` block by checking if its position falls between an opener and its paired closer. Flag VML orphans outside conditionals.

5. **`_validate_namespaces(html, has_vml)`** — If any VML/Office elements exist:
   - Check for `xmlns:v="urn:schemas-microsoft-com:vml"` on `<html>` tag
   - Check for `xmlns:o="urn:schemas-microsoft-com:office:office"` on `<html>` tag
   - Flag missing namespaces

6. **`_validate_ghost_tables(html)`** — Find ghost table patterns (MSO conditional containing `<table` with `width=`). Verify:
   - Ghost tables have explicit `width` attribute (numeric, no px)
   - Ghost tables have `role="presentation"`
   - Ghost table `<td>` elements have `valign` attribute
   - Only flag when multi-column div patterns exist nearby (within 500 chars of a `max-width` div)

7. **`validate_mso_conditionals(html: str) -> MSOValidationResult`** — Public API. Runs all sub-validators, aggregates issues. This is the function the Outlook Fixer agent (11.13) will import.

### Step 3: Register Custom Check Functions in `app/qa_engine/custom_checks.py`

Add bridge functions at the bottom of the existing `custom_checks.py` that delegate to `mso_parser.py`:

```python
# ─── MSO Fallback Custom Checks ─── (delegates to mso_parser module)

from app.qa_engine.mso_parser import validate_mso_conditionals


def _mso_balanced_pairs(
    doc: HtmlElement, raw_html: str, config: QACheckConfig | None
) -> tuple[list[str], float]:
    result = validate_mso_conditionals(raw_html)
    issues = [i.message for i in result.issues if i.category == "balanced_pair"]
    deduction = _resolve("deduction_unbalanced_pair", 0.25, config) * len(issues)
    return issues, deduction


def _mso_conditional_syntax(
    doc: HtmlElement, raw_html: str, config: QACheckConfig | None
) -> tuple[list[str], float]:
    result = validate_mso_conditionals(raw_html)
    issues = [i.message for i in result.issues if i.category == "syntax"]
    deduction = _resolve("deduction_invalid_syntax", 0.15, config) * len(issues)
    return issues, deduction


# ... similar for vml_nesting, namespace_validation, ghost_table_validation,
#     conditionals_present, namespaces_present

def _resolve(key: str, default: float, config: QACheckConfig | None) -> float:
    if config:
        return float(config.params.get(key, default))
    return default
```

Register all 7 custom functions with `register_custom_check()`.

**Important**: The `validate_mso_conditionals()` call is cached per raw_html within a single check run — don't call it 7 times. Use a module-level `_last_result` pattern or pass the result through. Best approach: compute once in the first custom function call and cache by hash.

### Step 4: Rewrite `app/qa_engine/checks/fallback.py`

Replace the stub with the rule-engine pattern (identical structure to `accessibility.py`):

```python
"""MSO fallback rendering check — validates Outlook conditional comments, VML nesting,
namespace declarations, and ghost table structure.

Uses the shared rule engine with rules loaded from rules/mso_fallback.yaml.
Complex checks delegated to custom functions in mso_parser.py via custom_checks.py.
"""

from __future__ import annotations

from pathlib import Path

# Import custom checks to ensure MSO check functions are registered
import app.qa_engine.custom_checks  # noqa: F401
from app.qa_engine.check_config import QACheckConfig
from app.qa_engine.rule_engine import RuleEngine, load_rules
from app.qa_engine.schemas import QACheckResult

_RULES_PATH = Path(__file__).parent.parent / "rules" / "mso_fallback.yaml"


class FallbackCheck:
    """Validates MSO conditional comments, VML nesting, and namespace declarations.

    Loads rules from rules/mso_fallback.yaml covering:
    A: Conditional Balance (2 rules)
    B: VML Nesting (1 rule)
    C: Namespace Declarations (1 rule)
    D: Ghost Tables (1 rule)
    E: Presence Checks (3 rules)
    """

    name = "fallback"

    def __init__(self) -> None:
        self._rules = load_rules(_RULES_PATH)
        self._engine = RuleEngine(self._rules)

    async def run(self, html: str, config: QACheckConfig | None = None) -> QACheckResult:
        if not html or not html.strip():
            return QACheckResult(
                check_name=self.name,
                passed=False,
                score=0.0,
                details="Empty HTML document",
                severity="error",
            )

        # Note: lxml strips comments, so most MSO checks use raw_html via custom functions.
        # We still parse DOM for the rule engine interface consistency.
        try:
            from lxml import html as lxml_html
            doc = lxml_html.document_fromstring(html)
        except Exception:
            return QACheckResult(
                check_name=self.name,
                passed=False,
                score=0.0,
                details="HTML could not be parsed",
                severity="error",
            )

        issues, total_deduction = self._engine.evaluate(doc, html, config)

        score = max(0.0, round(1.0 - total_deduction, 2))
        passed = len(issues) == 0
        return QACheckResult(
            check_name=self.name,
            passed=passed,
            score=score,
            details="; ".join(issues) if issues else None,
            severity="warning" if not passed else "info",
        )
```

## MSO Parser Caching Strategy

Since all 7 custom functions call `validate_mso_conditionals()` on the same raw HTML within a single check run, add caching:

```python
# In mso_parser.py or custom_checks.py
_mso_cache: dict[int, MSOValidationResult] = {}

def _get_cached_result(raw_html: str) -> MSOValidationResult:
    key = hash(raw_html)
    if key not in _mso_cache:
        _mso_cache[key] = validate_mso_conditionals(raw_html)
    return _mso_cache[key]

def clear_mso_cache() -> None:
    _mso_cache.clear()
```

Clear cache at the start of each `FallbackCheck.run()` call (or use WeakValueDict).

# Plan: 35.10 Incremental Conversion & Section Caching

## Context

A full Figma-to-email conversion for a 6-section email takes 2–5s. When a designer changes one section's text, re-converting all 6 wastes 80% of work. Section-level caching reduces re-conversion to <1s for incremental changes — critical for live preview sync (35.9).

## Implementation Steps

### Step 1: Config — add cache settings to `DesignSyncConfig`

`app/core/config.py` — add to `DesignSyncConfig`:
```python
# Section cache (35.10)
section_cache_enabled: bool = True
section_cache_memory_max: int = 500       # In-memory LRU entries
section_cache_redis_ttl: int = 3600       # Redis TTL seconds (1h)
```

### Step 2: Create `app/design_sync/section_cache.py`

**`compute_section_hash()`** — deterministic SHA-256 of section content:
- Input: `EmailSection`, `ExtractedTokens`, `container_width: int`, `target_clients: list[str] | None`
- Canonical JSON: sort keys, round floats to 2dp, stable ordering of lists
- Hash components: section fields (node_id, section_type, texts content, images, buttons, dimensions, padding, bg_color), token subset (colors hex values, typography families/sizes, spacing values), container_width, sorted target_clients
- Returns hex digest string

**`SectionCacheEntry`** — frozen dataclass:
```python
@dataclass(frozen=True)
class SectionCacheEntry:
    html: str
    images: tuple[dict[str, str], ...]   # Frozen for hashability
    generated_at: str                     # ISO timestamp
```

**`SectionCache`** class:
- `__init__(max_memory: int = 500, redis_ttl: int = 3600)` — config-driven
- In-memory: `dict[str, SectionCacheEntry]` with manual LRU eviction (same pattern as `section_adapter.py:252`)
- Redis key pattern: `section_cache:{connection_id}:{section_hash}`
- `async get(connection_id: str, section_hash: str) -> SectionCacheEntry | None` — memory first, then Redis
- `async set(connection_id: str, section_hash: str, entry: SectionCacheEntry) -> None` — write-through (memory + Redis)
- `async get_many(connection_id: str, hashes: dict[str, str]) -> dict[str, SectionCacheEntry]` — batch get, returns {section_node_id: entry}
- `async invalidate_connection(connection_id: str) -> int` — clear all entries for connection (Redis SCAN + DEL, memory sweep), returns count
- `clear_memory()` — clear in-memory cache (for tests)

Redis interaction: use `get_redis()` from `app.core.redis`, catch `RedisError` and fall back to memory-only (non-fatal).

### Step 3: Wire cache into `DesignConverterService`

**Add `_section_cache` class attribute** (lazy singleton):
```python
_section_cache: SectionCache | None = None

@staticmethod
def _get_cache() -> SectionCache:
    if DesignConverterService._section_cache is None:
        settings = get_settings()
        DesignConverterService._section_cache = SectionCache(
            max_memory=settings.design_sync.section_cache_memory_max,
            redis_ttl=settings.design_sync.section_cache_redis_ttl,
        )
    return DesignConverterService._section_cache
```

**Modify `convert()` signature** — add `connection_id: str | None = None`:
- If `connection_id` and `section_cache_enabled`: after layout analysis, compute per-section hashes
- Pass hashes + connection_id down to `_convert_with_components()`

**Modify `_convert_with_components()`** — add `connection_id: str | None`, `section_hashes: dict[str, str] | None`:
- Before rendering loop: `cached = await cache.get_many(connection_id, section_hashes)` if cache enabled
- In loop: skip `renderer.render_section()` for cached sections, use `cached[node_id].html` + `cached[node_id].images`
- After loop: `await cache.set()` for freshly rendered sections
- Log `design_sync.conversion_cache_stats` with `hit_count`, `miss_count`, `hit_rate`

**Problem:** `convert()` is sync, but Redis is async. Two options:
1. Make `convert()` async (breaking change — many callsites)
2. Keep memory-only cache in sync `convert()`, async Redis in `convert_mjml()` only

**Decision:** Keep `convert()` sync with **memory-only** cache. `convert_mjml()` (already async) gets **memory + Redis**. This avoids breaking the 46+ callsites of sync `convert()`.

For sync `convert()` → `_convert_with_components()`:
- Use memory cache only (no Redis await)
- `cache.get_sync(connection_id, hash)` / `cache.set_sync(...)` methods on `SectionCache`

For async `convert_mjml()` → `_convert_mjml_from_layout()`:
- Full memory + Redis cache

### Step 4: Wire cache into `convert_mjml()` path

`_convert_mjml_from_layout()` already iterates sections for template rendering. Add caching around the per-section template render:
- Before: compute section hashes
- Cache hit: use cached MJML fragment per section
- Cache miss: render section, store in cache
- Assemble full MJML from cached + fresh fragments, then compile

Note: The MJML path caches **pre-compilation MJML fragments** per section, not final HTML. The compilation step always runs on the assembled document (MJML compiler needs full doc context).

### Step 5: Add `ConversionResult` cache metadata

Extend `ConversionResult` with optional cache stats:
```python
@dataclass(frozen=True)
class ConversionResult:
    html: str
    sections_count: int
    warnings: list[str] = field(default_factory=list)
    layout: DesignLayoutDescription | None = None
    compatibility_hints: list[CompatibilityHint] = field(default_factory=list)
    images: list[dict[str, str]] = field(default_factory=list)
    cache_hit_rate: float | None = None  # 0.0-1.0, None if cache disabled
```

### Step 6: Cache invalidation

Invalidation triggers in `DesignConverterService`:
- `target_clients` change → different hash (automatic, no explicit invalidation needed)
- `container_width` change → different hash (automatic)
- Token structural change → different hash (automatic)

Explicit invalidation via admin endpoint (Step 7) for:
- Manual cache clear
- Forced re-conversion after component template updates

Hash-based approach means most invalidation is **implicit** — changed input = different hash = cache miss.

### Step 7: Admin cache-clear endpoint

`app/design_sync/routes.py` — add:
```python
@router.delete("/connections/{connection_id}/cache", status_code=status.HTTP_200_OK)
@limiter.limit("10/minute")
async def clear_section_cache(
    connection_id: int,
    request: Request,
    current_user: User = Depends(require_role("admin")),
) -> dict[str, int]:
    """Clear cached conversion results for a connection."""
    cache = DesignConverterService._get_cache()
    cleared = await cache.invalidate_connection(str(connection_id))
    return {"cleared_entries": cleared}
```

`app/design_sync/schemas.py` — add `CacheClearResponse`:
```python
class CacheClearResponse(BaseModel):
    cleared_entries: int
```

### Step 8: Tests — `app/design_sync/tests/test_section_cache.py`

Test categories (~25 tests):

**Hash computation (6 tests):**
- Same section + tokens → same hash (deterministic)
- Different text content → different hash
- Different bg_color → different hash
- Different container_width → different hash
- Different target_clients → different hash
- Float rounding: 10.001 vs 10.002 → same hash (2dp rounding)

**Memory cache (5 tests):**
- Set + get returns entry
- Cache miss returns None
- LRU eviction at max capacity
- `clear_memory()` empties cache
- `invalidate_connection()` removes only matching entries

**Redis cache (5 tests, mocked):**
- Set writes to Redis with TTL
- Get reads from Redis, deserializes entry
- Redis failure falls back gracefully (no exception)
- `get_many()` batch retrieval
- `invalidate_connection()` scans + deletes Redis keys

**Integration with converter (6 tests):**
- Convert with cache enabled → cache miss → all sections rendered → cache populated
- Convert again (same input) → cache hit → sections from cache (verify `render_section` not called for cached)
- Change one section's text → partial cache hit (N-1 hits, 1 miss)
- `cache_hit_rate` in ConversionResult metadata
- Cache disabled (`section_cache_enabled=False`) → no caching, normal conversion
- `connection_id=None` → no caching

**Admin endpoint (3 tests):**
- `DELETE /connections/{id}/cache` clears entries, returns count
- Non-admin gets 403
- Rate limited

Use factories from `test_component_matcher.py`: `_make_section()`, `_text()`, `_image()`, `_button()`.
Use token factory from `test_mjml_convert.py`: `_make_tokens()`.
Mock Redis with `AsyncMock` per reporting test pattern.

## Preflight Warnings

- `DesignConverterService` is **stateless** — no `__init__` params. Cache must be class-level or injected via method params.
- `convert()` is **sync** — cannot use Redis directly. Memory-only cache for sync path.
- `ConversionResult` is **frozen** — add `cache_hit_rate` field with default `None` (backward-compatible).
- `ExtractedTokens` is **frozen** with `list` fields (not tuple) — hash computation must handle mutable contents.
- `EmailSection` is **not frozen** — hash must capture all relevant mutable fields.

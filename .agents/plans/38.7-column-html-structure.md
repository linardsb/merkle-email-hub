# Plan: 38.7 Column HTML Structure — `<div class="column">` Pattern

## Context

Replace the converter's column wrapper from `<table class="column" style="display:inline-block">` to `<div class="column" style="display:inline-block">`, matching all 16 golden components. The current table wrapper doesn't respond to the mobile CSS `.column { display: block !important; }` — columns stay side-by-side on phones. The extra table nesting also adds 2+ unnecessary levels (7+ deep vs 5).

## Research Summary

### Current Implementation: `converter.py:1328-1446`

`_render_multi_column_row()` generates columns as:
```html
<table class="column" role="presentation" width="100%"
 cellpadding="0" cellspacing="0" border="0"
 style="display:inline-block;max-width:{col_width}px;width:100%;vertical-align:top;
 border-collapse:collapse;mso-table-lspace:0pt;mso-table-rspace:0pt;">
  <tr><td>{content}</td></tr>
</table>
```

**B1 comment (line 1391-1393):** Explains current choice — "Use `<table class="column">` instead of `<div class="column">` + inner `<table>` to avoid sanitizer double-wrapping." This concern is addressed by 38.7 — the golden templates prove `<div>` is the correct pattern.

### Target Pattern (from `email-templates/components/column-layout-2.html`)

```html
<div class="column" style="display: inline-block; max-width: 300px; width: 100%; vertical-align: top;">
  <table role="presentation" width="100%" cellpadding="0" cellspacing="0" border="0"
   style="border-collapse: collapse; mso-table-lspace: 0pt; mso-table-rspace: 0pt;">
    <tr><td data-slot="col_1" style="padding: 0;">{content}</td></tr>
  </table>
</div>
```

### Step 2: Add asymmetric gutter padding

Add `_col_padding()` method if not already present:

```python
def _col_padding(self, col_index: int, col_count: int, gutter: float = 8.0) -> str:
    if col_count <= 1:
        return "0"
    if col_index == 0:
        return f"0 {gutter}px 0 0"
    if col_index == col_count - 1:
        return f"0 0 0 {gutter}px"
    half = gutter / 2
    return f"0 {half}px 0 {half}px"
```

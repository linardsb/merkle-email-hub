# Plan: 43.3 Judge Skill Files for Domain Knowledge Accumulation

## Context

Corrections (43.2) address specific past mistakes from one calibration run. Skills address **systemic knowledge gaps** — patterns judges consistently struggle with across multiple traces. For example, if the scaffolder judge repeatedly misjudges MSO conditional nesting, a skill file teaches it the structural rules once. Separates ephemeral corrections from durable knowledge.

## Research Summary

### Prompt Assembly Order (current → proposed)

```
SYSTEM_PROMPT_TEMPLATE:
  ├─ criteria_block       (numbered criteria list)
  ├─ corrections_block    (43.2, from traces/corrections/)
  ├─ skills_block         (NEW — 43.3, from judges/skills/)  ← INSERT HERE
  └─ IMPORTANT footer     (strict judging instructions)
───
USER CONTENT:
  ├─ Agent input (brief/question/HTML)
  ├─ Golden references (per-criterion examples)
  ├─ Design context (Figma metadata, optional)
  └─ Agent output (HTML/answer)
```

Skills go **after corrections, before IMPORTANT** — knowledge → examples → mistakes → rules.

### Skill Mapping Design

Per the spec, skills map **criteria → skill files** (not agent → skills). A YAML manifest per judge:

```yaml
# judges/skills/scaffolder_skills.yaml
name: scaffolder_judge
skills:
  mso_conditional_correctness: [mso_conditional_patterns.md]
  email_layout_patterns: [email_layout_detection.md]
  dark_mode_readiness: [dark_mode_completeness.md]
```

`load_judge_skills(agent_name, criterion)` returns concatenated content for matching criteria.

### Step 3: Add skill loading functions to `base.py`

After `format_corrections_section()` (line 220), add:

```python
@lru_cache(maxsize=16)
def _load_skills_manifest_cached(path_str: str, _mtime: float) -> dict[str, Any]:
    """Load and cache skill manifest YAML. Cache key includes mtime for staleness."""
    with Path(path_str).open() as f:
        data: dict[str, Any] = yaml.safe_load(f) or {}
    return data


@lru_cache(maxsize=32)
def _load_skill_file_cached(path_str: str, _mtime: float) -> str:
    """Load and cache a skill markdown file. Cache key includes mtime."""
    return Path(path_str).read_text()


def load_judge_skills(agent_name: str, criterion: str) -> str:
    """Load skill content for a specific agent+criterion pair.

    Reads from judges/skills/{agent}_skills.yaml to find mapped skill files,
    then loads and concatenates their content within per-file token budget.

    Returns empty string if no manifest, no mapping, or skills disabled.
    """
    if not _skills_enabled:
        return ""

    manifest_path = _SKILLS_DIR / f"{agent_name}_skills.yaml"
    if not manifest_path.exists():
        return ""

    manifest = _load_skills_manifest_cached(
        str(manifest_path), manifest_path.stat().st_mtime
    )
    skills_map: dict[str, list[str]] = manifest.get("skills", {})
    file_names: list[str] = skills_map.get(criterion, [])
    if not file_names:
        return ""

    per_file_budget = 1000 * 4  # 1000 tokens per skill file
    parts: list[str] = []
    for fname in file_names:
        skill_path = _SKILLS_DIR / fname
        if not skill_path.exists():
            continue
        content = _load_skill_file_cached(str(skill_path), skill_path.stat().st_mtime)
        if len(content) > per_file_budget:
            content = content[:per_file_budget] + "\n[truncated]"
        parts.append(content)

    return "\n\n".join(parts)


def format_skills_section(agent_name: str, criteria: list[JudgeCriteria]) -> str:
    """Build domain knowledge section from skill files for all criteria.

    Loads relevant skills for each criterion and concatenates within total budget.
    Returns empty string if no skills found or skills disabled.
    """
    if not _skills_enabled:
        return ""

    total_budget = _SKILLS_TOKEN_BUDGET * 4
    parts: list[str] = []
    used = 0

    for c in criteria:
        skill_content = load_judge_skills(agent_name, c.name)
        if not skill_content:
            continue
        entry = f"### {c.name}\n{skill_content}"
        if used + len(entry) > total_budget:
            break
        parts.append(entry)
        used += len(entry)

    if not parts:
        return ""

    header = (
        "## DOMAIN KNOWLEDGE REFERENCE\n"
        "Use this reference material for consistent evaluation:\n"
    )
    return header + "\n\n" + "\n\n".join(parts)
```

### Step 4: Update `build_system_prompt()` to inject skills

```python
def build_system_prompt(
    criteria: list[JudgeCriteria],
    agent_name: str,
) -> str:
    """Build system prompt with criteria, corrections, and skills injected."""
    criteria_block = build_criteria_block(criteria)
    corrections = format_corrections_section(agent_name)
    corrections_block = f"\n{corrections}\n" if corrections else ""
    skills = format_skills_section(agent_name, criteria)
    skills_block = f"\n{skills}\n" if skills else ""
    return SYSTEM_PROMPT_TEMPLATE.format(
        criteria_block=criteria_block,
        corrections_block=corrections_block,
        skills_block=skills_block,
    )
```

### Step 6: Create skill files directory and 5 initial skill files

Create `app/ai/agents/evals/judges/skills/` with:

**`mso_conditional_patterns.md`** (~800 tokens) — MSO nesting rules:
- Valid patterns: `<!--[if mso]>`, `<!--[if !mso]><!-->`/`<!--<![endif]-->`, `<!--[if gte mso 9]>`
- `<!--[if !mso]><!-->` looks unbalanced but is correct (NOT a false positive)
- Nesting depth rules: max 3 levels, each must close before parent
- Common false positives: split conditionals across table cells (valid pattern)

**`email_layout_detection.md`** (~600 tokens) — Layout div distinction:
- FAIL: `<div style="width:300px">` or `<div style="display:flex">` for layout
- PASS: `<div style="text-align:center">` wrapper inside `<td>` (no layout CSS)
- PASS: `<div>` inside `<!--[if mso]>` blocks (MSO ghost tables)
- Key rule: check for width/flex/float/grid/columns on div/p — presence = layout = FAIL

**`dark_mode_completeness.md`** (~700 tokens) — Minimum viable dark mode:
- Required: `<meta name="color-scheme" content="light dark">` + `<meta name="supported-color-schemes" content="light dark">`
- Required: `@media (prefers-color-scheme: dark)` media query with color overrides
- Required: `[data-ogsc]`/`[data-ogsb]` Outlook selectors for each color override
- Partial = FAIL: meta without media query, media query without Outlook selectors

**`esp_syntax_validation.md`** (~700 tokens) — ESP delimiter rules:
- Liquid (Braze/Shopify): `{{ }}` for output, `{% %}` for logic. Nested `{{ }}` inside `{% %}` is valid
- AMPscript (SFMC): `%%[ ]%%` blocks, `%%=v(@var)=%%` inline. `CONCAT()` with nested quotes is valid
- Handlebars (Iterable): `{{ }}` output, `{{#each}}...{{/each}}` blocks
- Common false positives: `{{ user.custom_attributes["key"] }}` nested brackets, `%%=CONCAT(...)=%%` complex expressions

**`copy_quality_boundaries.md`** (~600 tokens) — Pass/fail calibration:
- PASS: clear, grammatically correct, matches requested tone, no spam triggers
- FAIL: grammatical errors, wrong tone (casual when professional requested), spam trigger words
- Boundary: "good enough" copy that's functional but not compelling = PASS (not a judge's job to demand excellence)
- Key: judge correctness and appropriateness, not creativity or marketing effectiveness

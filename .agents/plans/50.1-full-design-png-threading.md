# Plan: 50.1 Full-Design PNG Threading (Gap 9)

## Context

The converter exports a global `design.png` for training-case archival but **discards it before classification**. Per `docs/architecture/opus-figma-to-html-process.md` Gap 9: every classification decision is local to a single section (`design_screenshots: dict[str, bytes]` keyed by per-section node_id), so relational properties — *"this card is white-on-lime"*, *"this row is left-aligned in its outer wrapper"* — cannot be detected. This blocks Rules 4, 5, 6 (PNG-required) and degrades 50.4, 50.5 (low-confidence VLM fallback).

**Doc TL;DR:** "without this, the converter is structurally limited to ~85% fidelity on any non-trivial design."

```python
async def _export_full_frame_png(
    self,
    file_ref: str,
    access_token: str,
    target_node_id: str,
) -> bytes | None:
    """Export the root frame as a 2x PNG. Returns None on failure (graceful degradation)."""
    try:
        exported = await self.export_images(
            file_ref=file_ref,
            access_token=access_token,
            node_ids=[target_node_id],
            format="png",
            scale=2.0,
        )
        img = exported.get(target_node_id)
        if not img:
            return None
        return await self.download_image_bytes(img)
    except Exception:
        logger.warning("design_sync.figma.full_frame_png_failed", node_id=target_node_id, exc_info=True)
        return None
```

Wire into `sync_tokens_and_structure` immediately after `structure = DesignFileStructure(...)` at line 567 (resolve to mutated copy or rebuild via `dataclasses.replace`).

## Implementation Steps

### Step 1: Add field to `DesignFileStructure`

Edit `app/design_sync/protocol.py:159-164` per spec above.

### Step 2: Decide where to fetch the PNG

**Option A** (chosen): inside `sync_tokens_and_structure`. PNG is part of the structure contract; every consumer that holds a `DesignFileStructure` gets it for free.

**Option B** (rejected): inside `build_document`. Requires every downstream caller to opt in, defeats the "structure carries the PNG" invariant.

Trade-off: Option A adds one API call to every `sync_tokens_and_structure` invocation, which fires from the token-only path too. Mitigation: gate behind `target_node_id is not None` — token-only callers (no specific frame) skip the fetch.

```python
# After line 567 in service.py:
design_image: bytes | None = None
if target_node_id and get_settings().design_sync.full_design_png_enabled:
    design_image = await self._export_full_frame_png(file_ref, access_token, target_node_id)

structure = DesignFileStructure(
    file_name=file_name,
    pages=pages,
    design_image=design_image,
)
```

### Step 3: Add `_export_full_frame_png` helper

Per spec above. Place near `export_frame_screenshots` (line 1613) for symmetry.

### Step 4: Thread through layout_analyzer + component_matcher + visual_verify

All three sites accept the new param via keyword-only arg. Default `None` preserves callers that haven't been updated.

### Step 5: Wire into converter_service

`_convert_with_components` reads `structure.design_image`, threads to all three downstream calls. No new state — pass-through.

### Step 6: Visual verify — low-confidence VLM fallback

When `match_section` confidence < `vlm_low_confidence_threshold`:

```python
if global_design_image is not None and match.confidence < threshold:
    # Pass global + per-section crop to VLM with section bbox prompt
    refined = await vlm.classify_with_global_context(
        section_crop=design_screenshots.get(section.node_id),
        global_image=global_design_image,
        section_y_range=(section.y_position, section.y_position + section.height),
        candidate_components=candidates,
    )
    if refined.confidence > match.confidence:
        match = refined
```

This is the single behavioral change in 50.1 — everything else is pass-through. The downstream gaps (50.4, 50.5, 50.7, 52.x) consume the PNG to do the actual work.

## Dependencies & Blockers

- **Blocks:** 50.4 (nested-card bg sampling), 50.5 (low-confidence VLM with global context), 50.7 (physical-card detection — bbox + sibling visual signals), 52.8 (Rule 4: visible divider absent), 52.9 (Rule 5: asset name vs content)
- **Depends on:** Nothing — entry point is pure pass-through
- **Risk:** PNG can be 2-5 MB; held in memory for conversion lifecycle. Mitigate via `bytes` (released when `DesignFileStructure` goes out of scope) — do not cache in module globals

## Out of Scope

- Per-section image cropping from the global PNG (defer to 50.4 + 52.8)
- VLM prompt schema for global-context classification (defer to 50.5 — this plan only wires the bytes through)
- Caching the PNG in Redis or asset_cache — bytes are request-scoped only
- Image asset resolution mode toggle (Gap 7) — defer to Phase 53.3

### Step 8: Document Converter Gaps

For each failing case, analyze the diff between `actual.html` and `expected.html`:

| Gap Category | What to Look For | Example |
|---|---|---|
| Section classification | Wrong component matched | Hero classified as full-width-image |
| Color override | Missing brand color injection | Default `#1A1A1A` not replaced with `#1e3932` |
| Font override | Missing font swap | Default sans-serif not replaced with SoDo Sans |
| VML buttons | Missing Outlook fallback | `<a>` without `v:roundrect` wrapper |
| Column layout | Wrong column count/width | 2-col detected as single column |
| Slot filling | Missing/wrong text content | Slot unfilled or wrong text hierarchy |
| Dark mode | Missing dark mode overrides | No `[data-ogsc]` selectors |
| Mobile classes | Missing responsive classes | No `.column` class on column divs |

Save analysis to `data/debug/converter-gap-analysis.md` for Phase 40.3–40.6.

## Preflight Warnings

1. **Manifest `sections` count must be exact** — `TestSnapshotSectionCount` asserts `result.sections_count == manifest.sections`. If the converter splits/merges sections differently than the reference, the count will mismatch. This is a bug to fix, not a test to skip.
2. **Image paths in expected.html** — References use `../../design-assets/{id}/` relative paths. Ensure design-asset directories map correctly to case IDs (connection ID ≠ case ID).
3. **Expected.html is the reference HTML as-is** — Do NOT normalize or modify the hand-built reference. The test's `_normalize_html()` handles comparison tolerance.
4. **Cases 7/8/9 deferred** — The task mentions newsletter, transactional, and minimal cases but no reference HTMLs exist for them. These require finding suitable Figma designs or creating them from scratch — out of scope for the core 3-case activation.

**Manifest schema** (`component_manifest.yaml`):
```yaml
- slug: string          # REQUIRED — maps to {slug}.html
  name: string          # REQUIRED
  description: string   # REQUIRED
  category: action|content|structure|interactive|animation|typography|utility
  compatibility: full|partial_samsung|partial_interactive
  slot_definitions:     # optional — auto-detected from data-slot attrs if omitted
    - slot_id: string
      slot_type: body|text|headline|image|cta|content
      selector: "[data-slot='slot_id']"
      required: bool
  default_tokens: {}    # optional
  inject_target: body   # default; or head_style, structural, attribute
```

**HTML template pattern** (from `hero-block.html`, `product-card.html`):
- `<!-- Component: {slug} -->` comment header
- `<!--[if mso]>` VML wrapper for Outlook
- `<table role="presentation" width="100%" cellpadding="0" cellspacing="0" border="0">`
- `data-slot="slot_id"` on content elements
- Inline styles on every element, `font-family: Arial, sans-serif` fallback
- Dark mode via classes: `dark-bg`, `dark-text`, `dm_fontc` etc.
- `mso-line-height-rule: exactly` on spacing
- `<p style="margin:0 0 Npx 0;">` for text inside `<td>` (accessible)
- `<h2>` headings with inline styles inside `<td>` (accessible)
- No `<div>` for layout (width/flex/float)

**Loading pipeline:** `component_manifest.yaml` → `file_loader._load_manifest()` (lru_cached) → `seeds._build_all_seeds()` → DB seeding. Slots auto-detected via `_DATA_SLOT_RE = re.compile(r'data-slot="([^"]+)"')`.

## Implementation Steps

### Step 1: Create 61 HTML template files

For each new component, create `email-templates/components/{slug}.html` following this exact pattern:

```html
<!-- Component: {slug} -->
<!--[if mso]>
<table role="presentation" width="600" align="center" cellpadding="0" cellspacing="0" border="0"><tr><td>
<![endif]-->
<table role="presentation" width="100%" cellpadding="0" cellspacing="0" border="0"
       style="background-color: #ffffff;">
  <tr>
    <td style="padding: 24px; font-family: Arial, sans-serif;">
      <!-- content with data-slot attrs -->
    </td>
  </tr>
</table>
<!--[if mso]>
</td></tr></table>
<![endif]-->
```

**Rules for every file:**
- `<table>/<tr>/<td>` only — NO `<div>`/`<p>` for layout (width/flex/float)
- `<p style="margin:0 0 10px 0;">` for text paragraphs inside `<td>`
- `<h1>`–`<h6>` with inline styles for headings inside `<td>`
- `role="presentation"` on all layout tables
- `cellpadding="0" cellspacing="0" border="0"` on all tables
- `data-slot="{slot_id}"` on every replaceable content element
- Inline `font-family: Arial, sans-serif` fallback on every text element
- `display: block` on all `<img>` tags
- `alt` text on all images (not generic like "image")
- MSO conditional wrapper (`<!--[if mso]>`) on complex components
- `mso-line-height-rule: exactly` on spacing tds
- Dark mode classes where appropriate (`dark-bg`, `dark-text`)

**Order of creation** — batch by complexity:
1. Simple (dividers, spacers, text variants, footer/header variants): 7 files
2. Medium (zigzag, cards, testimonials, bio, event, video placeholder, social proof, announcement, app download, nav): 30 files
3. Complex (pricing tables, card grids, mosaic, countdown timers, FAQ, loyalty, survey, progressive disclosure, sidebar): 24 files

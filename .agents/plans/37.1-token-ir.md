# 37.1: Token Intermediate Representation (IR)

## Goal

Define an ESP-agnostic token format. Parse any ESP's personalization syntax into normalized IR nodes, emit from IR to any target ESP syntax. This is the foundation for cross-ESP token rewriting.

## IR Schema

Frozen dataclasses in `app/connectors/token_ir.py`:

```
TokenIR:
  variables: tuple[VariableToken, ...]
  conditionals: tuple[ConditionalToken, ...]
  loops: tuple[LoopToken, ...]

VariableToken:
  name: str                    # normalized name: "first_name" (ESP-agnostic)
  filters: tuple[FilterExpr, ...]  # chained filters
  fallback: str | None
  source_syntax: str           # original raw token string
  source_span: tuple[int, int] # char offsets in source HTML

ConditionalToken:
  variable: str
  operator: Literal["exists", "eq", "neq", "gt", "lt", "contains"]
  value: str | None
  body_html: str
  else_html: str | None
  source_span: tuple[int, int]

LoopToken:
  item_name: str       # "item"
  collection: str      # "event.items"
  body_html: str
  source_span: tuple[int, int]

FilterExpr:
  name: str            # normalized: "default", "uppercase", "date", "currency", "truncate"
  args: tuple[str, ...]

ESPPlatform = Literal["braze", "sfmc", "adobe_campaign", "klaviyo", "hubspot", "mailchimp", "sendgrid", "activecampaign", "iterable", "brevo"]
```

## Parser Registry

```python
PARSERS: dict[ESPPlatform, Callable[[str], TokenIR]] = {
    "braze": parse_braze,
    "sfmc": parse_sfmc,
    "klaviyo": parse_klaviyo,
    "hubspot": parse_hubspot,
    "mailchimp": parse_mailchimp,
    "iterable": parse_iterable,
    "sendgrid": parse_sendgrid,         # same as iterable (Handlebars)
    "activecampaign": parse_activecampaign,
    "brevo": parse_brevo,              # same as klaviyo (Django/Jinja2)
    "adobe_campaign": parse_adobe,
}
```

Each parser: extract tokens via regex → normalize variable names (strip platform prefixes like `contact.`, `person.`, `${}`) → parse filter chains → build IR nodes with source spans.

**Reuse existing patterns** from `personalisation_validator.py:_PLATFORM_PATTERNS` where possible. Do NOT duplicate regex — import and extend.

## Emitter Registry

```python
EMITTERS: dict[ESPPlatform, Callable[[TokenIR, str], str]] = { ... }
```

Each emitter: iterate IR nodes sorted by source_span descending (right-to-left to preserve offsets) → replace each span with target ESP syntax.

### Filter mapping table

| IR filter | Braze | Klaviyo | HubSpot | SFMC | Mailchimp | Iterable/SendGrid |
|-----------|-------|---------|---------|------|-----------|-------------------|
| `default` | `\|default:"X"` | `\|default:"X"` | `\|default("X")` | `IIF(Empty(@v),"X",@v)` | N/A | `{{defaultIfEmpty X}}` |
| `uppercase` | `\|upcase` | `\|upper` | `\|upper` | `Uppercase(@v)` | `\|upper` | N/A |
| `lowercase` | `\|downcase` | `\|lower` | `\|lower` | `Lowercase(@v)` | `\|lower` | N/A |
| `date` | `\|date:"%b %d"` | `\|date:"fmt"` | `\|datetimeformat("fmt")` | `Format(@d,"fmt")` | N/A | N/A |
| `truncate` | `\|truncate:N` | `\|truncatechars:N` | `\|truncate(N)` | `Substring(@v,1,N)` | N/A | N/A |

When a filter has no equivalent on target ESP: emit `<!-- MANUAL: {filter} not supported on {target_esp} -->` + raw variable value. Add warning to result.

### Step 2: Update openapi-ts config for v0.66+ compatibility

The `client` top-level option was moved to plugins in recent versions. Update the config:

```typescript
// cms/packages/sdk/openapi-ts.config.ts
import { defineConfig } from "@hey-api/openapi-ts";

export default defineConfig({
  input: "openapi.json",
  output: {
    path: "src/client",
  },
  plugins: [
    "@hey-api/typescript",
    "@hey-api/sdk",
    {
      name: "@hey-api/client-fetch",
      bundle: true,
    },
  ],
});
```

**Key changes:**
- `input` changed from URL to local file path (`openapi.json`) â€” no running backend required
- `client` removed, replaced with `plugins` array
- `output.format: "prettier"` removed (handled by plugins internally)
- Added `@hey-api/typescript` and `@hey-api/sdk` explicitly

**Fallback:** If v0.66.0 doesn't support the `plugins` array, keep the original `client` format but change `input` to local file:

```typescript
export default defineConfig({
  client: "@hey-api/client-fetch",
  input: "openapi.json",
  output: {
    path: "src/client",
  },
});
```

## Troubleshooting

### If `plugins` config fails with v0.66.0
The plugins API was introduced gradually. If v0.66.0 doesn't support it, use the legacy format:
```typescript
export default defineConfig({
  client: "@hey-api/client-fetch",
  input: "openapi.json",
  output: { path: "src/client" },
});
```

### If generated client doesn't export `client` instance
The web app imports `{ client }` from the SDK. If the generated code doesn't export a `client` instance, create a thin wrapper:
```typescript
// cms/packages/sdk/src/index.ts
export { client } from "./client/index";
export type * from "./client/types.gen";
```
And update package.json exports to point to this wrapper.

### If @hey-api/client-fetch version mismatch
The `@hey-api/client-fetch@^0.8.0` may need to match the `@hey-api/openapi-ts` version. Check the hey-api docs for compatible version pairs.

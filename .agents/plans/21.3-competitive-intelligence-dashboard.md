# Plan: 21.3 Competitive Intelligence Dashboard & Tests

## Context
Phase 21 introduces Real-Time Ontology Sync & Competitive Intelligence. The backend is complete (21.1–21.2): `CanIEmailSyncService` fetches/diffs Can I Email data, `RenderingChangeDetector` detects visual rendering changes across email clients, and `CompetitiveReport` provides audience-scoped feasibility analysis. This task builds the frontend dashboard to surface all of this to users.

**Backend endpoints available:**
- `GET /api/v1/ontology/sync-status` → `SyncStatusResponse` (last sync time, commit SHA, feature count, errors)
- `POST /api/v1/ontology/sync` → `SyncReportResponse` (changelog, new properties, updated levels — admin only, dry_run=true default)
- `GET /api/v1/ontology/competitive-report` → `CompetitiveReportResponse` (hub advantages, gaps, opportunities, full matrix)
- `GET /api/v1/ontology/clients` → `EmailClientResponse[]` (all email clients from registry)

**No rendering change detection endpoint exists yet** — only the background poller. We'll skip the rendering changelog component for now and note it as a TODO pending a `GET /api/v1/ontology/rendering-changes` endpoint in a future task.

### Step 2: SWR Hooks (`cms/apps/web/src/hooks/use-ontology.ts`)

```typescript
"use client";

import useSWR from "swr";
import useSWRMutation from "swr/mutation";
import { fetcher } from "@/lib/swr-fetcher";
import { longMutationFetcher } from "@/lib/mutation-fetcher";
import type { ApiError } from "@/lib/api-error";
import type {
  SyncStatusResponse,
  SyncReportResponse,
  CompetitiveReportResponse,
  EmailClientSchema,
} from "@/types/ontology";

/** GET /api/v1/ontology/sync-status — poll sync state. */
export function useOntologySyncStatus() {
  return useSWR<SyncStatusResponse>(
    "/api/v1/ontology/sync-status",
    fetcher,
    { refreshInterval: 60_000 }, // poll every 60s
  );
}

/** POST /api/v1/ontology/sync — trigger manual sync (admin only). */
export function useOntologySync() {
  return useSWRMutation<SyncReportResponse, ApiError, string, { dry_run?: boolean }>(
    "/api/v1/ontology/sync",
    longMutationFetcher,
  );
}

/** GET /api/v1/ontology/competitive-report — audience-scoped report. */
export function useCompetitiveReport(clientIds?: string[]) {
  const params = new URLSearchParams();
  if (clientIds?.length) {
    for (const id of clientIds) {
      params.append("client_ids", id);
    }
  }
  const key = clientIds?.length
    ? `/api/v1/ontology/competitive-report?${params.toString()}`
    : "/api/v1/ontology/competitive-report";

  return useSWR<CompetitiveReportResponse>(key, fetcher);
}

/** GET /api/v1/ontology/clients — all email clients from registry. */
export function useEmailClients() {
  return useSWR<EmailClientSchema[]>("/api/v1/ontology/clients", fetcher);
}
```

# Plan: 46.3 ESP Connector Key Rotation

## Context

SFMC and Braze have per-key rate limits for send and content API operations. During campaign pushes (bulk template upload + list segmentation + send), a single key can exhaust its quota. Wire `CredentialPool` (Phase 46.1) into ESP connectors so export/push operations rotate across multiple API keys per provider.

**Pool credential flow (new):** `get_credential_pool("braze")` → `pool.get_key()` → `CredentialLease` → extract key → build `dict[str, str]` → `provider.export()` → `lease.report_success/failure()`.

## Implementation Steps

### Step 1: Add pool-aware credential resolution to each connector

Each connector service gets:
1. Optional `_pool: CredentialPool | None` initialized in `__init__()` (same pattern as `AnthropicProvider`)
2. A `_credentials_from_pool()` helper that calls `pool.get_key()` and builds the provider-specific credential dict
3. In `export()`: if `credentials is None` and `_pool` exists → use pool; else use passed credentials

**Braze & Taxi** (simple auth — bare API key in pool):
```python
def __init__(self) -> None:
    settings = get_settings()
    self._base_url = settings.esp_sync.braze_base_url
    self._pool: CredentialPool | None = None
    if settings.credentials.enabled and "braze" in settings.credentials.pools:
        self._pool = get_credential_pool("braze")

async def _lease_credentials(self) -> tuple[dict[str, str], CredentialLease]:
    """Get credentials from pool. Raises NoHealthyCredentialsError if exhausted."""
    lease = await self._pool.get_key()  # type: ignore[union-attr]
    return {"api_key": lease.key}, lease
```

**SFMC & Adobe** (OAuth — JSON-encoded credential dict in pool):
```python
async def _lease_credentials(self) -> tuple[dict[str, str], CredentialLease]:
    lease = await self._pool.get_key()  # type: ignore[union-attr]
    creds: dict[str, str] = json.loads(lease.key)
    return creds, lease
```

### Step 2: Modify `export()` in each connector to use pool + report lifecycle

Pattern for each connector's `export()`:
```python
async def export(self, html: str, name: str, credentials: dict[str, str] | None = None) -> str:
    lease: CredentialLease | None = None

    # Pool rotation: use pool if no explicit credentials passed and pool is configured
    if credentials is None and self._pool is not None:
        credentials, lease = await self._lease_credentials()

    if credentials is not None:
        # ... existing HTTP call logic ...
        try:
            resp = await resilient_request(client, "POST", url, ...)
            resp.raise_for_status()
            if lease:
                await lease.report_success()
            return external_id
        except httpx.HTTPStatusError as exc:
            if lease:
                await lease.report_failure(exc.response.status_code)
            raise ExportFailedError(...) from exc

    # Mock fallback (no credentials, no pool)
    return f"braze_cb_{name.lower().replace(' ', '_')}"
```

### Step 3: Update ConnectorService for pool-aware dispatch

In `app/connectors/service.py`, the `export()` method currently always resolves credentials from DB when `connection_id` is provided. Add pool fallback:

```python
async def export(self, data: ExportRequest, user: User) -> ExportResponse:
    provider = self._get_provider(data.connector_type)
    
    credentials: dict[str, str] | None = None
    if data.connection_id is not None:
        # DB credentials take priority (explicit connection)
        credentials = await self._resolve_credentials(data.connection_id, user)
    # If no connection_id, provider will try its own pool (if configured)
    
    try:
        external_id = await provider.export(html, data.content_block_name, credentials)
    except NoHealthyCredentialsError:
        raise ExportFailedError(
            f"All API keys exhausted for {data.connector_type} — "
            "retry after cooldown or add more keys"
        )
```

**Key design decision:** DB credentials (from `connection_id`) always take priority over pool. Pool is used only when no explicit connection is provided — this is the "service-level API key" scenario for automated/bulk operations.

## Preflight Warnings

- SFMC/Adobe `_token_cache` is a `ClassVar` — each pool key will hash to a different cache key naturally (via `_cache_key()`), so multi-key rotation won't cause cache collisions.
- `resilient_request()` has its own retry logic — pool cooldowns are additive, not conflicting. A 429 from `resilient_request` will both trigger the resilient retry AND inform the pool via `report_failure()`.
- Pre-existing 5 pyright errors in `app/connectors/` — do not increase.

## Security Checklist

- No new endpoints added — this is internal wiring only
- Pool keys come from `CREDENTIALS__POOLS` env var (same security boundary as existing `CREDENTIALS__*`)
- `CredentialLease.key_hash` (SHA-256 prefix) used in logs — raw keys never logged
- `NoHealthyCredentialsError` message does not leak key values
- JSON-decoded credentials (SFMC/Adobe) validated by existing provider auth flow

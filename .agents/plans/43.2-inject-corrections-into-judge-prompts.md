# Plan: 43.2 Inject Corrections into Judge Prompt Template

## Context

Phase 43.1 generates per-agent correction YAML files (`traces/corrections/{agent}_judge_corrections.yaml`) from calibration disagreements. Phase 43.2 injects these corrections into judge prompts as few-shot anti-examples — showing the judge its own past mistakes with correct answers. False positives are prioritized (they let bad output through).

Add `format_corrections_section()` after `format_golden_section()` (after base.py:146):

```python
def format_corrections_section(agent_name: str) -> str:
    """Build correction anti-examples section for a given agent.

    Loads from traces/corrections/{agent}_judge_corrections.yaml.
    Prioritizes false positive corrections (more damaging — they let bad output through).
    Returns empty string if no corrections file or corrections disabled.
    """
    if not _corrections_enabled:
        return ""

    corrections_dir = Path(__file__).resolve().parents[3] / "traces" / "corrections"
    path = corrections_dir / f"{agent_name}_judge_corrections.yaml"

    if not path.exists():
        return ""

    data = _load_corrections_cached(str(path), path.stat().st_mtime)
    corrections: list[dict[str, Any]] = data.get("corrections", [])
    if not corrections:
        return ""

    # Prioritize false positives over false negatives
    fp = [c for c in corrections if c.get("type") == "false_positive"]
    fn = [c for c in corrections if c.get("type") == "false_negative"]
    ordered = fp + fn

    char_budget = _CORRECTIONS_TOKEN_BUDGET * 4
    parts: list[str] = []
    used = 0

    for i, c in enumerate(ordered, 1):
        entry = (
            f"{i}. **{c['criterion']}** on trace {c['trace_id']}:\n"
            f"   You said: {c['judge_said']}. Correct answer: {c['correct_answer']}.\n"
            f"   Your reasoning was: \"{c.get('judge_reasoning', '')[:200]}\"\n"
            f"   The mistake: {c.get('pattern', '')}"
        )
        if used + len(entry) > char_budget:
            break
        parts.append(entry)
        used += len(entry)

    if not parts:
        return ""

    header = (
        "## CORRECTION EXAMPLES (from prior calibration)\n"
        "You previously made these mistakes. Learn from them:\n"
    )
    return header + "\n" + "\n\n".join(parts)


@lru_cache(maxsize=16)
def _load_corrections_cached(path_str: str, mtime: float) -> dict[str, Any]:
    """Load and cache correction YAML. Cache key includes mtime for staleness."""
    with Path(path_str).open() as f:
        data: dict[str, Any] = yaml.safe_load(f) or {}
    return data
```

```python
def build_system_prompt(
    criteria: list[JudgeCriteria],
    agent_name: str,
) -> str:
    """Build system prompt with criteria and corrections injected."""
    criteria_block = build_criteria_block(criteria)
    corrections = format_corrections_section(agent_name)
    corrections_block = f"\n{corrections}\n" if corrections else ""
    return SYSTEM_PROMPT_TEMPLATE.format(
        criteria_block=criteria_block,
        corrections_block=corrections_block,
    )
```

## Preflight Warnings

- `test_golden_prompt_injection.py` calls `judge.build_prompt()` — if it asserts on exact prompt structure (e.g., expects no corrections block), it may need a `set_corrections_enabled(False)` fixture or the test will see a `{corrections_block}` placeholder change.
- `SYSTEM_PROMPT_TEMPLATE` is exported and used in 10+ files. Adding `{corrections_block}` placeholder means any code that calls `.format(criteria_block=...)` WITHOUT `corrections_block=` will raise `KeyError`. The `build_system_prompt()` wrapper avoids this, but any direct template usage outside judge files must be found and updated.
- The `import_annotator/evals/judges/import_judge.py` file imports from base.py — verify it follows the same pattern and update accordingly.
- Module-level `_corrections_enabled` toggle is not thread-safe. Fine for CLI usage but would need `threading.local()` if used in async server context. Current eval pipeline is CLI-only.

## Security Checklist

- No new endpoints — this is CLI/eval-only code
- Correction YAML files are local, generated from our own calibration data — no user input reaches prompt surface
- `yaml.safe_load()` used (not `yaml.load()`) — prevents arbitrary code execution
- Path construction uses `Path` with hardcoded directory structure — no path traversal

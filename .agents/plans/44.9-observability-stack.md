# Plan: 44.9 Observability Stack for Local Development

## Context

Debugging multi-service issues (app, maizzle-builder, mock-esp, cms) requires tailing 4+ log streams. `structlog` already emits JSON to stdout — Loki ingests it natively. A pre-configured Grafana dashboard makes local debugging 10x faster and serves as a production observability template.

## Implementation Steps

### Step 1: Create `observability/` directory structure

```
observability/
├── loki-config.yaml
├── promtail-config.yaml
├── dashboards/
│   └── email-hub.json
└── provisioning/
    ├── datasources/
    │   └── datasource.yaml
    └── dashboards/
        └── dashboard.yaml
```

### Step 2: Create `observability/loki-config.yaml`

Minimal local-dev config:
- `auth_enabled: false`
- BoltDB shipper for index, filesystem for chunks
- `retention_period: 168h` (7 days)
- Listen on `:3100`
- Local storage under `/loki/` inside container (ephemeral volume)

### Step 3: Create `observability/promtail-config.yaml`

- Scrape Docker container logs via `/var/run/docker.sock`
- Relabel: extract `container_name` from Docker labels
- Pipeline stages:
  1. `docker{}` — parse Docker JSON log wrapper
  2. `json` — parse structlog fields (`level`, `event`, `request_id`, `logger`)
  3. `labels` — promote `level`, `event` to Loki labels
  4. `timestamp` — parse ISO 8601 from structlog `timestamp` field
- Non-JSON logs (cms, nginx, db) pass through as raw text with `container_name` label

### Step 4: Create `observability/dashboards/email-hub.json`

Grafana dashboard JSON with panels:

| Panel | Type | Query | Purpose |
|-------|------|-------|---------|
| Log Stream | Logs | `{container_name=~".+"}` | All service logs, filterable |
| Error Rate | Time series | `rate({level="error"}[5m])` by `container_name` | Error spikes |
| Request Latency | Time series | Extract `duration_seconds` from `event="request.http_completed"` | p50/p95/p99 |
| Status Codes | Bar gauge | `count_over_time({event="request.http_completed"} \| json \| status_code=~"5..")[5m]` | 5xx vs 4xx vs 2xx |
| Service Health | Stat | `count_over_time({event=~".*health.*"}[1m])` per service | Liveness |
| Agent Eval | Logs | `{container_name="merkle-email-hub-app-1"} \|= "eval"` | Agent evaluation events |

Dashboard settings: auto-refresh 10s, time range last 1h, Loki datasource.

### Step 5: Create Grafana provisioning files

**`observability/provisioning/datasources/datasource.yaml`:**
```yaml
apiVersion: 1
datasources:
  - name: Loki
    type: loki
    access: proxy
    url: http://loki:3100
    isDefault: true
```

**`observability/provisioning/dashboards/dashboard.yaml`:**
```yaml
apiVersion: 1
providers:
  - name: default
    type: file
    options:
      path: /var/lib/grafana/dashboards
```

### Step 6: Create `docker-compose.observability.yml`

```yaml
services:
  loki:
    image: grafana/loki:3.4
    ports:
      - "3100:3100"
    volumes:
      - ./observability/loki-config.yaml:/etc/loki/local-config.yaml
    command: -config.file=/etc/loki/local-config.yaml
    healthcheck:
      test: ["CMD-SHELL", "wget -qO- http://localhost:3100/ready || exit 1"]
      interval: 10s
      timeout: 5s
      retries: 3

  promtail:
    image: grafana/promtail:3.4
    volumes:
      - /var/run/docker.sock:/var/run/docker.sock:ro
      - ./observability/promtail-config.yaml:/etc/promtail/config.yml
    command: -config.file=/etc/promtail/config.yml
    depends_on:
      loki:
        condition: service_healthy

  grafana:
    image: grafana/grafana:11.6
    ports:
      - "3333:3000"
    volumes:
      - ./observability/dashboards/:/var/lib/grafana/dashboards/
      - ./observability/provisioning/:/etc/grafana/provisioning/
    environment:
      GF_AUTH_ANONYMOUS_ENABLED: "true"
      GF_AUTH_ANONYMOUS_ORG_ROLE: Admin
      GF_DASHBOARDS_DEFAULT_HOME_DASHBOARD_PATH: /var/lib/grafana/dashboards/email-hub.json
    depends_on:
      loki:
        condition: service_healthy
```

No shared network needed — compose merge inherits the `default` network from `docker-compose.yml`, so Promtail can reach all service containers and Grafana can reach Loki.

### Step 7: Add Makefile targets

```makefile
dev-observe:  ## Start all services + observability stack (Grafana :3333, Loki :3100)
	docker compose -f docker-compose.yml -f docker-compose.observability.yml up --build

grafana:  ## Open Grafana dashboard in browser
	open http://localhost:3333
```

## Preflight Warnings

- Promtail needs Docker socket access (`/var/run/docker.sock:ro`) — works on macOS Docker Desktop and Linux. May need adjustment for rootless Docker or Podman.
- Grafana anonymous admin is dev-only. Production must use proper auth.
- Loki 3.4 uses TSDB index by default (not BoltDB). Config must match the version's expected schema.
- Dashboard JSON panel queries depend on structlog field names — if `app/core/logging.py` processors change, dashboard queries break.

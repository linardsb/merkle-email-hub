# Plan: 49.4 Token Override Element-Type Expansion

## Context

`_apply_token_overrides()` in `component_renderer.py:407` dispatches to `_replace_heading_font()`, `_replace_body_font()`, `_replace_heading_color()`, `_replace_body_color()` — all targeting only `<td data-slot="heading">` and `<td data-slot="body">`. Two problems:

1. **Narrow data-slot match:** Only `"heading"` and `"body"` matched. 23 components use `data-slot="heading"` but many use `"headline"`, `"title"`, `"subtext"`. Only 4 components have `data-slot="body"` — others use `"body_text"`, `"description"`, `"caption"`.
2. **No class-based fallback:** Components without `data-slot` but with semantic classes (`hero-title`, `textblock-heading`, `artcard-body`, etc.) are never matched. The reference template (`reframe-2025.html`) uses `class="dark-text"` on `<td>` with inline styles — no `data-slot` attrs at all.

Result: Figma-extracted fonts/colors (`Helvetica`, `#000000`) are ignored; component template defaults (`Arial`, `#333333`, `#555555`) survive in output.

### Step 2: Rewrite `_replace_*` methods (`component_renderer.py:442-470`)

Each method becomes a two-pass replacement: data-slot first, then class-based fallback.

```python
def _replace_heading_font(self, html_str: str, font: str) -> str:
    """Replace font-family on heading elements (data-slot or semantic class)."""
    safe = html.escape(font, quote=True)
    repl = rf"\g<1>font-family:{safe}\g<2>"
    result = _HEADING_SLOT_FONT_RE.sub(repl, html_str)
    return _HEADING_CLASS_FONT_RE.sub(repl, result)

def _replace_body_font(self, html_str: str, font: str) -> str:
    """Replace font-family on body elements (data-slot or semantic class)."""
    safe = html.escape(font, quote=True)
    repl = rf"\g<1>font-family:{safe}\g<2>"
    result = _BODY_SLOT_FONT_RE.sub(repl, html_str)
    return _BODY_CLASS_FONT_RE.sub(repl, result)

def _replace_heading_color(self, html_str: str, color: str) -> str:
    """Replace color on heading elements. Negative lookbehind avoids background-color."""
    safe = html.escape(color, quote=True)
    repl = rf"\g<1>color:{safe}\g<2>"
    result = _HEADING_SLOT_COLOR_RE.sub(repl, html_str)
    return _HEADING_CLASS_COLOR_RE.sub(repl, result)

def _replace_body_color(self, html_str: str, color: str) -> str:
    """Replace color on body elements. Negative lookbehind avoids background-color."""
    safe = html.escape(color, quote=True)
    repl = rf"\g<1>color:{safe}\g<2>"
    result = _BODY_SLOT_COLOR_RE.sub(repl, html_str)
    return _BODY_CLASS_COLOR_RE.sub(repl, result)
```

## Preflight Warnings

- Existing tests in `TestTokenOverrides` use `renderer.render_section()` end-to-end with real component templates.

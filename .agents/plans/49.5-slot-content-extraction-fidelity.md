# Plan: 49.5 Per-Node Slot Content Extraction Fidelity

## Context

The converter pipeline produces placeholder text ("Section Heading", "Editorial heading", "Lorem ipsum") even when actual content exists in the Figma design. Root cause is a 3-stage content loss:

1. **Flat extraction** — `_extract_texts()` at `layout_analyzer.py:860` walks the entire section node tree, dumping all TEXT nodes into one flat `EmailSection.texts` list. Parent-child grouping is lost.
2. **First-N slot fill** — `_build_slot_fills()` at `component_matcher.py:486` dispatches to per-slug builders (e.g. `_fills_text_block` at :791) that call `_first_heading()`/`_first_body()`, grabbing `texts[0]`/`texts[1]` and silently dropping the rest.
3. **Silent defaults** — When a SlotFill isn't generated (content dropped), the component template's placeholder text survives in the rendered output.

The fix must be **generic** — it must improve extraction for any Figma design, not just specific templates. REFRAME 2025 is used as a reference case only.

### Content Flow Through Pipeline

```
Figma JSON → DesignNode tree
  ↓ _get_section_candidates() — unwraps single wrapper, returns top-level frames
  ↓ analyze_layout() — per candidate:
      _extract_buttons() → buttons
      _extract_texts(node, exclude=button_ids) → flat texts list
      _detect_content_hierarchy() → marks is_heading via 1.3× median
      _extract_images() → flat images list
      _detect_column_layout_with_groups() → column_groups (column-level grouping)
      _compute_content_roles() → role hints
      → EmailSection(texts=flat, images=flat, column_groups=col-level)
  ↓ detect_repeating_groups() — groups similar adjacent sections (49.1)
  ↓ match_all() → per section: _score_candidates() → slug, _build_slot_fills() → fills
  ↓ renderer._fill_slots() → HTML
```

**Problem**: `_extract_texts()` at :860 recursively walks the entire node tree via `_walk_for_texts()` at :871, collecting ALL TEXT nodes into one flat list. When a section has 5 child groups (e.g., 5 "reason" blocks), all 10 texts (5 headings + 5 bodies) end up in one list. `_first_heading()` grabs `texts[0]`, `_first_body()` grabs the first non-heading — the other 8 texts are dropped.

**Existing grouping**: `column_groups` already preserves per-column structure (used by `_build_column_fills_from_groups` at :1132). But this only works for multi-column layouts. Single-column sections with multiple child groups (like stacked icon+text blocks) have no grouping.

### Integration with 49.1/49.2 (Sibling Detector)

The sibling detector groups **adjacent sections** that share structural signatures. This is complementary to content_groups which groups **child nodes within a single section**. When a section has 5 repeated child blocks but they're all inside one Figma frame, the sibling detector won't split them — they remain one `EmailSection` with all content flattened.

Content groups feed into the slot filler so each group maps to one template instance (for repeating patterns within a section), while sibling groups feed into the renderer so each group gets a container wrapper.

### Step 2: Add `_extract_content_groups()` function (`layout_analyzer.py`)

Add after `_extract_buttons()` (~line 1072):

```python
def _extract_content_groups(
    node: DesignNode,
    *,
    button_name_hints: list[str] | None = None,
) -> list[ContentGroup]:
    """Extract content groups from direct child frames of a section node.

    Each direct child FRAME/GROUP/COMPONENT that contains at least one TEXT or IMAGE
    node becomes a ContentGroup. This preserves the parent-child relationship that
    flat extraction loses.

    Only produces groups when the section has 2+ qualifying child frames — if there's
    only one child frame (or all content is at the root level), returns empty list
    to signal the caller should use flat extraction.
    """
```

Logic:
- Iterate `node.children` — for each child that is FRAME/GROUP/COMPONENT:
  - Extract buttons, texts (excluding button nodes), images from that child subtree
  - If child has any content (texts or images or buttons), create a `ContentGroup`
- If result has < 2 groups, return `[]` (not enough structure to warrant grouping)
- For each group, assign `role_hint` to texts: largest font = "heading", smallest font (if < 14px or < 0.7× median) = "label", text inside button-like frames = "cta", rest = "body"
- Set `source_frame_id` on each TextBlock to the child frame's node_id

### Step 4: Add `_assign_role_hints()` helper (`layout_analyzer.py`)

Add near `_detect_content_hierarchy()` at line ~1100:

```python
def _assign_role_hints(texts: list[TextBlock], frame_id: str) -> list[TextBlock]:
    """Assign role_hint and source_frame_id to text blocks within a group."""
    if not texts:
        return texts
    sizes = [t.font_size for t in texts if t.font_size is not None]
    if not sizes:
        return [dataclasses.replace(t, source_frame_id=frame_id, role_hint="body") for t in texts]

    max_size = max(sizes)
    median_size = statistics.median(sizes)
    label_threshold = min(14.0, median_size * 0.7)

    result: list[TextBlock] = []
    for t in texts:
        hint = "body"
        if t.font_size is not None:
            if t.font_size >= max_size and t.font_size > median_size * 1.2:
                hint = "heading"
            elif t.font_size <= label_threshold:
                hint = "label"
        result.append(dataclasses.replace(t, source_frame_id=frame_id, role_hint=hint))
    return result
```

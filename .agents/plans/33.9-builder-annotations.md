# Plan: 33.9 Builder Annotations for Visual Builder Sync

## Context

The Figma design sync pipeline produces HTML without builder annotations (`data-section-id`, `data-component-name`, `data-slot-name`). The frontend sync engine's Strategy 1 (annotated HTML) never matches ‚Äî it falls through to Strategy 2 (structural analysis), producing `SectionNode` objects with `componentId=0` and empty `slotValues`. The visual builder shows placeholders instead of actual Figma content. Fix: annotate converter output at generation time.

## Implementation Steps

### Step 1: Add slot counter type and helper

In `converter.py`, add a helper to generate slot names with dedup:

```python
def _next_slot_name(counter: dict[str, int], slot_type: str) -> str:
    """Generate unique slot name: 'body', 'body_2', 'body_3', etc."""
    count = counter.get(slot_type, 0) + 1
    counter[slot_type] = count
    return slot_type if count == 1 else f"{slot_type}_{count}"
```

### Step 2: Modify `_render_semantic_text()` (converter.py :313)

Add `slot_counter: dict[str, int] | None = None` parameter. When provided:
- Headings (`<h1>`-`<h3>`): inject `data-slot-name="{_next_slot_name(counter, 'heading')}"`
- Body `<p>`: inject `data-slot-name="{_next_slot_name(counter, 'body')}"`
- Multi-line `<p>` tags: each gets its own slot name

### Step 3: Modify `_render_button()` (converter.py :368)

Add `slot_counter: dict[str, int] | None = None` parameter. When provided:
- Inject `data-slot-name="{_next_slot_name(counter, 'cta')}"` on the `<a>` tag

### Step 4: Modify `node_to_email_html()` (converter.py :446)

Add parameters:
- `section_index: int | None = None` ‚Äî for `data-section-id` on section root
- `slot_counter: dict[str, int] | None = None` ‚Äî shared per-section counter

Changes:
- Pass `slot_counter` to `_render_semantic_text()` and `_render_button()` calls
- IMAGE nodes: add `data-slot-name="{_next_slot_name(counter, 'image')}"` alongside existing `data-node-id`
- FRAME nodes at depth 0 (section root): add `data-component-name="{html.escape(node.name)}"` on the outer `<table>`
- Thread `slot_counter` through recursive calls to children

### Step 5: Modify `DesignConverterService.convert()` (converter_service.py :231)

For each frame in the conversion loop:
- Create a fresh `slot_counter: dict[str, int] = {}` per section
- Pass `section_index=idx` and `slot_counter=slot_counter` to `node_to_email_html()`
- Modify the `<tr><td>` wrapper:
  ```python
  section_parts.append(
      f'<tr data-section-id="section_{idx}"><td>\n{section_html}\n</td></tr>'
  )
  ```

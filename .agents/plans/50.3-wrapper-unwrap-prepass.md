# Plan: 50.3 Wrapper Unwrap Pre-Pass (Gap 1)

## Context

A `mj-wrapper` (or any FRAME with a coloured fill + multiple `mj-section` children) is treated as **one** `EmailSection` today. This is the root cause of:
- **Wrapper #5 / #6 in LEGO email** — heading + 2 cards / heading + 4 product rows collapse into one component.
- **Gap 4** — composite footer wrapper has 4 heterogeneous children (user-info row + white card + links + legal); converter sees one section, emits one `_fills_footer` join.
- **Gap 6** — hero text wrapper #3 and hero image wrapper #2 are separate `mj-wrapper`s; their relationship is lost.

Per `docs/architecture/opus-figma-to-html-process.md` Gap 1 + Part 4 §2:
> *"the `mj-wrapper` boundary is not the section boundary. A `mj-wrapper` is a Figma layout convenience; it routinely groups a section heading with the N cards beneath it on the same coloured background... Two-stage layout: wrapper unwrap → section classify."*

### Modify: `app/design_sync/figma/layout_analyzer.py:385-410` — extend `_get_section_candidates`

Add a second unwrap pass:

```python
def _get_section_candidates(
    structure: DesignFileStructure,
    *,
    naming: NamingConvention,
) -> list[tuple[DesignNode, str | None, str | None]]:
    """Returns list of (section_node, container_bg, parent_wrapper_id) tuples."""
    # Existing single-frame unwrap returns [(node, None, None)] for one root frame
    candidates = _existing_unwrap(structure)

    # NEW: second pass — when a candidate is a coloured wrapper with ≥2 mj-section children,
    # replace the wrapper with its children and propagate bg
    if naming != NamingConvention.MJML:
        return candidates  # Heuristic gated to MJML files (extends in Phase 51 for descriptive)

    expanded: list[tuple[DesignNode, str | None, str | None]] = []
    for node, _container_bg, _parent_id in candidates:
        if _is_container_wrapper(node):
            wrapper_bg = _resolve_fill_hex(node.fills) if node.fills else None
            for child in node.children:
                if _is_mj_section(child):
                    expanded.append((child, wrapper_bg, node.node_id))
        else:
            expanded.append((node, _container_bg, _parent_id))
    return expanded


def _is_container_wrapper(node: DesignNode) -> bool:
    """A container wrapper has a non-default fill AND ≥2 mj-section children."""
    has_fill = bool(node.fills) and _resolve_fill_hex(node.fills) is not None
    section_children = [c for c in node.children if _is_mj_section(c)]
    return has_fill and len(section_children) >= 2


def _is_mj_section(node: DesignNode) -> bool:
    """An mj-section child of a wrapper. Matches name convention OR structural shape."""
    name_lower = (node.name or "").lower()
    if "mj-section" in name_lower or "mj-wrapper" in name_lower:
        return True
    # Structural fallback: a FRAME with no fill OR same as wrapper, ≥1 content child
    return node.node_type == DesignNodeType.FRAME and bool(node.children)
```

Per master plan Out-of-band Decision #1: collapse to `container_bg` to match 50.4's `inner_bg`. Reject `wrapper_bg_color` — would force a rename in 50.4. ✓

### Step 2: Add `EmailSection` fields

Per spec above. Place after `child_content_groups`, before any optional VLM-related fields, to keep MJML-related fields grouped.

### Step 3: Extend `_get_section_candidates` with `_is_container_wrapper` predicate

Tuple-shape return changes; existing callers must unpack 3-tuples instead of nodes. Use `dataclasses.replace` if any positional construction lurks (audit grep before writing).

### Step 4: Wire renderer

For 50.3, just emit the outer `<td bgcolor>` wrapping. 50.4 expands this to inner-table layer.

### Step 5: Tests

10 tests per table. Synthetic Figma JSON fixtures via `_make_design_node` factory. Validate against existing 4 regression cases — must remain passing with flag enabled.

### Step 6: Coordinate with 50.4

50.4 introduces `<td class="_outer"><table class="_inner">` layered structure. The 50.3 renderer change is provisional — 50.4 will replace the simple `<td bgcolor>` wrap with the proper outer/inner pair. Mark the 50.3 renderer block with a `# Phase 50.3 — superseded by 50.4` comment so the next dev knows to fold it.

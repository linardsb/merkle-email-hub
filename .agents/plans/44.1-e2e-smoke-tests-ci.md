# Plan: 44.1 E2E Smoke Tests in CI

## Context

Playwright E2E tests exist (10 spec files, ~33 tests) but skip CI entirely. UI regressions only surface during manual testing. A `@smoke`-tagged subset (8-10 critical paths) runs on every PR in <3 minutes, catching the most damaging regressions without bloating CI.

## Implementation Steps

### Step 1: Tag smoke tests

Add `@smoke` to test descriptions in the 8 selected tests. Playwright supports tag filtering via `--grep @smoke` on test titles:

```typescript
// auth.spec.ts — change:
test('login with valid credentials', ...)
// to:
test('login with valid credentials @smoke', ...)
```

Apply to all 8 tests listed in the smoke candidates table above. Match exact test names by reading each file.

### Step 2: Add `e2e:smoke` script to package.json

`cms/apps/web/package.json` — add to `scripts`:
```json
"e2e:smoke": "playwright test --grep @smoke"
```

### Step 3: Add `e2e-smoke` Makefile target

Add after existing `e2e` target in `Makefile`:
```makefile
.PHONY: e2e-smoke
e2e-smoke:  ## Run smoke E2E tests (Chromium, @smoke tagged)
	cd cms && pnpm --filter web e2e:smoke
```

### Step 4: Add `e2e-smoke` CI job

Add to `.github/workflows/ci.yml` after existing jobs. Key design decisions:

- **Depends on `backend` + `frontend`** — code quality gates pass first
- **Services:** PostgreSQL 15 + Redis 7 (same as production stack)
- **Backend startup:** `uv run uvicorn` backgrounded, health-checked before tests
- **Frontend:** Playwright's `webServer` config handles `pnpm dev` automatically
- **Chromium only** — fastest browser, sufficient for smoke
- **Artifact upload on failure** — traces + screenshots for debugging
- **Artifact retention:** 7 days (traces may contain rendered HTML)

```yaml
e2e-smoke:
  name: E2E Smoke Tests
  runs-on: ubuntu-latest
  needs: [backend, frontend]
  services:
    postgres:
      image: postgres:15
      env:
        POSTGRES_USER: test
        POSTGRES_PASSWORD: test
        POSTGRES_DB: test
      ports: ['5432:5432']
      options: --health-cmd pg_isready --health-interval 5s --health-timeout 5s --health-retries 5
    redis:
      image: redis:7
      ports: ['6379:6379']
      options: --health-cmd "redis-cli ping" --health-interval 5s --health-timeout 5s --health-retries 5
  env:
    DATABASE__URL: postgresql+asyncpg://test:test@localhost:5432/test
    REDIS__URL: redis://localhost:6379/0
    AUTH__JWT_SECRET_KEY: test-secret-for-ci
    CI: true
  steps:
    - uses: actions/checkout@v4
    - uses: actions/setup-python@v5
      with:
        python-version: '3.12'
    - name: Install uv
      uses: astral-sh/setup-uv@v4
    - run: uv sync
    - uses: pnpm/action-setup@v4
    - uses: actions/setup-node@v4
      with:
        node-version: '22'
        cache: pnpm
        cache-dependency-path: cms/pnpm-lock.yaml
    - run: cd cms && pnpm install --frozen-lockfile
    - run: cd cms && pnpm exec playwright install --with-deps chromium
    - name: Run database migrations
      run: uv run alembic upgrade head
    - name: Start backend
      run: |
        uv run uvicorn app.main:app --host 0.0.0.0 --port 8891 &
        echo "Waiting for backend..."
        for i in $(seq 1 30); do
          curl -sf http://localhost:8891/api/v1/health && break
          sleep 2
        done
    - name: Run smoke E2E tests
      run: cd cms && pnpm --filter web e2e:smoke
    - uses: actions/upload-artifact@v4
      if: failure()
      with:
        name: playwright-traces
        path: |
          cms/apps/web/test-results/
          cms/apps/web/playwright-report/
        retention-days: 7
```

### Step 5: Verify locally

1. `make e2e-smoke` — runs 8 tagged tests on Chromium
2. `make e2e` — still runs all 33 tests (no regression)
3. `make e2e-all-browsers` — still runs full matrix

## Preflight Warnings

- **Backend must be running for E2E** — `global-setup.ts` health-checks `:8891` with 10 retries at 2s intervals. CI job starts uvicorn in background with 30-retry loop.
- **`reuseExistingServer: !process.env.CI`** — In CI, Playwright starts its own `pnpm dev`. Backend is separate.
- **WebSocket tests skipped** — `collaboration.spec.ts` is not in smoke set (flaky WS in CI).
- **Playwright `--grep @smoke`** filters on test title strings. Tags must be in the `test()` name, not `describe()` block.
- **Artifact paths** — `test-results/` contains traces/videos, `playwright-report/` contains HTML report.

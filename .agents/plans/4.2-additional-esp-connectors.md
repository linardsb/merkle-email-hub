# Plan: Additional ESP Connectors (SFMC, Adobe Campaign, Taxi for Email)

## Context

The export dialog currently only supports Raw HTML and Braze tabs (see screenshot). The roadmap (Task 4.2) requires SFMC, Adobe Campaign, and Taxi for Email connectors. All three follow the same build → export flow as Braze — they use the same `POST /api/v1/connectors/export` endpoint with a different `connector_type`. The backend `connector_type` column is already an unconstrained `String(50)`, so no DB migration is needed.

The key design insight: rather than copy-pasting the Braze tab 3 more times, we refactor the export dialog to be **data-driven** using a connector config array. Each ESP connector has identical UX: name input → build → export → success/error. Only the labels, placeholder text, and icon differ.

**3c.** Replace the separate `brazeState` / `contentBlockName` / `errorMessage` / `externalId` state with a single state record keyed by connector ID. This way each ESP tab has independent state:

```ts
interface EspState {
  dialogState: DialogState;
  name: string;
  errorMessage: string | null;
  externalId: string | null;
}

const initialEspState: EspState = {
  dialogState: "idle",
  name: "",
  errorMessage: null,
  externalId: null,
};

const [espStates, setEspStates] = useState<Record<string, EspState>>({});

// Helper to get state for a connector
const getEspState = (id: string): EspState => espStates[id] ?? initialEspState;

// Helper to update one connector's state
const updateEspState = (id: string, patch: Partial<EspState>) => {
  setEspStates((prev) => ({
    ...prev,
    [id]: { ...(prev[id] ?? initialEspState), ...patch },
  }));
};
```

**3d.** Replace the `handleBrazeExport` callback with a generic `handleEspExport(config: ConnectorConfig)` that:
- Reads `getEspState(config.id).name` for the name
- Calls `triggerBuild` then `triggerExport` with `connector_type: config.id`
- Uses `updateEspState(config.id, { dialogState: "building" })` etc.
- Uses `t(config.successMsg)` / `t(config.errorMsg)` for toasts
- Creates `ExportHistoryRecord` with `platform: config.id` (cast to ConnectorPlatform)

**3e.** Reset ALL esp states when dialog closes:

**3g.** Replace the tab content area. Keep the raw_html tab as-is. For ESP tabs, render a generic `<EspTabContent>` inline block:

```tsx
{activeTab === "raw_html" ? (
  /* ... existing raw HTML content unchanged ... */
) : (
  (() => {
    const cfg = ESP_CONNECTORS.find((c) => c.id === activeTab)!;
    const state = getEspState(cfg.id);
    const isNameValid = state.name.trim().length >= 1 && state.name.trim().length <= 200;
    const isBusy = state.dialogState === "building" || state.dialogState === "exporting";

    return (
      <div className="space-y-4">
        {/* Name Input */}
        <div>
          <label htmlFor={`${cfg.id}-name`} className="mb-1.5 block text-sm font-medium text-foreground">
            {t(cfg.nameLabel)}
          </label>
          <input
            id={`${cfg.id}-name`}
            type="text"
            value={state.name}
            onChange={(e) => updateEspState(cfg.id, { name: e.target.value })}
            placeholder={t(cfg.namePlaceholder)}
            maxLength={200}
            disabled={isBusy}
            className="w-full rounded-md border border-input-border bg-input-bg px-3 py-2 text-sm text-foreground placeholder:text-foreground-muted focus:border-interactive focus:outline-none focus:ring-1 focus:ring-interactive disabled:opacity-50"
          />
          <p className="mt-1 text-xs text-foreground-muted">{t(cfg.nameHint)}</p>
        </div>

        {/* State-based UI — building */}
        {state.dialogState === "building" && (
          <div className="flex items-center gap-2 text-sm text-foreground-muted">
            <Loader2 className="h-4 w-4 animate-spin" />
            {t("building")}
          </div>
        )}

        {/* State-based UI — exporting */}
        {state.dialogState === "exporting" && (
          <div className="flex items-center gap-2 text-sm text-foreground-muted">
            <Loader2 className="h-4 w-4 animate-spin" />
            {t(cfg.exportingMsg)}
          </div>
        )}

        {/* State-based UI — success */}
        {state.dialogState === "success" && (
          <div className="rounded-md border border-status-success/20 bg-status-success/5 p-3">
            <div className="flex items-center gap-2 text-sm font-medium text-status-success">
              <CheckCircle2 className="h-4 w-4" />
              {t(cfg.successMsg)}
            </div>
            {state.externalId && (
              <p className="mt-1 text-xs text-foreground-muted">
                {t("externalId")}: {state.externalId}
              </p>
            )}
          </div>
        )}

        {/* State-based UI — error */}
        {state.dialogState === "error" && (
          <div className="space-y-2">
            <div className="rounded-md border border-status-danger/20 bg-status-danger/5 p-3">
              <div className="flex items-center gap-2 text-sm font-medium text-status-danger">
                <XCircle className="h-4 w-4" />
                {state.errorMessage ?? t(cfg.errorMsg)}
              </div>
            </div>
            <button
              type="button"
              onClick={() => updateEspState(cfg.id, { dialogState: "idle", errorMessage: null })}
              className="text-sm font-medium text-interactive hover:underline"
            >
              {t("retry")}
            </button>
          </div>
        )}

        {/* Export Button */}
        {(state.dialogState === "idle" || state.dialogState === "error") && (
          <button
            type="button"
            onClick={() => handleEspExport(cfg)}
            disabled={!isNameValid || !compiledHtml}
            className="flex w-full items-center justify-center gap-2 rounded-md bg-interactive px-4 py-2.5 text-sm font-medium text-foreground-inverse transition-colors hover:bg-interactive-hover disabled:opacity-50"
          >
            <cfg.icon className="h-4 w-4" />
            {t(cfg.exportButton)}
          </button>
        )}

        {!compiledHtml && state.dialogState === "idle" && (
          <p className="text-xs text-foreground-muted">{t("noCompiledHtml")}</p>
        )}
      </div>
    );
  })()
)}
```

## What This Does NOT Include (Backend — Separate Task)

The backend `app/connectors/service.py` has `SUPPORTED_CONNECTORS = {"braze"}` which will reject requests for new connector types. The frontend export flow will show an error toast when the backend rejects unsupported connectors — this is correct and expected. When backend connector implementations are added later (Task 4.2 backend side), they just need to:
1. Add to `SUPPORTED_CONNECTORS` set
2. Create `app/connectors/sfmc/`, `app/connectors/adobe/`, `app/connectors/taxi/` subdirectories
3. Add dispatch branches in `ConnectorService.export()`

The frontend will work immediately once those are added — no additional frontend changes needed.

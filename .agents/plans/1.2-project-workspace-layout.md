# Plan: 1.2 Project Workspace Layout

## Context

Task 1.2 creates the 3-pane workspace at `/projects/[id]/workspace` — the core editing environment where developers write email HTML (left), see compiled preview (right), and interact with AI agents (bottom). This task builds the **layout shell only** — Monaco editor (1.3), live preview (1.4), and AI chat (2.5) are separate tasks that plug into the panels created here.

**From TODO.md:** "Split-pane workspace at `/projects/[id]/workspace`. Three resizable panels: editor (left), preview (center/right), AI chat (bottom, collapsible). Use a resizable pane library compatible with React 19."

**Security requirements:**
- Route guard: verify user has access to project before rendering
- Project ID validated server-side (no client-side trust)

## Architecture Decisions

1. **Resizable panels**: Use `react-resizable-panels` — the standard React library for resizable panes, works with React 19, zero dependencies, accessible
2. **Route structure**: `/projects/[id]/workspace` as a dedicated route group outside `(dashboard)` since the workspace has its own full-screen layout (no sidebar)
3. **Layout**: Horizontal split (editor | preview) with a collapsible vertical bottom panel (AI chat)
4. **Project data**: Use existing `useProject(id)` hook for project access verification
5. **Workspace layout**: Full viewport height, no sidebar — workspace gets all available space. A minimal top toolbar provides project context and navigation back to dashboard.

### Step 3: Create workspace layout (full-screen, no sidebar)

**File:** `cms/apps/web/src/app/[locale]/projects/[id]/workspace/layout.tsx`

This layout replaces the default sidebar layout with a full-screen workspace. It wraps children without the sidebar navigation — the workspace gets 100% viewport.

```tsx
export default function WorkspaceLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <div className="flex h-screen flex-col overflow-hidden bg-surface">
      {children}
    </div>
  );
}
```

Key points:
- Server component (no "use client" needed)
- `h-screen` + `overflow-hidden` — workspace fills viewport, panels handle their own scrolling
- No sidebar — workspace is full-screen
- The parent `[locale]/layout.tsx` still provides SessionProvider, SWRConfig, etc. but its sidebar will show because it wraps all routes. We need to handle this.

**Important:** The `[locale]/layout.tsx` has a hardcoded sidebar. The workspace route lives outside `(dashboard)` but still under `[locale]`, so the sidebar will still render. We need to modify the `[locale]/layout.tsx` to conditionally hide the sidebar for workspace routes, OR restructure routing.

**Recommended approach:** Use a route group. Move the sidebar layout into `(dashboard)` layout instead of the locale layout:
- Move the sidebar + main content shell from `[locale]/layout.tsx` into `[locale]/(dashboard)/layout.tsx`
- `[locale]/layout.tsx` keeps only providers (SessionProvider, SWRConfig, ThemeProvider, etc.)
- Workspace routes at `[locale]/projects/[id]/workspace/` get full-screen layout

// layout.tsx — providers only
export default async function LocaleLayout({ children, params }) {
  const { locale } = await params;
  const messages = await getMessages();
  const swrConfig = { revalidateOnFocus: false, dedupingInterval: 5000 };

  return (
    <html lang={locale} suppressHydrationWarning>
      <body className="bg-surface text-foreground antialiased">
        <ThemeProvider attribute="class" defaultTheme="system" enableSystem>

            <NextIntlClientProvider messages={messages}>
              <SWRConfig value={swrConfig}>

                <Toaster position="top-right" />

### Step 6: Create workspace page with resizable panels

**File:** `cms/apps/web/src/app/[locale]/projects/[id]/workspace/page.tsx`

```tsx
"use client";

import { use } from "react";
import { useTranslations } from "next-intl";
import { useRouter, useParams } from "next/navigation";
import {
  Panel,
  PanelGroup,
  PanelResizeHandle,
} from "react-resizable-panels";
import { useProject } from "@/hooks/use-projects";
import { WorkspaceToolbar } from "@/components/workspace/workspace-toolbar";
import { EditorPanel } from "@/components/workspace/editor-panel";
import { PreviewPanel } from "@/components/workspace/preview-panel";
import { ChatPanel } from "@/components/workspace/chat-panel";
import { ChevronUp, ChevronDown, GripVertical, GripHorizontal } from "lucide-react";
import { useState } from "react";

export default function WorkspacePage() {
  const t = useTranslations("workspace");
  const params = useParams<{ id: string; locale: string }>();
  const projectId = Number(params.id);
  const { data: project, isLoading, error } = useProject(projectId);
  const [chatCollapsed, setChatCollapsed] = useState(false);

  // Loading state
  if (isLoading) {
    return (
      <div className="flex h-full items-center justify-center">
        <p className="text-sm text-muted-foreground">{t("loading")}</p>
      </div>
    );
  }

  // Error / not found / no access
  if (error || !project) {
    return (
      <div className="flex h-full items-center justify-center">
        <p className="text-sm text-destructive">
          {error?.status === 403 ? t("noAccess") : t("notFound")}
        </p>
      </div>
    );
  }

  return (
    <>
      <WorkspaceToolbar projectName={project.name} />

      {/* Main workspace area */}
      <PanelGroup direction="vertical" className="flex-1">
        {/* Top: Editor + Preview (horizontal split) */}
        <Panel defaultSize={75} minSize={40}>
          <PanelGroup direction="horizontal">
            {/* Editor panel */}
            <Panel defaultSize={50} minSize={25}>
              <EditorPanel />
            </Panel>

            {/* Vertical resize handle */}
            <PanelResizeHandle className="flex w-1.5 items-center justify-center bg-border transition-colors hover:bg-primary/50 data-[resize-handle-active]:bg-primary/50">
              <GripVertical className="h-4 w-4 text-muted-foreground" />
            </PanelResizeHandle>

            {/* Preview panel */}
            <Panel defaultSize={50} minSize={25}>
              <PreviewPanel />
            </Panel>
          </PanelGroup>
        </Panel>

        {/* Horizontal resize handle + collapse toggle */}
        <PanelResizeHandle className="flex h-1.5 items-center justify-center bg-border transition-colors hover:bg-primary/50 data-[resize-handle-active]:bg-primary/50">
          <GripHorizontal className="h-4 w-4 text-muted-foreground" />
        </PanelResizeHandle>

        {/* Bottom: AI Chat (collapsible) */}
        <Panel
          defaultSize={25}
          minSize={0}
          collapsible
          collapsedSize={0}
          onCollapse={() => setChatCollapsed(true)}
          onExpand={() => setChatCollapsed(false)}
        >
          <ChatPanel />
        </Panel>
      </PanelGroup>

      {/* Collapsed chat toggle */}
      {chatCollapsed && (
        <button
          type="button"
          onClick={() => setChatCollapsed(false)}
          className="flex w-full items-center justify-center gap-2 border-t border-border bg-card py-1.5 text-xs text-muted-foreground transition-colors hover:bg-accent hover:text-foreground"
        >
          <ChevronUp className="h-3.5 w-3.5" />
          {t("expandChat")}
        </button>
      )}
    </>
  );
}
```

**Key design decisions:**
- Outer `PanelGroup direction="vertical"` splits top (editor+preview) from bottom (chat)
- Inner `PanelGroup direction="horizontal"` splits editor from preview
- Chat panel is collapsible with `collapsible` + `collapsedSize={0}`
- When collapsed, a slim bar at the bottom allows re-expanding
- Resize handles have visible grip icons and hover/active states
- Project data fetched via `useProject(id)` — 403 from backend means no access
- The `useProject` hook already calls `GET /api/v1/projects/{id}` which does server-side membership verification

**Note on collapse toggle:** The `react-resizable-panels` library handles collapse via `PanelResizeHandle` double-click or programmatic API. The collapsed state button is a UX affordance — the actual collapse is handled by the Panel's `collapsible` prop. To programmatically expand, we need a ref:

Update the page to use `useRef` for the chat panel:
```tsx
import { type ImperativePanelHandle } from "react-resizable-panels";
// ...
const chatPanelRef = useRef<ImperativePanelHandle>(null);
// On the Panel: ref={chatPanelRef}
// On expand button: onClick={() => chatPanelRef.current?.expand()}
```

# Plan: 48.7 Scaffolder Component-Tree Generation Mode

## Context

Free-form HTML generation has ~85-95% structural reliability. Component-tree mode constrains the LLM to selecting from 150+ tested components and filling typed slots — bringing structural reliability to ~99%. The tree is compiled deterministically in 48.8. Borrowed from **json-render** guardrailed generation pattern.

### Current Pipeline Flow (structured mode)

```
brief → ScaffolderPipeline.execute()
  Pass 1 (layout): brief → LLM → TemplateSelection + SectionDecisions
  Pass 2 (content): brief + template slots → LLM → SlotFills     } parallel
  Pass 3 (design):  brief + brand_config → LLM → DesignTokens    }
→ EmailBuildPlan → TemplateAssembler.assemble() → HTML → XSS sanitize → QA
```

### Tree Mode Pipeline Flow (new)

```
brief → ScaffolderPipeline.execute_tree()
  Pass 1 (layout): brief + component manifest → LLM → ComponentSelection[]
  Pass 2 (content): brief + per-component slot_defs → LLM → slot fills per section
  Pass 3 (design):  unchanged → DesignTokens
→ EmailTree (assembled from passes) → validate_tree_against_manifest() → return
```

Key difference: layout pass selects **components** (from 150 manifest slugs) instead of **templates** (from GoldenTemplate registry). Content pass fills **component-specific slots** instead of template-wide slots.

### Component Manifest Data Shape

Each entry in `COMPONENT_SEEDS` (from `file_loader.py`):
```python
{"slug": "hero-full-width", "name": "Hero Full Width", "category": "hero",
 "description": "...", "compatibility": {...},
 "slot_definitions": [
   {"slot_id": "heading", "slot_type": "content", "selector": "[data-slot='heading']", "required": False},
   {"slot_id": "cta_url", "slot_type": "cta", ...},
 ],
 "default_tokens": {"bg_color": "#FF6B35", ...},
 "html_source": "...",  # full HTML — NOT needed in prompt
}
```

### EmailTree Model (existing, `tree_schema.py`)

- `EmailTree(metadata: TreeMetadata, sections: list[TreeSection])`
- `TreeSection(component_slug, slot_fills: dict[str, SlotValue], style_overrides, children, custom_html)`
- `SlotValue` = `TextSlot | ImageSlot | ButtonSlot | HtmlSlot` (discriminated union on `type`)
- `validate_tree_against_manifest(tree, manifest_slugs, slot_definitions)` returns `list[str]` errors
- `__custom__` slug allowed for fallback (custom_html field)

### Step 4: TreeBuilder — deterministic EmailTree assembly

New file `app/ai/agents/scaffolder/tree_builder.py`:

```python
class TreeBuilder:
    """Assemble EmailTree from tree-mode pipeline outputs."""

    def __init__(self, manifest_slugs: set[str], slot_definitions: dict[str, list[str]]) -> None:
        self._manifest_slugs = manifest_slugs
        self._slot_definitions = slot_definitions

    def build(
        self,
        component_selections: list[ComponentSelection],
        slot_fills_by_section: dict[int, dict[str, SlotValue]],
        design_tokens: DesignTokens,
        subject: str,
        preheader: str,
    ) -> EmailTree:
        """Build EmailTree from 3-pass outputs. Validates against manifest."""
```

**`ComponentSelection`** dataclass (in same file):
```python
@dataclasses.dataclass(frozen=True)
class ComponentSelection:
    component_slug: str
    rationale: str = ""
```

**`_infer_slot_type(slot_id: str, content: str, slot_type: str) -> SlotValue`** — maps raw content to `TextSlot|ImageSlot|ButtonSlot|HtmlSlot` based on `slot_type` from manifest and content heuristics:
- `slot_type == "cta"` or has href → `ButtonSlot`
- `slot_type == "image"` or starts with `http` + image extension → `ImageSlot`
- Has HTML tags → `HtmlSlot`
- Default → `TextSlot`

**`_build_manifest_index()` → `tuple[set[str], dict[str, list[str]]]`** — module-level cached helper that reads `COMPONENT_SEEDS` and returns `(slugs, slot_defs)`.

### Step 5: Pipeline — add `execute_tree()` method

`app/ai/agents/scaffolder/pipeline.py` — add method to `ScaffolderPipeline`:

```python
async def execute_tree(
    self,
    brief: str,
    brand_config: dict[str, object] | None = None,
) -> EmailTree:
    """Execute 3-pass pipeline in tree mode. Returns EmailTree."""
```

**Pass 1 (tree layout):**
- Build manifest prompt via `build_component_manifest_prompt()`
- System prompt: `build_tree_layout_system_prompt(manifest_prompt, skills=[])`
- LLM returns `{"sections": [{"component_slug": "hero-full-width", ...}], "subject": "...", "preheader": "..."}`
- Parse → `list[ComponentSelection]`

**Pass 2 (tree content) — parallel with Pass 3:**
- For each section, build per-component content prompt with that component's slot_definitions
- Batch into single LLM call (all sections in one prompt) to minimize latency
- LLM returns `{"sections": [{"index": 0, "fills": {"heading": {"type": "text", "text": "..."}, ...}}]}`
- Parse → `dict[int, dict[str, SlotValue]]`

**Pass 3 (design) — reuse `_design_pass()` unchanged**

**Assembly:**
- Call `TreeBuilder.build()` with all 3 outputs
- Run `validate_tree_against_manifest()` — if errors, log warning + attempt `__custom__` fallback for unknown slugs

**New method `_process_tree()`:**
```python
async def _process_tree(
    self,
    req: ScaffolderRequest,
    provider: LLMProvider,
    model: str,
    model_id: str,
) -> ScaffolderResponse:
    pipeline = ScaffolderPipeline(provider, model)
    tree = await pipeline.execute_tree(req.brief, brand_config=req.brand_config)

    # QA on compiled HTML (if tree compiler available — 48.8)
    qa_results = None
    qa_passed = None
    if self._should_run_qa(req):
        # Defer to 48.8 — tree compiler not yet available
        pass

    return ScaffolderResponse(
        html="",  # No HTML in tree mode (compiled in 48.8)
        tree=tree.model_dump(),
        model=model_id,
        confidence=0.9,  # Higher confidence in constrained mode
        skills_loaded=[],
        mso_warnings=[],
    )
```


## Preflight Warnings

- `ScaffolderResponse.html` is currently `str` (required). Making it `str = ""` may affect existing consumers that assume non-empty HTML. Check all `ScaffolderResponse` references.
- `_process_structured()` in service.py has no early return guard — tree routing must be added at the top of the method.
- Manifest prompt for 150 components at ~50 chars each ≈ 7500 chars ≈ 1875 tokens — well within 8000 budget. But full slot_definitions push it higher. Monitor actual token usage.
- `_load_manifest()` is `@lru_cache` — safe to call in prompt builder without performance concern.

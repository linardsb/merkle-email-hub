# Plan: 47.8 Custom Component Generation (AI Fallback)

## Context

Even with 150+ components, some Figma designs have unique layouts (5-column icon grids, brand-specific heroes) that don't match any template well. When `ComponentMatch.confidence < 0.6`, generate a one-off email-safe HTML section via the Scaffolder agent instead of using a poorly-matched template.

### Step 3: Integration in converter_service.py

In `_convert_with_components()` render loop (~line 687), before `renderer.render_section(match)`:

```python
# Custom component generation for low-confidence matches
custom_gen_count = 0
settings = get_settings()
generator: CustomComponentGenerator | None = None
if settings.design_sync.custom_component_enabled:
    from app.design_sync.custom_component_generator import CustomComponentGenerator
    generator = CustomComponentGenerator()

# In the for-match loop:
rendered: RenderedSection | None = None
if (
    generator is not None
    and match.confidence < settings.design_sync.custom_component_confidence_threshold
    and custom_gen_count < settings.design_sync.custom_component_max_per_email
):
    try:
        rendered = await generator.generate(
            section=match.section, tokens=tokens
        )
        custom_gen_count += 1
        warnings.append(f"Section {match.section_idx}: custom-generated (confidence={match.confidence:.2f})")
    except Exception:
        logger.warning("design_sync.custom_component_failed",
                       section_idx=match.section_idx, exc_info=True)
        rendered = None

if rendered is None:
    rendered = renderer.render_section(match)
```

**CRITICAL — sync/async resolution (preflight finding):**
`_convert_with_components()` is sync (line 641), `convert_document()` is sync (line 241), and callers in `import_service.py:223` and `converter_service.py:442` call it synchronously. `design_screenshots` is accepted by `convert_document()` but explicitly discarded (`_ = design_screenshots`, line 267) — it's only used in the async verification path.

**Approach:** Keep `CustomComponentGenerator.generate()` as a **sync** method. Use `ScaffolderService` synchronously via `asyncio.get_event_loop().run_until_complete()` if needed, or restructure the generator to build the brief + context synchronously and defer the actual scaffolder call. Alternatively, collect low-confidence sections, return from `_convert_with_components()`, and have the caller (which IS async in `import_service.py`) run async generation as a post-step. **Recommended: make `_convert_with_components()` async** since its only callers (`convert_document` at line 295 and `convert` at line 442) can also be made async — `import_service.py:223` already awaits the broader flow.

## Preflight Warnings

- **Sync/async chain (CRITICAL):** `convert_document()` → `_convert_with_components()` are both sync. Callers: `import_service.py:223` (async context), `converter_service.py:442` (sync shim). Making `_convert_with_components()` async requires propagating up. See Step 3 notes above.
- **RenderedSection required fields:** `component_slug: str` and `section_idx: int` are required (no defaults). Generator must set these — use `"custom-generated"` slug and the match's `section_idx`.
- **`propagated_bgcolor: str | None = None`** — optional field on `RenderedSection`, can be omitted.
- **Section cache:** custom-generated sections should also be cached if `connection_id` + `section_hashes` available.
- **`design_screenshots` not threaded:** `convert_document()` accepts `design_screenshots` but discards it (line 267). To pass screenshots to the generator, thread it through to `_convert_with_components()`.
- **ScaffolderResponse confirmed:** `.html: str` field exists (schemas.py:50). Also has `.model: str`, `.confidence: float | None`.

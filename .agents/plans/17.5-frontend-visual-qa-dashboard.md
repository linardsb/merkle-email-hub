# Plan: 17.5 Frontend Visual QA Dashboard

## Context
Phase 17 added backend screenshot capture (17.1), visual diff with baselines (17.2), and VLM defect analysis (17.3). The Visual QA Dashboard brings this data into the workspace UI — side-by-side client screenshots, diff overlays, baseline management, and accept/reject flows for VLM suggestions.

**Scope decisions:**
- **Entity types:** Both `golden_template` (workspace) AND `component_version` (component library) are supported
- **Defect annotations (17.3):** Component is created but NOT wired to backend yet — will be connected when VLM agent endpoints are finalized
- **Layout:** Compact sidebar summary + full-width dialog (confirmed)

**Architecture decision:** The workspace QA panel is 320px (`w-80`) — too narrow for side-by-side screenshots. Pattern: add a compact "Visual QA" summary tab in the QA panel sidebar, with a "View Details" button that opens a full-width `VisualQADialog` (56rem, matching existing `VisualRegressionDialog` pattern). Baseline management lives in a dedicated `BaselineManager` component within the dialog.

## API Endpoints Consumed

| Method | Path | Rate Limit | Purpose |
|--------|------|------------|---------|
| POST | `/api/v1/rendering/screenshots` | 5/min | Capture screenshots for 5 clients |
| POST | `/api/v1/rendering/visual-diff` | 10/min | Compare baseline vs current image |
| GET | `/api/v1/rendering/baselines/{entity_type}/{entity_id}` | 30/min | List stored baselines |
| PUT | `/api/v1/rendering/baselines/{entity_type}/{entity_id}` | 10/min | Upsert baseline image |

**Client profiles:** `gmail_web`, `outlook_2019`, `apple_mail`, `outlook_dark`, `mobile_ios`

**Entity types:** `component_version`, `golden_template`

### Step 2: Create SWR Hooks

**File:** `cms/apps/web/src/hooks/use-visual-qa.ts`

```typescript
"use client";

import useSWR from "swr";
import useSWRMutation from "swr/mutation";
import { fetcher } from "@/lib/swr-fetcher";
import { longMutationFetcher } from "@/lib/mutation-fetcher";
import type {
  ScreenshotRequest,
  ScreenshotResponse,
  VisualDiffRequest,
  VisualDiffResponse,
  BaselineListResponse,
  BaselineUpdateRequest,
  BaselineResponse,
} from "@/types/rendering";

// Capture screenshots for current HTML across all client profiles
export function useCaptureScreenshots() {
  return useSWRMutation<ScreenshotResponse, Error, string, ScreenshotRequest>(
    "/api/v1/rendering/screenshots",
    longMutationFetcher
  );
}

// Run visual diff between baseline and current image
export function useVisualDiff() {
  return useSWRMutation<VisualDiffResponse, Error, string, VisualDiffRequest>(
    "/api/v1/rendering/visual-diff",
    longMutationFetcher
  );
}

// Fetch baselines for an entity
export function useBaselines(entityType: string | null, entityId: number | null) {
  const key =
    entityType && entityId
      ? `/api/v1/rendering/baselines/${entityType}/${entityId}`
      : null;
  return useSWR<BaselineListResponse>(key, fetcher);
}

// Update a baseline image
export function useUpdateBaseline(entityType: string, entityId: number) {
  return useSWRMutation<BaselineResponse, Error, string, BaselineUpdateRequest>(
    `/api/v1/rendering/baselines/${entityType}/${entityId}`,
    async (url: string, { arg }: { arg: BaselineUpdateRequest }) => {
      const { authFetch } = await import("@/lib/auth-fetch");
      const res = await authFetch(url, {
        method: "PUT",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify(arg),
      });
      if (!res.ok) throw new Error("Failed to update baseline");
      return res.json();
    }
  );
}
```

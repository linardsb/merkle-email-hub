# Plan: 49.7 CTA Fidelity — Button Color/Shape Extraction

## Context

The converter extracts button text and URL but discards visual properties (bg color, text color, border radius, border, icon) present on Figma nodes. All CTAs render with template defaults (`#e84e0f`/`#1a1a1a`, `border-radius:25px/10px`) regardless of the design. The REFRAME 2025 case exposes this: 3 distinct CTAs (`#c6fc6a`, `#ffbaf3`, `#06d5ff`, `border-radius:6px`) all collapse to one orange button.

This must work for **any** Figma design, not just REFRAME 2025.

## Implementation Steps

### Step 1: Extend ButtonElement dataclass

`layout_analyzer.py:90-102` — add 3 fields after `text_color`:

```python
stroke_color: str | None = None      # Border stroke hex
stroke_weight: float | None = None   # Border stroke width (px)
icon_url: str | None = None          # Exported icon asset URL
```

### Step 2: Enhance button extraction

`layout_analyzer.py:1053-1067` — in `_walk_for_buttons()`, after existing property extraction:

- Read `node.stroke_color` and `node.stroke_weight` (already on `DesignNode` protocol)
- Scan children for icon nodes: iterate `node.children`, find child with name containing `"icon"` (case-insensitive) and type `RECTANGLE`/`VECTOR`/`FRAME` with dimensions ≤64px. Store as `icon_url` (the node_id, to be resolved to asset URL later by the caller)
- Pass new fields to `ButtonElement` constructor

### Step 3: Emit CTA token overrides

`component_matcher.py:1250-1304` — in `_build_token_overrides()`, after existing override extraction:

```python
# CTA button overrides from first button
if section.buttons:
    btn = section.buttons[0]
    if btn.fill_color and _HEX_COLOR_RE.match(btn.fill_color):
        overrides.append(TokenOverride("background-color", "_cta", btn.fill_color))
    if btn.text_color and _HEX_COLOR_RE.match(btn.text_color):
        overrides.append(TokenOverride("color", "_cta", btn.text_color))
    if btn.border_radius is not None:
        overrides.append(TokenOverride("border-radius", "_cta", f"{btn.border_radius:.0f}px"))
    if btn.stroke_color and _HEX_COLOR_RE.match(btn.stroke_color):
        overrides.append(TokenOverride("border-color", "_cta", btn.stroke_color))
    if btn.stroke_weight is not None:
        overrides.append(TokenOverride("border-width", "_cta", f"{btn.stroke_weight:.0f}px"))
```

### Step 5: Add `_cta` target handler in renderer

`component_renderer.py:524-618` — in `_apply_token_overrides()`, add handler block:

```python
elif target == "_cta":
    if prop == "background-color":
        # Replace bgcolor attr on table/td
        result = re.sub(r'bgcolor="[^"]*"', f'bgcolor="{val}"', result)
        # Replace background-color in inline styles
        result = self._replace_css_prop_all(result, "background-color", val)
        # Update VML fillcolor
        result = re.sub(r'fillcolor="[^"]*"', f'fillcolor="{val}"', result)
    elif prop == "color":
        # Replace color on <a> tags inside CTA
        result = self._replace_cta_text_color(result, val)
    elif prop == "border-radius":
        result = self._replace_css_prop_all(result, "border-radius", val)
        # Update VML arcsize: radius_px / height * 100
        result = self._update_vml_arcsize(result, val)
    elif prop == "border-color":
        result = re.sub(r'strokecolor="[^"]*"', f'strokecolor="{val}"', result)
        result = self._replace_css_prop_all(result, "border-color", val)
    elif prop == "border-width":
        result = self._replace_css_prop_all(result, "border-width", val)
```

New helper methods on `ComponentRenderer`:

- `_replace_css_prop_all(html, prop, val)` — replace ALL occurrences of a CSS prop (not just first like `_replace_first_css_prop`)
- `_replace_cta_text_color(html, val)` — replace `color:` on `<a>` elements inside CTA context (use existing `_SLOT_STYLE_RE` pattern approach)
- `_update_vml_arcsize(html, radius_val)` — convert `Npx` radius to VML `arcsize="N%"` (approximate: `arcsize = round(radius / height * 100)`, default height 48)

## Preflight Warnings

- `_walk_for_buttons()` at line 1030 checks `node.type in (DesignNodeType.FRAME, DesignNodeType.COMPONENT, DesignNodeType.INSTANCE)` — icon children may be `RECTANGLE`/`VECTOR` type, so icon detection must check children of these parent types
- `_build_column_fill_html()` hardcodes `font-weight:bold` — some designs may want `font-weight:400` (REFRAME uses 400). Consider extracting from button text node later, but out of scope for 49.7
- The `_replace_css_prop_all()` helper must not accidentally modify CSS props in MSO conditional comments that should only be touched by VML-specific logic — scope replacements carefully
- Snapshot baselines for cases 5/6/10 may need regeneration if any CTA components are present in those cases

## Security Checklist

No new endpoints. All button property values pass through:
- `_HEX_COLOR_RE` validation for colors (prevents CSS injection)
- `_safe_color()` with fallback for untrusted values
- Numeric values (`border_radius`, `stroke_weight`) are floats — format with `:.0f`, no injection risk
- `icon_url` is a node_id reference, not a user-supplied URL — resolved to asset path by existing asset pipeline

# Plan: 44.2 Dependency Update Automation (Renovate)

## Context

100+ production dependencies with no automated update tooling. `uv.lock` and `pnpm-lock.yaml` pin exact versions — good for reproducibility, bad for security when CVEs hit pinned versions. The existing `make check` gate (lint + types + tests + security) catches breakage, so automated PRs are safe to merge when CI passes.

### CI Pipeline (validates Renovate PRs)

CI runs on all PRs against `main` (`.github/workflows/ci.yml`):
1. Backend: ruff format + ruff check + mypy + pyright + bandit + pytest
2. Frontend: ESLint + Prettier + TypeScript + tests (pnpm `--frozen-lockfile`)
3. Migration lint (squawk, PR-only)
4. Commit message lint (conventional commits, PR-only)

Plus Semgrep SAST on all PRs (`.github/workflows/semgrep.yml`).

### Existing Automation

- **None** — no `renovate.json*`, `.renovaterc*`, or `.github/dependabot.yml`
- Pre-commit hooks configured but separate from CI
- 7 pnpm security overrides already in `cms/package.json`

### Special Version Considerations

- `anthropic>=0.43.0` — may have breaking API changes in major bumps
- `openai>=1.0.0` — same concern
- `pnpm@10.28.2` pinned in `packageManager` field
- Node 20 (maizzle) vs Node 22 (CMS) — different major versions
- All Python deps use `>=` only (no upper bounds) — Renovate-friendly

## Implementation Steps

### Step 1: Create `renovate.json5`

Create at repo root with the following configuration:

```json5
{
  "$schema": "https://docs.renovatebot.com/renovate-schema.json",
  "extends": [
    "config:recommended",
    "helpers:pinGitHubActionDigests",
    ":separateMajorReleases"
  ],
  "timezone": "Europe/Riga",
  "schedule": ["before 8am on Monday"],
  "labels": ["dependencies"],
  "prConcurrentLimit": 5,
  "prHourlyLimit": 2,
  "commitMessagePrefix": "chore(deps):",

  // Manager enablement
  "enabledManagers": [
    "pep621",
    "pip_requirements",
    "pnpm",
    "npm",
    "dockerfile",
    "github-actions"
  ],

  // Lock file maintenance
  "lockFileMaintenance": {
    "enabled": true,
    "schedule": ["before 6am on Monday"],
    "commitMessageAction": "refresh"
  },

  "packageRules": [
    // ── Security patches: auto-merge if CI passes ──
    {
      "description": "Auto-merge security patches after CI green",
      "matchUpdateTypes": ["patch"],
      "matchCategories": ["security"],
      "automerge": true,
      "automergeType": "pr",
      "platformAutomerge": true
    },

    // ── Python minor+patch: weekly grouped PR ──
    {
      "description": "Group Python minor+patch updates",
      "matchManagers": ["pep621", "pip_requirements"],
      "matchUpdateTypes": ["minor", "patch"],
      "groupName": "python-minor",
      "schedule": ["before 8am on Monday"]
    },

    // ── Node.js minor+patch: weekly grouped PR ──
    {
      "description": "Group Node.js minor+patch updates",
      "matchManagers": ["pnpm", "npm"],
      "matchUpdateTypes": ["minor", "patch"],
      "groupName": "node-minor",
      "schedule": ["before 8am on Monday"]
    },

    // ── AI SDKs: separate PRs, manual review ──
    {
      "description": "AI SDK updates need manual review (potential API changes)",
      "matchPackageNames": ["anthropic", "openai"],
      "groupName": "ai-sdks",
      "automerge": false,
      "labels": ["dependencies", "ai"]
    },

    // ── Docker base images: grouped, manual review ──
    {
      "description": "Docker image updates in separate PR",
      "matchManagers": ["dockerfile"],
      "groupName": "docker-images",
      "automerge": false,
      "labels": ["dependencies", "infrastructure"]
    },

    // ── GitHub Actions: grouped, manual review ──
    {
      "description": "Pin and group GitHub Actions updates",
      "matchManagers": ["github-actions"],
      "groupName": "github-actions",
      "automerge": false,
      "labels": ["dependencies", "ci"]
    },

    // ── Major versions: individual PRs, never auto-merge ──
    {
      "description": "Major versions need manual review",
      "matchUpdateTypes": ["major"],
      "automerge": false,
      "labels": ["dependencies", "breaking"]
    },

    // ── Dev dependencies: more permissive auto-merge ──
    {
      "description": "Auto-merge dev dependency patches",
      "matchDepTypes": ["devDependencies", "dev"],
      "matchUpdateTypes": ["patch"],
      "automerge": true,
      "automergeType": "pr",
      "platformAutomerge": true
    }
  ],

  // Pin GitHub Actions to full SHA for supply-chain security
  "pinDigests": true,

  // Post-update commands for lock file regeneration
  "postUpdateOptions": [
    "pnpmDedupe"
  ]
}
```

### Step 2: Validate config

Run `npx renovate-config-validator renovate.json5` to ensure the config is valid.

### Step 3: Enable Renovate on the repository

Two options (choose one):
- **Option A (recommended):** Install Renovate GitHub App from https://github.com/apps/renovate — zero maintenance, hosted by Mend
- **Option B (self-hosted):** Add `.github/workflows/renovate.yml` workflow using `renovatebot/github-action`

### Step 4: Verify CI compatibility

The existing CI pipeline already runs on all PRs, so Renovate PRs will automatically get:
- Full backend checks (lint, types, tests, security)
- Full frontend checks (ESLint, Prettier, TypeScript, tests)
- Semgrep SAST
- Conventional commit validation (Renovate's `commitMessagePrefix: "chore(deps):"` matches the `chore` type)

No CI changes needed — the existing pipeline validates Renovate PRs out of the box.

## Renovate Behavior Summary

| Update Type | Grouping | Auto-merge | Schedule | Labels |
|-------------|----------|------------|----------|--------|
| Security patch | Ungrouped | Yes (CI must pass) | Immediate | `dependencies` |
| Python minor/patch | `python-minor` | No | Monday 8am | `dependencies` |
| Node minor/patch | `node-minor` | No | Monday 8am | `dependencies` |
| AI SDKs (any) | `ai-sdks` | No | Monday 8am | `dependencies`, `ai` |
| Docker images | `docker-images` | No | Monday 8am | `dependencies`, `infrastructure` |
| GitHub Actions | `github-actions` | No | Monday 8am | `dependencies`, `ci` |
| Major (any) | Individual PRs | No | Monday 8am | `dependencies`, `breaking` |
| Dev dep patches | Per-group | Yes (CI must pass) | Monday 8am | `dependencies` |
| Lock file maintenance | N/A | No | Monday 6am | `dependencies` |

## Key Design Decisions

1. **`pep621` not `pip_requirements`** for main Python deps — Renovate reads `pyproject.toml` directly and updates `uv.lock` via its built-in uv support
2. **Both `pnpm` and `npm` managers** — CMS uses pnpm workspace, maizzle-builder uses npm
3. **AI SDKs isolated** — anthropic/openai get their own group because major versions can break prompt/response formats
4. **`pinDigests: true`** — GitHub Actions pinned to SHA to prevent supply-chain attacks via tag mutation
5. **`pnpmDedupe`** in postUpdateOptions — keeps lock file clean
6. **Lock file maintenance** — weekly refresh catches transitive dependency updates
7. **`prConcurrentLimit: 5`** — prevents PR flood on first run
8. **Conventional commits** — `chore(deps):` prefix passes the commit message lint hook

## Security Checklist

- [x] No secrets exposed to Renovate (it only reads dependency manifests)
- [x] GitHub Actions pinned to SHA digests (`pinDigests: true`)
- [x] Auto-merge only for security patches + dev patches, both require CI green
- [x] Major versions never auto-merge
- [x] AI SDK updates require manual review
- [x] All Renovate PRs go through full CI pipeline (`make check` equivalent)
- [x] Conventional commit format enforced on Renovate commits

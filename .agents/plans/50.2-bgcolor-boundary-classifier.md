### Modify: `app/design_sync/bgcolor_propagator.py` (rewrite as classifier)

Add a new function (keep existing `propagate_adjacent_bgcolor` for backward-compat â€” Phase 41.2 callers still use it; deprecate in Phase 53):

```python
from typing import Literal

BoundaryRelation = Literal["continuous_with_above", "continuous_with_below", "hard_break", "unknown"]

@dataclass(frozen=True)
class SectionBoundary:
    """Per-section boundary classification."""
    section_node_id: str
    boundary_above: BoundaryRelation
    boundary_below: BoundaryRelation
    sampled_top_color: str | None  # hex
    sampled_bottom_color: str | None

def classify_section_boundaries(
    sections: list[EmailSection],
    *,
    global_design_image: bytes | None,
    delta_threshold: int = 5,  # max per-channel RGB delta to consider "continuous"
) -> dict[str, SectionBoundary]:
    """Sample top + bottom edges of each section in the global PNG, classify boundaries.

    Returns dict keyed by section.node_id. Sections with no PNG or
    out-of-bounds y_position get `unknown` for both boundaries.
    """
```

Implementation: walk sections in y-position order; for each pair `(s_n, s_n+1)`:
- Sample 5px-tall band at `s_n.bottom = s_n.y + s_n.height - 5`
- Sample 5px-tall band at `s_n+1.top = s_n+1.y + 5`
- If `max_per_channel_delta(bottom, top) â‰¤ delta_threshold` â†’ both `continuous`
- Else â†’ both `hard_break`

## Implementation Steps

### Step 1: Define `SectionBoundary` + `BoundaryRelation`

In `bgcolor_propagator.py`, add the dataclass + Literal type at module top.

### Step 2: Implement `classify_section_boundaries`

Reuse `image_sampler.sample_edge_color` for the pixel reads. Compute per-channel delta as `max(abs(r1-r2), abs(g1-g2), abs(b1-b2))`. Default threshold=5 matches Pass 2 of doc Part 1 (12-pixel sample, "Î” â‰¤ 5 in each channel").

### Step 3: Add 4 fields to `EmailSection`

Frozen dataclass â€” order matters; place after `child_content_groups` to maintain field-position stability for any positional-arg callers (none should exist; named-arg construction is the convention).

### Step 4: Wire into `analyze_layout`

After section construction, before return. Use `dataclasses.replace` to rebuild frozen `EmailSection` instances with boundary fields populated.

### Step 5: Tests

Use synthetic 600Ă—100 PNG fixtures with `Pillow.Image.new(...)` â€” drop a 5px band of one color at y=0-5 and another at y=95-100. Build sections with matching y_position/height. Assert boundaries.

# Plan: 35.6 AI Visual Fidelity Scoring Pipeline

## Context

After converting a Figma design to HTML, automatically capture a rendered screenshot and compare it against the Figma frame image. Produces per-section fidelity scores (0–100%) using SSIM and flags regions where converter output drifts from design intent. Closes the feedback loop: design → convert → render → score → identify drift → fix converter.

### Step 6: Visual scorer — SSIM engine

`app/design_sync/visual_scorer.py` — core comparison module:

```python
@dataclass(frozen=True)
class SectionScore:
    section_id: str
    section_name: str
    section_type: str
    ssim: float               # 0.0–1.0
    y_start: int              # pixel row in image
    y_end: int

@dataclass(frozen=True)
class FidelityScore:
    overall: float            # 0.0–1.0 (mean SSIM of all sections)
    sections: list[SectionScore]
    diff_image: bytes | None  # Red-highlighted diff overlay PNG

def score_fidelity(
    figma_image: bytes,
    html_image: bytes,
    sections: list[EmailSection],
    *,
    blur_sigma: float = 1.0,
    win_size: int = 7,
) -> FidelityScore:
```

**Algorithm:**
1. Load both PNGs via Pillow → numpy arrays (grayscale for SSIM)
2. Resize shorter image height to match taller (pad with white)
3. Apply Gaussian blur (σ=`blur_sigma`) to both — smooths anti-aliasing diffs
4. Compute full-image SSIM via `skimage.metrics.structural_similarity()`
5. For each `EmailSection`: compute y-pixel bounds from `y_position`/`height` relative to design height → slice both arrays → per-section SSIM
6. Generate diff image: absolute pixel difference > 20% luminance → red overlay, encode as PNG
7. Return `FidelityScore(overall=mean(section_ssims), sections=[...], diff_image=diff_bytes)`

**Section y-mapping:** `EmailSection.y_position` is in design coordinates. Scale factor = `image_height / design_total_height`. Compute: `y_start = int(section.y_position * scale)`, `y_end = int((section.y_position + section.height) * scale)`.

```python
async def download_image_bytes(
    self,
    exported: ExportedImage,
) -> bytes:
    """Download image bytes from Figma CDN URL."""
    async with httpx.AsyncClient(timeout=httpx.Timeout(30.0)) as client:
        resp = await client.get(exported.url)
        resp.raise_for_status()
        return resp.content
```

```python
    ) -> FidelityScore:
```

**Flow:**
1. Export Figma frame images for top-level section node IDs via `FigmaDesignSyncService.export_images()`
2. Download image bytes via `download_image_bytes()`
3. Stitch section images into single full-design image (vertically concatenate by y-position)
4. Render HTML screenshot via `capture_screenshot()` with a `gmail_web` profile (600px width, 2x scale)
5. Call `score_fidelity(figma_bytes, html_bytes, layout.sections)`
6. Store result in `design_import.fidelity_json`
7. Return `FidelityScore`

## Preflight Warnings

- `scikit-image` `structural_similarity` requires `win_size` ≤ smallest image dimension. Use `win_size=7` (not default 11) for small section crops.
- `structural_similarity` expects float64 arrays. Convert via `img.astype(np.float64) / 255.0` or set `data_range=255`.
- Figma CDN URLs expire after ~14 days. Scoring must happen promptly after export, or re-export.
- `EmailSection.y_position` may be `None` for sections with unknown layout. Skip those sections in per-section scoring.

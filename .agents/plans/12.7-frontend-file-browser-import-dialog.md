# Plan: 12.7 Frontend File Browser & Import Dialog

## Context
Backend tasks 12.1–12.6 are complete. The `/design-sync` page currently shows connections, tokens, and a connect dialog. Task 12.7 adds a **file browser tree**, a **tabbed import dialog** (Import Design + Extract Components), and **thumbnail previews** — turning the design-sync page into the full design-to-email import hub.

The `/figma` page is a legacy duplicate — this plan consolidates into `/design-sync` only and redirects `/figma` there.

## Decisions
- **One page**: `/design-sync` is the single entry point for all design tools (Figma, Sketch, Canva)
- **Dialog with tabs**: "Import Design" tab (4-step wizard) and "Extract Components" tab (3-step wizard) inside one Dialog
- **Thumbnails**: Export node thumbnails via `POST /connections/export-images` when file tree loads
- **Polling**: 2s `refreshInterval` on `GET /imports/{id}` during conversion step
- **Consolidation**: `/figma` page redirects to `/design-sync`; figma-specific hooks/components become unused (removal deferred to cleanup)

## Backend API Surface (all under `/api/v1/design-sync`)
| Method | Path | Purpose |
|--------|------|---------|
| GET | `/connections/{id}/file-structure?depth=N` | File tree (pages → frames → nodes) |
| GET | `/connections/{id}/components` | Component list |
| POST | `/connections/export-images` | Export node thumbnails (returns CDN URLs) |
| POST | `/connections/generate-brief` | Generate brief from selected nodes |
| POST | `/imports` | Create import with brief (StartImportRequest) |
| GET | `/imports/{id}` | Poll import status |
| PATCH | `/imports/{id}/brief` | Edit brief before conversion |
| POST | `/imports/{id}/convert` | Trigger Scaffolder conversion (202) |
| POST | `/connections/{id}/extract-components` | Extract components (202) |

### 2. Hooks — `cms/apps/web/src/hooks/use-design-sync.ts` (MODIFY)
Add 7 new hooks below existing ones. Follow existing SWR patterns:

```ts
// -- File structure (GET, conditional on connectionId) --
export function useDesignFileStructure(connectionId: number | null, depth?: number) {
  const depthParam = depth ? `?depth=${depth}` : "";
  return useSWR<DesignFileStructure>(
    connectionId ? `/api/v1/design-sync/connections/${connectionId}/file-structure${depthParam}` : null,
    fetcher,
  );
}

// -- Components list (GET, conditional) --
export function useDesignComponents(connectionId: number | null) {
  return useSWR<DesignComponentList>(
    connectionId ? `/api/v1/design-sync/connections/${connectionId}/components` : null,
    fetcher,
  );
}

// -- Export images (POST mutation) --
export function useExportImages() {
  return useSWRMutation<ImageExportResult, Error, string, ExportImagesArg>(
    "/api/v1/design-sync/connections/export-images",
    mutationFetcher,
  );
}
// type ExportImagesArg = { connection_id: number; node_ids: string[]; format?: string; scale?: number }

// -- Generate brief (POST mutation) --
export function useGenerateBrief() {
  return useSWRMutation<GeneratedBrief, Error, string, GenerateBriefArg>(
    "/api/v1/design-sync/connections/generate-brief",
    mutationFetcher,
  );
}
// type GenerateBriefArg = { connection_id: number; selected_node_ids: string[]; include_tokens?: boolean }

// -- Create import (POST mutation) --
export function useCreateDesignImport() {
  return useSWRMutation<DesignImport, Error, string, CreateImportArg>(
    "/api/v1/design-sync/imports",
    mutationFetcher,
  );
}
// type CreateImportArg = { connection_id: number; brief: string; selected_node_ids: string[]; template_name?: string }

// -- Get import (GET with polling) --
export function useDesignImport(importId: number | null, polling?: boolean) {
  return useSWR<DesignImport>(
    importId ? `/api/v1/design-sync/imports/${importId}` : null,
    fetcher,
    { refreshInterval: polling ? 2000 : 0 },
  );
}

// -- Update import brief (PATCH mutation, inline fetcher) --
export function useUpdateImportBrief(importId: number | null) {
  return useSWRMutation<DesignImport, Error, string, { generated_brief: string }>(
    importId ? `/api/v1/design-sync/imports/${importId}/brief` : "",
    async (url: string, { arg }: { arg: { generated_brief: string } }) => {
      const res = await authFetch(url, {
        method: "PATCH",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify(arg),
      });
      if (!res.ok) throw new ApiError(res.status, "Failed to update brief");
      return res.json();
    },
  );
}

// -- Trigger conversion (POST mutation, long timeout) --
export function useConvertImport(importId: number | null) {
  return useSWRMutation<DesignImport, Error, string, { run_qa?: boolean; output_mode?: string }>(
    importId ? `/api/v1/design-sync/imports/${importId}/convert` : "",
    longMutationFetcher,
  );
}

// -- Extract components (POST mutation) --
export function useExtractComponents(connectionId: number | null) {
  return useSWRMutation<ExtractComponentsResult, Error, string, { component_ids?: string[]; generate_html?: boolean }>(
    connectionId ? `/api/v1/design-sync/connections/${connectionId}/extract-components` : "",
    mutationFetcher,
  );
}
```

Import `longMutationFetcher` from `@/lib/mutation-fetcher`, `authFetch` from `@/lib/auth-fetch`, `ApiError` from `@/lib/api-error`. Add required arg types.

### 4. Design Import Dialog — `cms/apps/web/src/components/design-sync/design-import-dialog.tsx` (CREATE)
Tabbed dialog with "Import Design" and "Extract Components" workflows.

**Props:**
```ts

```

**Dialog structure:**
- `max-w-[48rem]` (wider than forms dialog since it contains a file tree)
- Two tabs at top: "Import Design" | "Extract Components"
- Tab content renders the appropriate wizard

**Import Design tab — 4-step wizard:**
State machine: `select-frames` → `review-brief` → `converting` → `result`

**Step 1: Select Frames**
- Embed `<DesignFileBrowser>` component
- "Next" button (disabled until ≥1 node selected)
- On Next: call `useGenerateBrief` with selected node IDs → advance to step 2

**Step 2: Review Brief**
- Show layout summary line (from `GeneratedBrief.layout_summary`)
- Show sections detected count
- Editable `<textarea>` pre-filled with `GeneratedBrief.brief`
- "Back" button, "Start Conversion" button
- On Start: call `useCreateDesignImport` with `{ connection_id, brief, selected_node_ids }` → then call `useConvertImport` → advance to step 3

**Step 3: Converting**
- Spinner + "Converting your design to an email template…" message
- Progress indicators: show import status from polling (`useDesignImport(importId, true)`)
- Status badges for each phase: extracting → converting → completed
- If `status === "completed"` → auto-advance to step 4
- If `status === "failed"` → show error message + "Try Again" button (goes back to step 2)

**Step 4: Result**
- Success icon + "Template created successfully"
- Show `result_template_id` — link to workspace: `/projects/{project_id}/workspace?template={result_template_id}`
- "Open in Workspace" primary button
- "Close" secondary button
- Show list of imported assets (thumbnails + names)

**Extract Components tab — 3-step wizard:**
State machine: `select-components` → `extracting` → `result`

**Step 1: Select Components**
- Use `useDesignComponents(connectionId)` to fetch list
- Grid of component cards (thumbnail, name, description, containing page)
- Checkboxes for batch selection. "Select All" toggle
- "Extract Selected" button (disabled until ≥1 selected)
- On Extract: call `useExtractComponents` → advance to step 2

**Step 2: Extracting**
- Spinner + progress. Poll via `useDesignImport(importId, true)` using the returned `import_id`
- On completed → advance to step 3

**Step 3: Result**
- Success message + count of extracted components
- "View in Component Library" link to `/components`
- "Close" button

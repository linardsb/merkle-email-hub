# Plan: 3.3 Dashboard Homepage Enhancement

## Context

The current dashboard at `/[locale]/(dashboard)/page.tsx` is a bare scaffold: 3 stat cards (projects count, orgs count, hardcoded "Healthy"), a project grid, and a static "Getting Started" card. Task 3.3 upgrades it to show real operational data: QA quality at a glance, recent activity across modules, component library stats, and approval status — turning it into a useful daily command center.

**Design constraint:** No new backend endpoints. All data comes from existing paginated GET endpoints via existing SWR hooks. Client-side aggregation where needed. No backend changes.

**Available data sources (existing hooks):**
- `useProjects()` → project list + total count
- `useOrgs()` → org count
- `useComponents({ pageSize: 1 })` → component count via `.total`
- `useQADashboard()` → totalRuns, avgScore, passRate, overrideCount (fetches 50 results)
- `useApprovals(projectId)` → approval list with status field (pending/approved/rejected/revision_requested)
- `useQAResults({ pageSize: 5 })` → latest 5 QA results for activity feed

**SDK types available:**
- `ProjectResponse` — `id, name, description, status, is_active, created_at, updated_at`
- `ApprovalResponse` — `id, build_id, project_id, status, created_at, updated_at`
- `ComponentResponse` — `id, name, slug, description, category, latest_version, created_at`
- `QAResultResponse` — `id, overall_score, passed, checks_passed, checks_total, created_at` (local type)

**Key implementation details:**

1. **Stats cards pattern** (same as existing but 4 columns):
```tsx
<div className="rounded-lg border border-card-border bg-card-bg p-6">
  <div className="flex items-center gap-2">
    <Icon className="h-4 w-4 text-foreground-muted" />
    <p className="text-sm font-medium text-foreground-muted">{label}</p>
  </div>
  <p className={`mt-2 text-3xl font-semibold ${colorClass}`}>{value}</p>
</div>
```

2. **Pending approvals count** — derived from first project's approvals:
```typescript
const firstProjectId = projects?.items?.[0]?.id ?? null;
const { data: approvals } = useApprovals(firstProjectId);
const pendingCount = approvals?.filter((a) => a.status === "pending").length ?? 0;
```

3. **Quality Overview mini trend** — 10 dots from scoreTrend:
```tsx
<div className="flex items-center gap-1">
  {qaMetrics.scoreTrend.slice(-10).map((point, i) => (
    <div
      key={i}
      className={`h-2 w-2 rounded-full ${

      }`}
      title={`${Math.round(point.score * 100)}%`}
    />
  ))}
</div>
```

4. **Recent activity list** — from `useQAResults({ page: 1, pageSize: 5 })`:
```tsx
{recentQA?.items.map((result) => (
  <div key={result.id} className="flex items-center justify-between py-2">
    <div className="flex items-center gap-2">
      <div className={`h-2 w-2 rounded-full ${result.passed ? "bg-status-success" : "bg-status-danger"}`} />
      <span className="text-sm text-foreground">{t("qaRun")}</span>
      <span className="text-xs text-foreground-muted">
        {t("scoreLabel", { score: Math.round(result.overall_score * 100) })}
      </span>
    </div>
    <span className="text-xs text-foreground-muted">
      {new Date(result.created_at).toLocaleDateString(undefined, {
        month: "short", day: "numeric", hour: "2-digit", minute: "2-digit",
      })}
    </span>
  </div>
))}
```

# Plan: 37.5 Complete Human Labeling with Improved Judges

## Context

The eval pipeline's trustworthiness depends on calibrated judges. Without human labels, we can't measure judge accuracy (TPR/TNR), which means we can't trust the eval gates that control skill updates, regression detection, or production monitoring. The golden references (37.1-37.3) improved judge prompts ÔÇö human labels confirm whether accuracy actually improved.

**Status:** Labeling has been completed and results consolidated into `email-templates/components/` and `email-templates/components/golden-references/`.

## Implementation Steps

### Step 1: Regenerate label scaffolds from current verdicts
```bash
make eval-labels
```
Ensures all 7 agents with verdicts have `{agent}_human_labels.jsonl` files.

### Step 2: Run judges on missing agents
```bash
make eval-judge  # Generates verdicts for knowledge + innovation
make eval-labels # Re-scaffold to include new agents
```

### Step 3: Human labeling (manual, ~2-4 hours)
Open `docs/eval-labeling-tool.html` in browser. Label 540 rows:
- **Priority 1:** 15 criteria with >20% flip rate (see table above)
- **Priority 2:** Remaining criteria with >10% flip rate
- **Priority 3:** Spot-check low-flip criteria
Export labeled JSONL back to `traces/`.

### Step 4: Run calibration
```bash
make eval-calibrate      # TPR/TNR per judge criterion
make eval-qa-calibrate   # Agreement per QA check
```

### Step 5: Iterate on failing criteria
If TPR < 0.85 or TNR < 0.80 on any criterion:
1. Add more golden references for that criterion to `golden-references/`
2. Update `index.yaml` with new mappings
3. Adjust judge prompt framing if needed
4. Re-run `make eval-rejudge` Ôćĺ `make eval-calibrate`

### Step 6: Error analysis & documentation
```bash
make eval-analysis       # Failure taxonomy
make eval-regression     # Baseline comparison
```
Document calibration results in TODO.md completion record.

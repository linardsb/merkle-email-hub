# Plan: 22.2 Prompt Template Store

## Context
Agent system prompts are currently loaded from SKILL.md files on disk at module import time (e.g. `scaffolder/prompt.py` reads `SKILL.md` into `_SKILL_CONTENT` at import). The existing A/B override mechanism (`skill_override.py`) is in-memory only — lost on restart, no history, no variant tracking.

This phase adds a versioned database store for agent prompts with A/B variant support. Agents load prompts at runtime via `PromptStore.get_active(agent_id, variant)`, falling back to SKILL.md when the store is disabled or empty. Versions are tracked with rollback capability.

## Design Decisions
- **Opt-in via config**: `AI__PROMPT_STORE_ENABLED=false` by default. When disabled, zero overhead — existing SKILL.md path is untouched.
- **Fallback-first**: `get_active()` returns `None` when no DB prompt exists. Callers always fall back to SKILL.md. This means the store is purely additive — no data loss risk.
- **No seeding migration**: Seeding SKILL.md content into DB during migration is fragile (files may not exist in all environments). Instead, provide a `POST /api/v1/prompts/seed` admin endpoint that reads current SKILL.md files and creates initial versions.
- **Variant model**: Each template has an `agent_id` (e.g. "scaffolder"), `variant` (e.g. "control", "experimental"), and `version` (auto-incrementing per agent+variant). Only one version per agent+variant is `active=True`.
- **Integration point**: `skill_override.py` already provides `get_override(agent) -> str | None` checked by every agent's `prompt.py`. The prompt store hooks into the same pattern — `get_override` is extended to check the DB store when enabled, before falling back to the in-memory override dict.

### Step 3: Create model, repository, and service
File: `app/ai/prompt_store.py`

**Model** — `PromptTemplate(Base)`:
- `id: Mapped[int]` — PK
- `agent_id: Mapped[str]` — e.g. "scaffolder", "dark_mode"
- `variant: Mapped[str]` — e.g. "default", "experimental"
- `version: Mapped[int]` — auto-incremented per agent+variant
- `content: Mapped[str]` — full prompt text
- `description: Mapped[str | None]` — human-readable change note
- `active: Mapped[bool]` — only one active per agent+variant
- `created_by: Mapped[str | None]` — user email who created it
- `created_at: Mapped[datetime]` — server-default now()

**Repository** — `PromptTemplateRepository(db: AsyncSession)`:
- `get_active(agent_id: str, variant: str = "default") -> PromptTemplate | None` — fetch active prompt for agent+variant
- `get_by_id(template_id: int) -> PromptTemplate | None`
- `list_versions(agent_id: str, variant: str = "default", limit: int = 20) -> list[PromptTemplate]` — ordered by version desc
- `list_agents() -> list[str]` — distinct agent_ids
- `create(agent_id: str, variant: str, content: str, description: str | None, created_by: str | None) -> PromptTemplate` — auto-version: `SELECT COALESCE(MAX(version), 0) + 1` for agent+variant
- `activate(template_id: int) -> PromptTemplate` — deactivate all other versions for same agent+variant, activate this one
- `rollback(agent_id: str, variant: str = "default") -> PromptTemplate | None` — activate the previous version (version = active.version - 1)

**Service** — `PromptStoreService`:
- `get_prompt(agent_id: str, variant: str = "default") -> str | None` — delegates to repo, returns content string or None
- `seed_from_skill_files() -> dict[str, int]` — reads all SKILL.md files from `AGENT_NAMES`, creates version 1 "default" variant for each agent that doesn't already have a DB entry. Returns `{agent_id: version}` for seeded agents.

Key patterns:
- Repository follows `RoutingHistoryRepository` pattern (constructor takes `AsyncSession`)
- Service methods are `async` and take `db: AsyncSession` parameter (not stored)
- Use `select()` + `func.coalesce()` for version auto-increment
- Use `sa.update()` with `where()` for bulk deactivation before activation

### Step 6: Integrate with skill_override.py
Modify `app/ai/agents/skill_override.py`:

```python
"""Runtime SKILL.md content override for A/B testing and prompt store.

Priority: prompt store (DB) > in-memory override > SKILL.md on disk.
"""
from __future__ import annotations

from app.core.config import get_settings
from app.core.logging import get_logger

logger = get_logger(__name__)

_overrides: dict[str, str] = {}

# Cache for prompt store lookups (populated by sync caller)
_store_cache: dict[str, str] = {}


def set_override(agent: str, content: str) -> None:
    """Set SKILL.md content override for an agent."""
    _overrides[agent] = content


def clear_override(agent: str) -> None:
    """Remove SKILL.md content override for an agent."""
    _overrides.pop(agent, None)


def clear_all_overrides() -> None:
    """Remove all overrides (cleanup after A/B test)."""
    _overrides.clear()


def set_store_cache(agent: str, content: str) -> None:
    """Cache a prompt store lookup result."""
    _store_cache[agent] = content


def clear_store_cache(agent: str | None = None) -> None:
    """Clear prompt store cache (all or specific agent)."""
    if agent is None:
        _store_cache.clear()
    else:
        _store_cache.pop(agent, None)


def get_override(agent: str) -> str | None:
    """Get prompt override for an agent.

    Priority: prompt store cache > in-memory A/B override > None (use SKILL.md).
    Prompt store is only checked when AI__PROMPT_STORE_ENABLED=true.
    """
    settings = get_settings()
    if settings.ai.prompt_store_enabled and agent in _store_cache:
        return _store_cache[agent]
    return _overrides.get(agent)
```

**Important**: The prompt store cache is populated asynchronously. The `BlueprintEngine` (or a startup hook) pre-loads active prompts into `_store_cache` via `PromptStoreService`. This avoids async DB calls in the synchronous `get_override()` path.

### Step 7: Add cache pre-loading
Add a helper function in `app/ai/prompt_store.py`:

```python
async def preload_prompt_store_cache(db: AsyncSession) -> None:
    """Load all active prompt templates into the skill_override cache.

    Called at startup and after any prompt mutation (create/activate/rollback).
    """
    settings = get_settings()
    if not settings.ai.prompt_store_enabled:
        return

    from app.ai.agents.skill_override import clear_store_cache, set_store_cache

    clear_store_cache()
    repo = PromptTemplateRepository(db)
    agents = await repo.list_agents()
    for agent_id in agents:
        template = await repo.get_active(agent_id)
        if template:
            set_store_cache(agent_id, template.content)
            logger.info(
                "prompt_store.cache_loaded",
                agent_id=agent_id,
                version=template.version,
                variant=template.variant,
            )
```

Call `preload_prompt_store_cache()` in route handlers after mutations (`create`, `activate`, `rollback`, `seed`).

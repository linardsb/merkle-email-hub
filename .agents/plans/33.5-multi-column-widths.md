# Plan: 33.5 Multi-Column Layout & Proportional Width Calculation

## Context

Every nested `<table>` gets `width="100%"` (converter.py:399). In a two-column HORIZONTAL layout (e.g., 200px + 400px in a 600px parent), both columns render full-width and stack vertically instead of sitting side by side. The `_group_into_rows()` function uses a 10px y-tolerance to guess rows, but: (a) auto-layout nodes often lack absolute positions (`y=None`), (b) 11px offset splits columns into separate rows, (c) it ignores `layoutMode: "HORIZONTAL"` from Figma. No MSO ghost table wrapping exists for multi-column rows.

### Target HTML pattern (from codebase skill docs + memory)

Multi-column rows must use the **hybrid inline-block + MSO ghost table** pattern:

```html
<td style="font-size:0;text-align:center;mso-line-height-rule:exactly;">
  <!--[if mso]><table role="presentation" width="100%" cellpadding="0" cellspacing="0" border="0"><tr><td width="200" valign="top"><![endif]-->
  <div class="column" style="display:inline-block;max-width:200px;width:100%;vertical-align:top;">
    <table role="presentation" width="100%" cellpadding="0" cellspacing="0" border="0"
      style="border-collapse:collapse;mso-table-lspace:0pt;mso-table-rspace:0pt;">
      {child_html}
    </table>
  </div>
  <!--[if mso]></td><td width="400" valign="top"><![endif]-->
  <div class="column" style="display:inline-block;max-width:400px;width:100%;vertical-align:top;">
    <table role="presentation" width="100%" ...>
      {child_html}
    </table>
  </div>
  <!--[if mso]></td></tr></table><![endif]-->
</td>
```

Key rules from skill docs:
- MSO ghost table: `cellpadding="0" cellspacing="0" border="0"`, `mso-table-lspace:0pt;mso-table-rspace:0pt`
- NEVER nest MSO conditionals — each ghost table is self-contained per row
- `display:inline-block` requires `vertical-align:top` and explicit `max-width` in px
- Individual columns are NOT sections — the parent gets the section annotation
- Outlook ignores `max-width` — MSO ghost table provides fixed pixel widths
- Gap between columns: MSO uses spacer `<td width="N">`, modern uses `padding-left`

### Step 2: Add `_calculate_column_widths()` helper (converter.py, after `_group_into_rows`)

```python
def _calculate_column_widths(
    children: list[DesignNode],
    container_width: float,
    gap: float = 0,
) -> list[int]:
```

Logic:
1. Count children: `n = len(children)`
2. Calculate total gap: `total_gap = gap * (n - 1)` — gap applies between columns, not at edges
3. Available width: `avail = container_width - total_gap`
4. If ALL children have `width` set: calculate proportional px widths:
   - `total_child_w = sum(c.width for c in children)`
   - Each child: `px = round(avail * (child.width / total_child_w))`
   - Adjust last column to absorb rounding: `widths[-1] = int(avail) - sum(widths[:-1])`
5. If NONE have width: distribute equally: `px = int(avail / n)`, last gets remainder
6. If MIXED: use known widths proportionally, split remainder equally among unknowns
7. Return `list[int]` of pixel widths (one per child)

### Step 3: Add `_render_multi_column_row()` helper (converter.py, after `_calculate_column_widths`)

```python
def _render_multi_column_row(
    children: list[DesignNode],
    *,
    column_widths: list[int],
    gap: int,
    indent: int,
    props_map: dict[str, _NodeProps] | None,
    parent_bg: str | None,
    parent_font: str | None,
    section_map: dict[str, EmailSection] | None,
    button_ids: set[str] | None,
    text_meta: dict[str, TextBlock] | None,
    current_section: EmailSection | None,
    body_font_size: float,
    compat: ConverterCompatibility | None,
    _depth: int,
    container_width: int,
) -> list[str]:
```

Generates the hybrid inline-block + MSO ghost table pattern. Returns list of HTML lines.

Logic:
1. `pad = "  " * indent`
2. Open outer `<tr>`:
   ```
   <tr>
     <td style="font-size:0;text-align:center;mso-line-height-rule:exactly;">
   ```
3. Open MSO ghost table:
   ```
   <!--[if mso]><table role="presentation" width="{container_width}"
     cellpadding="0" cellspacing="0" border="0"
     style="border-collapse:collapse;mso-table-lspace:0pt;mso-table-rspace:0pt;">
   <tr><![endif]-->
   ```
4. For each `(child, col_width)` in `zip(children, column_widths)`:
   - If NOT first child and `gap > 0`: emit MSO spacer `<!--[if mso]><td width="{gap}"></td><![endif]-->`
   - Emit MSO column open: `<!--[if mso]><td width="{col_width}" valign="top"><![endif]-->`
   - Emit modern wrapper div: `<div class="column" style="display:inline-block;max-width:{col_width}px;width:100%;vertical-align:top;">`
   - Emit inner table open: `<table role="presentation" width="100%" cellpadding="0" cellspacing="0" border="0" style="border-collapse:collapse;mso-table-lspace:0pt;mso-table-rspace:0pt;">`
   - Recurse: `child_html = node_to_email_html(child, ...)` — pass all kwargs, increment `_depth`
   - If child is TEXT: wrap in `<tr>{child_html}</tr>`, else wrap in `<tr><td>{child_html}</td></tr>`
   - Close inner table, div
   - Emit MSO column close: `<!--[if mso]></td><![endif]-->`
5. Close MSO ghost table: `<!--[if mso]></tr></table><![endif]-->`
6. Close outer `</td></tr>`

Add `display:inline-block` compatibility warning via `compat.check_and_warn("display", value="inline-block", context="Multi-column layout")` once per row (not per column).

Currently, every row is rendered as a simple `<tr>` with `<td>` per child. Change the logic to:

1. After building `rows` (line 502), iterate `rows` as before
2. For each row, check if it's a **multi-column row**: `len(row) > 1`
3. If multi-column:
   - Determine effective parent width: use `node.width` if available, else `container_width`
   - Call `_calculate_column_widths(row, effective_width, gap=gap_px)`

4. If single-column (len == 1): keep existing rendering unchanged (lines 525–562)
5. This means the `width="100%"` on the outer `<table>` (line 399) stays — it's correct for the section wrapper. Only the inner column structure changes.

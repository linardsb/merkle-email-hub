# Plan: 11.3 Accessibility — WCAG AA Coverage + Shared Rule Engine

## Context

The current accessibility check (`app/qa_engine/checks/accessibility.py`) is the weakest check in the QA gate — only 3 regex-based checks (lang, alt, role=presentation). Task 11.3 replaces it with a comprehensive DOM-parsed validator covering 24 WCAG AA checks across 8 groups.

**Key architectural change:** Instead of hardcoding checks in Python (like 11.2 did), we build a **shared rule engine** that loads check definitions from YAML files. Both the accessibility check AND the html_validation check use the same engine — different rule files, same evaluator. This also means refactoring `html_validation.py` (11.2) to be rule-driven.

## Rule Engine Architecture

### `app/qa_engine/rule_engine.py`

A generic engine that loads YAML rule definitions and evaluates them against an lxml DOM tree. Both `accessibility.py` and `html_validation.py` delegate to this engine.

```python
"""Shared rule engine — loads YAML rule definitions and evaluates against lxml DOM.

Used by accessibility check (WCAG AA rules) and html_validation check (email structure rules).
Rule YAML files also serve as RAG knowledge base entries for agent context.
"""

from __future__ import annotations

import re
from dataclasses import dataclass, field
from pathlib import Path
from typing import Any

import yaml
from lxml import html as lxml_html
from lxml.html import HtmlElement

from app.qa_engine.check_config import QACheckConfig


@dataclass(frozen=True)
class Rule:
    """A single validation rule loaded from YAML."""
    id: str
    group: str
    name: str
    check_type: str            # One of the ~15 evaluator types
    message: str
    wcag: str = ""             # WCAG criterion (empty for non-a11y rules)
    level: str = ""            # A, AA, AAA (empty for non-a11y rules)
    selector: str = ""         # CSS-like selector for lxml xpath conversion
    attr: str = ""             # Single attribute to check
    attrs: tuple[str, ...] = () # Multiple attributes to check
    value: str = ""            # Expected attribute value
    values: tuple[str, ...] = () # Multiple acceptable values
    patterns: tuple[str, ...] = () # Regex/text patterns to match
    deduction_key: str = ""    # Key in defaults.yaml params
    default_deduction: float = 0.10
    per_element: bool = False  # Deduct per matching element vs once
    cap_key: str = ""          # Key for max_issues_reported in config


@dataclass
class RuleResult:
    """Result of evaluating a single rule."""
    rule_id: str
    passed: bool
    issues: list[str] = field(default_factory=list)
    deduction: float = 0.0


def load_rules(yaml_path: Path) -> list[Rule]:
    """Load rules from a YAML file."""
    with open(yaml_path) as f:
        data = yaml.safe_load(f)
    rules: list[Rule] = []
    for entry in data.get("rules", []):
        rules.append(Rule(
            id=entry["id"],
            group=entry["group"],
            name=entry["name"],
            check_type=entry["check"],
            message=entry["message"],
            wcag=entry.get("wcag", ""),
            level=entry.get("level", ""),
            selector=entry.get("selector", ""),
            attr=entry.get("attr", ""),
            attrs=tuple(entry.get("attrs", ())),
            value=entry.get("value", ""),
            values=tuple(entry.get("values", ())),
            patterns=tuple(entry.get("patterns", ())),
            deduction_key=entry.get("deduction_key", ""),
            default_deduction=entry.get("default_deduction", 0.10),
            per_element=entry.get("per_element", False),
            cap_key=entry.get("cap_key", ""),
        ))
    return rules


# --- Check type registry ---
# Each check type is a function: (doc, raw_html, rule, config) -> RuleResult

_CHECK_TYPES: dict[str, CheckFn] = {}  # populated via register_check_type()


class RuleEngine:
    """Evaluates a set of rules against an HTML document."""

    def __init__(self, rules: list[Rule]) -> None:
        self.rules = rules

    def evaluate(
        self,
        doc: HtmlElement,
        raw_html: str,
        config: QACheckConfig | None = None,
    ) -> tuple[list[str], float]:
        """Run all rules, return (issues, total_deduction)."""
        all_issues: list[str] = []
        total_deduction = 0.0

        for rule in self.rules:
            check_fn = _CHECK_TYPES.get(rule.check_type)
            if check_fn is None:
                continue  # Unknown check type — skip gracefully
            result = check_fn(doc, raw_html, rule, config)
            if not result.passed:
                all_issues.extend(result.issues)
                total_deduction += result.deduction

        return all_issues, total_deduction
```

### Check Types (~15 evaluators)

Each check type is a pure function registered in `_CHECK_TYPES`. These cover all patterns needed by both rule files:

| Check Type | Description | Used By |
|-----------|-------------|---------|
| `attr_present` | Element has required attribute(s) | img alt, img width/height, table cellpadding |
| `attr_value` | Attribute matches expected value | role="presentation", border="0" |
| `attr_empty` | Attribute must be empty string | alt="" on tracking pixels |
| `attr_pattern` | Attribute matches regex | lang matches BCP-47 pattern |
| `attr_absent` | Attribute must NOT be present | no summary on layout tables |
| `element_present` | At least one element matching selector exists | headings exist, unsubscribe link |
| `element_absent` | No elements matching selector | no `<base>` tag, no nested `<a>` |
| `element_count` | Count constraint (max/min) on matching elements | single `<h1>`, single `<head>` |
| `parent_has` | Parent element has specific attribute/value | `<th>` has scope, `<li>` inside `<ul>`/`<ol>` |
| `children_match` | Children meet criteria | `<details>` first child is `<summary>` |
| `text_content` | Element text matches/excludes patterns | generic link text, empty links |
| `sibling_check` | Consecutive siblings share attribute | redundant links with same href |
| `style_contains` | CSS in `<style>` blocks matches pattern | outline:none, @media prefers-color-scheme |
| `raw_html_pattern` | Regex on raw HTML string | doctype, MSO conditionals, nested `<a>` |
| `custom` | Named Python function for complex logic | heading hierarchy, tracking pixel detection, preview padding |

The `custom` type delegates to named Python functions for checks that are too complex for declarative rules (heading level sequence, tracking pixel heuristics, dark mode color analysis, etc.). These are registered by name:

```python
_CUSTOM_CHECKS: dict[str, CustomCheckFn] = {
    "heading_hierarchy": check_heading_hierarchy,
    "tracking_pixel_alt": check_tracking_pixel_alt,
    "preview_padding_aria": check_preview_padding_aria,
    "dark_unsafe_colors": check_dark_unsafe_colors,
    "layout_table_heuristic": check_layout_table_heuristic,
    "unclosed_tags": check_unclosed_tags,
    "block_in_inline": check_block_in_inline,
    # ... etc
}
```

```python
class AccessibilityCheck:
    """Validates email HTML for WCAG AA accessibility using lxml DOM parsing.

    Implements 24 checks across 8 groups:
    A (1-2): Language
    B (3-5): Table Semantics
    C (6-8): Image Accessibility
    D (9-11): Heading Hierarchy
    E (12-14): Link Accessibility
    F (15-20): Content Semantics & Screen Reader
    G (21-22): Dark Mode Contrast
    H (23-24): AMP Form Accessibility
    """

    name = "accessibility"

    async def run(self, html: str, config: QACheckConfig | None = None) -> QACheckResult:
        issues: list[str] = []
        score = 1.0

        if not html or not html.strip():
            return QACheckResult(
                check_name=self.name, passed=False, score=0.0,
                details="Empty HTML document", severity="error",
            )

        try:
            doc = lxml_html.document_fromstring(html)
        except Exception:
            return QACheckResult(
                check_name=self.name, passed=False, score=0.0,
                details="HTML could not be parsed", severity="error",
            )

        # --- Group A: Language ---
        score = self._check_lang_present(doc, config, issues, score)
        score = self._check_lang_valid(doc, config, issues, score)

        # --- Group B: Table Semantics ---
        score = self._check_table_role_presentation(doc, config, issues, score)
        score = self._check_table_mixed_signals(doc, config, issues, score)
        score = self._check_data_table_headers(doc, config, issues, score)

        # --- Group C: Image Accessibility ---
        score = self._check_img_alt(doc, config, issues, score)
        score = self._check_tracking_pixel_alt(doc, config, issues, score)
        score = self._check_linked_img_alt(doc, config, issues, score)

        # --- Group D: Heading Hierarchy ---
        score = self._check_headings_present(doc, config, issues, score)
        score = self._check_multiple_h1(doc, config, issues, score)
        score = self._check_heading_levels(doc, config, issues, score)

        # --- Group E: Link Accessibility ---
        score = self._check_generic_link_text(doc, config, issues, score)
        score = self._check_empty_links(doc, config, issues, score)
        score = self._check_redundant_links(doc, config, issues, score)

        # --- Group F: Content Semantics & Screen Reader ---
        score = self._check_semantic_emphasis(doc, config, issues, score)
        score = self._check_preview_padding_aria(doc, html, config, issues, score)
        score = self._check_spacer_aria(doc, config, issues, score)
        score = self._check_separator_aria(doc, config, issues, score)
        score = self._check_outline_removal(doc, html, config, issues, score)
        score = self._check_br_spacing(doc, config, issues, score)

        # --- Group G: Dark Mode Contrast ---
        score = self._check_dark_unsafe_colors(doc, html, config, issues, score)
        score = self._check_dark_no_overrides(doc, html, config, issues, score)

        # --- Group H: AMP Form Accessibility ---
        score = self._check_input_label(doc, config, issues, score)
        score = self._check_required_aria(doc, config, issues, score)

        score = max(0.0, score)
        passed = len(issues) == 0
        return QACheckResult(
            check_name=self.name,
            passed=passed,
            score=round(score, 2),
            details="; ".join(issues) if issues else None,
            severity="warning" if not passed else "info",
        )
```

#### Per-check method specs:

**A1: `_check_lang_present`**
- Find `<html>` element in DOM, check for `lang` attribute
- If missing: issue + deduction

**A2: `_check_lang_valid`**
- If `lang` attr exists, validate against `_LANG_PATTERN` (BCP-47)
- Empty string or invalid pattern → issue

**B3: `_check_table_role_presentation`**
- Find ALL `<table>` elements in `<body>`
- Heuristic for "layout table": no `role="table"`, no `<caption>`, no `<th>`, no `summary` attr
- If a table looks like layout (no data-table signals) and lacks `role="presentation"` → issue
- Cap at `max_table_issues_reported`; deduction per table

**B4: `_check_table_mixed_signals`**
- Find tables WITH `role="presentation"` that also contain `<th>`, `<caption>`, or `summary` attr
- These conflicting signals confuse screen readers → issue per table

**B5: `_check_data_table_headers`**
- Find tables with `role="table"` or containing `<caption>` / `<th>` (explicitly data tables)
- Check that `<th>` elements have `scope="col"` or `scope="row"`
- Missing scope → issue

**C6: `_check_img_alt`**
- Find ALL `<img>` in DOM
- Check each for `alt` attribute presence (note: `alt=""` IS valid — empty alt for decorative)
- If `alt` attr is missing entirely → issue
- Cap at `max_img_issues_reported`; deduction per image
- Extract partial `src` for context in issue message (basename, max 40 chars)

**C7: `_check_tracking_pixel_alt`**
- Find `<img>` with tracking pixel signals:
  - `width="1"` and `height="1"` (or style `width:1px; height:1px`)
  - `width="0"` / `height="0"`
  - `display:none` in style
  - `visibility:hidden` in style
- These MUST have `alt=""` (empty, not missing, not non-empty)
- If `alt` is present but non-empty → issue (tracking pixel has descriptive alt)
- If `alt` is missing → issue (should be `alt=""`)

**C8: `_check_linked_img_alt`**
- Find `<a>` tags containing `<img>` children
- If the `<a>` contains ONLY an `<img>` (no text siblings), the `<img>` `alt` is the link's accessible name
- If that `alt` is empty (`""`) or missing → issue (linked image with no accessible name)
- Note: if `<a>` has text content alongside the image, the text provides the name — no issue

**D9: `_check_headings_present`**
- Search body for any h1–h6 elements
- If zero headings found → issue (email has no navigable structure for screen readers)

**D10: `_check_multiple_h1`**
- Count `<h1>` elements in body
- If > 1 → issue with count

**D11: `_check_heading_levels`**
- Collect all heading elements in document order, extract level numbers
- Walk sequence: if level jumps by >1 (e.g., h1→h3 or h2→h5) → issue per skip
- Cap at `max_heading_issues_reported`

**E12: `_check_generic_link_text`**
- Find all `<a>` elements with `href`
- Extract visible text content (`.text_content().strip().lower()`)
- If text matches `_GENERIC_LINK_TEXT` exactly → issue
- Cap at `max_link_issues_reported`

**E13: `_check_empty_links`**
- Find all `<a>` elements with `href`
- An `<a>` has accessible name if: (a) has non-whitespace text content, (b) has `aria-label`, or (c) contains `<img>` with non-empty `alt`
- If none of the above → issue (empty link)
- Cap at `max_link_issues_reported`

**E14: `_check_redundant_links`**
- Find consecutive `<a>` elements (siblings or close in DOM) with the same `href`
- Common pattern: `<a href="X"><img ...></a><a href="X">Product Name</a>`
- Detect: iterate all `<a>` in body, group by parent element; within each parent, check for consecutive `<a>` children with identical `href` → issue per pair

**F15: `_check_semantic_emphasis`**
- Find `<b>` and `<i>` elements in body (not in `<style>` or `<script>`)
- If found → single issue noting count: "{N} `<b>`/`<i>` tags found — use `<strong>`/`<em>` for semantic emphasis"
- Single deduction (not per-element — this is a pattern issue)

**F16: `_check_preview_padding_aria`**
- Detect the common preheader padding pattern: repeated `&zwnj;` (U+200C) and `&nbsp;` (U+00A0)
- Search for elements/text nodes containing 3+ consecutive `\u200c` or `\u00a0` characters
- If found and the containing element lacks `aria-hidden="true"` → issue
- Use raw HTML search for the `&zwnj;&nbsp;` pattern as well as unicode equivalents

**F17: `_check_spacer_aria`**
- Find `<td>` / `<div>` elements whose only content is `&nbsp;` (single non-breaking space) or empty
- Also find `<img>` with `src` containing "spacer" or "blank" (common naming)
- If these lack `aria-hidden="true"` → issue per element
- Cap at `max_content_issues_reported`

**F18: `_check_separator_aria`**
- Find text nodes containing common separator characters: `|`, `•`, `·`, `–`, `—`
- These are typically between footer links: `<span>|</span>` or just `|` text
- Check elements whose ONLY content is a single separator character
- If the element lacks `aria-hidden="true"` → issue
- Cap at `max_content_issues_reported`

**F19: `_check_outline_removal`**
- Search `<style>` elements and inline `style` attributes for:
  - `outline: none` or `outline:none`
  - `outline: 0` or `outline:0`
- If found without a compensating focus style (`:focus` rule in same style block) → issue
- Simple heuristic: if `outline: none` OR `outline: 0` appears anywhere in styles → issue

**F20: `_check_br_spacing`**
- Find consecutive `<br>` elements (2+ `<br>` tags with no intervening content)
- Look for `<br>` whose next sibling is also `<br>` (or `<br>` followed immediately by `<br>` in same parent)
- If found → issue per occurrence (capped)
- Message: "Consecutive `<br>` for spacing — use `<p>` or table cell padding"

## Implementation Notes

### Table Layout Detection Heuristic (B3)
A table is considered a **layout table** if it has NONE of these data-table signals:
- `role="table"` or `role="grid"`
- Contains `<caption>` child
- Contains `<th>` child
- Has `summary` attribute
- Contains `<thead>`, `<tbody>`, `<tfoot>`

If none present AND no `role="presentation"` → flag it.

### Tracking Pixel Detection (C7)
Match `<img>` with ANY of:
- `width="1"` AND `height="1"` (attributes)
- `width="0"` or `height="0"`
- inline style containing `width:1px` + `height:1px` (or `0px`)
- inline style containing `display:none` or `visibility:hidden`

### Preview Padding Detection (F16)
Look for text sequences of 3+ consecutive ZWNJ (`\u200c`) or NBSP (`\u00a0`) characters. Common pattern:
```html
<div style="max-height:0;overflow:hidden;">&zwnj;&nbsp;&zwnj;&nbsp;...</div>
```
Walk text nodes; if a node's text matches the pattern and the node's parent lacks `aria-hidden="true"`, flag it.

### Dark Mode Unsafe Colors Detection (G21)
- Collect all `<style>` content from `<head>`
- Find `@media (prefers-color-scheme: dark)` blocks (regex extraction of content between braces)
- Within dark mode blocks, extract `color:` and `background-color:` / `background:` values
- Flag pure black/white pairs: `#000000`/`#000`/`black` with `#ffffff`/`#fff`/`white`
- These invert unpredictably across Outlook.com, Gmail dark mode — recommend `#f0f0f0` on `#1a1a1a` instead
- Also scan `[data-ogsc]` and `[data-ogsb]` targeted selectors (Outlook.com dark mode overrides)

### Dark Mode Missing Overrides Detection (G22)
- Check if `<meta name="color-scheme" content="light dark">` or `<meta name="supported-color-schemes">` exists
- If dark mode meta is declared, search all `<style>` blocks for `@media (prefers-color-scheme: dark)`
- If the media query is missing entirely → issue (meta declares dark support but no styles provide overrides)
- If the media query exists but contains no `color` or `background` properties → issue (empty dark mode block)

### AMP Form Input Label Detection (H23)
- Find all `<input>`, `<select>`, `<textarea>` elements (excluding `type="hidden"`, `type="submit"`, `type="button"`)
- For each, check if it has an associated label:
  - (a) `<label for="X">` where X matches the input's `id` attribute
  - (b) Input is nested inside a `<label>` element
  - (c) Input has `aria-label` or `aria-labelledby` attribute
- If no `placeholder` attr exists either → strong issue
- If only `placeholder` exists (no label) → issue: "placeholder alone is insufficient"
- Cap at `max_form_issues_reported`
- **Skip entirely if no form elements found** — no deductions for non-AMP emails

### AMP Required Field Aria Detection (H24)
- Find all elements with `required` attribute
- Check each for `aria-required="true"` — if missing → issue
- Also check the closest parent/sibling for a visible text indicator of required status (heuristic: look for text containing "required" or "*" with an associated legend/label explaining it)
- Cap at `max_form_issues_reported`
- **Skip entirely if no form elements found**

### Consecutive BR Detection (F20)
Walk children of each element. For each `<br>` found, check if the next sibling (ignoring whitespace-only text) is also `<br>`. Pattern:
```python
for el in doc.iter():
    children = list(el)
    for i, child in enumerate(children):
        if child.tag == "br" and i + 1 < len(children):
            # Check if tail text between br tags is empty
            tail = (child.tail or "").strip()
            if not tail and children[i + 1].tag == "br":
                # Consecutive br found
```

### Redundant Link Detection (E14)
For each parent element, collect `<a>` children in order. If two consecutive `<a>` children share the same `href` (after stripping trailing slashes and normalizing) → flag as redundant. Common email pattern:
```html
<td>
  <a href="https://shop.com/item"><img src="item.jpg" alt="Item"></a>
  <a href="https://shop.com/item">Item Name</a>
</td>
```

# Plan: 49.6 Per-Email Token Scoping from Shared Figma Files

## Context

When a Figma file contains multiple email designs (common in design system files), `sync_tokens_and_structure()` extracts tokens from the **entire file** — producing 254 colors and 25 font families when the target email uses only ~3 colors and 1 font. This causes `convert_typography()` and `convert_colors_to_palette()` to make wrong heuristic choices (e.g., picking Inter instead of Helvetica for the REFRAME email). The fix: scope token extraction to the target frame's subtree when a `node_id` is provided.

## Design Decision: Subtree Extraction vs Post-Filter

Two approaches considered:

**A. Post-filter** — extract globally, then filter tokens by matching against subtree node references.
- Pro: simpler, no changes to walk methods
- Con: must build node ID set from raw Figma JSON, then cross-reference style IDs → still needs tree walk

**B. Scoped walk** — extract a subtree from `file_data["document"]` first, then run existing parse methods on that subtree instead of the root.
- Pro: **zero changes to walk methods** — just pass a different root node; published style lookups automatically scoped because `_walk_for_style_property()` starts from the given node
- Con: must find the target node in the raw Figma JSON tree

**Chosen: B (scoped walk).** Extract the target frame subtree from `file_data["document"]` and pass it as the walk root. This is the simplest approach — existing `_walk_for_*` methods work unchanged. The only new code is a subtree finder.

## Implementation Steps

### Step 1: Add `_find_subtree()` to `figma/service.py`

Add a module-level helper that walks the raw Figma JSON tree to find a node by ID and returns it (with all descendants intact):

```python
def _find_subtree(document: dict[str, Any], node_id: str) -> dict[str, Any] | None:
    """Find a node by ID in the Figma document tree, returning the full subtree."""
    if not isinstance(document, dict):
        return None
    if str(document.get("id", "")) == node_id:
        return document
    for child in document.get("children", []):
        result = _find_subtree(child, node_id)
        if result is not None:
            return result
    return None
```

Place after `_MAX_WALK_DEPTH` constant (~line 40).

### Step 3: Update `FigmaDesignSyncService.sync_tokens_and_structure()` (`service.py:415`)

Add `target_node_id` parameter. After `file_data` is fetched (line 441), determine the walk root:

```python
async def sync_tokens_and_structure(
    self, file_ref: str, access_token: str, *, target_node_id: str | None = None
) -> tuple[ExtractedTokens, DesignFileStructure]:
    # ... existing fetch logic (lines 418-442) ...

    settings = get_settings()
    token_scoping = settings.design_sync.token_scoping_enabled and target_node_id is not None

    # Determine walk root for token extraction
    document_root = file_data.get("document", {})
    if token_scoping:
        subtree = _find_subtree(document_root, target_node_id)
        walk_root = subtree if subtree is not None else document_root
        if subtree is None:
            logger.warning(
                "design_sync.figma.token_scoping_target_not_found",
                target_node_id=target_node_id,
            )
    else:
        walk_root = document_root
```

Then replace all `file_data.get("document", {})` references in the token extraction section (lines 475-485) with a scoped `file_data` dict:

```python
    # Build scoped file_data for parse methods (they expect {"document": ..., "styles": ...})
    scoped_file_data: dict[str, Any] = {
        "document": walk_root,
        "styles": file_data.get("styles", {}),
    }

    # ... existing variables logic (lines 447-466, uses API endpoint — not affected) ...

    if variables_source:
        colors = var_colors
        stroke_colors: list[ExtractedColor] = []
        grad_list: list[ExtractedGradient] = []
        self._walk_for_colors(walk_root, [], set(), gradients=grad_list, bg_hex=bg_hex)
        gradients = grad_list
    else:
        colors, stroke_colors, gradients = self._parse_colors(
            scoped_file_data, styles_data, bg_hex=bg_hex
        )

    typography = self._parse_typography(scoped_file_data, styles_data)
    spacing = self._parse_spacing(scoped_file_data)
```

This works because:
- `_parse_colors()` reads `file_data.get("document", {})` as walk root and `file_data.get("styles", {})` for published styles
- `_find_fills_for_style()` / `_find_type_style_for_style()` read `file_data.get("document", {})` — scoped automatically
- `_walk_for_colors()` / `_walk_for_typography()` / `_walk_for_spacing()` take a `node` param — already scoped via `walk_root`

**Key insight:** Published styles in `file_data["styles"]` are kept unfiltered (they're file-level metadata with names), but the tree walk that resolves style references (`_walk_for_style_property`) only searches within `walk_root`. So a style defined in Frame B won't be found when walking Frame A's subtree — it naturally falls through to Phase 2 (node walk), which is also scoped.

## Preflight Warnings

- `_parse_colors()` receives `file_data` dict — when passing `scoped_file_data`, the `"styles"` key must be preserved for Phase 1 published style matching to work (it reads `file_data.get("styles", {})`).
- `_walk_for_colors()` at line 475 is called with `file_data.get("document", {})` directly (not through `_parse_colors`) — must use `walk_root` there too.
- Variables path (lines 454-466) fetches from a separate API endpoint — NOT affected by document tree scoping. No changes needed.
- `_walk_for_style_property()` is called by `_find_fills_for_style()` / `_find_type_style_for_style()` which pass `file_data.get("document", {})` — automatically scoped when `scoped_file_data` is used.

- `_find_subtree()` is pure recursive lookup on parsed JSON — no injection vector

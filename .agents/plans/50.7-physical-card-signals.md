# Plan: 50.7 Physical-Card Identity-Exception Signals (Rule 9 Prep)

## Context

Phase 52.7 will ship Rule 9 (dark-mode contrast on nested cards): every nested coloured surface gets its own `class="card-bg-N"` + matching dark-mode override. But Rule 9 has an **identity exception**:

> *"When a card visually represents a physical identity object — a membership card with logo + barcode, a credit-card-style coupon, a boarding-pass mock-up — the design intent is 'this looks like a real plastic card, **always**'. Flip the surrounding bg, but leave the card itself white (no `card-bg` class)."*

Without this exception, the LEGO Insiders membership card (footer #16) would flip dark in dark mode — wrong by design intent.

Per the doc, telltale FRAME-tree signals (≥2 must fire to opt-out):
1. Aspect ratio matches a physical card — 1.586:1 (ID-1) / 2:1 (loyalty card) / 4:3 (boarding pass)
2. Contains a barcode, QR code, or strip-of-numbers asset
3. Contains a logo asset on a perfectly white field
4. Has its own `cornerRadius` distinct from surrounding sections (typical 16–24 px)

This plan ships the detection so Rule 9 (52.7) can read `EmailSection.is_physical_card_surface` and skip the auto-flip. Detection is FRAME-tree only — no PNG.

**Heuristic detection (no AI, no PNG):**
- **Aspect ratio**: card frame's `width / height` matches one of three ratios within ±5% tolerance.
- **Barcode strip**: child image with `width / height ≥ 4:1` (typical barcodes are ~1:5 to 1:8).
- **QR code**: child image with `width ≈ height` (1:1 ratio ±10%) AND ≤ 200px square.
- **Logo on white field**: child image's parent FRAME has `fills[0]=#ffffff` AND image fills less than 60% of parent.
- **Distinct cornerRadius**: card frame's `cornerRadius` ≥ 16 AND differs from immediate sibling cards.

### New: `app/design_sync/figma/physical_card_detector.py` (~180 lines)

```python
"""Detect physical-card-surface FRAMEs (Rule 9 identity exception, Phase 50.7).

A physical card is a FRAME that represents a real plastic card / membership card /
boarding pass / loyalty card. Identified by ≥2 of 4 signals:
1. Aspect ratio matches ID-1 (1.586:1), loyalty (2:1), or boarding-pass (4:3)
2. Contains a barcode strip (width:height ≥ 4:1)
3. Contains a logo on a perfectly white field (parent FRAME #ffffff, image < 60%)
4. Has its own cornerRadius ≥ 16, distinct from sibling sections

Pure FRAME-tree read — no I/O, no PNG.
"""

from __future__ import annotations
from dataclasses import dataclass

from app.design_sync.protocol import DesignNode

# Aspect ratio targets (with ±5% tolerance)
_ID_1_RATIO = 1.586  # Credit card / driver's license
_LOYALTY_RATIO = 2.0  # Many loyalty cards
_BOARDING_PASS_RATIO = 4 / 3
_RATIO_TOLERANCE = 0.05  # ±5%

# Barcode strip aspect threshold
_BARCODE_MIN_RATIO = 4.0  # width / height ≥ 4:1

# QR code aspect threshold
_QR_RATIO_TOLERANCE = 0.1  # ±10% of square
_QR_MAX_SIZE_PX = 200

# Logo-on-white threshold
_LOGO_MAX_FILL_RATIO = 0.6  # image fills ≤ 60% of parent

# Distinct corner radius threshold
_CARD_MIN_RADIUS = 16

# Number of signals required to opt OUT of dark-mode flip
_MIN_SIGNALS = 2


@dataclass(frozen=True)
class PhysicalCardDetection:
    is_physical: bool
    signals: tuple[str, ...]


def detect_physical_card_surface(
    card_frame: DesignNode,
    *,
    sibling_radii: list[float] | None = None,
) -> PhysicalCardDetection:
    """Return whether this FRAME represents a physical card surface."""
    signals: list[str] = []

    # Signal 1: aspect ratio
    bb = card_frame.absolute_bounding_box
    if bb and bb.width > 0 and bb.height > 0:
        ratio = bb.width / bb.height
        if _is_within(ratio, _ID_1_RATIO, _RATIO_TOLERANCE):
            signals.append("aspect_ratio_id_1")
        elif _is_within(ratio, _LOYALTY_RATIO, _RATIO_TOLERANCE):
            signals.append("aspect_ratio_loyalty")
        elif _is_within(ratio, _BOARDING_PASS_RATIO, _RATIO_TOLERANCE):
            signals.append("aspect_ratio_boarding_pass")

    # Signal 2: barcode child
    if _has_barcode_child(card_frame):
        signals.append("barcode_child")

    # Signal 3: logo on white field
    if _has_logo_on_white_field(card_frame):
        signals.append("logo_on_white_field")

    # Signal 4: distinct cornerRadius
    if card_frame.corner_radius and card_frame.corner_radius >= _CARD_MIN_RADIUS:
        if sibling_radii is None or not _is_in_sibling_set(card_frame.corner_radius, sibling_radii):
            signals.append("distinct_corner_radius")

    is_physical = len(signals) >= _MIN_SIGNALS
    return PhysicalCardDetection(is_physical=is_physical, signals=tuple(signals))


def _is_within(value: float, target: float, tolerance: float) -> bool:
    return abs(value - target) / target <= tolerance


def _has_barcode_child(card: DesignNode) -> bool:
    for child in _walk_image_descendants(card):
        bb = child.absolute_bounding_box
        if not bb or bb.height == 0:
            continue
        if (bb.width / bb.height) >= _BARCODE_MIN_RATIO:
            return True
    return False


def _has_logo_on_white_field(card: DesignNode) -> bool:
    """A logo is an image inside a parent FRAME with #ffffff fill, image area < 60%."""
    for child in card.children:
        if not _has_white_fill(child):
            continue
        for grandchild in child.children:
            if not _is_image_node(grandchild):
                continue
            parent_area = _area(child)
            child_area = _area(grandchild)
            if parent_area > 0 and (child_area / parent_area) <= _LOGO_MAX_FILL_RATIO:
                return True
    return False


# (helpers _walk_image_descendants, _has_white_fill, _is_image_node, _area, _is_in_sibling_set
#  follow the same FRAME-tree predicate pattern — implemented per existing layout_analyzer
#  helpers like _resolve_fill_hex etc.)
```

## Dependencies & Blockers

- **Depends on:** 50.4 (only fires on sections with `inner_bg`)
- **Blocks:** 52.7 (Rule 9 dark-mode flip reads `is_physical_card_surface` to opt out)
- **Risk:** False positives — a non-card section with 2 coincidental signals fires. Mitigation: tune `_MIN_SIGNALS=2` based on real designs in Phase 53.6 reference set; consider raising to 3 if false positives appear

## Out of Scope

- Image-content recognition (does this PNG actually contain a barcode?) — defer to Phase 52.9 (Rule 5: asset name vs content) which uses PNG perceptual hash
- Multi-card sections (e.g. credit-card-row component) — current detection is per-section
- Auto dark-mode override of physical cards (Rule 9 itself) — Phase 52.7

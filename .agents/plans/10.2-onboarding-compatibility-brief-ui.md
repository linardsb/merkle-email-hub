# Plan: 10.2 Onboarding Compatibility Brief UI

## Context

Phase 9.5 built `app/projects/onboarding.py` which generates rich compatibility briefs (executive summary, per-client CSS profiles, cross-client risk matrix) from the email ontology. Phase 10.1 added the `target_clients` selector. Now we need a frontend UI to display this compatibility data so developers can reference "what CSS is safe across my target clients?" while building.

**Key architecture insight**: The backend POST `/projects/{id}/onboarding-brief` only triggers async Cognee graph storage (202 Accepted). There is no GET endpoint returning structured JSON. The plan adds a **GET endpoint** that returns structured JSON by calling the ontology directly (always in-memory, no Cognee dependency).

## Backend Addition (Required)

A new GET endpoint is needed since the existing POST only fires off async Cognee storage.

### New GET endpoint in `app/projects/routes.py`

```python
@router.get("/projects/{project_id}/compatibility-brief")
@limiter.limit("30/minute")
async def get_compatibility_brief(
    request: Request,
    project_id: int,
    service: ProjectService = Depends(get_service),
    current_user: User = Depends(get_current_user),
) -> CompatibilityBriefResponse:
    """Return structured compatibility brief for project's target clients."""
    project = await service.verify_project_access(project_id, current_user)
    if not project.target_clients:
        raise AppError("No target clients configured", status_code=400)
    brief = generate_compatibility_brief(project.target_clients)
    if not brief:
        raise AppError("No valid clients found", status_code=404)
    return CompatibilityBriefResponse(...)
```

## Implementation Steps

### Step 1: Backend — Structured Brief Generator

Create `app/projects/compatibility_brief.py` with `generate_compatibility_brief(client_ids) -> CompatibilityBrief | None`. Logic mirrors `onboarding.py` but returns dataclasses instead of markdown:

1. Load ontology via `load_ontology()`
2. Resolve clients (skip unknown, log warnings)
3. For each client: gather unsupported properties with fallbacks → `ClientProfile`
4. Build risk matrix: properties unsupported by 2+ clients → `RiskMatrixEntry`
5. Detect dark mode warning (Word engine in targets)
6. Return `CompatibilityBrief` or `None` if no valid clients

### Step 2: Backend — Schemas + Endpoint

Add Pydantic schemas to `app/projects/schemas.py` (as above).

Add GET `/projects/{project_id}/compatibility-brief` to `app/projects/routes.py`:
- Auth required (all roles can read — viewer included)
- Rate limited 30/min
- Calls `verify_project_access()` for BOLA protection
- Returns 400 if no target_clients, 404 if no valid clients

```typescript
import useSWR from "swr";
import { fetcher } from "@/lib/swr-fetcher";
import type { CompatibilityBriefResponse } from "@merkle-email-hub/sdk";

export function useCompatibilityBrief(projectId: number | null) {
  return useSWR<CompatibilityBriefResponse>(
    projectId ? `/api/v1/projects/${projectId}/compatibility-brief` : null,
    fetcher,
    { revalidateOnFocus: false, dedupingInterval: 600_000 }
  );
}
```

Also add `useRegenerateBrief(projectId)` mutation hook that POSTs to the existing regenerate endpoint:

```typescript
export function useRegenerateBrief(projectId: number) {
  const [isRegenerating, setIsRegenerating] = useState(false);
  const regenerate = useCallback(async () => {
    setIsRegenerating(true);
    try {
      await authFetch(`/api/v1/projects/${projectId}/onboarding-brief`, {
        method: "POST",
      });
    } finally {
      setIsRegenerating(false);
    }
  }, [projectId]);
  return { regenerate, isRegenerating };
}
```

**Key implementation details:**
- Uses `useCompatibilityBrief(projectId)` for data
- Loading state with skeleton (not spinner)
- Empty state if `targetClients` is null/empty: "Configure target clients first"
- Engine badges use semantic colors:
  - WebKit → `bg-accent text-accent-foreground`
  - Blink → `bg-accent text-accent-foreground`
  - Word → `bg-destructive/10 text-destructive` (highest risk)
  - Gecko → `bg-accent text-accent-foreground`
  - Custom → `bg-muted text-muted-foreground`
- Dark mode warning: `bg-destructive/10 border-destructive/20 text-destructive` alert
- Accordion from shadcn/ui (`@merkle-email-hub/ui/components/accordion`)
- Table uses standard HTML table with Tailwind styling
- "Regenerate Brief" calls POST endpoint, shows toast, revalidates SWR

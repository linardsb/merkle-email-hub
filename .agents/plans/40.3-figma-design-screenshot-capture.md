# Plan: 40.3 — Figma Design Screenshot Capture

## Context

The manual workflow of screenshotting Figma, pasting into chat, and eyeballing differences doesn't scale. An automated pixel-level comparison needs both images (design source + rendered HTML) in a standardized format. The Figma Images API renders exact frames at configurable scale, producing pixel-perfect references without browser artifacts.

Three hand-built reference HTMLs already exist with known Figma node IDs — these are the highest-value visual regression cases.

### Existing Figma Images API Pattern (`service.py:1460-1560`)

The `FigmaDesignSyncService` already implements the full Figma Images API flow:
- `export_images(file_ref, access_token, node_ids, format="png", scale=2.0)` → `list[ExportedImage]`
- Endpoint: `GET /v1/images/{file_ref}` with `ids`, `format`, `scale` params
- Auth: `{"X-Figma-Token": access_token}` header
- Error handling: 403 → `SyncFailedError`, 429 → rate limit, non-200 → generic error
- `download_image_bytes(exported)` → raw PNG bytes via CDN URL

**Key insight:** We can reuse `FigmaDesignSyncService.export_images()` + `download_image_bytes()` directly in `extract.py` instead of duplicating the API call logic.

### Step 4: Implement `_capture_design_image()` helper

New async function in `extract.py`, placed after `_fetch_figma_json()` (~line 91):

```python
async def _capture_design_image(
    file_key: str,
    token: str,
    node_id: str,
    output_dir: Path,
) -> tuple[Path | None, int | None, int | None]:
    """Capture a PNG screenshot of the Figma design node.

    Returns (image_path, width, height) or (None, None, None) on failure.
    """
    from app.design_sync.figma.service import FigmaDesignSyncService

    figma_node_id = node_id.replace("-", ":")
    service = FigmaDesignSyncService()
    scale = _get_scale()

    try:
        exported = await service.export_images(
            file_key, token, [figma_node_id], format="png", scale=scale,
        )
        if not exported:
            logger.warning("diagnose.image_capture_empty", node_id=node_id)
            return None, None, None

        image_bytes = await service.download_image_bytes(exported[0])
    except Exception:
        logger.warning("diagnose.image_capture_failed", node_id=node_id, exc_info=True)
        return None, None, None

    image_path = output_dir / "design.png"
    image_path.write_bytes(image_bytes)

    # Read dimensions
    width, height = _read_png_dimensions(image_bytes)
    logger.info(
        "diagnose.image_captured",
        path=str(image_path),
        size_bytes=len(image_bytes),
        width=width,
        height=height,
    )
    return image_path, width, height
```

### Step 5: PNG dimension reader (no PIL dependency)

Read width/height from PNG header (bytes 16-24) to avoid a PIL import in the extract script:

```python
def _read_png_dimensions(data: bytes) -> tuple[int | None, int | None]:
    """Read width and height from PNG IHDR chunk (no PIL needed)."""
    import struct
    if len(data) < 24 or data[:8] != b"\x89PNG\r\n\x1a\n":
        return None, None
    width, height = struct.unpack(">II", data[16:24])
    return width, height
```

# Plan: 43.6 End-to-End Validation — Re-Judge with Corrections A/B

## Context

Final validation step in Phase 43 (Judge Feedback Loop). Re-runs the full judge pipeline with corrections enabled (43.1–43.2) and compares TPR/TNR against the 37.5 baseline. A second pass without corrections (`--no-corrections`) provides A/B comparison. This proves the feedback loop actually improves accuracy and establishes the first calibration baseline for the regression gate (43.5).

## Implementation Steps

### Step 1: Create `correction_impact.py`

New module: `app/ai/agents/evals/correction_impact.py`

**Functions:**

```
load_calibration_pair(
    with_dir: Path,       # traces dir from run WITH corrections
    without_dir: Path,    # traces dir from run WITHOUT corrections
) -> tuple[dict[str, dict[str, float]], dict[str, dict[str, float]]]
```
- Reuses `calibration_tracker.load_calibration_reports()` + `extract_metrics()` for both dirs.

```
classify_impact(
    tpr_delta: float,
    tnr_delta: float,
    regression_threshold: float = 0.03,
) -> str
```
- Returns `"improved"` if either TPR or TNR increased >0 and neither regressed >threshold.
- Returns `"degraded"` if either TPR or TNR dropped >threshold.
- Returns `"no_change"` otherwise.

```
build_impact_report(
    with_metrics: dict[str, dict[str, float]],
    without_metrics: dict[str, dict[str, float]],
    regression_threshold: float = 0.03,
) -> dict[str, Any]
```
- Iterates all criteria keys (union of both).
- Per criterion: `tpr_without`, `tpr_with`, `tpr_delta`, `tnr_without`, `tnr_with`, `tnr_delta`, `verdict`.
- Summary: `criteria_improved`, `criteria_degraded`, `criteria_unchanged`, `overall_recommendation` (`"accept"` if 0 degraded, `"review"` if any degraded).

```
main() -> None  # CLI entry point
```
- argparse: `--with-corrections DIR`, `--without-corrections DIR`, `--output PATH`, `--threshold FLOAT`
- Loads both calibration sets, builds report, writes JSON, prints summary, exits 1 if any degraded.

### Step 4: Manual Execution Sequence

This is the actual validation workflow (run by user, requires LLM):

1. `make eval-corrections` — generate correction YAMLs from 37.5 labels
2. `make eval-rejudge` — re-judge WITH corrections (default)
3. `make eval-calibrate` — calibrate against 37.5 human labels
4. `make eval-judge-no-corrections` — re-judge WITHOUT corrections
5. `make eval-calibrate-no-corrections` — calibrate no-corrections verdicts
6. `make eval-correction-impact` — generate impact report
7. Review `traces/correction_impact_report.json`
8. If any criterion degraded >3pp: adjust correction YAML, re-run steps 2-6
9. `make eval-calibration-gate --update-baseline` — save final baseline

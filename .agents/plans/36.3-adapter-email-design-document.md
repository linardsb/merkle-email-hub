# Plan: 36.3 ŌĆö Refactor Figma + Penpot Adapters to Produce EmailDesignDocument

## Context

Currently, adapters (`FigmaDesignSyncService`, `PenpotDesignSyncService`) return raw `(ExtractedTokens, DesignFileStructure)` tuples. Layout analysis happens downstream in `EmailDesignDocument.from_legacy()` ŌĆö called by both `converter_service.convert()` and `service.sync_connection()`. This means provider-specific concerns (node grouping, section detection) leak into the converter layer.

**Goal:** Each adapter gains a `build_document()` method that returns `EmailDesignDocument` directly, encapsulating tree normalization, layout analysis, and token validation. The converter receives pre-analyzed sections. Stub providers (Sketch, Canva) use the legacy path unchanged.

## Implementation Steps

### Step 1: Add `build_document()` to FigmaDesignSyncService

In `app/design_sync/figma/service.py`, add after `sync_tokens_and_structure()` (~line 310):

```python
async def build_document(
    self,
    file_ref: str,
    access_token: str,
    *,
    selected_nodes: list[str] | None = None,
    connection_config: dict[str, Any] | None = None,
    target_clients: list[str] | None = None,
) -> EmailDesignDocument:
    """Build a complete EmailDesignDocument from a Figma file.

    Encapsulates: API fetch ŌåÆ token extraction ŌåÆ tree normalization ŌåÆ
    layout analysis ŌåÆ document assembly.  The converter receives
    pre-analyzed sections.
    """
    from app.design_sync.email_design_document import EmailDesignDocument
    from app.design_sync.figma.tree_normalizer import normalize_tree
    from app.design_sync.token_transforms import validate_and_transform

    # 1. Fetch tokens + structure (single Figma API call)
    tokens, structure = await self.sync_tokens_and_structure(file_ref, access_token)

    # 2. Validate and transform tokens
    tokens, token_warnings = validate_and_transform(
        tokens, target_clients=target_clients
    )

    # 3. Normalize tree
    structure, _stats = normalize_tree(structure)

    # 4. Build document (from_legacy handles layout analysis + width derivation)
    document = EmailDesignDocument.from_legacy(
        structure,
        tokens,
        selected_nodes=selected_nodes,
        connection_config=connection_config,
        source_provider="figma",
        _pre_normalized=True,
    )

    return document, tokens, token_warnings
```

**Return type note:** The method returns `tuple[EmailDesignDocument, ExtractedTokens, list[TokenWarning]]` because `sync_connection` needs all three (tokens for the legacy `tokens_json` column, warnings for compatibility hints). Use a named tuple or just return the triple ŌĆö callers destructure it.

Refined signature:

```python
from app.design_sync.token_transforms import TokenWarning

async def build_document(
    self,
    file_ref: str,
    access_token: str,
    *,
    selected_nodes: list[str] | None = None,
    connection_config: dict[str, Any] | None = None,
    target_clients: list[str] | None = None,
) -> tuple[EmailDesignDocument, ExtractedTokens, list[TokenWarning]]:
```

### Step 2: Add `build_document()` to PenpotDesignSyncService

In `app/design_sync/penpot/service.py`, add after `sync_tokens_and_structure()` (~line 135):

Same pattern as Figma. Only difference: pass `raw_file_data` to `normalize_tree()` and `from_legacy()` since Penpot structure may need it, and `source_provider="penpot"`.

```python
async def build_document(
    self,
    file_ref: str,
    access_token: str,
    *,
    selected_nodes: list[str] | None = None,
    connection_config: dict[str, Any] | None = None,
    target_clients: list[str] | None = None,
) -> tuple[EmailDesignDocument, ExtractedTokens, list[TokenWarning]]:
    from app.design_sync.email_design_document import EmailDesignDocument
    from app.design_sync.figma.tree_normalizer import normalize_tree
    from app.design_sync.token_transforms import validate_and_transform

    tokens, structure = await self.sync_tokens_and_structure(file_ref, access_token)
    tokens, token_warnings = validate_and_transform(
        tokens, target_clients=target_clients
    )
    structure, _stats = normalize_tree(structure)
    document = EmailDesignDocument.from_legacy(
        structure,
        tokens,
        selected_nodes=selected_nodes,
        connection_config=connection_config,
        source_provider="penpot",
        _pre_normalized=True,
    )
    return document, tokens, token_warnings
```

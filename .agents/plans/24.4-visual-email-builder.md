# Plan: Phase 24.4 — Visual Email Builder (Component Palette & Canvas)

## Context
70% of email creation at agencies is done by non-developers using drag-and-drop tools. The Hub currently requires HTML knowledge. A visual builder using the existing component library means every component built for the code editor is automatically available visually — no duplicate work. This is the single biggest user-base expansion feature.

**Key discovery:** EditorPanel already has a `"code" | "visual"` tab bar — the "Visual" tab currently renders `LiquidBuilderPanel` (Liquid block builder). Phase 24.4 replaces this with a proper email section builder that uses email components from the component library.

**Existing infrastructure:**
- `@dnd-kit/core`, `@dnd-kit/sortable`, `@dnd-kit/utilities` already installed
- `dompurify` already installed
- `LiquidBuilderPanel` pattern: palette (left sidebar w-36) + canvas (dnd-kit sortable) + preview (bottom third) — same layout pattern we'll follow
- `PreviewIframe` uses `srcdoc` + `sandbox=""` + zoom/viewport support — reuse this pattern
- `useComponents()` hook fetches from `GET /api/v1/components/?page=&page_size=&category=&search=`
- `ComponentResponse` has: `id`, `name`, `slug`, `description`, `category`, `latest_version`, `compatibility_badge`
- `VersionResponse` has: `html_source`, `css_source`, `slot_definitions`, `default_tokens`
- `useComponentVersions(componentId)` fetches versions for a component

### Step 2: Builder Hooks (`cms/apps/web/src/hooks/use-builder.ts`)

```typescript
"use client";

import { useReducer, useCallback, useMemo, useRef } from "react";
import DOMPurify from "dompurify";
import type {
  BuilderSection,
  BuilderState,
  BuilderAction,
  HistoryEntry,
} from "@/types/visual-builder";

const MAX_HISTORY = 50;

function pushHistory(state: BuilderState): HistoryEntry[] {
  const newHistory = state.history.slice(0, state.historyIndex + 1);
  newHistory.push({ sections: structuredClone(state.sections), timestamp: Date.now() });
  if (newHistory.length > MAX_HISTORY) newHistory.shift();
  return newHistory;
}

function builderReducer(state: BuilderState, action: BuilderAction): BuilderState {
  switch (action.type) {
    case "ADD_SECTION": {
      const history = pushHistory(state);
      const sections = [...state.sections];
      const idx = action.atIndex ?? sections.length;
      sections.splice(idx, 0, action.section);
      return { ...state, sections, history, historyIndex: history.length - 1 };
    }
    case "REMOVE_SECTION": {
      const history = pushHistory(state);
      const sections = state.sections.filter((s) => s.id !== action.sectionId);
      const selectedSectionId =
        state.selectedSectionId === action.sectionId ? null : state.selectedSectionId;
      return { ...state, sections, selectedSectionId, history, historyIndex: history.length - 1 };
    }
    case "DUPLICATE_SECTION": {
      const history = pushHistory(state);
      const idx = state.sections.findIndex((s) => s.id === action.sectionId);
      if (idx === -1) return state;
      const original = state.sections[idx];
      const duplicate: BuilderSection = {
        ...structuredClone(original),
        id: crypto.randomUUID(),
      };
      const sections = [...state.sections];
      sections.splice(idx + 1, 0, duplicate);
      return { ...state, sections, history, historyIndex: history.length - 1 };
    }
    case "MOVE_SECTION": {
      const history = pushHistory(state);
      const sections = [...state.sections];
      const [moved] = sections.splice(action.fromIndex, 1);
      sections.splice(action.toIndex, 0, moved);
      return { ...state, sections, history, historyIndex: history.length - 1 };
    }
    case "UPDATE_SECTION": {
      const history = pushHistory(state);
      const sections = state.sections.map((s) =>
        s.id === action.sectionId ? { ...s, ...action.updates } : s
      );
      return { ...state, sections, history, historyIndex: history.length - 1 };
    }
    case "SELECT_SECTION":
      return { ...state, selectedSectionId: action.sectionId };
    case "SET_SECTIONS": {
      const history = pushHistory(state);
      return { ...state, sections: action.sections, history, historyIndex: history.length - 1 };
    }
    case "UNDO": {
      if (state.historyIndex <= 0) return state;
      const newIndex = state.historyIndex - 1;
      return {
        ...state,
        sections: structuredClone(state.history[newIndex].sections),
        historyIndex: newIndex,
      };
    }
    case "REDO": {
      if (state.historyIndex >= state.history.length - 1) return state;
      const newIndex = state.historyIndex + 1;
      return {
        ...state,
        sections: structuredClone(state.history[newIndex].sections),
        historyIndex: newIndex,
      };
    }
    default:
      return state;
  }
}

const INITIAL_STATE: BuilderState = {
  sections: [],
  selectedSectionId: null,
  history: [{ sections: [], timestamp: Date.now() }],
  historyIndex: 0,
};

export function useBuilderState() {
  const [state, dispatch] = useReducer(builderReducer, INITIAL_STATE);

  const addSection = useCallback(
    (section: BuilderSection, atIndex?: number) =>
      dispatch({ type: "ADD_SECTION", section, atIndex }),
    []
  );
  const removeSection = useCallback(
    (sectionId: string) => dispatch({ type: "REMOVE_SECTION", sectionId }),
    []
  );
  const duplicateSection = useCallback(
    (sectionId: string) => dispatch({ type: "DUPLICATE_SECTION", sectionId }),
    []
  );
  const moveSection = useCallback(
    (fromIndex: number, toIndex: number) =>
      dispatch({ type: "MOVE_SECTION", fromIndex, toIndex }),
    []
  );
  const updateSection = useCallback(
    (sectionId: string, updates: Partial<BuilderSection>) =>
      dispatch({ type: "UPDATE_SECTION", sectionId, updates }),
    []
  );
  const selectSection = useCallback(
    (sectionId: string | null) => dispatch({ type: "SELECT_SECTION", sectionId }),
    []
  );
  const undo = useCallback(() => dispatch({ type: "UNDO" }), []);
  const redo = useCallback(() => dispatch({ type: "REDO" }), []);

  const canUndo = state.historyIndex > 0;
  const canRedo = state.historyIndex < state.history.length - 1;

  return {
    ...state,
    addSection,
    removeSection,
    duplicateSection,
    moveSection,
    updateSection,
    selectSection,
    undo,
    redo,
    canUndo,
    canRedo,
  };
}

/**
 * Assembles sections into a complete email HTML document.
 * Debounced — returns compiled HTML after 200ms of no changes.
 */
export function useBuilderPreview(sections: BuilderSection[]) {
  const timerRef = useRef<ReturnType<typeof setTimeout>>();
  const [html, setHtml] = useState<string | null>(null);

  // Need to import useState
  // Actually, use useMemo with a debounced effect pattern
  // For simplicity, assemble synchronously — debouncing is in the preview component
  const assembled = useMemo(() => {
    if (sections.length === 0) return null;

    const sectionHtmlParts = sections.map((s) => {
      const sectionCss = s.css ? `<style>${s.css}</style>` : "";
      return `<!-- section: ${DOMPurify.sanitize(s.componentName)} -->\n${sectionCss}\n${s.html}`;
    });

    return `<!DOCTYPE html>
<html>
<head>
<meta charset="utf-8">
<meta name="viewport" content="width=device-width, initial-scale=1.0">
<style>
  body { margin: 0; padding: 0; font-family: Arial, Helvetica, sans-serif; }
  table { border-collapse: collapse; }
  img { border: 0; display: block; }
</style>
</head>
<body>
<center>
<table role="presentation" width="100%" cellpadding="0" cellspacing="0" style="max-width:600px;margin:0 auto;">
${sectionHtmlParts.join("\n")}
</table>
</center>
</body>
</html>`;
  }, [sections]);

  return assembled;
}
```

**Note:** The hook file needs `useState` imported too. The actual implementation should use `useMemo` for synchronous assembly (the debouncing happens in the preview iframe via `useDeferredValue` or a timer in the component).

### Step 13: CLAUDE.md for builder package

Create `cms/apps/web/src/components/builder/CLAUDE.md` with brief description of the builder architecture, component relationships, and the one-way data flow (palette → canvas → HTML output).

## Key Design Decisions

1. **One-way data flow:** Visual builder → HTML output only. We do NOT parse HTML back into sections. If the user edits code in the Code tab, switching to Builder shows a warning. This avoids the complexity of HTML→section parsing.

2. **Component version fetching:** When a component is dropped, we lazy-fetch its latest version's `html_source` via `useComponentVersions()`. Results are cached in a `Map` ref for the session.

3. **No new dependencies:** `@dnd-kit/*` and `dompurify` are already installed. No new packages needed.

4. **Reuse PreviewIframe pattern:** The builder preview uses the same `srcdoc` + `sandbox=""` + zoom pattern as the existing `PreviewIframe` component.

5. **LiquidBuilderPanel preserved:** Not deleting the liquid builder — just swapping the dynamic import in EditorPanel. The liquid builder files remain available.

6. **Tab rename:** "Visual" → "Builder" to differentiate from the Liquid block builder and better describe the drag-and-drop experience.

7. **Inline editing deferred to 24.5:** Double-click contenteditable editing and the property panel are Phase 24.5. This phase focuses on palette, canvas, DnD, preview, undo/redo.

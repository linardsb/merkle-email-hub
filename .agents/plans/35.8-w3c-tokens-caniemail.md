# Plan: Phase 35.8 — W3C Design Tokens & caniemail.com Integration

## Context

W3C Design Tokens v1.0 reached stable Oct 2025; Figma ships native W3C import/export Nov 2026. Supporting W3C now future-proofs for Figma, Tokens Studio, Penpot, and Sketch plugins. Separately, `compatibility.py` uses our ontology YAML which requires manual updates — caniemail.com tracks 303+ HTML/CSS features with community-maintained data.

## Implementation Steps

### Step 1: W3C Design Tokens parser — `app/design_sync/w3c_tokens.py`

```python
from __future__ import annotations
from dataclasses import dataclass
from typing import Any
from app.core.logging import get_logger
from app.design_sync.protocol import (
    ExtractedColor, ExtractedGradient, ExtractedSpacing,
    ExtractedTokens, ExtractedTypography,
)
from app.design_sync.token_transforms import TokenWarning

logger = get_logger(__name__)

_MAX_NESTING_DEPTH = 20
_MAX_ALIAS_DEPTH = 10
_MAX_FILE_SIZE = 1_048_576  # 1 MB

# W3C type → handler mapping
_W3C_COLOR_TYPES = frozenset({"color"})
_W3C_DIMENSION_TYPES = frozenset({"dimension"})
_W3C_FONT_TYPES = frozenset({"fontFamily", "fontWeight", "fontSize"})
_W3C_GRADIENT_TYPES = frozenset({"gradient"})
_W3C_IGNORED_TYPES = frozenset({"duration", "cubicBezier", "strokeStyle", "transition"})
_W3C_KNOWN_TYPES = _W3C_COLOR_TYPES | _W3C_DIMENSION_TYPES | _W3C_FONT_TYPES | _W3C_GRADIENT_TYPES | _W3C_IGNORED_TYPES | frozenset({"shadow", "border", "number"})

@dataclass(frozen=True)
class W3cParseResult:
    tokens: ExtractedTokens
    warnings: list[TokenWarning]
```

Key functions:
- `parse_w3c_tokens(tokens_json: dict[str, Any]) -> W3cParseResult` — main entry
- `_validate_w3c_schema(obj: dict[str, Any], depth: int = 0) -> list[str]` — schema validation with depth limit
- `_resolve_alias(ref: str, root: dict[str, Any], visited: frozenset[str] = frozenset()) -> tuple[Any, str | None]` — cycle-safe alias resolution (frozenset for visited, max 10 hops)
- `_walk_tokens(obj: dict[str, Any], path: str, root: dict[str, Any], *, depth: int = 0) -> _CollectedTokens` — recursive walk collecting tokens by type
- `_parse_color(name: str, value: Any) -> ExtractedColor | None` — hex, rgba(), hsl() support
- `_parse_dimension(name: str, value: str) -> ExtractedSpacing | None` — px/rem parsing
- `_parse_gradient(name: str, value: dict[str, Any]) -> ExtractedGradient | None` — W3C gradient composite
- `_collect_dark_mode(obj: dict[str, Any], root: dict[str, Any]) -> list[ExtractedColor]` — `$extensions.mode` detection

Security: reject unknown `$type` (warning, not error). Depth limit 20. Alias chain limit 10. No `eval`/`exec`.

### Step 2: W3C Design Tokens exporter — `app/design_sync/w3c_export.py`

    """Convert ExtractedTokens to W3C Design Tokens v1.0 JSON."""

Logic:
- Group `colors` → `{"color": {"<name>": {"$type": "color", "$value": "#RRGGBB"}}}`. If `opacity < 1.0`, emit `rgba()`.
- Group `typography` → `{"typography": {"<name>": {"family": {"$type": "fontFamily", ...}, "size": {"$type": "fontSize", ...}, ...}}}`.
- Group `spacing` → `{"spacing": {"<name>": {"$type": "dimension", "$value": "<N>px"}}}`.
- Group `gradients` → `{"gradient": {"<name>": {"$type": "gradient", "$value": {...}}}}`.
- `dark_colors` → `$extensions.mode.dark` on corresponding color tokens.
- `_w3c_color_value(hex: str, opacity: float) -> str` — "#RRGGBB" or "rgba(R, G, B, A)".

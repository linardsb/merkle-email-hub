# Plan: 47.7 Extended Component Matcher Scoring

## Context

New component types added in 47.6 (countdown-timer, testimonial, pricing-table, video-placeholder, event-card, faq-accordion, zigzag-layout) need detection signals in the scorer. The existing `_score_candidates()` handles product-grid/image-gallery/category-nav but can't distinguish these new types.

## Implementation Steps

### Step 1: Add `_score_extended_candidates()` after `_score_candidates()` (~line 325)

```python
def _score_extended_candidates(
    section: EmailSection,
    has_images: bool,
    has_texts: bool,
    has_buttons: bool,
    has_headings: bool,
) -> tuple[str, float]:
```

**Detection signals per component type:**

| Component | Signal | Score |
|-----------|--------|-------|
| `countdown-timer` | 3+ short numeric texts matching `\d{1,2}[:\.]?\d{0,2}` pattern OR texts containing time-unit words (hours/min/sec/days), has_headings | 0.92 |
| `testimonial` | Text containing `"` or `"` or `"`, short body (≤200 chars), 1 small image (w≤100, h≤100 — avatar), optionally has_headings for author | 0.9 |
| `pricing-table` | Text containing currency symbols (`$€£¥`), 2+ col_groups with texts, has_buttons | 0.93 |
| `video-placeholder` | 1 image with 16:9±tolerance aspect ratio (1.6–1.9), has_buttons (play CTA), ≤2 texts | 0.88 |
| `event-card` | Date pattern in text (`\d{1,2}[/\-\.]\d{1,2}` or month names), has_images, has_texts | 0.85 |
| `faq-accordion` | 3+ texts where alternating texts have `?` in odd-indexed items (Q&A pairs), no images | 0.88 |
| `zigzag-alternating` | 2+ col_groups where image position alternates (odd groups: img col 0, even groups: img col 1 or vice versa) | 0.9 |

**Algorithm:** Same as `_score_candidates()` — build `candidates: list[tuple[str, float]]`, return highest. If no candidates, return `("text-block", 0.0)` so caller can compare.

### Step 2: Wire into `_match_by_type()` at line 213

Change:
```python
if st == EmailSectionType.CONTENT:
    return _score_candidates(section, has_images, has_texts, has_buttons, has_headings)
```
To:
```python
if st == EmailSectionType.CONTENT:
    slug, confidence = _score_candidates(section, has_images, has_texts, has_buttons, has_headings)
    ext_slug, ext_confidence = _score_extended_candidates(
        section, has_images, has_texts, has_buttons, has_headings,
    )
    if ext_confidence > confidence:
        return ext_slug, ext_confidence
    return slug, confidence
```

## Preflight Warnings

- Existing tests assert exact confidence values (e.g., `== 0.95`). Extended scorer must NOT change scores for existing components.
- The `_match_by_type()` call site returns `_score_candidates()` directly — modifying to two-step compare means both functions run on every CONTENT section. Keep `_score_extended_candidates()` fast (no I/O, just regex + field checks).
- `_all_images_are_icons()` at line 241 already redirects to `navigation-bar` — ensure countdown/testimonial with small images don't collide with icon heuristic (icon check runs BEFORE `_score_candidates`).

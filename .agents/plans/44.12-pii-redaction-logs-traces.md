# Plan: 44.12 PII Redaction in Logs and Eval Traces

## Implementation Steps

### Step 1: Core Redactor (`app/core/redaction.py`)

Reuse patterns from `app/ai/sanitize.py:14-23` but keep module independent (core layer shouldn't import ai layer). Copy the compiled patterns.

```python
"""PII redaction for logs and eval traces."""
import re
from typing import Any

from structlog.typing import EventDict, WrappedLogger

# Compiled at module level — ~0 cost after first import
_PII_PATTERNS: list[tuple[str, re.Pattern[str]]] = [
    ("EMAIL", re.compile(r"[\w.+-]+@[\w-]+\.[\w.-]+", re.IGNORECASE)),
    ("PHONE_INTL", re.compile(r"\+\d{1,3}[\s.-]?\(?\d{1,4}\)?[\s.-]?\d{3,4}[\s.-]?\d{3,4}")),
    ("PHONE", re.compile(r"\b\d{3}[-.]?\d{3}[-.]?\d{4}\b")),
    ("SSN", re.compile(r"\b\d{3}-\d{2}-\d{4}\b")),
    ("CREDIT_CARD", re.compile(r"\b(?:\d{4}[-\s]?){3}\d{4}\b")),
]

# Keys whose string values should always be redacted
_SENSITIVE_KEYS = frozenset({"brief", "html", "password", "secret", "api_key", "token"})

def redact_pii(text: str) -> str:
    """Replace PII patterns with placeholder tokens."""
    for label, pattern in _PII_PATTERNS:
        text = pattern.sub(f"[{label}]", text)
    return text

def redact_value(value: Any, *, key: str = "") -> Any:
    """Recursively redact PII from a value."""
    if isinstance(value, str):
        return redact_pii(value)
    if isinstance(value, dict):
        return {k: redact_value(v, key=k) for k, v in value.items()}
    if isinstance(value, list):
        return [redact_value(item) for item in value]
    return value

def redact_event_dict(
    _logger: WrappedLogger, _method_name: str, event_dict: EventDict
) -> EventDict:
    """Structlog processor that redacts PII from log event dicts."""
    for key, value in event_dict.items():
        if key in ("timestamp", "level", "request_id", "logger"):
            continue  # skip metadata keys
        if isinstance(value, str):
            event_dict[key] = redact_pii(value)
    return event_dict
```

Design decisions:
- Keep processor lightweight — only redact string values at top level of event_dict (not deep recursion, logs are flat)
- `redact_value()` is the recursive variant for trace dicts (deeper structures)
- `_SENSITIVE_KEYS` reserved for future use (full-value redaction vs pattern-based)

```python
from app.core.redaction import redact_event_dict

def setup_logging(log_level: str = "INFO", *, pii_redaction: bool = True) -> None:
    processors = [
        add_request_id,
        structlog.contextvars.merge_contextvars,
        structlog.processors.add_log_level,
        structlog.processors.TimeStamper(fmt="iso"),
        structlog.processors.StackInfoRenderer(),
        structlog.processors.format_exc_info,
    ]
    if pii_redaction:
        processors.append(redact_event_dict)
    processors.append(structlog.processors.JSONRenderer())
    # ... rest unchanged
```

## Preflight Warnings

- `app/ai/sanitize.py` has identical PII patterns — we intentionally duplicate them in `app/core/redaction.py` to maintain layer separation (core cannot import ai). Add a comment referencing the source.
- `runner.py` trace writes happen in 4 places (2 normal + 2 adversarial) — all 4 must be wrapped.
- `setup_logging()` is called at app startup and uses `cache_logger_on_first_use=True` — changing processor chain requires app restart, not dynamic toggle.
- The structlog processor runs on every log call (465+ sites). Keep it fast: string-only top-level check, no recursion.

# Plan: 11.22.8 — Agent Role Redefinition (Tighten Specialisation)

## Context

Current agents overlap and conflict: Scaffolder generates dark mode CSS, then Dark Mode agent overwrites it. Five agents (Dark Mode, Outlook Fixer, Accessibility, Personalisation, Content) still generate raw HTML via `_post_process()` → `extract_html()` → `sanitize_html_xss()`, producing non-deterministic output that drives the 16.7% eval pass rate.

The Scaffolder (11.22.1–11.22.7) already migrated to structured output: `_process_structured()` → 3-pass `ScaffolderPipeline` → `EmailBuildPlan` JSON → `TemplateAssembler` (deterministic). This task migrates the remaining agents to the same pattern: **LLM returns structured decisions; deterministic code assembles the output**.

**Depends on:** 11.22.2 (structured schemas), 11.22.4 (repair pipeline), 11.22.7 (TemplateComposer)
**Parent plan:** `.agents/plans/11.22-deterministic-agent-architecture.md`

## Architecture Change

```
BEFORE (per downstream agent):
  HTML in → LLM rewrites HTML → _post_process → QA

AFTER (per downstream agent):
  EmailBuildPlan in → LLM returns structured decisions JSON → merge into plan → deterministic assembly → QA
```

Downstream agents no longer receive or produce raw HTML. They receive the `EmailBuildPlan` from the Scaffolder and return structured decision schemas that modify specific plan fields. The final `TemplateAssembler.assemble()` call is the single HTML generation point.

## Key Decisions

| # | Decision | Rationale |
|---|----------|-----------|
| 1 | Downstream agents modify `EmailBuildPlan` fields, not HTML | Eliminates HTML corruption (html_preservation 10% → 99%+) |
| 2 | New per-agent decision schemas in `app/ai/agents/schemas/` | Type-safe structured output per agent role |
| 3 | Blueprint nodes updated to pass `EmailBuildPlan` through chain | Single assembly point at end of pipeline |
| 4 | Knowledge & Innovation unchanged | Advisory agents, no HTML in/out |
| 5 | Content agent operates on `SlotFill` fields | Text-only, already doesn't do HTML extraction |
| 6 | Outlook Fixer becomes diagnostic-only | Golden templates handle MSO; agent reports template bugs |

**`app/ai/agents/schemas/dark_mode_decisions.py`:**
```python
"""Structured decisions for the Dark Mode agent."""
from pydantic import BaseModel, Field

class DarkColorOverride(BaseModel):
    """Single dark-mode color override."""
    token_name: str = Field(description="Design token to override, e.g. 'primary_color'")
    light_value: str = Field(description="Original light-mode hex value")
    dark_value: str = Field(description="Recommended dark-mode hex value")
    reasoning: str = Field(description="Why this override (contrast, readability, brand)")

class DarkModeDecisions(BaseModel):
    """Dark Mode agent structured output — color remapping decisions only."""
    color_overrides: list[DarkColorOverride] = Field(default_factory=list)
    background_dark: str = Field(default="#1a1a2e", description="Dark mode body background")
    text_dark: str = Field(default="#e0e0e0", description="Dark mode body text color")
    enable_prefers_color_scheme: bool = Field(default=True)
    confidence: float = Field(ge=0.0, le=1.0)
    reasoning: str = Field(description="Overall dark mode strategy explanation")
```

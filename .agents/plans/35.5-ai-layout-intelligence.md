# Plan: 35.5 — AI Layout Intelligence & Semantic Detection

## Context

The current `layout_analyzer.py` classifies sections via `_SECTION_PATTERNS` regex + positional heuristics. This works for well-named designs (MJML, descriptive) but fails for generic names (`Frame 1`, `Group 42`). Phase 35.5 adds:
1. **AI Layout Classifier** — LLM fallback when heuristic classification yields `UNKNOWN` or low confidence
2. **Semantic Content Detector** — detects content roles (logo, social, unsubscribe, legal) from text/structure, not names
3. **Position Intelligence** — enhanced positional heuristics in existing analyzer (no LLM)

## Research Summary

### Current Classification Pipeline (`layout_analyzer.py:176-276`)

```
analyze_layout() → _classify_section() → one of:
  ├─ _classify_mj_section()    (MJML convention)
  ├─ _classify_by_name()       (regex match on frame name)
  ├─ _classify_by_content()    (content heuristics for GENERIC convention)
  └─ _classify_by_position()   (height/index fallback)
```

Key observations:
- `_classify_by_content()` (line 484) already does content-based inference for GENERIC convention — images, texts, buttons, font sizes
- `_classify_by_position()` (line 530) uses height + index — simple but effective
- `EmailSection` is frozen dataclass (line 94) — no `confidence` or `content_roles` fields yet
- `analyze_layout()` is sync — new AI modules must be async, called separately

### Step 2: Position intelligence in `_classify_by_content()`

`layout_analyzer.py:484-517` — enhance existing heuristic (no LLM, pure logic):

| Signal | Current | Enhanced |
|--------|---------|----------|
| Single large image + text overlay at top | Only checks `not has_texts` | Allow texts if height > 300 |
| Section with `©`/`unsubscribe` text | Not checked | → `FOOTER` regardless of position |
| Section with all social platform URLs | Not checked | → `SOCIAL` |
| Section between HERO and FOOTER | Not checked | Boost `CONTENT` probability |
| Button-only section | Not checked | → `CTA` |

These are zero-cost heuristic improvements that reduce LLM fallback calls.

```python
    reasoning: str

async def classify_sections_batch(

    *,
    node_map: dict[str, DesignNode],

) -> list[SectionClassification]:
```

**Prompt construction** — compact structural summary per section:
- Node tree: types + dimensions + colors (not raw JSON)
- Sibling context: `"This is section 3 of 6. Previous: HERO. Next: UNKNOWN."`
- Text snippets: first 100 chars per text block
- Image count + dimensions
- Total: ~200 tokens input per section

**Caching:**
```python
_classification_cache: dict[str, SectionClassification] = {}

def _section_cache_key(section: EmailSection) -> str:
    data = f"{section.node_name}|{section.width}|{section.height}|{len(section.texts)}|{len(section.images)}"
    for t in section.texts[:3]:
        data += f"|{t.content[:50]}"
    return hashlib.sha256(data.encode()).hexdigest()[:16]
```

**LLM call:**
- `resolve_model("lightweight")` → Haiku
- JSON mode response: `{ "section_type": str, "column_layout": str, "confidence": float, "reasoning": str }`
- Validate `section_type` against `EmailSectionType` enum — reject if invalid
- Batch: one LLM call per unknown section (not batched in single prompt — context isolation)

### Step 4: Create `ai_content_detector.py`

```python
# app/design_sync/ai_content_detector.py
"""Semantic content role detection — heuristic first, LLM fallback."""
```

**Content roles** (Literal type):
```python
ContentRole = Literal[
    "logo", "social_links", "unsubscribe_link", "legal_text",
    "navigation", "preheader", "view_in_browser", "address",
]
```

**Heuristic pass** (no LLM, regex-based):

| Role | Pattern |
|------|---------|
| `unsubscribe_link` | text contains "unsubscribe" (case-insensitive) |
| `legal_text` | text contains "©", "copyright", "all rights reserved" |
| `social_links` | URLs matching `facebook.com\|twitter.com\|x.com\|instagram.com\|linkedin.com\|youtube.com` |
| `logo` | First section, single image with height < 100px |
| `navigation` | 4+ short text blocks (< 30 chars each) |
| `preheader` | First section, single short text (< 200 chars), small font |
| `view_in_browser` | text contains "view in browser", "view online" |
| `address` | text matches address-like patterns (street number + name, city + state/zip) |

**LLM pass** — only for sections where heuristic detected nothing AND section type is UNKNOWN:
- Same lightweight model as classifier
- Input: text content + position + image dimensions
- Output: `list[ContentRole]` (structured output)
- Cache by text content hash

**Integration function:**
```python
@dataclass(frozen=True)
class ContentRoleAnnotation:
    section_node_id: str
    roles: tuple[str, ...]
    source: Literal["heuristic", "llm"]

async def detect_content_roles(
    sections: list[EmailSection],
) -> list[ContentRoleAnnotation]:
```

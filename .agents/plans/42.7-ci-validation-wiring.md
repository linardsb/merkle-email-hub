## Implementation Steps

### Step 1: ESLint Rule — Hardcoded `refreshInterval` Detection

Add `no-restricted-syntax` rule to `cms/apps/web/eslint.config.mjs:52-83` (project-specific rules section):

```javascript
// Enforce POLL.* constants for refreshInterval (Phase 42.7)
"no-restricted-syntax": [
  "error",
  {
    selector: "Property[key.name='refreshInterval'][value.type='Literal'][value.value!=0]",
    message: "Use POLL.* constants from @/lib/swr-constants instead of hardcoded refreshInterval values. See Phase 42.4.",
  },
],
```

This AST selector catches `refreshInterval: <number>` where number != 0, but allows:
- `refreshInterval: POLL.background` (Identifier, not Literal)
- `refreshInterval: interval` (variable reference)
- `refreshInterval: (data) => ...` (ArrowFunctionExpression)
- `refreshInterval: isActive ? ... : ...` (ConditionalExpression — only caught if branches are literals)
- `refreshInterval: 0` (explicitly allowed, means "disabled")

For the ternary case (`isActive ? 5_000 : 30_000`), the `ConditionalExpression` type won't match the `Literal` selector — it passes. But if someone writes `refreshInterval: 5000` directly, it'll be caught.

**Edge case:** The ternary `isActive ? 5_000 : 30_000` in `use-workflows.ts:21` won't be caught by the AST selector. To properly enforce POLL constants there, we migrate it in Step 2.

import { useSmartPolling } from "@/hooks/use-smart-polling";
import { POLL, SWR_PRESETS } from "@/lib/swr-constants";

export function useWorkflows() {
  const interval = useSmartPolling(POLL.background);
  return useSWR<WorkflowListResponse>(BASE, fetcher, {
    refreshInterval: interval,
    ...SWR_PRESETS.polling,

export function useWorkflowStatus(executionId: string | null, isActive = false) {
  const interval = useSmartPolling(isActive ? POLL.frequent : POLL.status);
  return useSWR<WorkflowStatus>(
    executionId ? `${BASE}/${executionId}` : null,
    fetcher,
    { refreshInterval: interval, ...SWR_PRESETS.polling },
  );
}

### Step 4: Makefile `lint-polling` Target

Add after `lint-fe` (line 90):

```makefile
lint-polling: ## Check for hardcoded polling intervals in hooks
	@cd cms && pnpm --filter web lint --no-error-on-unmatched-pattern 2>&1 \
		| grep -q "refreshInterval" \
		&& { echo "FAIL: Hardcoded refreshInterval found (use POLL.* constants)"; exit 1; } \
		|| echo "OK: All polling intervals use POLL constants"
```

**Alternative (simpler, grep-based):**

```makefile
lint-polling: ## Check for hardcoded polling intervals in hooks
	@! grep -rn 'refreshInterval:\s*[0-9]' cms/apps/web/src/hooks/ \
		--include='*.ts' --include='*.tsx' \
		--exclude-dir='__tests__' \
		| grep -v '// polling-ok' \
		&& echo "OK: All polling intervals use POLL constants" \
		|| { echo "FAIL: Hardcoded refreshInterval found"; exit 1; }
```

**Decision:** Use the ESLint rule as primary enforcement (catches at lint time), add the `lint-polling` Makefile target as a secondary grep-based check that also catches test file drift. The ESLint rule already excludes test files via the test relaxation section.

Actually, simplest approach: rely solely on the ESLint `no-restricted-syntax` rule (Step 1). It runs during `pnpm --filter web lint` which is already in `check-fe`. Add a dedicated `lint-polling` Makefile target that's a grep-based fallback for files ESLint doesn't cover:

### Step 5: Backend ETag CI Test

Create `app/core/tests/test_etag_ci.py` — validates ETag middleware is active on the real app:

```python
"""CI gate: ETag middleware is registered and functional."""

from __future__ import annotations

from fastapi.testclient import TestClient

from app.main import app


class TestETagMiddlewareCI:
    """Verify ETag middleware is wired into the production app."""

    def setup_method(self) -> None:
        self.client = TestClient(app, raise_server_exceptions=False)

    def test_etag_header_on_health(self) -> None:
        """Health endpoint returns ETag header (middleware active)."""
        resp = self.client.get("/api/v1/health")
        assert resp.status_code == 200
        assert "ETag" in resp.headers, "ETag middleware not registered in app"

    def test_304_on_health_etag_match(self) -> None:
        """Health endpoint returns 304 when ETag matches."""
        resp1 = self.client.get("/api/v1/health")
        etag = resp1.headers["ETag"]
        resp2 = self.client.get(
            "/api/v1/health", headers={"If-None-Match": etag}
        )
        assert resp2.status_code == 304

    def test_cache_control_header(self) -> None:
        """ETag responses include Cache-Control: no-cache, must-revalidate."""
        resp = self.client.get("/api/v1/health")
        assert resp.headers.get("Cache-Control") == "no-cache, must-revalidate"
```

These tests import the real `app` and verify middleware registration — if someone removes `ETagMiddleware` from `setup_middleware()`, these fail. They run as part of `make test` (no integration marker needed).

# Plan: 48.10 Synthetic Adversarial Email Generator

## Context

The golden reference library (Phase 37) contains real-world "good" emails. To measure QA check false negatives and strengthen meta-evaluation (48.9), we need emails with **known defects** — targeted, realistic mutations that a human reviewer would catch but automated checks might miss. This is distinct from the chaos engine (random corruption) and property testing (invariant validation).

## Research Summary

### QA Engine Structure
- 14 checks in `app/qa_engine/checks/`, each implementing `QACheckProtocol.run(html, config) -> QACheckResult`
- Check names: `html_validation`, `css_support`, `css_audit`, `file_size`, `link_validation`, `spam_score`, `dark_mode`, `accessibility`, `fallback`, `image_optimization`, `brand_compliance`, `personalisation_syntax`, `deliverability`, `liquid_syntax`
- `QACheckResult`: `check_name: str`, `passed: bool`, `score: float`, `details: str | None`, `severity: str`

### Existing Patterns to Follow
- **Property testing generators** (`app/qa_engine/property_testing/generators.py`): `EmailConfig` frozen dataclass + `build_email()` deterministic builder — same architecture for defect injection
- **Evals synthetic data** (`app/ai/agents/evals/synthetic_data_*.py`): Per-dimension test cases with `expected_challenges` — same labeling pattern for `expected_failures`
- **Adversarial evals** (`app/ai/agents/evals/adversarial.py`): `AdversarialCase` dataclass, YAML fixtures, 7 attack types — reference for defect taxonomy

### Golden Templates (15 built-in)
`app/ai/templates/library/`: `newsletter_1col`, `newsletter_2col`, `promotional_hero`, `promotional_grid`, `promotional_split`, `event_invitation`, `event_reminder`, `transactional_receipt`, `transactional_shipping`, `transactional_welcome`, `retention_winback`, `retention_survey`, `announcement_company`, `announcement_product`, `minimal_text`

### Config Pattern
Nested Pydantic on `Settings`: `qa_chaos: QAChaosConfig`, `qa_property_testing: QAPropertyTestingConfig`, etc. at `config.py:816-823`. Env prefix: `QA_SYNTHETIC__`.

### Step 2: Core Module (`app/qa_engine/synthetic_generator.py`)

```
Dataclasses:
  SyntheticEmail(frozen):
    id: str                          # e.g. "accessibility_001"
    html: str
    expected_failures: dict[str, bool]  # check_name -> should_fail
    defect_category: str             # matches check name
    defect_description: str          # human-readable
    difficulty: Literal["easy", "medium", "hard"]
    base_template: str               # source template name

  SyntheticManifest(frozen):
    generated_at: str                # ISO datetime
    count: int
    emails: list[SyntheticEmail]     # metadata only (no html)

Class SyntheticEmailGenerator:
  __init__(base_templates: list[str] | None = None):
    - Default: load all 15 golden templates from TemplateRegistry
    - Store as dict[str, str] mapping name -> html
    - _injectors: dict[str, Callable] mapping check_name -> injector fn

  generate_adversarial_set() -> list[SyntheticEmail]:
    - Iterate TARGET_CHECKS (11 checks, skip css_audit/deliverability/css_support)
    - Actually: include css_support, skip css_audit + deliverability (13 → 11 target)
    - Call generate_for_check(check_name, count) for each
    - Return flat list

  generate_for_check(check_name: str, count: int = 5) -> list[SyntheticEmail]:
    - Pick `count` templates round-robin from base_templates
    - Apply _injectors[check_name](html, difficulty, idx) for each
    - difficulty cycles: easy(0,1) medium(2,3) hard(4)
    - Return list of SyntheticEmail

  save(emails: list[SyntheticEmail], output_dir: Path) -> Path:
    - Write each email as {id}.html
    - Write manifest.json (SyntheticManifest without html bodies)
    - Return output_dir path
```

### Step 3: Defect Injectors (11 functions in same module)

Each injector: `_inject_{check}(html: str, difficulty: str, idx: int) -> tuple[str, str]` returning `(mutated_html, description)`.

| Injector | Strategy | Notes |
|----------|----------|-------|
| `_inject_file_size` | Insert base64 `<img>` data to push to 101/102/103KB | Use `_PAD_CHAR = "A"` base64 padding; measure with `len(html.encode())` |
| `_inject_accessibility` | Easy: remove `alt=""` → `alt` attr deletion. Medium: set low contrast `color:#777` on `#fff`. Hard: remove `role="presentation"` + `lang` attr | lxml parse + modify + serialize |
| `_inject_link_validation` | Easy: `javascript:void(0)` href. Medium: `http://` (not https). Hard: `mailto:` with header injection | Regex replacement on first `<a href=` |
| `_inject_dark_mode` | Easy: remove `@media (prefers-color-scheme:dark)` block. Medium: hardcode `color:#000` without override. Hard: remove `color-scheme` meta + all dark mode | Regex for style block removal |
| `_inject_image_optimization` | Easy: missing `width`/`height`. Medium: `width="2000"`. Hard: non-retina `width="600"` with data-src 1200px | lxml attr manipulation on `<img>` |
| `_inject_liquid_syntax` | Easy: unclosed `{% if %}`. Medium: nested `{{ }}` inside `{% %}`. Hard: mixed Braze+SFMC syntax | String insertion at specific positions |
| `_inject_personalisation` | Easy: unescaped `%%first_name%%` in href. Medium: AMPscript in subject. Hard: unclosed `%%[` block | String insertion |
| `_inject_spam_score` | Easy: ALL CAPS subject. Medium: `font-size:0` hidden text. Hard: multiple `!!!` + spam keywords | Regex on `<title>` + body insertion |
| `_inject_brand_compliance` | Easy: off-brand `#FF0000` color. Medium: wrong font-family. Hard: both + missing logo | Regex color/font replacement |
| `_inject_fallback` | Easy: remove one `<!--[if mso]>` block. Medium: remove VML namespace. Hard: remove ALL MSO conditionals | Regex removal |
| `_inject_html_validation` | Easy: remove `<!DOCTYPE>`. Medium: unclosed `<td>`. Hard: remove `<meta charset>` + `<title>` | String manipulation on doctype/head |

### Step 4: Template Loading

```python
def _load_base_templates() -> dict[str, str]:
    """Load golden templates from library directory."""
    library = Path(__file__).resolve().parent.parent / "ai" / "templates" / "library"
    templates: dict[str, str] = {}
    for html_file in sorted(library.glob("*.html")):
        templates[html_file.stem] = html_file.read_text()
    return templates
```

### Step 5: Tests (`app/qa_engine/tests/test_synthetic_generator.py`)

12 tests across 3 groups:

**Group A — Generator mechanics (4 tests):**

7. `test_liquid_syntax_has_unclosed_tag` — `{% if %}` without `{% endif %}`
8. `test_dark_mode_removes_media_query` — no `prefers-color-scheme` in output
9. `test_link_validation_injects_javascript_href` — `javascript:` in href

**Group C — Integration (3 tests):**

11. `test_expected_failures_labels_are_correct` — each email's `expected_failures[category] == True`
12. `test_round_trip_check_detects_defect` — run actual QA check on injected email, verify it fails

## Preflight Warnings

- `test_checks.py` is 82K lines — avoid reading it fully; only grep for specific check test patterns
- Property testing generators use `frozen=True` dataclass — follow same pattern for `SyntheticEmail`
- Golden templates are loaded from filesystem, not DB — use `Path` not `TemplateRegistry` (avoids async/DB dependency)
- File size injection must account for UTF-8 encoding (`len(html.encode("utf-8"))`, not `len(html)`)

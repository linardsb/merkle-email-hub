# Plan: 37.3 Wire Golden References into Judge Prompts

## Context

Zero-shot judging forces judges to infer "correct" from text descriptions alone. Few-shot judging with concrete golden reference examples is empirically more accurate — judges pattern-match against known-good HTML. This wires the 37.2 golden reference loader into all 7 HTML-evaluating judges.

## Research Summary

**Judge architecture:** `Judge` protocol in `base.py` defines `build_prompt(JudgeInput) -> str`. Each judge builds: `SYSTEM_PROMPT_TEMPLATE` (with `{criteria_block}`) + `---` separator + user content sections. No base class changes needed — each judge assembles its own prompt string.

**Golden reference loader (37.2, complete):** `app/ai/agents/evals/golden_references.py`
- `get_references_for_criterion(name) -> list[tuple[str, str]]` — max 3 `(name, html)` pairs
- `get_references_for_agent(name) -> list[GoldenReference]` — full objects
- `GoldenReference` dataclass: `name`, `html`, `criteria`, `agents`, `verified_date`, `source_file`
- Constants: `_MAX_SNIPPET_LINES=80`, `_MAX_SNIPPETS_PER_CRITERION=3`

**14 golden references** in `email-templates/components/golden-references/` indexed by `index.yaml`.

## Implementation Steps

### Step 1: Add shared helper in `base.py`

Add to `app/ai/agents/evals/judges/base.py`:

```python
_GOLDEN_TOKEN_BUDGET = 2000  # ~8000 chars at ~4 chars/token

def format_golden_section(
    criteria_names: list[str],
    *,
    framing: str = "standard",
) -> str:
    """Build golden reference prompt section for given criteria.

    Args:
        criteria_names: Criterion names to fetch references for.
        framing: "standard" | "inverted". Inverted adds "do NOT flag" clause.

    Returns:
        Formatted section string, or empty string if no references found.
    """
    from app.ai.agents.evals.golden_references import get_references_for_criterion

    snippets: list[tuple[str, str]] = []
    for name in criteria_names:
        snippets.extend(get_references_for_criterion(name))

    if not snippets:
        return ""

    # Deduplicate by name (same ref may appear for multiple criteria)
    seen: set[str] = set()
    unique: list[tuple[str, str]] = []
    for ref_name, html in snippets:
        if ref_name not in seen:
            seen.add(ref_name)
            unique.append((ref_name, html))

    # Enforce token budget (~4 chars per token)
    char_budget = _GOLDEN_TOKEN_BUDGET * 4
    parts: list[str] = []
    used = 0
    for i, (ref_name, html) in enumerate(unique, 1):
        entry = f"### Example {i}: {ref_name}\n```html\n{html}\n```"
        if used + len(entry) > char_budget:
            break
        parts.append(entry)
        used += len(entry)

    if not parts:
        return ""

    if framing == "inverted":
        header = (
            "## GOLDEN REFERENCE EXAMPLES\n"
            "The following are verified-correct email HTML patterns. "
            "These patterns are CORRECT — do NOT flag them as issues.\n"
        )
    else:
        header = (
            "## GOLDEN REFERENCE EXAMPLES\n"
            "The following are verified-correct email HTML patterns. "
            "Use them as your standard for what \"correct\" looks like "
            "when evaluating the criteria above.\n"
        )

    return header + "\n" + "\n\n".join(parts)
```

## Preflight Warnings

- **LRU cache:** `load_golden_references()` is cached. Tests must clear cache via `load_golden_references.cache_clear()` in autouse fixture (pattern exists in `test_golden_references.py`).
- **Import cycle risk:** `format_golden_section` in `base.py` imports from `golden_references.py`. Use lazy import (inside function body) to avoid circular imports since `golden_references.py` doesn't import from `judges/`.
- **Prompt length:** Golden section adds up to ~2000 tokens. Existing prompts are well under context limits — no overflow risk.
- **No `Judge` protocol change:** The `build_prompt(JudgeInput) -> str` signature is unchanged. Golden refs are fetched internally by each judge, not passed as a parameter.

## Security Checklist

- No new endpoints — internal prompt assembly only
- Golden reference HTML is static repo content, not user-supplied — no injection risk
- Token budget cap prevents prompt overflow
- No new config, secrets, or external API calls

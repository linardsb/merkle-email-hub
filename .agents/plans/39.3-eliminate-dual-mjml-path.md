# Plan: 39.3 Eliminate Dual MJML Generation Path

## Context

Two MJML generation paths exist: `mjml_generator.py` (454 LOC, programmatic) and `mjml_template_engine.py` (231 LOC, Jinja2). They produce different output for the same input. Fixing one leaves the other broken. Phase 38.4 aligned templates with per-text properties ‚Äî the template path is now strictly better. This plan unifies to a single template-based path.

### Current Dual-Path Logic (`converter_service.py:458-581`)

```
_convert_mjml_from_layout(layout, tokens, *, use_templates=True, ...):
  if use_templates:
    try: engine.render_email(sections, ctx)
    except: mjml_str = None  # silent fallback
  if mjml_str is None:
    mjml_str = generate_mjml(layout, tokens)  # programmatic fallback
  compiled = compile_mjml(mjml_str)
  html = inject_section_markers(compiled.html, layout)
```

Remove the `generate_mjml` import entirely.

### Step 3: Simplify `_convert_mjml_from_layout` in `converter_service.py`

Remove the fallback pattern. The method becomes:

```python
async def _convert_mjml_from_layout(self, layout, tokens, *, container_width=600, target_clients=None):
    engine = get_engine()
    ctx = build_template_context(tokens, container_width=container_width)
    # Extract preheader
    preheader = ""
    body_sections: list[EmailSection] = []
    for s in layout.sections:
        if s.section_type == EmailSectionType.PREHEADER:
            preheader = " ".join(t.content for t in s.texts) if s.texts else ""
        else:
            body_sections.append(s)
    mjml_str = engine.render_email(body_sections, ctx, preheader=preheader)
    # ... cache check, compile, inject markers (unchanged)
```

- Remove `use_templates` parameter from signature
- Remove `try/except` that catches template errors and falls back
- Remove `if mjml_str is None: mjml_str = generate_mjml(...)` block
- Let template errors propagate (they indicate real bugs now)

## Preflight Warnings

1. **Fallback removal is intentional**: Template errors should now be loud failures, not silently swallowed. This is correct ‚Äî hiding bugs via fallback is why the dual path persisted.
2. **`inject_section_markers` is the only survivor**: Double-check it has no hidden dependencies on `mjml_generator.py` internals.
3. **Count-based test assertions**: Some tests in `test_mjml_templates.py` use `result.count(...)` ‚Äî these may need updating if the template engine's output format differs slightly from what was tested against the programmatic path.
4. **`converter_service.py` callers**: Verify no external caller passes `use_templates=False` explicitly. Grep for this.
